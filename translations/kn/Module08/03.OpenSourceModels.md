# ಸೆಷನ್ 3: ಓಪನ್-ಸೋರ್ಸ್ ಮಾದರಿ ಅನ್ವೇಷಣೆ ಮತ್ತು ನಿರ್ವಹಣೆ

## ಅವಲೋಕನ

ಈ ಸೆಷನ್ ಫೌಂಡ್ರಿ ಲೋಕಲ್‌ನೊಂದಿಗೆ ಪ್ರಾಯೋಗಿಕ ಮಾದರಿ ಅನ್ವೇಷಣೆ ಮತ್ತು ನಿರ್ವಹಣೆಗೆ ಕೇಂದ್ರೀಕರಿಸುತ್ತದೆ. ಲಭ್ಯವಿರುವ ಮಾದರಿಗಳನ್ನು ಪಟ್ಟಿ ಮಾಡುವುದನ್ನು, ವಿಭಿನ್ನ ಆಯ್ಕೆಗಳನ್ನು ಪರೀಕ್ಷಿಸುವುದನ್ನು ಮತ್ತು ಮೂಲಭೂತ ಕಾರ್ಯಕ್ಷಮತೆ ಲಕ್ಷಣಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದನ್ನು ನೀವು ಕಲಿಯುತ್ತೀರಿ. ಈ ವಿಧಾನವು ಫೌಂಡ್ರಿ CLI ಮೂಲಕ ಕೈಯಿಂದ ಅನ್ವೇಷಣೆಯನ್ನು ಒತ್ತಾಯಿಸುತ್ತದೆ, ನಿಮ್ಮ ಬಳಕೆ ಪ್ರಕರಣಗಳಿಗೆ ಸರಿಯಾದ ಮಾದರಿಗಳನ್ನು ಆಯ್ಕೆ ಮಾಡಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.

## ಕಲಿಕೆಯ ಉದ್ದೇಶಗಳು

- ಮಾದರಿ ಅನ್ವೇಷಣೆ ಮತ್ತು ನಿರ್ವಹಣೆಗೆ ಫೌಂಡ್ರಿ CLI ಆಜ್ಞೆಗಳನ್ನು ನಿಪುಣತೆ ಪಡೆಯುವುದು  
- ಮಾದರಿ ಕ್ಯಾಶೆ ಮತ್ತು ಸ್ಥಳೀಯ ಸಂಗ್ರಹಣಾ ಮಾದರಿಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು  
- ಬೇಗನೆ ಪರೀಕ್ಷಿಸಿ ವಿಭಿನ್ನ ಮಾದರಿಗಳನ್ನು ಹೋಲಿಸುವುದನ್ನು ಕಲಿಯುವುದು  
- ಮಾದರಿ ಆಯ್ಕೆ ಮತ್ತು ಬೆಂಚ್ಮಾರ್ಕಿಂಗ್‌ಗಾಗಿ ಪ್ರಾಯೋಗಿಕ ಕಾರ್ಯಪ್ರವಾಹಗಳನ್ನು ಸ್ಥಾಪಿಸುವುದು  
- ಫೌಂಡ್ರಿ ಲೋಕಲ್ ಮೂಲಕ ಲಭ್ಯವಿರುವ ಬೆಳೆಯುತ್ತಿರುವ ಮಾದರಿ ಪರಿಸರವನ್ನು ಅನ್ವೇಷಿಸುವುದು  

## ಪೂರ್ವಾಪೇಕ್ಷಿತಗಳು

- ಸೆಷನ್ 1: ಫೌಂಡ್ರಿ ಲೋಕಲ್‌ನೊಂದಿಗೆ ಪ್ರಾರಂಭಿಸುವಿಕೆ ಪೂರ್ಣಗೊಳಿಸಲಾಗಿದೆ  
- ಫೌಂಡ್ರಿ ಲೋಕಲ್ CLI ಸ್ಥಾಪಿತ ಮತ್ತು ಪ್ರವೇಶಿಸಬಹುದಾಗಿದೆ  
- ಮಾದರಿ ಡೌನ್‌ಲೋಡ್‌ಗಳಿಗೆ ಸಾಕಷ್ಟು ಸಂಗ್ರಹಣಾ ಜಾಗ (ಮಾದರಿಗಳು 1GB ರಿಂದ 20GB+ ವರೆಗೆ ಇರಬಹುದು)  
- ಮಾದರಿ ಪ್ರಕಾರಗಳು ಮತ್ತು ಬಳಕೆ ಪ್ರಕರಣಗಳ ಮೂಲಭೂತ ಅರ್ಥ  

## ಭಾಗ 6: ಕೈಯಿಂದ ವ್ಯಾಯಾಮ

### ವ್ಯಾಯಾಮ: ಮಾದರಿ ಅನ್ವೇಷಣೆ ಮತ್ತು ಹೋಲಿಕೆ

ನಿಮ್ಮ ಸ್ವಂತ ಮಾದರಿ ಮೌಲ್ಯಮಾಪನ ಸ್ಕ್ರಿಪ್ಟ್ ಅನ್ನು Sample 03 ಆಧಾರಿತವಾಗಿ ರಚಿಸಿ:

```cmd
REM create_model_test.cmd
@echo off
echo Model Discovery and Testing Script
echo =====================================

echo.
echo Step 1: List available models
foundry model list

echo.
echo Step 2: Check what's cached
foundry cache list

echo.
echo Step 3: Start phi-4-mini for testing
foundry model run phi-4-mini --verbose

echo.
echo Step 4: Test with a simple prompt
curl -X POST http://localhost:8000/v1/chat/completions ^
  -H "Content-Type: application/json" ^
  -d "{\"model\":\"phi-4-mini\",\"messages\":[{\"role\":\"user\",\"content\":\"Hello, please introduce yourself.\"}],\"max_tokens\":100}"

echo.
echo Model test complete!
```

### ನಿಮ್ಮ ಕಾರ್ಯ

1. **Sample 03 ಸ್ಕ್ರಿಪ್ಟ್ ಅನ್ನು ಚಾಲನೆ ಮಾಡಿ**: `samples\03\list_and_bench.cmd`  
2. **ವಿಭಿನ್ನ ಮಾದರಿಗಳನ್ನು ಪ್ರಯತ್ನಿಸಿ**: ಕನಿಷ್ಠ 3 ವಿಭಿನ್ನ ಮಾದರಿಗಳನ್ನು ಪರೀಕ್ಷಿಸಿ  
3. **ಕಾರ್ಯಕ್ಷಮತೆಯನ್ನು ಹೋಲಿಸಿ**: ವೇಗ ಮತ್ತು ಪ್ರತಿಕ್ರಿಯೆ ಗುಣಮಟ್ಟದ ವ್ಯತ್ಯಾಸಗಳನ್ನು ಗಮನಿಸಿ  
4. **ನಿರ್ಣಯಗಳನ್ನು ದಾಖಲೆ ಮಾಡಿ**: ಸರಳ ಹೋಲಿಕೆ ಚಾರ್ಟ್ ರಚಿಸಿ  

### ಉದಾಹರಣೆಯ ಹೋಲಿಕೆ ಸ್ವರೂಪ

```
Model Comparison Results:
========================
phi-4-mini:        Fast (~2s), good for general chat
qwen2.5-7b:       Slower (~5s), better reasoning  
deepseek-r1:      Medium (~3s), excellent for code

switch to qwen2.5-7b for production reasoning tasks.
```

## ಭಾಗ 7: ಸಮಸ್ಯೆ ಪರಿಹಾರ ಮತ್ತು ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು

### ಸಾಮಾನ್ಯ ಸಮಸ್ಯೆಗಳು ಮತ್ತು ಪರಿಹಾರಗಳು

**ಮಾದರಿ ಪ್ರಾರಂಭವಾಗುತ್ತಿಲ್ಲ:**  
```cmd
REM Check service status
foundry service status

REM Restart service if needed
foundry service stop
foundry service start

REM Try with verbose output
foundry model run phi-4-mini --verbose
```
  
**ಸ್ಮೃತಿ ಕೊರತೆ:**  
- ಸಣ್ಣ ಮಾದರಿಗಳಿಂದ ಪ್ರಾರಂಭಿಸಿ (`phi-4-mini`)  
- ಇತರ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಮುಚ್ಚಿ  
- ನಿಯಮಿತವಾಗಿ ಮಿತಿಗಳನ್ನು ತಲುಪುತ್ತಿದ್ದರೆ RAM ಅನ್ನು ಅಪ್‌ಗ್ರೇಡ್ ಮಾಡಿ  

**ನೀವು ಕಾರ್ಯಕ್ಷಮತೆ ನಿಧಾನವಾಗಿದೆ:**  
- ಮಾದರಿ ಸಂಪೂರ್ಣವಾಗಿ ಲೋಡ್ ಆಗಿರುವುದನ್ನು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ (ವಿವರವಾದ ಔಟ್‌ಪುಟ್ ಪರಿಶೀಲಿಸಿ)  
- ಅಗತ್ಯವಿಲ್ಲದ ಹಿನ್ನೆಲೆ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಮುಚ್ಚಿ  
- ವೇಗವಾದ ಸಂಗ್ರಹಣೆಯನ್ನು ಪರಿಗಣಿಸಿ (SSD)  

### ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು

1. **ಸಣ್ಣದರಿಂದ ಪ್ರಾರಂಭಿಸಿ**: ಸೆಟ್‌ಅಪ್ ಪರಿಶೀಲಿಸಲು `phi-4-mini` ಬಳಸಿ  
2. **ಒಂದು ಮಾದರಿ ಒಂದೇ ಸಮಯದಲ್ಲಿ**: ಹೊಸ ಮಾದರಿಗಳನ್ನು ಪ್ರಾರಂಭಿಸುವ ಮೊದಲು ಹಳೆಯ ಮಾದರಿಗಳನ್ನು ನಿಲ್ಲಿಸಿ  
3. **ಸಂಪನ್ಮೂಲಗಳನ್ನು ಗಮನಿಸಿ**: ಸ್ಮೃತಿ ಬಳಕೆಯನ್ನು ಗಮನದಲ್ಲಿಡಿ  
4. **ನಿರಂತರವಾಗಿ ಪರೀಕ್ಷಿಸಿ**: ನ್ಯಾಯಸಮ್ಮತ ಹೋಲಿಕೆಗಾಗಿ ಒಂದೇ ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ಬಳಸಿ  
5. **ಫಲಿತಾಂಶಗಳನ್ನು ದಾಖಲೆ ಮಾಡಿ**: ನಿಮ್ಮ ಬಳಕೆ ಪ್ರಕರಣಗಳಿಗೆ ಮಾದರಿ ಕಾರ್ಯಕ್ಷಮತೆಯ ಬಗ್ಗೆ ಟಿಪ್ಪಣಿಗಳನ್ನು ಇಡಿ  

## ಭಾಗ 8: ಮುಂದಿನ ಹಂತಗಳು ಮತ್ತು ಉಲ್ಲೇಖಗಳು

### ಸೆಷನ್ 4 ಗೆ ಸಿದ್ಧತೆ

- **ಸೆಷನ್ 4 ಕೇಂದ್ರೀಕರಣ**: ಆಪ್ಟಿಮೈಜೆಷನ್ ಸಾಧನಗಳು ಮತ್ತು ತಂತ್ರಗಳು  
- **ಪೂರ್ವಾಪೇಕ್ಷಿತಗಳು**: ಮಾದರಿ ಬದಲಾವಣೆ ಮತ್ತು ಮೂಲಭೂತ ಕಾರ್ಯಕ್ಷಮತೆ ಪರೀಕ್ಷೆಯಲ್ಲಿ ಆರಾಮದಾಯಕತೆ  
- **ಶಿಫಾರಸು ಮಾಡಲಾಗಿದೆ**: ಈ ಸೆಷನ್‌ನಿಂದ 2-3 ಪ್ರಿಯ ಮಾದರಿಗಳನ್ನು ಗುರುತಿಸಿ  

### ಹೆಚ್ಚುವರಿ ಸಂಪನ್ಮೂಲಗಳು

- **[Foundry Local ಡಾಕ್ಯುಮೆಂಟೇಶನ್](https://learn.microsoft.com/azure/ai-foundry/foundry-local/)**: ಅಧಿಕೃತ ಡಾಕ್ಯುಮೆಂಟೇಶನ್  
- **[CLI ರೆಫರೆನ್ಸ್](https://learn.microsoft.com/azure/ai-foundry/foundry-local/reference/reference-cli)**: ಸಂಪೂರ್ಣ ಆಜ್ಞೆ ರೆಫರೆನ್ಸ್  
- **[Model Mondays](https://aka.ms/model-mondays)**: ವಾರದ ಮಾದರಿ ಸ್ಪಾಟ್‌ಲೈಟ್‌ಗಳು  
- **[Foundry Local GitHub](https://github.com/microsoft/Foundry-Local)**: ಸಮುದಾಯ ಮತ್ತು ಸಮಸ್ಯೆಗಳು  
- **[Sample 03: Model Discovery](samples/03/README.md)**: ಕೈಯಿಂದ ಉದಾಹರಣಾ ಸ್ಕ್ರಿಪ್ಟ್  

### ಪ್ರಮುಖ ಪಾಠಗಳು

✅ **ಮಾದರಿ ಅನ್ವೇಷಣೆ**: ಲಭ್ಯವಿರುವ ಮಾದರಿಗಳನ್ನು ಅನ್ವೇಷಿಸಲು `foundry model list` ಬಳಸಿ  
✅ **ವೇಗದ ಪರೀಕ್ಷೆ**: ತ್ವರಿತ ಮೌಲ್ಯಮಾಪನಕ್ಕೆ `list_and_bench.cmd` ಮಾದರಿ  
✅ **ಕಾರ್ಯಕ್ಷಮತೆ ಮೇಲ್ವಿಚಾರಣೆ**: ಮೂಲಭೂತ ಸಂಪನ್ಮೂಲ ಬಳಕೆ ಮತ್ತು ಪ್ರತಿಕ್ರಿಯೆ ಸಮಯದ ಅಳೆಯುವಿಕೆ  
✅ **ಮಾದರಿ ಆಯ್ಕೆ**: ಬಳಕೆ ಪ್ರಕರಣದ ಪ್ರಕಾರ ಮಾದರಿಗಳನ್ನು ಆಯ್ಕೆ ಮಾಡುವ ಪ್ರಾಯೋಗಿಕ ಮಾರ್ಗಸೂಚಿಗಳು  
✅ **ಕ್ಯಾಶೆ ನಿರ್ವಹಣೆ**: ಸಂಗ್ರಹಣೆ ಮತ್ತು ಕ್ಲೀನ್‌ಅಪ್ ಪ್ರಕ್ರಿಯೆಗಳ ಅರ್ಥ  

ನೀವು ಈಗ ಫೌಂಡ್ರಿ ಲೋಕಲ್‌ನ ಸರಳ CLI ವಿಧಾನವನ್ನು ಬಳಸಿ ನಿಮ್ಮ AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗೆ ಸೂಕ್ತ ಮಾದರಿಗಳನ್ನು ಅನ್ವೇಷಿಸಲು, ಪರೀಕ್ಷಿಸಲು ಮತ್ತು ಆಯ್ಕೆ ಮಾಡಲು ಪ್ರಾಯೋಗಿಕ ಕೌಶಲ್ಯಗಳನ್ನು ಹೊಂದಿದ್ದೀರಿ.
