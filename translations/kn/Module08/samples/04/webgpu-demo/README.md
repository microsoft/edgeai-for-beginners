# WebGPU + ONNX Runtime ಡೆಮೊ

ಈ ಡೆಮೊ WebGPU ಬಳಸಿ ಹಾರ್ಡ್‌ವೇರ್ ವೇಗವರ್ಧನೆಗಾಗಿ ಮತ್ತು ONNX Runtime ವೆಬ್ ಬಳಸಿ ಬ್ರೌಸರ್‌ನಲ್ಲಿ ನೇರವಾಗಿ AI ಮಾದರಿಗಳನ್ನು ಹೇಗೆ ಚಾಲನೆ ಮಾಡುವುದು ಎಂದು ತೋರಿಸುತ್ತದೆ.

## ಇದು ಏನು ತೋರಿಸುತ್ತದೆ

- **ಬ್ರೌಸರ್ ಆಧಾರಿತ AI**: ಮಾದರಿಗಳನ್ನು ಸಂಪೂರ್ಣವಾಗಿ ಬ್ರೌಸರ್‌ನಲ್ಲಿ ಚಾಲನೆ ಮಾಡಿ
- **WebGPU ವೇಗವರ್ಧನೆ**: ಲಭ್ಯವಿದ್ದರೆ ಹಾರ್ಡ್‌ವೇರ್ ವೇಗವರ್ಧಿತ ಇನ್ಫರೆನ್ಸ್
- **ಗೌಪ್ಯತೆ-ಪ್ರಥಮ**: ನಿಮ್ಮ ಸಾಧನದಿಂದ ಯಾವುದೇ ಡೇಟಾ ಹೊರಬರದು
- **ಶೂನ್ಯ ಸ್ಥಾಪನೆ**: ಯಾವುದೇ ಹೊಂದಾಣಿಕೆಯ ಬ್ರೌಸರ್‌ನಲ್ಲಿ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ
- **ಸುಮಧುರ ಬ್ಯಾಕಪ್**: WebGPU ಲಭ್ಯವಿಲ್ಲದಿದ್ದರೆ CPU ಗೆ ಬ್ಯಾಕಪ್ ಆಗುತ್ತದೆ

## ಅಗತ್ಯತೆಗಳು

**ಬ್ರೌಸರ್ ಹೊಂದಾಣಿಕೆ:**
- Chrome/Edge 113+ WebGPU ಸಕ್ರಿಯಗೊಂಡಿರುವುದು
- WebGPU ಸ್ಥಿತಿಯನ್ನು ಪರಿಶೀಲಿಸಿ: `chrome://gpu`
- WebGPU ಸಕ್ರಿಯಗೊಳಿಸಿ: `chrome://flags/#enable-unsafe-webgpu`

## ಡೆಮೊ ಚಾಲನೆ

### ಆಯ್ಕೆ 1: ಸ್ಥಳೀಯ ಸರ್ವರ್ (ಶಿಫಾರಸು ಮಾಡಲಾಗಿದೆ)

```cmd
# Navigate to the demo directory
cd Module08\samples\04\webgpu-demo

# Start a local server
python -m http.server 5173

# Open browser to http://localhost:5173
```

### ಆಯ್ಕೆ 2: VS ಕೋಡ್ ಲೈವ್ ಸರ್ವರ್

1. VS ಕೋಡ್‌ನಲ್ಲಿ "Live Server" ವಿಸ್ತರಣೆ ಸ್ಥಾಪಿಸಿ
2. `index.html` ಮೇಲೆ ರೈಟ್-ಕ್ಲಿಕ್ ಮಾಡಿ → "Open with Live Server"
3. ಡೆಮೊ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಬ್ರೌಸರ್‌ನಲ್ಲಿ ತೆರೆಯುತ್ತದೆ

## ನೀವು ಏನು ನೋಡುತ್ತೀರಿ

1. **WebGPU ಪತ್ತೆ**: ಬ್ರೌಸರ್ ಹೊಂದಾಣಿಕೆಯನ್ನು ಪರಿಶೀಲಿಸುತ್ತದೆ
2. **ಮಾದರಿ ಲೋಡ್**: MNIST ವರ್ಗೀಕರಣೆಯನ್ನು ಡೌನ್‌ಲೋಡ್ ಮಾಡಿ ಪ್ರಾರಂಭಿಸುತ್ತದೆ
3. **ಇನ್ಫರೆನ್ಸ್ ಕಾರ್ಯಾಚರಣೆ**: ಮಾದರಿ ಮೇಲೆ ಭವಿಷ್ಯವಾಣಿ ನಡೆಸುತ್ತದೆ
4. **ಕಾರ್ಯಕ್ಷಮತೆ ಅಳೆಯುವಿಕೆ**: ಲೋಡ್ ಸಮಯ ಮತ್ತು ಇನ್ಫರೆನ್ಸ್ ವೇಗವನ್ನು ತೋರಿಸುತ್ತದೆ
5. **ಫಲಿತಾಂಶ ಪ್ರದರ್ಶನ**: ಭವಿಷ್ಯವಾಣಿ ವಿಶ್ವಾಸ ಮತ್ತು ಕಚ್ಚಾ ಔಟ್‌ಪುಟ್‌ಗಳು

## ನಿರೀಕ್ಷಿತ ಕಾರ್ಯಕ್ಷಮತೆ

| ಕಾರ್ಯಾಚರಣೆ ಒದಗಿಸುವವರು | ಮಾದರಿ ಲೋಡ್ | ಇನ್ಫರೆನ್ಸ್ | ಟಿಪ್ಪಣಿಗಳು |
|-------------------|------------|-----------|-------|
| **WebGPU** | ~2-5 ಸೆಕೆಂಡುಗಳು | ~10-50 ಮಿಲಿ ಸೆಕೆಂಡುಗಳು | ಹಾರ್ಡ್‌ವೇರ್ ವೇಗವರ್ಧಿತ |
| **CPU (WASM)** | ~2-5 ಸೆಕೆಂಡುಗಳು | ~50-200 ಮಿಲಿ ಸೆಕೆಂಡುಗಳು | ಸಾಫ್ಟ್‌ವೇರ್ ಬ್ಯಾಕಪ್ |

## ಸಮಸ್ಯೆ ಪರಿಹಾರ

**WebGPU ಲಭ್ಯವಿಲ್ಲ:**
- Chrome/Edge 113+ ಗೆ ನವೀಕರಿಸಿ
- `chrome://flags` ನಲ್ಲಿ WebGPU ಸಕ್ರಿಯಗೊಳಿಸಿ
- GPU ಡ್ರೈವರ್‌ಗಳು ನವೀಕರಿಸಿರುವುದನ್ನು ಪರಿಶೀಲಿಸಿ
- ಡೆಮೊ ಸ್ವಯಂಚಾಲಿತವಾಗಿ CPU ಗೆ ಬ್ಯಾಕಪ್ ಆಗುತ್ತದೆ

**ಲೋಡಿಂಗ್ ದೋಷಗಳು:**
- ನೀವು HTTP ಮೂಲಕ ಸೇವೆ ನೀಡುತ್ತಿರುವುದನ್ನು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ (file:// ಅಲ್ಲ)
- ಮಾದರಿ ಡೌನ್‌ಲೋಡ್‌ಗೆ ನೆಟ್‌ವರ್ಕ್ ಸಂಪರ್ಕ ಪರಿಶೀಲಿಸಿ
- ONNX ಮಾದರಿಯನ್ನು CORS ತಡೆಗಟ್ಟುತ್ತಿಲ್ಲ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ

**ಕಾರ್ಯಕ್ಷಮತೆ ಸಮಸ್ಯೆಗಳು:**
- WebGPU CPU ಗಿಂತ ಬಹಳ ವೇಗವನ್ನು ಒದಗಿಸುತ್ತದೆ
- ಮೊದಲ ಚಾಲನೆ ಮಾದರಿ ಡೌನ್‌ಲೋಡ್ ಕಾರಣದಿಂದ ನಿಧಾನವಾಗಬಹುದು
- ಮುಂದಿನ ಚಾಲನೆಗಳು ಬ್ರೌಸರ್ ಕ್ಯಾಶೆ ಬಳಸುತ್ತವೆ

## Foundry Local ಜೊತೆಗೆ ಏಕೀಕರಣ

ಈ WebGPU ಡೆಮೊ Foundry Local ನೊಂದಿಗೆ ಪೂರಕವಾಗಿದೆ ಮತ್ತು ತೋರಿಸುತ್ತದೆ:

- **ಗ್ರಾಹಕ-ಪಾರ್ಶ್ವ ಇನ್ಫರೆನ್ಸ್** ಪರಮ ಗೌಪ್ಯತೆಗಾಗಿ
- **ಆಫ್‌ಲೈನ್ ಸಾಮರ್ಥ್ಯಗಳು** ಇಂಟರ್ನೆಟ್ ಲಭ್ಯವಿಲ್ಲದಾಗ
- **ಎಡ್ಜ್ ನಿಯೋಜನೆ** ಸಂಪನ್ಮೂಲ-ಮಿತಿಯ ಪರಿಸರಗಳಿಗೆ
- **ಹೈಬ್ರಿಡ್ ವಾಸ್ತುಶಿಲ್ಪಗಳು** ಸ್ಥಳೀಯ ಮತ್ತು ಸರ್ವರ್ ಇನ್ಫರೆನ್ಸ್ ಸಂಯೋಜನೆ

ಉತ್ಪಾದನಾ ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗೆ ಪರಿಗಣಿಸಿ:
- ಸರ್ವರ್-ಪಾರ್ಶ್ವ ಇನ್ಫರೆನ್ಸ್‌ಗೆ Foundry Local ಬಳಸಿ
- ಗ್ರಾಹಕ-ಪಾರ್ಶ್ವ ಪೂರ್ವ/ಪೋಸ್ಟ್ ಪ್ರೊಸೆಸಿಂಗ್‌ಗೆ WebGPU ಬಳಸಿ
- ಸ್ಥಳೀಯ/ದೂರಸ್ಥ ಇನ್ಫರೆನ್ಸ್ ನಡುವೆ ಬುದ್ಧಿವಂತ ಮಾರ್ಗದರ್ಶನ ಅನುಷ್ಠಾನಗೊಳಿಸಿ

## ತಾಂತ್ರಿಕ ವಿವರಗಳು

**ಬಳಸಿದ ಮಾದರಿ:**
- MNIST ಅಂಕಿ ವರ್ಗೀಕರಣೆ (ONNX ಫಾರ್ಮ್ಯಾಟ್)
- ಇನ್ಪುಟ್: 28x28 ಗ್ರೇಸ್ಕೇಲ್ ಚಿತ್ರಗಳು
- ಔಟ್‌ಪುಟ್: 10-ವರ್ಗದ ಸಾಧ್ಯತೆ ವಿತರಣಾ
- ಗಾತ್ರ: ~500KB (ವೇಗದ ಡೌನ್‌ಲೋಡ್)

**ONNX Runtime Web:**
- GPU ವೇಗವರ್ಧನೆಗಾಗಿ WebGPU ಕಾರ್ಯಾಚರಣೆ ಒದಗಿಸುವವರು
- CPU ಬ್ಯಾಕಪ್‌ಗೆ WASM ಕಾರ್ಯಾಚರಣೆ ಒದಗಿಸುವವರು
- ಸ್ವಯಂಚಾಲಿತ ಆಪ್ಟಿಮೈಜೆಷನ್ ಮತ್ತು ಗ್ರಾಫ್ ಆಪ್ಟಿಮೈಜೆಷನ್

**ಬ್ರೌಸರ್ APIಗಳು:**
- ಹಾರ್ಡ್‌ವೇರ್ ಪ್ರವೇಶಕ್ಕೆ WebGPU
- ಹಿನ್ನೆಲೆ ಪ್ರಕ್ರಿಯೆಗೆ Web Workers (ಭವಿಷ್ಯ ಸುಧಾರಣೆ)
- ಪರಿಣಾಮಕಾರಿ ಗಣನೆಗೆ WebAssembly

## ಮುಂದಿನ ಹಂತಗಳು

- ಕಸ್ಟಮ್ ONNX ಮಾದರಿಗಳೊಂದಿಗೆ ಪ್ರಯತ್ನಿಸಿ
- ನಿಜವಾದ ಚಿತ್ರ ಅಪ್ಲೋಡ್ ಮತ್ತು ವರ್ಗೀಕರಣವನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಿ
- ದೊಡ್ಡ ಮಾದರಿಗಳಿಗಾಗಿ ಸ್ಟ್ರೀಮಿಂಗ್ ಇನ್ಫರೆನ್ಸ್ ಸೇರಿಸಿ
- ಕ್ಯಾಮೆರಾ/ಮೈಕ್ರೋಫೋನ್ ಇನ್ಪುಟ್ ಜೊತೆಗೆ ಏಕೀಕರಿಸಿ
