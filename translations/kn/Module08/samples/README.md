# Module 08 ಮಾದರಿಗಳು: Foundry ಸ್ಥಳೀಯ ಅಭಿವೃದ್ಧಿ ಮಾರ್ಗದರ್ಶಿ

## ಅವಲೋಕನ

ಈ ಸಮಗ್ರ ಸಂಗ್ರಹವು **Foundry ಸ್ಥಳೀಯ SDK** ಮತ್ತು **ಶೆಲ್ ಕಮಾಂಡ್** ವಿಧಾನಗಳನ್ನು ಎರಡನ್ನೂ ಪ್ರದರ್ಶಿಸುತ್ತದೆ, ಉತ್ಪಾದನೆಗೆ ಸಿದ್ಧವಾದ AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿರ್ಮಿಸಲು. ಪ್ರತಿ ಮಾದರಿ ಎಡ್ಜ್ AI ಅಭಿವೃದ್ಧಿಯ ವಿಭಿನ್ನ ಅಂಶಗಳನ್ನು ತೋರಿಸುತ್ತದೆ, ಮೂಲ REST ಏಕೀಕರಣದಿಂದ ಪ್ರಗತಿಶೀಲ ಬಹು-ಏಜೆಂಟ್ ವ್ಯವಸ್ಥೆಗಳವರೆಗೆ.

## ಅಭಿವೃದ್ಧಿ ವಿಧಾನ: SDK ವಿರುದ್ಧ ಶೆಲ್ ಕಮಾಂಡ್‌ಗಳು

### Foundry ಸ್ಥಳೀಯ SDK ಅನ್ನು ಬಳಸುವಾಗ:

- **ಕಾರ್ಯಕ್ರಮ ನಿಯಂತ್ರಣ**: ಏಜೆಂಟ್ ಜೀವನಚಕ್ರ, ಮೌಲ್ಯಮಾಪನ ಅಥವಾ ನಿಯೋಜನೆ ಕಾರ್ಯಪ್ರವಾಹಗಳ ಮೇಲೆ ಸಂಪೂರ್ಣ ನಿಯಂತ್ರಣ ಬೇಕಾದಾಗ
- **ಕಸ್ಟಮ್ ಉಪಕರಣಗಳು**: Foundry ಸ್ಥಳೀಯದ ಸುತ್ತಲೂ ಸ್ವಯಂಚಾಲನೆ ನಿರ್ಮಿಸುವಾಗ (CI/CD ಏಕೀಕರಣ, ಬಹು-ಏಜೆಂಟ್ ಸಂಯೋಜನೆ)
- **ಸೂಕ್ಷ್ಮ ಪ್ರವೇಶ**: ವಿವರವಾದ ಏಜೆಂಟ್ ಮೆಟಾಡೇಟಾ, ಆವೃತ್ತಿ ಅಥವಾ ಮೌಲ್ಯಮಾಪನ ರನ್ನರ್ ನಿಯಂತ್ರಣ ಅಗತ್ಯವಿದ್ದಾಗ
- **ಪೈಥಾನ್ ಏಕೀಕರಣ**: ಪೈಥಾನ್-ಭಾರಿತ ಪರಿಸರಗಳಲ್ಲಿ ಕೆಲಸ ಮಾಡುತ್ತಿರುವಾಗ ಅಥವಾ Foundry ಲಾಜಿಕ್ ಅನ್ನು ವ್ಯಾಪಕ ಅಪ್ಲಿಕೇಶನ್‌ಗಳಲ್ಲಿ ಸಂಯೋಜಿಸುವಾಗ
- **ಉದ್ಯಮ ಕಾರ್ಯಪ್ರವಾಹಗಳು**: Microsoft ಉಲ್ಲೇಖ معمارياتಕ್ಕೆ ಹೊಂದಿಕೆಯಾಗುವ ಮಡ್ಯೂಲರ್ ಕಾರ್ಯಪ್ರವಾಹಗಳು ಮತ್ತು ಪುನರಾವರ್ತನೀಯ ಮೌಲ್ಯಮಾಪನ ಪೈಪ್ಲೈನ್‌ಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸುವಾಗ

### ಶೆಲ್ ಕಮಾಂಡ್‌ಗಳನ್ನು ಬಳಸುವಾಗ:

- **ತ್ವರಿತ ಪರೀಕ್ಷೆ**: ಸ್ಥಳೀಯವಾಗಿ ವೇಗವಾಗಿ ಪರೀಕ್ಷೆ ನಡೆಸುವುದು, ಕೈಯಿಂದ ಏಜೆಂಟ್ ಪ್ರಾರಂಭಿಸುವುದು ಅಥವಾ ಸೆಟ್‌ಅಪ್ ಪರಿಶೀಲನೆ
- **CLI ಸರಳತೆ**: ಏಜೆಂಟ್‌ಗಳನ್ನು ಪ್ರಾರಂಭ/ನಿಲ್ಲಿಸುವುದು, ಲಾಗ್‌ಗಳನ್ನು ಪರಿಶೀಲಿಸುವುದು ಅಥವಾ ಮೂಲ ಮೌಲ್ಯಮಾಪನಗಳಿಗೆ ಸರಳ CLI ಕಾರ್ಯಾಚರಣೆಗಳು ಬೇಕಾದಾಗ
- **ತೂಕಹೀನ ಸ್ವಯಂಚಾಲನೆ**: ಸಂಪೂರ್ಣ SDK ಏಕೀಕರಣ ಅಗತ್ಯವಿಲ್ಲದೆ ಸರಳ ಸ್ವಯಂಚಾಲನೆ ಸ್ಕ್ರಿಪ್ಟಿಂಗ್
- **ತ್ವರಿತ ಪುನರಾವರ್ತನೆ**: ಡಿಬಗ್ ಮತ್ತು ಅಭಿವೃದ್ಧಿ ಚಕ್ರಗಳು, ವಿಶೇಷವಾಗಿ ನಿರ್ಬಂಧಿತ ಪರಿಸರಗಳಲ್ಲಿ ಅಥವಾ ಸಂಪನ್ಮೂಲ ಗುಂಪು ಮಟ್ಟದ ನಿಯೋಜನೆಗಳಲ್ಲಿ
- **ಸೆಟ್‌ಅಪ್ ಮತ್ತು ಮಾನ್ಯತೆ**: ಪ್ರಾಥಮಿಕ ಪರಿಸರ ಸಂರಚನೆ ಮತ್ತು ತ್ವರಿತ ಪರಿಶೀಲನೆ ಕಾರ್ಯಗಳು

## ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು ಮತ್ತು ಶಿಫಾರಸು ಮಾಡಿದ ಕಾರ್ಯಪ್ರವಾಹ

ಏಜೆಂಟ್ ಜೀವನಚಕ್ರ ನಿರ್ವಹಣೆ, ಅವಲಂಬನೆ ಟ್ರ್ಯಾಕಿಂಗ್ ಮತ್ತು ಕನಿಷ್ಠ ಹಕ್ಕು ಪುನರಾವರ್ತನೀಯತೆಯ ತತ್ವಗಳ ಆಧಾರದ ಮೇಲೆ:

### ಹಂತ 1: ಆಧಾರ ಮತ್ತು ಸೆಟ್‌ಅಪ್
1. ಪ್ರಾಥಮಿಕ ಸೆಟ್‌ಅಪ್ ಮತ್ತು ತ್ವರಿತ ಮಾನ್ಯತೆಗಾಗಿ **ಶೆಲ್ ಕಮಾಂಡ್‌ಗಳಿಂದ ಪ್ರಾರಂಭಿಸಿ**
2. CLI ಉಪಕರಣಗಳು ಮತ್ತು ಮೂಲ ಮಾದರಿ ನಿಯೋಜನೆ ಬಳಸಿ **ಪರಿಸರವನ್ನು ಪರಿಶೀಲಿಸಿ**
3. ಸರಳ REST ಕರೆಗಳು ಮತ್ತು ಆರೋಗ್ಯ ಪರಿಶೀಲನೆಗಳೊಂದಿಗೆ **ಸಂಪರ್ಕವನ್ನು ಪರೀಕ್ಷಿಸಿ**

### ಹಂತ 2: ಅಭಿವೃದ್ಧಿ ಮತ್ತು ಏಕೀಕರಣ
1. **SDK ಗೆ ಪರಿವರ್ತಿಸಿ** ವ್ಯಾಪಕ, ಟ್ರೇಸಬಲ್ ಕಾರ್ಯಪ್ರವಾಹಗಳಿಗೆ
2. ಸಂಕೀರ್ಣ ಏಜೆಂಟ್ ಸಂವಹನಗಳಿಗೆ **ಕಾರ್ಯಕ್ರಮ ನಿಯಂತ್ರಣವನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಿ**
3. ಸಮುದಾಯಕ್ಕೆ ಸಿದ್ಧವಾದ ಟೆಂಪ್ಲೇಟುಗಳು ಮತ್ತು Azure OpenAI ಏಕೀಕರಣಕ್ಕಾಗಿ **ಕಸ್ಟಮ್ ಉಪಕರಣಗಳನ್ನು ನಿರ್ಮಿಸಿ**

### ಹಂತ 3: ಉತ್ಪಾದನೆ ಮತ್ತು ಪ್ರಮಾಣ
1. ಕಾರ್ಯಾಚರಣೆಗಳಿಗೆ CLI ಮತ್ತು ಅಪ್ಲಿಕೇಶನ್ ಲಾಜಿಕ್‌ಗೆ SDK ಅನ್ನು ಸಂಯೋಜಿಸುವ **ಹೈಬ್ರಿಡ್ ವಿಧಾನ**
2. ಮೇಲ್ವಿಚಾರಣೆ, ಲಾಗಿಂಗ್ ಮತ್ತು ನಿಯೋಜನೆ ಪೈಪ್ಲೈನ್‌ಗಳೊಂದಿಗೆ **ಉದ್ಯಮ ಏಕೀಕರಣ**
3. ಪುನಃಬಳಕೆ ಮಾಡಬಹುದಾದ ಟೆಂಪ್ಲೇಟುಗಳು ಮತ್ತು ಉತ್ತಮ ಅಭ್ಯಾಸಗಳ ಮೂಲಕ **ಸಮುದಾಯ ಕೊಡುಗೆ**
