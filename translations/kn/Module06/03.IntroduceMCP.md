# ವಿಭಾಗ 03 - ಮಾದರಿ ಸಂದರ್ಭ ಪ್ರೋಟೋಕಾಲ್ (MCP) ಏಕೀಕರಣ

## MCP (ಮಾದರಿ ಸಂದರ್ಭ ಪ್ರೋಟೋಕಾಲ್) ಪರಿಚಯ

ಮಾದರಿ ಸಂದರ್ಭ ಪ್ರೋಟೋಕಾಲ್ (MCP) ಎಂಬುದು AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಹೊರಗಿನ ವ್ಯವಸ್ಥೆಗಳಿಗೆ ಸಂಪರ್ಕಿಸುವ ಮುಕ್ತ ಮೂಲ ಮಾನಕವಾಗಿದೆ. MCP ಬಳಸಿ, Claude ಅಥವಾ ChatGPT ಮುಂತಾದ AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳು ಡೇಟಾ ಮೂಲಗಳು (ಉದಾ., ಸ್ಥಳೀಯ ಫೈಲ್‌ಗಳು, ಡೇಟಾಬೇಸ್‌ಗಳು), ಉಪಕರಣಗಳು (ಉದಾ., ಹುಡುಕಾಟ ಎಂಜಿನ್‌ಗಳು, ಕ್ಯಾಲ್ಕ್ಯುಲೇಟರ್‌ಗಳು), ಮತ್ತು ಕಾರ್ಯಪ್ರವಾಹಗಳು (ಉದಾ., ವಿಶೇಷ ಪ್ರಾಂಪ್ಟ್‌ಗಳು) ಗೆ ಸಂಪರ್ಕ ಹೊಂದಬಹುದು—ಅವು ಮುಖ್ಯ ಮಾಹಿತಿಯನ್ನು ಪ್ರವೇಶಿಸಲು ಮತ್ತು ಕಾರ್ಯಗಳನ್ನು ನಿರ್ವಹಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.

MCP ಅನ್ನು AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗೆ **USB-C ಪೋರ್ಟ್** ಎಂದು ಭಾವಿಸಬಹುದು. USB-C ಎಲೆಕ್ಟ್ರಾನಿಕ್ ಸಾಧನಗಳನ್ನು ಸಂಪರ್ಕಿಸುವ ಮಾನಕ ವಿಧಾನವನ್ನು ಒದಗಿಸುವಂತೆ, MCP AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಹೊರಗಿನ ವ್ಯವಸ್ಥೆಗಳಿಗೆ ಸಂಪರ್ಕಿಸುವ ಮಾನಕ ವಿಧಾನವನ್ನು ಒದಗಿಸುತ್ತದೆ.

### MCP ಏನು ಸಾಧ್ಯಮಾಡುತ್ತದೆ?

MCP AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗೆ ಶಕ್ತಿಶಾಲಿ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಅನ್ಲಾಕ್ ಮಾಡುತ್ತದೆ:

- **ವೈಯಕ್ತಿಕೃತ AI ಸಹಾಯಕರು**: ಏಜೆಂಟ್‌ಗಳು ನಿಮ್ಮ Google ಕ್ಯಾಲೆಂಡರ್ ಮತ್ತು Notion ಗೆ ಪ್ರವೇಶಿಸಿ, ಹೆಚ್ಚು ವೈಯಕ್ತಿಕೃತ AI ಸಹಾಯಕನಾಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸಬಹುದು
- **ಅತ್ಯಾಧುನಿಕ ಕೋಡ್ ರಚನೆ**: Claude Code ಫಿಗ್ಮಾ ವಿನ್ಯಾಸವನ್ನು ಬಳಸಿ ಸಂಪೂರ್ಣ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್ ರಚಿಸಬಹುದು
- **ಉದ್ಯಮ ಡೇಟಾ ಏಕೀಕರಣ**: ಉದ್ಯಮ ಚಾಟ್‌ಬಾಟ್‌ಗಳು ಸಂಸ್ಥೆಯ ವಿವಿಧ ಡೇಟಾಬೇಸ್‌ಗಳಿಗೆ ಸಂಪರ್ಕ ಹೊಂದಿ, ಬಳಕೆದಾರರಿಗೆ ಚಾಟ್ ಮೂಲಕ ಡೇಟಾ ವಿಶ್ಲೇಷಣೆ ಮಾಡಲು ಶಕ್ತಿ ನೀಡುತ್ತದೆ
- **ಸೃಜನಾತ್ಮಕ ಕಾರ್ಯಪ್ರವಾಹಗಳು**: AI ಮಾದರಿಗಳು Blender ನಲ್ಲಿ 3D ವಿನ್ಯಾಸಗಳನ್ನು ರಚಿಸಿ 3D ಪ್ರಿಂಟರ್ ಬಳಸಿ ಮುದ್ರಿಸಬಹುದು
- **ನೈಜ-ಸಮಯ ಮಾಹಿತಿ ಪ್ರವೇಶ**: ನವೀಕೃತ ಮಾಹಿತಿಗಾಗಿ ಹೊರಗಿನ ಡೇಟಾ ಮೂಲಗಳಿಗೆ ಸಂಪರ್ಕ
- **ಸಂಕೀರ್ಣ ಬಹು-ಹಂತ ಕಾರ್ಯಾಚರಣೆಗಳು**: ಹಲವಾರು ಉಪಕರಣಗಳು ಮತ್ತು ವ್ಯವಸ್ಥೆಗಳನ್ನು ಸಂಯೋಜಿಸುವ ಸುಕ್ಷ್ಮ ಕಾರ್ಯಪ್ರವಾಹಗಳನ್ನು ನಿರ್ವಹಿಸುವುದು

### MCP ಯಾಕೆ ಮುಖ್ಯ?

MCP ಪರಿಸರದಲ್ಲಿ ಹಲವಾರು ಲಾಭಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ:

**ವಿಕಸಕರಿಗೆ**: MCP AI ಅಪ್ಲಿಕೇಶನ್ ಅಥವಾ ಏಜೆಂಟ್ ನಿರ್ಮಿಸುವಾಗ ಅಥವಾ ಏಕೀಕರಿಸುವಾಗ ಅಭಿವೃದ್ಧಿ ಸಮಯ ಮತ್ತು ಸಂಕೀರ್ಣತೆಯನ್ನು ಕಡಿಮೆ ಮಾಡುತ್ತದೆ.

**AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗೆ**: MCP ಡೇಟಾ ಮೂಲಗಳು, ಉಪಕರಣಗಳು ಮತ್ತು ಅಪ್ಲಿಕೇಶನ್‌ಗಳ ಪರಿಸರಕ್ಕೆ ಪ್ರವೇಶ ಒದಗಿಸಿ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಹೆಚ್ಚಿಸುತ್ತದೆ ಮತ್ತು ಅಂತಿಮ ಬಳಕೆದಾರ ಅನುಭವವನ್ನು ಸುಧಾರಿಸುತ್ತದೆ.

**ಅಂತಿಮ ಬಳಕೆದಾರರಿಗೆ**: MCP ನಿಮ್ಮ ಡೇಟಾಗೆ ಪ್ರವೇಶ ಹೊಂದುವ ಮತ್ತು ಅಗತ್ಯವಿದ್ದಾಗ ನಿಮ್ಮ ಪರವಾಗಿ ಕ್ರಮಗಳನ್ನು ಕೈಗೊಳ್ಳುವ ಹೆಚ್ಚು ಸಾಮರ್ಥ್ಯವಿರುವ AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳು ಅಥವಾ ಏಜೆಂಟ್‌ಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ.

## MCP ನಲ್ಲಿ ಸಣ್ಣ ಭಾಷಾ ಮಾದರಿಗಳು (SLMs)

ಸಣ್ಣ ಭಾಷಾ ಮಾದರಿಗಳು AI ನಿಯೋಜನೆಗೆ ಪರಿಣಾಮಕಾರಿಯಾದ ವಿಧಾನವನ್ನು ಪ್ರತಿನಿಧಿಸುತ್ತವೆ, ಹಲವಾರು ಪ್ರಯೋಜನಗಳನ್ನು ನೀಡುತ್ತವೆ:

### SLM ಗಳ ಪ್ರಯೋಜನಗಳು
- **ಸಂಪನ್ಮೂಲ ದಕ್ಷತೆ**: ಕಡಿಮೆ ಗಣನೀಯ ಅಗತ್ಯಗಳು
- **ವೇಗವಾದ ಪ್ರತಿಕ್ರಿಯೆ ಸಮಯಗಳು**: ನೈಜ-ಸಮಯ ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗೆ ಕಡಿಮೆ ವಿಳಂಬ  
- **ಖರ್ಚು ಪರಿಣಾಮಕಾರಿತ್ವ**: ಕನಿಷ್ಠ ಮೂಲಸೌಕರ್ಯ ಅಗತ್ಯಗಳು
- **ಗೌಪ್ಯತೆ**: ಡೇಟಾ ಪ್ರಸರಣವಿಲ್ಲದೆ ಸ್ಥಳೀಯವಾಗಿ ಚಾಲನೆ ಮಾಡಬಹುದು
- **ಕಸ್ಟಮೈಜೆಷನ್**: ನಿರ್ದಿಷ್ಟ ಕ್ಷೇತ್ರಗಳಿಗೆ ಸುಲಭವಾಗಿ ಸೂಕ್ತಗೊಳಿಸಬಹುದು

### MCP ಜೊತೆಗೆ SLM ಗಳ ಉತ್ತಮ ಕಾರ್ಯಕ್ಷಮತೆ

SLM ಗಳು MCP ಜೊತೆಗೆ ಜೋಡಿಸಿದಾಗ, ಮಾದರಿಯ ತರ್ಕ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಹೊರಗಿನ ಉಪಕರಣಗಳಿಂದ ಹೆಚ್ಚಿಸಲಾಗುತ್ತದೆ, ಕಡಿಮೆ ಪ್ಯಾರಾಮೀಟರ್ ಸಂಖ್ಯೆಯನ್ನು ಹೆಚ್ಚಿದ ಕಾರ್ಯಕ್ಷಮತೆಯಿಂದ ಪೂರೈಸುತ್ತದೆ.

## Python MCP SDK ಅವಲೋಕನ

Python MCP SDK MCP-ಸಕ್ರಿಯ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿರ್ಮಿಸಲು ಆಧಾರ ಒದಗಿಸುತ್ತದೆ. SDK ಒಳಗೊಂಡಿದೆ:

- **ಗ್ರಾಹಕ ಗ್ರಂಥಾಲಯಗಳು**: MCP ಸರ್ವರ್‌ಗಳಿಗೆ ಸಂಪರ್ಕಿಸಲು
- **ಸರ್ವರ್ ಫ್ರೇಮ್ವರ್ಕ್**: ಕಸ್ಟಮ್ MCP ಸರ್ವರ್‌ಗಳನ್ನು ರಚಿಸಲು
- **ಪ್ರೋಟೋಕಾಲ್ ಹ್ಯಾಂಡ್ಲರ್‌ಗಳು**: ಸಂವಹನ ನಿರ್ವಹಿಸಲು
- **ಉಪಕರಣ ಏಕೀಕರಣ**: ಹೊರಗಿನ ಕಾರ್ಯಗಳನ್ನು ನಿರ್ವಹಿಸಲು

## ಪ್ರಾಯೋಗಿಕ ಅನುಷ್ಠಾನ: Phi-4 MCP ಗ್ರಾಹಕ

Microsoft ನ Phi-4 ಮಿನಿ ಮಾದರಿಯನ್ನು MCP ಸಾಮರ್ಥ್ಯಗಳೊಂದಿಗೆ ಏಕೀಕರಿಸಿ ನೈಜ ಜಗತ್ತಿನ ಅನುಷ್ಠಾನವನ್ನು ಪರಿಶೀಲಿಸೋಣ.

### MCP ವಾಸ್ತುಶಿಲ್ಪ ಅವಲೋಕನ

MCP ಒಂದು **ಗ್ರಾಹಕ-ಸರ್ವರ್ ವಾಸ್ತುಶಿಲ್ಪ** ಅನುಸರಿಸುತ್ತದೆ, ಇಲ್ಲಿ MCP ಹೋಸ್ಟ್ (Claude Code ಅಥವಾ Claude Desktop ಮುಂತಾದ AI ಅಪ್ಲಿಕೇಶನ್) ಒಂದು ಅಥವಾ ಹೆಚ್ಚು MCP ಸರ್ವರ್‌ಗಳಿಗೆ ಸಂಪರ್ಕಗಳನ್ನು ಸ್ಥಾಪಿಸುತ್ತದೆ. MCP ಹೋಸ್ಟ್ ಪ್ರತಿ MCP ಸರ್ವರ್‌ಗೆ ಒಂದು MCP ಗ್ರಾಹಕವನ್ನು ರಚಿಸುವ ಮೂಲಕ ಇದನ್ನು ಸಾಧಿಸುತ್ತದೆ.

#### ಪ್ರಮುಖ ಭಾಗವಹಿಸುವವರು

- **MCP ಹೋಸ್ಟ್**: ಒಂದು ಅಥವಾ ಹೆಚ್ಚು MCP ಗ್ರಾಹಕಗಳನ್ನು ಸಂಯೋಜಿಸಿ ನಿರ್ವಹಿಸುವ AI ಅಪ್ಲಿಕೇಶನ್
- **MCP ಗ್ರಾಹಕ**: MCP ಸರ್ವರ್‌ಗೆ ಸಂಪರ್ಕವನ್ನು ಕಾಯ್ದುಕೊಳ್ಳುವ ಮತ್ತು MCP ಹೋಸ್ಟ್ ಬಳಕೆಗಾಗಿ ಸರ್ವರ್‌ನಿಂದ ಸಂದರ್ಭವನ್ನು ಪಡೆಯುವ ಘಟಕ
- **MCP ಸರ್ವರ್**: MCP ಗ್ರಾಹಕರಿಗೆ ಸಂದರ್ಭವನ್ನು ಒದಗಿಸುವ ಪ್ರೋಗ್ರಾಂ

#### ಎರಡು-ಮಟ್ಟದ ವಾಸ್ತುಶಿಲ್ಪ

MCP ಎರಡು ವಿಭಿನ್ನ ಪದರಗಳನ್ನು ಹೊಂದಿದೆ:

**ಡೇಟಾ ಪದರ**: ಗ್ರಾಹಕ-ಸರ್ವರ್ ಸಂವಹನಕ್ಕಾಗಿ JSON-RPC ಆಧಾರಿತ ಪ್ರೋಟೋಕಾಲ್ ಅನ್ನು ವ್ಯಾಖ್ಯಾನಿಸುತ್ತದೆ, ಇದರಲ್ಲಿ:
- ಜೀವನಚಕ್ರ ನಿರ್ವಹಣೆ (ಸಂಪರ್ಕ ಆರಂಭ, ಸಾಮರ್ಥ್ಯ ಸಂಧಾನ)
- ಮೂಲ ಪ್ರಿಮಿಟಿವ್‌ಗಳು (ಉಪಕರಣಗಳು, ಸಂಪನ್ಮೂಲಗಳು, ಪ್ರಾಂಪ್ಟ್‌ಗಳು)
- ಗ್ರಾಹಕ ವೈಶಿಷ್ಟ್ಯಗಳು (ಸ್ಯಾಂಪ್ಲಿಂಗ್, ಎಲಿಸಿಟೇಶನ್, ಲಾಗಿಂಗ್)
- ಉಪಯುಕ್ತ ವೈಶಿಷ್ಟ್ಯಗಳು (ಅಧಿಸೂಚನೆಗಳು, ಪ್ರಗತಿ ಟ್ರ್ಯಾಕಿಂಗ್)

**ಸಂವಹನ ಪದರ**: ಸಂವಹನ ವಿಧಾನಗಳು ಮತ್ತು ಚಾನೆಲ್‌ಗಳನ್ನು ವ್ಯಾಖ್ಯಾನಿಸುತ್ತದೆ:
- **STDIO ಸಂವಹನ**: ಸ್ಥಳೀಯ ಪ್ರಕ್ರಿಯೆಗಳಿಗಾಗಿ ಸ್ಟ್ಯಾಂಡರ್ಡ್ ಇನ್‌ಪುಟ್/ಔಟ್‌ಪುಟ್ ಸ್ಟ್ರೀಮ್‌ಗಳನ್ನು ಬಳಸುತ್ತದೆ (ಅತ್ಯುತ್ತಮ ಕಾರ್ಯಕ್ಷಮತೆ, ಜಾಲತಾಣ ಭಾರವಿಲ್ಲದೆ)
- **ಸ್ಟ್ರೀಮಬಲ್ HTTP ಸಂವಹನ**: ದೂರದ ಸರ್ವರ್‌ಗಳಿಗೆ HTTP POST ಮತ್ತು ಐಚ್ಛಿಕ ಸರ್ವರ್-ಸೆಂಟ್ ಇವೆಂಟ್ಸ್ ಬಳಸಿ (ಸಾಮಾನ್ಯ HTTP ಪ್ರಮಾಣೀಕರಣವನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ)

### MCP ಮೂಲ ಪ್ರಿಮಿಟಿವ್‌ಗಳು

MCP AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗೆ ಹಂಚಿಕೊಳ್ಳಬಹುದಾದ ಸಾಂದರ್ಭಿಕ ಮಾಹಿತಿಯ ಪ್ರಕಾರಗಳನ್ನು ಮತ್ತು ನಿರ್ವಹಿಸಬಹುದಾದ ಕ್ರಿಯೆಗಳ ವ್ಯಾಪ್ತಿಯನ್ನು ನಿರ್ದಿಷ್ಟಪಡಿಸುವ ಪ್ರಿಮಿಟಿವ್‌ಗಳನ್ನು ವ್ಯಾಖ್ಯಾನಿಸುತ್ತದೆ.

#### ಸರ್ವರ್ ಪ್ರಿಮಿಟಿವ್‌ಗಳು

MCP ಸರ್ವರ್‌ಗಳು ಬಹಿರಂಗಪಡಿಸಬಹುದಾದ ಮೂರು ಮೂಲ ಪ್ರಿಮಿಟಿವ್‌ಗಳನ್ನು ವ್ಯಾಖ್ಯಾನಿಸುತ್ತದೆ:

**ಉಪಕರಣಗಳು**: AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳು ಕಾರ್ಯಗಳನ್ನು ನಿರ್ವಹಿಸಲು ಕರೆಮಾಡಬಹುದಾದ ಕಾರ್ಯನಿರ್ವಹಣೀಯ ಫಂಕ್ಷನ್‌ಗಳು
- ಉದಾಹರಣೆಗಳು: ಫೈಲ್ ಕಾರ್ಯಾಚರಣೆಗಳು, API ಕರೆಗಳು, ಡೇಟಾಬೇಸ್ ಪ್ರಶ್ನೆಗಳು
- ವಿಧಾನಗಳು: `tools/list`, `tools/call`
- ಡೈನಾಮಿಕ್ ಅನ್ವೇಷಣೆ ಮತ್ತು ಕಾರ್ಯನಿರ್ವಹಣೆಯನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ

**ಸಂಪನ್ಮೂಲಗಳು**: AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗೆ ಸಾಂದರ್ಭಿಕ ಮಾಹಿತಿಯನ್ನು ಒದಗಿಸುವ ಡೇಟಾ ಮೂಲಗಳು
- ಉದಾಹರಣೆಗಳು: ಫೈಲ್ ವಿಷಯಗಳು, ಡೇಟಾಬೇಸ್ ದಾಖಲೆಗಳು, API ಪ್ರತಿಕ್ರಿಯೆಗಳು
- ವಿಧಾನಗಳು: `resources/list`, `resources/read`
- ರಚಿಸಲಾದ ಡೇಟಾಗೆ ಪ್ರವೇಶವನ್ನು ಸಕ್ರಿಯಗೊಳಿಸುತ್ತದೆ

**ಪ್ರಾಂಪ್ಟ್‌ಗಳು**: ಭಾಷಾ ಮಾದರಿಗಳೊಂದಿಗೆ ಸಂವಹನವನ್ನು ರಚಿಸಲು ಸಹಾಯ ಮಾಡುವ ಪುನಃಬಳಕೆ ಮಾಡಬಹುದಾದ ಟೆಂಪ್ಲೇಟುಗಳು
- ಉದಾಹರಣೆಗಳು: ಸಿಸ್ಟಮ್ ಪ್ರಾಂಪ್ಟ್‌ಗಳು, ಫ್ಯೂ-ಶಾಟ್ ಉದಾಹರಣೆಗಳು
- ವಿಧಾನಗಳು: `prompts/list`, `prompts/get`
- AI ಸಂವಹನ ಮಾದರಿಗಳನ್ನು ಮಾನಕಗೊಳಿಸುತ್ತದೆ

#### ಗ್ರಾಹಕ ಪ್ರಿಮಿಟಿವ್‌ಗಳು

MCP ಗ್ರಾಹಕರು ಹೆಚ್ಚು ಸಮೃದ್ಧ ಸಂವಹನಗಳನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಲು ಬಹಿರಂಗಪಡಿಸಬಹುದಾದ ಪ್ರಿಮಿಟಿವ್‌ಗಳನ್ನು ಸಹ ವ್ಯಾಖ್ಯಾನಿಸುತ್ತದೆ:

**ಸ್ಯಾಂಪ್ಲಿಂಗ್**: ಸರ್ವರ್‌ಗಳು ಗ್ರಾಹಕನ AI ಅಪ್ಲಿಕೇಶನ್‌ನಿಂದ ಭಾಷಾ ಮಾದರಿ ಪೂರ್ಣಗೊಳಿಸುವಿಕೆಗಳನ್ನು ವಿನಂತಿಸಲು ಅನುಮತಿಸುತ್ತದೆ
- ವಿಧಾನ: `sampling/complete`
- ಮಾದರಿ-ಸ್ವತಂತ್ರ ಸರ್ವರ್ ಅಭಿವೃದ್ಧಿಯನ್ನು ಸಕ್ರಿಯಗೊಳಿಸುತ್ತದೆ
- ಹೋಸ್ಟ್‌ನ ಭಾಷಾ ಮಾದರಿಗೆ ಪ್ರವೇಶ ಒದಗಿಸುತ್ತದೆ

**ಎಲಿಸಿಟೇಶನ್**: ಸರ್ವರ್‌ಗಳು ಬಳಕೆದಾರರಿಂದ ಹೆಚ್ಚುವರಿ ಮಾಹಿತಿಯನ್ನು ವಿನಂತಿಸಲು ಅನುಮತಿಸುತ್ತದೆ
- ವಿಧಾನ: `elicitation/request`
- ಬಳಕೆದಾರ ಸಂವಹನ ಮತ್ತು ದೃಢೀಕರಣವನ್ನು ಸಕ್ರಿಯಗೊಳಿಸುತ್ತದೆ
- ಡೈನಾಮಿಕ್ ಮಾಹಿತಿಯ ಸಂಗ್ರಹಣೆಯನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ

**ಲಾಗಿಂಗ್**: ಸರ್ವರ್‌ಗಳು ಗ್ರಾಹಕರಿಗೆ ಲಾಗ್ ಸಂದೇಶಗಳನ್ನು ಕಳುಹಿಸಲು ಅನುಮತಿಸುತ್ತದೆ
- ಡಿಬಗ್ ಮತ್ತು ಮೇಲ್ವಿಚಾರಣೆಗೆ ಬಳಸಲಾಗುತ್ತದೆ
- ಸರ್ವರ್ ಕಾರ್ಯಾಚರಣೆಗಳ ದೃಶ್ಯತೆಯನ್ನು ಒದಗಿಸುತ್ತದೆ

### MCP ಪ್ರೋಟೋಕಾಲ್ ಜೀವನಚಕ್ರ

#### ಪ್ರಾರಂಭ ಮತ್ತು ಸಾಮರ್ಥ್ಯ ಸಂಧಾನ

MCP ಒಂದು ಸ್ಥಿತಿಗತ ಪ್ರೋಟೋಕಾಲ್ ಆಗಿದ್ದು, ಜೀವನಚಕ್ರ ನಿರ್ವಹಣೆಯನ್ನು ಅಗತ್ಯವಿದೆ. ಪ್ರಾರಂಭ ಪ್ರಕ್ರಿಯೆಯು ಹಲವಾರು ಪ್ರಮುಖ ಉದ್ದೇಶಗಳನ್ನು ಸೇವಿಸುತ್ತದೆ:

1. **ಪ್ರೋಟೋಕಾಲ್ ಆವೃತ್ತಿ ಸಂಧಾನ**: ಗ್ರಾಹಕ ಮತ್ತು ಸರ್ವರ್ ಎರಡೂ ಹೊಂದಾಣಿಕೆಯ ಪ್ರೋಟೋಕಾಲ್ ಆವೃತ್ತಿಗಳನ್ನು ಬಳಸುವುದನ್ನು ಖಚಿತಪಡಿಸುತ್ತದೆ (ಉದಾ., "2025-06-18")
2. **ಸಾಮರ್ಥ್ಯ ಅನ್ವೇಷಣೆ**: ಪ್ರತಿ ಪಕ್ಷವು ಬೆಂಬಲಿಸುವ ವೈಶಿಷ್ಟ್ಯಗಳು ಮತ್ತು ಪ್ರಿಮಿಟಿವ್‌ಗಳನ್ನು ಘೋಷಿಸುತ್ತದೆ
3. **ಗುಣಪತ್ರ ವಿನಿಮಯ**: ಗುರುತಿಸುವಿಕೆ ಮತ್ತು ಆವೃತ್ತಿ ಮಾಹಿತಿಯನ್ನು ಒದಗಿಸುತ್ತದೆ

```python
# ಉದಾಹರಣೆಯ ಪ್ರಾರಂಭಿಕ ವಿನಂತಿ
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "initialize",
  "params": {
    "protocolVersion": "2025-06-18",
    "capabilities": {
      "elicitation": {},  # ಗ್ರಾಹಕ ಬಳಕೆದಾರ ಸಂವಹನವನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ
      "sampling": {}      # ಗ್ರಾಹಕ LLM ಪೂರ್ಣಗೊಳಿಸುವಿಕೆಗಳನ್ನು ಒದಗಿಸಬಹುದು
    },
    "clientInfo": {
      "name": "edge-ai-client",
      "version": "1.0.0"
    }
  }
}
```

#### ಉಪಕರಣ ಅನ್ವೇಷಣೆ ಮತ್ತು ಕಾರ್ಯನಿರ್ವಹಣೆ

ಪ್ರಾರಂಭದ ನಂತರ, ಗ್ರಾಹಕರು ಉಪಕರಣಗಳನ್ನು ಅನ್ವೇಷಿಸಿ ಕಾರ್ಯನಿರ್ವಹಿಸಬಹುದು:

```python
# ಲಭ್ಯವಿರುವ ಸಾಧನಗಳನ್ನು ಕಂಡುಹಿಡಿಯಿರಿ
tools_response = await session.list_tools()

# ಒಂದು ಸಾಧನವನ್ನು ಕಾರ್ಯಗತಗೊಳಿಸಿ
result = await session.call_tool(
    "weather_current",
    {
        "location": "San Francisco",
        "units": "imperial"
    }
)
```

#### ನೈಜ-ಸಮಯ ಅಧಿಸೂಚನೆಗಳು

MCP ಡೈನಾಮಿಕ್ ನವೀಕರಣಗಳಿಗೆ ನೈಜ-ಸಮಯ ಅಧಿಸೂಚನೆಗಳನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ:

```python
# ಸಾಧನಗಳು ಬದಲಾಗುವಾಗ ಸರ್ವರ್ ಸೂಚನೆ ಕಳುಹಿಸುತ್ತದೆ
{
  "jsonrpc": "2.0",
  "method": "notifications/tools/list_changed"
}

# ಕ್ಲೈಂಟ್ ಸಾಧನ ಪಟ್ಟಿ ನವೀಕರಿಸುವ ಮೂಲಕ ಪ್ರತಿಕ್ರಿಯಿಸುತ್ತದೆ
await session.list_tools()  # ನವೀಕರಿಸಿದ ಸಾಧನಗಳನ್ನು ಪಡೆಯಿರಿ
```

### ಹಂತ 4: ಪ್ರೋಗ್ರಾಮ್ಯಾಟಿಕ್ ಬಳಕೆ

```python
import asyncio
from ghmodel_mcp_demo import OllamaClient, Phi4MiniMCPClient

async def automated_interaction():
    # MCP ಸರ್ವರ್ ಪರಿಮಾಣಗಳನ್ನು ಸಂರಚಿಸಿ
    server_params = StdioServerParameters(
        command="npx",
        args=["@playwright/mcp@latest"],
        env=None,
    )
    
    # MCP ಕ್ಲೈಂಟ್ ಮತ್ತು ಪ್ರಕ್ರಿಯೆ ಸಾಧನಗಳನ್ನು ರಚಿಸಿ
    async with Phi4MiniMCPClient(server_params) as mcp_client:
        tools = await process_mcp_tools(mcp_client)
        llm_client = OllamaClient()
        
        # ಸಾಧನ ಸಾಮರ್ಥ್ಯಗಳೊಂದಿಗೆ ಪ್ರತಿಕ್ರಿಯೆಯನ್ನು ರಚಿಸಿ
        response, messages = await llm_client.generate_response(
            "Help me automate a web task",
            tools
        )
        return response

# ಸ್ವಯಂಚಾಲಿತ ಕಾರ್ಯಾಚರಣೆ ನಡೆಸಿ
result = asyncio.run(automated_interaction())
print(result)
```

- ಸರಳ ಸಂರಚನೆ

## ದೋಷ ನಿರ್ವಹಣೆ ಮತ್ತು ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು

### ಸಮಗ್ರ ದೋಷ ನಿರ್ವಹಣೆ

ಅನುಷ್ಠಾನವು ಕೆಳಗಿನ ದೋಷ ನಿರ್ವಹಣೆಯನ್ನು ಒಳಗೊಂಡಿದೆ:

**ಸಂಪರ್ಕ ದೋಷಗಳು:**
- MCP ಸರ್ವರ್ ವೈಫಲ್ಯಗಳು
- ಜಾಲತಾಣ ಸಮಯ ಮೀರಿಕೆಗಳು
- ಸಂಪರ್ಕ ಸಮಸ್ಯೆಗಳು

**ಉಪಕರಣ ಕಾರ್ಯನಿರ್ವಹಣೆ ದೋಷಗಳು:**
- ಉಪಕರಣಗಳ ಕೊರತೆ
- ಪ್ಯಾರಾಮೀಟರ್ ಪರಿಶೀಲನೆ
- ಕಾರ್ಯನಿರ್ವಹಣಾ ವೈಫಲ್ಯಗಳು

**ಪ್ರತಿಕ್ರಿಯೆ ಪ್ರಕ್ರಿಯೆ ದೋಷಗಳು:**
- JSON ವಿಶ್ಲೇಷಣಾ ಸಮಸ್ಯೆಗಳು
- ಸ್ವರೂಪ ಅಸಮಾನತೆಗಳು
- LLM ಪ್ರತಿಕ್ರಿಯೆ ಅನೋಮಲಿಗಳು

### ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು

1. **ಸಂಪನ್ಮೂಲ ನಿರ್ವಹಣೆ**: ಅಸಿಂಕ್ರೋನಸ್ ಸಂದರ್ಭ ನಿರ್ವಾಹಕರನ್ನು ಬಳಸಿ
2. **ದೋಷ ನಿರ್ವಹಣೆ**: ಸಮಗ್ರ ಪ್ರಯತ್ನ-ಹಿಡಿತ ಬ್ಲಾಕ್‌ಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಿ
3. **ಲಾಗಿಂಗ್**: ಸೂಕ್ತ ಲಾಗಿಂಗ್ ಮಟ್ಟಗಳನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿ
4. **ಭದ್ರತೆ**: ಇನ್‌ಪುಟ್‌ಗಳನ್ನು ಪರಿಶೀಲಿಸಿ ಮತ್ತು ಔಟ್‌ಪುಟ್‌ಗಳನ್ನು ಶುದ್ಧೀಕರಿಸಿ
5. **ಕಾರ್ಯಕ್ಷಮತೆ**: ಸಂಪರ್ಕ ಪೂಲಿಂಗ್ ಮತ್ತು ಕ್ಯಾಶಿಂಗ್ ಬಳಸಿ

## MCP ಪರಿಸರ ಮತ್ತು ಅಭಿವೃದ್ಧಿ

### MCP ಯೋಜನೆ ವ್ಯಾಪ್ತಿ

ಮಾದರಿ ಸಂದರ್ಭ ಪ್ರೋಟೋಕಾಲ್ ಪರಿಸರವು ಹಲವಾರು ಪ್ರಮುಖ ಘಟಕಗಳನ್ನು ಒಳಗೊಂಡಿದೆ:

- **[MCP ನಿರ್ದಿಷ್ಟತೆ](https://modelcontextprotocol.io/specification/latest)**: ಗ್ರಾಹಕ ಮತ್ತು ಸರ್ವರ್‌ಗಳ ಅನುಷ್ಠಾನ ಅಗತ್ಯಗಳನ್ನು ವಿವರಿಸುವ ಅಧಿಕೃತ ನಿರ್ದಿಷ್ಟತೆ
- **[MCP SDKಗಳು](https://modelcontextprotocol.io/docs/sdk)**: MCP ಅನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸುವ ವಿವಿಧ ಪ್ರೋಗ್ರಾಮಿಂಗ್ ಭಾಷೆಗಳ SDKಗಳು
- MCP ಅಭಿವೃದ್ಧಿ ಉಪಕರಣಗಳು: MCP ಸರ್ವರ್‌ಗಳು ಮತ್ತು ಗ್ರಾಹಕಗಳನ್ನು ಅಭಿವೃದ್ಧಿಪಡಿಸಲು ಉಪಕರಣಗಳು, [MCP ಇನ್ಸ್‌ಪೆಕ್ಟರ್](https://github.com/modelcontextprotocol/inspector) ಸೇರಿದಂತೆ
- **[MCP ರೆಫರೆನ್ಸ್ ಸರ್ವರ್ ಅನುಷ್ಠಾನಗಳು](https://github.com/modelcontextprotocol/servers)**: MCP ಸರ್ವರ್‌ಗಳ ರೆಫರೆನ್ಸ್ ಅನುಷ್ಠಾನಗಳು

### MCP ಅಭಿವೃದ್ಧಿ ಪ್ರಾರಂಭಿಸುವುದು

MCP ಬಳಸಿ ನಿರ್ಮಾಣ ಪ್ರಾರಂಭಿಸಲು:

**ಸರ್ವರ್‌ಗಳನ್ನು ನಿರ್ಮಿಸಿ**: ನಿಮ್ಮ ಡೇಟಾ ಮತ್ತು ಉಪಕರಣಗಳನ್ನು ಬಹಿರಂಗಪಡಿಸಲು [MCP ಸರ್ವರ್‌ಗಳನ್ನು ರಚಿಸಿ](https://modelcontextprotocol.io/docs/develop/build-server)

**ಗ್ರಾಹಕಗಳನ್ನು ನಿರ್ಮಿಸಿ**: MCP ಸರ್ವರ್‌ಗಳಿಗೆ ಸಂಪರ್ಕ ಹೊಂದುವ [ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಅಭಿವೃದ್ಧಿಪಡಿಸಿ](https://modelcontextprotocol.io/docs/develop/build-client)

**ಸಿದ್ಧಾಂತಗಳನ್ನು ಕಲಿಯಿರಿ**: MCP ನ ಮೂಲ ಸಿದ್ಧಾಂತಗಳು ಮತ್ತು ವಾಸ್ತುಶಿಲ್ಪವನ್ನು [ಅರ್ಥಮಾಡಿಕೊಳ್ಳಿ](https://modelcontextprotocol.io/docs/learn/architecture)

## ಸಮಾರೋಪ

MCP ಜೊತೆಗೆ ಏಕೀಕೃತ SLM ಗಳು AI ಅಪ್ಲಿಕೇಶನ್ ಅಭಿವೃದ್ಧಿಯಲ್ಲಿ ಪರಿಕಲ್ಪನಾ ಬದಲಾವಣೆಯನ್ನು ಪ್ರತಿನಿಧಿಸುತ್ತವೆ. ಸಣ್ಣ ಮಾದರಿಗಳ ದಕ್ಷತೆಯನ್ನು ಹೊರಗಿನ ಉಪಕರಣಗಳ ಶಕ್ತಿಯೊಂದಿಗೆ ಸಂಯೋಜಿಸುವ ಮೂಲಕ, ವಿಕಸಕರು ಸಂಪನ್ಮೂಲ-ದಕ್ಷ ಮತ್ತು ಅತ್ಯಂತ ಸಾಮರ್ಥ್ಯವಿರುವ ಬುದ್ಧಿವಂತ ವ್ಯವಸ್ಥೆಗಳನ್ನು ರಚಿಸಬಹುದು.

ಮಾದರಿ ಸಂದರ್ಭ ಪ್ರೋಟೋಕಾಲ್ AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಹೊರಗಿನ ವ್ಯವಸ್ಥೆಗಳಿಗೆ ಸಂಪರ್ಕಿಸುವ ಮಾನಕ ವಿಧಾನವನ್ನು ಒದಗಿಸುತ್ತದೆ, USB-C ಎಲೆಕ್ಟ್ರಾನಿಕ್ ಸಾಧನಗಳಿಗೆ ವಿಶ್ವವ್ಯಾಪಿ ಸಂಪರ್ಕ ಮಾನಕವನ್ನು ಒದಗಿಸುವಂತೆ. ಈ ಮಾನಕೀಕರಣವು ಸಾಧ್ಯಮಾಡುತ್ತದೆ:

- **ಸರಳ ಏಕೀಕರಣ**: AI ಮಾದರಿಗಳನ್ನು ವಿಭಿನ್ನ ಡೇಟಾ ಮೂಲಗಳು ಮತ್ತು ಉಪಕರಣಗಳಿಗೆ ಸಂಪರ್ಕಿಸುವುದು
- **ಪರಿಸರ ವೃದ್ಧಿ**: ಒಮ್ಮೆ ನಿರ್ಮಿಸಿ, ಹಲವಾರು AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳಲ್ಲಿ ಬಳಸಿ
- **ಸಾಮರ್ಥ್ಯ ವೃದ್ಧಿ**: SLM ಗಳನ್ನು ಹೊರಗಿನ ಕಾರ್ಯಕ್ಷಮತೆಯಿಂದ ಹೆಚ್ಚಿಸುವುದು
- **ನೈಜ-ಸಮಯ ನವೀಕರಣಗಳು**: ಡೈನಾಮಿಕ್, ಪ್ರತಿಕ್ರಿಯಾಶೀಲ AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಬೆಂಬಲಿಸುವುದು

ಪ್ರಮುಖ ಅಂಶಗಳು:
- MCP AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳು ಮತ್ತು ಹೊರಗಿನ ವ್ಯವಸ್ಥೆಗಳನ್ನು ಸೇರುವ ಮುಕ್ತ ಮಾನಕವಾಗಿದೆ
- ಪ್ರೋಟೋಕಾಲ್ ಉಪಕರಣಗಳು, ಸಂಪನ್ಮೂಲಗಳು ಮತ್ತು ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ಮೂಲ ಪ್ರಿಮಿಟಿವ್‌ಗಳಾಗಿ ಬೆಂಬಲಿಸುತ್ತದೆ
- ನೈಜ-ಸಮಯ ಅಧಿಸೂಚನೆಗಳು ಡೈನಾಮಿಕ್, ಪ್ರತಿಕ್ರಿಯಾಶೀಲ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಸಕ್ರಿಯಗೊಳಿಸುತ್ತವೆ
- ಸರಿಯಾದ ಜೀವನಚಕ್ರ ನಿರ್ವಹಣೆ ಮತ್ತು ದೋಷ ನಿರ್ವಹಣೆ ಉತ್ಪಾದನಾ ಬಳಕೆಗೆ ಅಗತ್ಯ
- ಪರಿಸರವು ಸಮಗ್ರ SDKಗಳು ಮತ್ತು ಅಭಿವೃದ್ಧಿ ಉಪಕರಣಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ

- **[AI ಏಜೆಂಟ್ ಅಭಿವೃದ್ಧಿ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು](https://arxiv.org/abs/2309.02427)** - ಏಜೆಂಟ್ ವಾಸ್ತುಶಿಲ್ಪಗಳ ಮೇಲೆ ಸಂಶೋಧನೆ

ಈ ವಿಭಾಗವು ನಿಮ್ಮ ಸ್ವಂತ SLM-ಚಾಲಿತ MCP ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿರ್ಮಿಸಲು ಆಧಾರವನ್ನು ಒದಗಿಸುತ್ತದೆ, ಸ್ವಯಂಚಾಲನೆ, ಡೇಟಾ ಪ್ರಕ್ರಿಯೆ ಮತ್ತು ಬುದ್ಧಿವಂತಿಕೆಯ ವ್ಯವಸ್ಥೆ ಸಂಯೋಜನೆಗೆ ಅವಕಾಶಗಳನ್ನು ತೆರೆಯುತ್ತದೆ.
