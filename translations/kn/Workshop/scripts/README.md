# ಕಾರ್ಯಾಗಾರ ಸ್ಕ್ರಿಪ್ಟ್‌ಗಳು

ಈ ಡೈರೆಕ್ಟರಿ ಕಾರ್ಯಾಗಾರ ಸಾಮಗ್ರಿಗಳ ಗುಣಮಟ್ಟ ಮತ್ತು ಸತತತೆಯನ್ನು ನಿರ್ವಹಿಸಲು ಬಳಸುವ ಸ್ವಯಂಚಾಲಿತ ಮತ್ತು ಬೆಂಬಲ ಸ್ಕ್ರಿಪ್ಟ್‌ಗಳನ್ನು ಹೊಂದಿದೆ.

## ವಿಷಯಗಳು

| ಫೈಲ್ | ಉದ್ದೇಶ |
|------|---------|
| `lint_markdown_cli.py` | ಡಿಪ್ರಿಕೇಟೆಡ್ ಫೌಂಡ್ರಿ ಲೋಕಲ್ CLI ಕಮಾಂಡ್ ಮಾದರಿಗಳನ್ನು ತಡೆಯಲು ಮಾರ್ಕ್‌ಡೌನ್ ಕೋಡ್ ಫೆನ್ಸುಗಳನ್ನು ಲಿಂಟ್ ಮಾಡುತ್ತದೆ. |
| `export_benchmark_markdown.py` | ಬಹು-ಮಾದರಿ ವಿಳಂಬ ಬೆಂಚ್‌ಮಾರ್ಕ್ ನಡೆಸಿ ಮಾರ್ಕ್‌ಡೌನ್ + JSON ವರದಿಗಳನ್ನು ಹೊರತರುತ್ತದೆ. |

## 1. ಮಾರ್ಕ್‌ಡೌನ್ CLI ಮಾದರಿ ಲಿಂಟರ್

`lint_markdown_cli.py` ಎಲ್ಲಾ ಅನುವಾದವಲ್ಲದ `.md` ಫೈಲ್‌ಗಳಲ್ಲಿ ನಿಷೇಧಿತ ಫೌಂಡ್ರಿ ಲೋಕಲ್ CLI ಮಾದರಿಗಳನ್ನು **ಕೋಡ್ ಫೆನ್ಸುಗಳ ಒಳಗೆ** (``` ... ```) ಸ್ಕ್ಯಾನ್ ಮಾಡುತ್ತದೆ. ಮಾಹಿತಿ ಪ್ರಬಂಧದಲ್ಲಿ ಇತಿಹಾಸಾತ್ಮಕ ಸನ್ನಿವೇಶಕ್ಕಾಗಿ ಡಿಪ್ರಿಕೇಟೆಡ್ ಕಮಾಂಡ್‌ಗಳನ್ನು ಉಲ್ಲೇಖಿಸಬಹುದು.

### ಡಿಪ್ರಿಕೇಟೆಡ್ ಮಾದರಿಗಳು (ಕೋಡ್ ಫೆನ್ಸುಗಳ ಒಳಗೆ ತಡೆಯಲ್ಪಟ್ಟಿವೆ)

ಲಿಂಟರ್ ಡಿಪ್ರಿಕೇಟೆಡ್ CLI ಮಾದರಿಗಳನ್ನು ತಡೆಯುತ್ತದೆ. ಬದಲಿಗೆ ಆಧುನಿಕ ಪರ್ಯಾಯಗಳನ್ನು ಬಳಸಿ.

### ಅಗತ್ಯ ಬದಲಾವಣೆಗಳು
| ಡಿಪ್ರಿಕೇಟೆಡ್ | ಬದಲಿಗೆ ಬಳಸಿ |
|------------|-------------|
| `foundry model chat <a> "..."` | `foundry model run <a> --prompt "..."` |
| `foundry model list --running` | `foundry model list` |
| `foundry model list --cached` | `foundry cache list` |
| `foundry model stats` | ಬೆಂಚ್‌ಮಾರ್ಕ್ ಸ್ಕ್ರಿಪ್ಟ್ + ಸಿಸ್ಟಮ್ ಉಪಕರಣಗಳು (`Task Manager`, `nvidia-smi`) |
| `foundry model benchmark` | `samples/session03/benchmark_oss_models.py` |
| `foundry model list --available` | `foundry model list` |

### ನಿರ್ಗಮನ ಕೋಡ್‌ಗಳು
| ಕೋಡ್ | ಅರ್ಥ |
|------|---------|
| 0 | ಯಾವುದೇ ಉಲ್ಲಂಘನೆ ಕಂಡುಬಂದಿಲ್ಲ |
| 1 | ಒಂದು ಅಥವಾ ಹೆಚ್ಚು ಡಿಪ್ರಿಕೇಟೆಡ್ ಮಾದರಿಗಳು ಕಂಡುಬಂದಿವೆ |

### ಸ್ಥಳೀಯವಾಗಿ ಚಾಲನೆ
ರಿಪೊರಿಟರಿ ರೂಟ್‌ನಿಂದ (ಶಿಫಾರಸು ಮಾಡಲಾಗಿದೆ):

ವಿಂಡೋಸ್ (ಪವರ್‌ಶೆಲ್):
```powershell
python Workshop\scripts\lint_markdown_cli.py --verbose
```

ಮ್ಯಾಕ್‌ಒಎಸ್ / ಲಿನಕ್ಸ:
```bash
python Workshop/scripts/lint_markdown_cli.py --verbose
```

### ಪೂರ್ವ-ಕಮಿಟ್ ಹೂಕ್ (ಐಚ್ಛಿಕ)
```bash
echo "python Workshop/scripts/lint_markdown_cli.py" > .git/hooks/pre-commit
chmod +x .git/hooks/pre-commit
```
ಇದು ಡಿಪ್ರಿಕೇಟೆಡ್ ಮಾದರಿಗಳನ್ನು ಪರಿಚಯಿಸುವ ಕಮಿಟ್‌ಗಳನ್ನು ತಡೆಯುತ್ತದೆ.

### CI ಏಕೀಕರಣ
GitHub Action ವರ್ಕ್‌ಫ್ಲೋ (`.github/workflows/markdown-cli-lint.yml`) ಪ್ರತಿ ಪುಷ್ ಮತ್ತು ಪುಲ್ ರಿಕ್ವೆಸ್ಟ್‌ಗೆ `main` ಮತ್ತು `Reactor` ಶಾಖೆಗಳಲ್ಲಿ ಲಿಂಟರ್ ಅನ್ನು ಚಾಲನೆ ಮಾಡುತ್ತದೆ. ವಿಫಲವಾದ ಕೆಲಸಗಳನ್ನು ಮರ್ಜ್ ಮಾಡುವ ಮೊದಲು ಸರಿಪಡಿಸಬೇಕು.

### ಹೊಸ ಡಿಪ್ರಿಕೇಟೆಡ್ ಮಾದರಿಗಳನ್ನು ಸೇರಿಸುವುದು
1. `lint_markdown_cli.py` ತೆರೆಯಿರಿ.
2. `DEPRECATED` ಪಟ್ಟಿಗೆ `(regex, suggestion)` ಟ್ಯೂಪಲ್ ಅನ್ನು ಸೇರಿಸಿ. ರಾ ಸ್ಟ್ರಿಂಗ್ ಬಳಸಿ ಮತ್ತು ಅಗತ್ಯವಿದ್ದಲ್ಲಿ `\b` ಪದ ಸೀಮೆಗಳನ್ನೂ ಸೇರಿಸಿ.
3. ಲಿಂಟರ್ ಅನ್ನು ಸ್ಥಳೀಯವಾಗಿ ಚಾಲನೆ ಮಾಡಿ ಪತ್ತೆಮಾಡುವಿಕೆಯನ್ನು ಪರಿಶೀಲಿಸಿ.
4. ಕಮಿಟ್ ಮಾಡಿ ಪುಷ್ ಮಾಡಿ; CI ಹೊಸ ನಿಯಮವನ್ನು ಜಾರಿಗೆ ತರುತ್ತದೆ.

ಉದಾಹರಣೆಯ ಸೇರ್ಪಡೆ:
```python
DEPRECATED.append((r"\\bfoundry\\s+experimental\\s+foo\\b", "Remove experimental foo usage"))
```

### ವಿವರಣಾತ್ಮಕ ಉಲ್ಲೇಖಗಳನ್ನು ಅನುಮತಿಸುವುದು
ಕೆವಲ ಕೋಡ್ ಫೆನ್ಸುಗಳು ಜಾರಿಗೆ ಬರುವುದರಿಂದ, ನೀವು ಡಿಪ್ರಿಕೇಟೆಡ್ ಕಮಾಂಡ್‌ಗಳನ್ನು ಕಥನ ಪಠ್ಯದಲ್ಲಿ ಸುರಕ್ಷಿತವಾಗಿ ವಿವರಿಸಬಹುದು. ನೀವು ಅವುಗಳನ್ನು ವ್ಯತ್ಯಾಸಕ್ಕಾಗಿ ಫೆನ್ಸಿನೊಳಗೆ ತೋರಿಸಬೇಕಾದರೆ, ತ್ರಿಪುಟ ಬ್ಯಾಕ್ಟಿಕ್ ಇಲ್ಲದ ಫೆನ್ಸು ಬ್ಲಾಕ್ (ಉದಾ: ಇಂದೆಂಟ್ ಅಥವಾ ಉಲ್ಲೇಖ) ಸೇರಿಸಿ ಅಥವಾ ಪ್ಸ್ಯೂಡೋ ರೂಪಕ್ಕೆ ಮರುಬರೆಯಿರಿ.

### ನಿರ್ದಿಷ್ಟ ಫೈಲ್‌ಗಳನ್ನು ಬಿಟ್ಟುಹೋಗುವುದು (ಅಧಿಕೃತ)
ಅಗತ್ಯವಿದ್ದರೆ, ಪುರಾತನ ಉದಾಹರಣೆಗಳನ್ನು ರಿಪೊ ಹೊರಗಿನ ಬೇರೆ ಫೈಲ್‌ನಲ್ಲಿ ಇಟ್ಟುಕೊಳ್ಳಿ ಅಥವಾ ಡ್ರಾಫ್ಟ್ ಮಾಡುವಾಗ ವಿಭಿನ್ನ ವಿಸ್ತರಣೆ ಹೊಂದಿಸಿ. ಅನುವಾದಿತ ಪ್ರತಿಗಳಿಗಾಗಿ ಉದ್ದೇಶಿತ ಬಿಟ್ಟುಹೋಗುವಿಕೆ ಸ್ವಯಂಚಾಲಿತ (ಪಥಗಳಲ್ಲಿ `translations` ಇದ್ದರೆ).

### ಸಮಸ್ಯೆ ಪರಿಹಾರ
| ಸಮಸ್ಯೆ | ಕಾರಣ | ಪರಿಹಾರ |
|-------|-------|-----------|
| ನೀವು ನವೀಕರಿಸಿದ ಸಾಲನ್ನು ಲಿಂಟರ್ ಗುರುತಿಸುತ್ತದೆ | Regex ತುಂಬಾ ವ್ಯಾಪಕವಾಗಿದೆ | ಹೆಚ್ಚುವರಿ ಪದ ಸೀಮೆ (`\b`) ಅಥವಾ ಆಂಕರ್‌ಗಳೊಂದಿಗೆ ಮಾದರಿಯನ್ನು ಸೀಮಿತಗೊಳಿಸಿ |
| CI ವಿಫಲವಾಗುತ್ತದೆ ಆದರೆ ಸ್ಥಳೀಯವಾಗಿ ಸರಿ | ವಿಭಿನ್ನ Python ಆವೃತ್ತಿ ಅಥವಾ ಅಕಮಿಟ್ ಮಾಡಿದ ಬದಲಾವಣೆಗಳು | ಸ್ಥಳೀಯವಾಗಿ ಮರುಚಾಲನೆ ಮಾಡಿ, ಸ್ವಚ್ಛ ಕೆಲಸದ ಮರವನ್ನು ಖಚಿತಪಡಿಸಿ, ವರ್ಕ್‌ಫ್ಲೋ Python ಆವೃತ್ತಿ (3.11) ಪರಿಶೀಲಿಸಿ |
| ತಾತ್ಕಾಲಿಕವಾಗಿ ಬೈಸ್ಪಾಸ್ ಮಾಡಬೇಕಿದೆ | ತುರ್ತು ತಿದ್ದುಪಡಿ | ತಕ್ಷಣ ತಿದ್ದುಪಡಿ ಮಾಡಿ; ತಾತ್ಕಾಲಿಕ ಶಾಖೆ ಮತ್ತು ನಂತರದ PR ಬಳಸಿ (ಬೈಸ್ಪಾಸ್ ಸ್ವಿಚ್‌ಗಳನ್ನು ಸೇರಿಸುವುದನ್ನು ತಪ್ಪಿಸಿ) |

### ತರ್ಕ
*ಪ್ರಸ್ತುತ* ಸ್ಥಿರ CLI ಮೇಲ್ಮೈಗೆ ದಾಖಲೆಗಳನ್ನು ಹೊಂದಿಸುವುದು ಕಾರ್ಯಾಗಾರದ ಅಡಚಣೆಯನ್ನು ತಡೆಯುತ್ತದೆ, ಕಲಿಯುವವರ ಗೊಂದಲವನ್ನು ತಪ್ಪಿಸುತ್ತದೆ ಮತ್ತು CLI ಉಪಕಮಾಂಡ್‌ಗಳ ಬದಲಿಗೆ ನಿರ್ವಹಿಸಲಾದ Python ಸ್ಕ್ರಿಪ್ಟ್‌ಗಳ ಮೂಲಕ ಕಾರ್ಯಕ್ಷಮತೆ ಅಳೆಯುವಿಕೆಯನ್ನು ಕೇಂದ್ರಗೊಳಿಸುತ್ತದೆ.

---
ಕಾರ್ಯಾಗಾರ ಗುಣಮಟ್ಟ ಉಪಕರಣ ಸರಣಿಯ ಭಾಗವಾಗಿ ನಿರ್ವಹಿಸಲಾಗಿದೆ. ಸುಧಾರಣೆಗಳಿಗೆ (ಉದಾ: ಸ್ವಯಂ-ತಿದ್ದುಪಡಿ ಸಲಹೆಗಳು ಅಥವಾ HTML ವರದಿ ತಯಾರಿಕೆ) ಸಮಸ್ಯೆ ತೆರೆಯಿರಿ ಅಥವಾ PR ಸಲ್ಲಿಸಿ.

## 2. ಮಾದರಿ ಮಾನ್ಯತೆ ಸ್ಕ್ರಿಪ್ಟ್

`validate_samples.py` ಎಲ್ಲಾ Python ಮಾದರಿ ಫೈಲ್‌ಗಳ ವಾಕ್ಯರಚನೆ, ಆಮದುಗಳು ಮತ್ತು ಉತ್ತಮ ಅಭ್ಯಾಸಗಳ ಅನುಕೂಲತೆಯನ್ನು ಪರಿಶೀಲಿಸುತ್ತದೆ.

### ಬಳಕೆ
```bash
# ಎಲ್ಲಾ ಮಾದರಿಗಳನ್ನು ಪರಿಶೀಲಿಸಿ
python scripts/validate_samples.py

# ನಿರ್ದಿಷ್ಟ ಸೆಷನ್ ಅನ್ನು ಪರಿಶೀಲಿಸಿ
python scripts/validate_samples.py --session 01

# ವಿವರವಾದ ಮೋಡ್ (ಉತ್ತಮ ಅಭ್ಯಾಸ ಎಚ್ಚರಿಕೆಗಳನ್ನು ಒಳಗೊಂಡಿದೆ)
python scripts/validate_samples.py --verbose

# ಸಾರಾಂಶ ಮಾತ್ರ
python scripts/validate_samples.py --summary
```

### ಇದು ಪರಿಶೀಲಿಸುವುದು
- ✅ Python ವಾಕ್ಯರಚನೆ ಮಾನ್ಯತೆ
- ✅ ಅಗತ್ಯ ಆಮದುಗಳು ಇದ್ದವು
- ✅ ದೋಷ ನಿರ್ವಹಣೆ ಅನುಷ್ಠಾನ (ವಿವರಾತ್ಮಕ ಮೋಡ್)
- ✅ ಪ್ರಕಾರ ಸೂಚನೆಗಳ ಬಳಕೆ (ವಿವರಾತ್ಮಕ ಮೋಡ್)
- ✅ ಕಾರ್ಯ ಡಾಕ್ಸ್ಟ್ರಿಂಗ್‌ಗಳು (ವಿವರಾತ್ಮಕ ಮೋಡ್)
- ✅ SDK ಉಲ್ಲೇಖ ಲಿಂಕ್‌ಗಳು (ವಿವರಾತ್ಮಕ ಮೋಡ್)

### ಪರಿಸರ ಚರಗಳು
- `SKIP_IMPORT_CHECK=1` - ಆಮದು ಪರಿಶೀಲನೆಯನ್ನು ಬಿಟ್ಟುಹೋಗಿ
- `SKIP_SYNTAX_CHECK=1` - ವಾಕ್ಯರಚನೆ ಪರಿಶೀಲನೆಯನ್ನು ಬಿಟ್ಟುಹೋಗಿ

### ನಿರ್ಗಮನ ಕೋಡ್‌ಗಳು
- `0` - ಎಲ್ಲಾ ಮಾದರಿಗಳು ಮಾನ್ಯತೆ ಪಾಸಾದವು
- `1` - ಒಂದು ಅಥವಾ ಹೆಚ್ಚು ಮಾದರಿಗಳು ವಿಫಲವಾದವು

## 3. ಮಾದರಿ ಪರೀಕ್ಷಾ ರನ್ನರ್

`test_samples.py` ಎಲ್ಲಾ ಮಾದರಿಗಳ ಮೇಲೆ ಸ್ಮೋಕ್ ಪರೀಕ್ಷೆಗಳನ್ನು ನಡೆಸಿ ಅವು ದೋಷರಹಿತವಾಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತವೆ ಎಂದು ಪರಿಶೀಲಿಸುತ್ತದೆ.

### ಬಳಕೆ
```bash
# ಎಲ್ಲಾ ಮಾದರಿಗಳನ್ನು ಪರೀಕ್ಷಿಸಿ
python scripts/test_samples.py

# ನಿರ್ದಿಷ್ಟ ಸೆಷನ್ ಅನ್ನು ಪರೀಕ್ಷಿಸಿ
python scripts/test_samples.py --session 01

# ವೇಗದ ಮೋಡ್ (ಕಡಿಮೆ ಸಮಯ ಮೀರಿಕೆಗಳು)
python scripts/test_samples.py --quick

# ವಿವರವಾದ ಮೋಡ್ (ಔಟ್‌ಪುಟ್ ತೋರಿಸಿ)
python scripts/test_samples.py --verbose
```

### ಪೂರ್ವಾಪೇಕ್ಷೆಗಳು
- ಫೌಂಡ್ರಿ ಲೋಕಲ್ ಸೇವೆ ಚಾಲನೆಯಲ್ಲಿದೆ: `foundry service start`
- ಮಾದರಿಗಳು ಲೋಡ್ ಆಗಿವೆ: `foundry model run phi-4-mini`
- ಅವಲಂಬನೆಗಳು ಸ್ಥಾಪಿಸಲಾಗಿದೆ: `pip install -r requirements.txt`

### ಇದು ಪರೀಕ್ಷಿಸುವುದು
- ✅ ಮಾದರಿ ರನ್‌ಟೈಮ್ ದೋಷಗಳಿಲ್ಲದೆ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ
- ✅ ಅಗತ್ಯ ಔಟ್‌ಪುಟ್ ಉತ್ಪಾದಿಸಲಾಗಿದೆ
- ✅ ವಿಫಲವಾದಾಗ ಸರಿಯಾದ ದೋಷ ನಿರ್ವಹಣೆ
- ✅ ಕಾರ್ಯಕ್ಷಮತೆ (ನಿರ್ವಹಣಾ ಸಮಯ)

### ಪರಿಸರ ಚರಗಳು
- `FOUNDRY_LOCAL_ALIAS=phi-4-mini` - ಪರೀಕ್ಷೆಗೆ ಬಳಸುವ ಮಾದರಿ
- `TEST_TIMEOUT=30` - ಪ್ರತಿ ಮಾದರಿಗಾಗಿ ಟೈಮೌಟ್ ಸೆಕೆಂಡುಗಳಲ್ಲಿ

### ನಿರೀಕ್ಷಿತ ವಿಫಲತೆಗಳು
ಕೆಲವು ಪರೀಕ್ಷೆಗಳು ಐಚ್ಛಿಕ ಅವಲಂಬನೆಗಳು ಸ್ಥಾಪಿತವಾಗದಿದ್ದರೆ ವಿಫಲವಾಗಬಹುದು (ಉದಾ: `ragas`, `sentence-transformers`). ಇವುಗಳನ್ನು ಸ್ಥಾಪಿಸಲು:
```bash
pip install sentence-transformers ragas datasets
```

### ನಿರ್ಗಮನ ಕೋಡ್‌ಗಳು
- `0` - ಎಲ್ಲಾ ಪರೀಕ್ಷೆಗಳು ಪಾಸಾದವು
- `1` - ಒಂದು ಅಥವಾ ಹೆಚ್ಚು ಪರೀಕ್ಷೆಗಳು ವಿಫಲವಾದವು

## 4. ಬೆಂಚ್‌ಮಾರ್ಕ್ ಮಾರ್ಕ್‌ಡೌನ್ ರಫ್ತು

ಸ್ಕ್ರಿಪ್ಟ್: `export_benchmark_markdown.py`

ಮಾದರಿಗಳ ಸಮೂಹಕ್ಕೆ ಪುನರಾವರ್ತನೀಯ ಕಾರ್ಯಕ್ಷಮತೆ ಟೇಬಲ್ ರಚಿಸುತ್ತದೆ.

### ಬಳಕೆ
```powershell
python Workshop\scripts\export_benchmark_markdown.py --models "qwen2.5-0.5b" --prompt "Explain retrieval augmented generation briefly." --rounds 3 --output benchmark_report.md
```

### ಔಟ್‌ಪುಟ್‌ಗಳು
| ಫೈಲ್ | ವಿವರಣೆ |
|------|-------------|
| `benchmark_report.md` | ಮಾರ್ಕ್‌ಡೌನ್ ಟೇಬಲ್ (ಸರಾಸರಿ, p95, ಟೋಕನ್ಸ್/ಸೆಕೆಂಡ್, ಐಚ್ಛಿಕ ಮೊದಲ ಟೋಕನ್) |
| `benchmark_report.json` | ವ್ಯತ್ಯಾಸ ಮತ್ತು ಇತಿಹಾಸಕ್ಕಾಗಿ ಕಚ್ಚಾ ಮೆಟ್ರಿಕ್ ಸರಣಿ |

### ಆಯ್ಕೆಗಳು
| ಫ್ಲ್ಯಾಗ್ | ವಿವರಣೆ | ಡೀಫಾಲ್ಟ್ |
|------|-------------|---------|
| `--models` | ಕಾಮಾ ವಿಭಜಿತ ಮಾದರಿ ಅಲಿಯಾಸ್‌ಗಳು | (ಅಗತ್ಯ) |

| `--fail-on-empty` | ಎಲ್ಲಾ ಬೆಂಚ್‌ಮಾರ್ಕ್ ವಿಫಲವಾದರೆ ನಾನ್-ಝೀರೋ ನಿರ್ಗಮನ | ಆಫ್ |
