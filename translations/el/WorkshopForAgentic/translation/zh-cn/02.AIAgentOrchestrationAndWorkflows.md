# Πράξη 2: Δημιουργήστε την ομάδα παραγωγής του podcast σας 🎬

![lab2](../../../../../translated_images/lab2.48444b9d549e7863.el.png)

## Ενταση του σεναρίου

Ο Alex (ο βοηθός AI σας από την Πράξη 1) είναι υπέροχος, αλλά ένας μόνο πράκτορας δεν μπορεί να τρέξει ολόκληρο στούντιο podcast. Χρειάζεστε μια *ομάδα*:
- 🔍 **Πράκτορας έρευνας**: ψάχνει στο διαδίκτυο για φρέσκες πληροφορίες
- ✍️ **Πράκτορας σεναρίου**: μετατρέπει την έρευνα σε συναρπαστικό διάλογο
- 👤 **Εσείς (επιμελητής)**: εγκρίνετε ή ζητάτε επανεγγραφή του σεναρίου

Καλώς ήρθατε στο **AI Ορχηστρωτήριο Πρακτόρων** — είστε ο σκηνοθέτης της δικής σας ομάδας AI. Σκεφτείτε το σαν τους Εκδικητές, αλλά για παραγωγή podcast.

## Τι είναι η ορχήστρωση πρακτόρων; (Απλή έκδοση)

Φανταστείτε ότι διαχειρίζεστε ένα εστιατόριο. Δεν κάνετε όλη τη δουλειά μόνος σας, σωστά; Έχετε:
- 🍳 Σεφ που μαγειρεύει
- 👨‍🍳 Βοηθό σεφ που προετοιμάζει
- 👩‍🍳 Σερβιτόρους που σερβίρουν

Η ορχήστρωση πρακτόρων είναι η ίδια ιδέα, αλλά με AI. Κάθε πράκτορας έχει την ειδικότητά του και εσείς τα συγχρονίζετε για ένα μεγαλύτερο στόχο. Κανένας πράκτορας δεν υπερφορτώνεται, και η δουλειά γίνεται πιο γρήγορα.

### Αναλογία μπάντας 🎸

Οι πράκτορες AI σας είναι σαν μια μπάντα:
- **Τραγουδιστής**: ο κύριος πράκτορας για τις εργασίες που έρχονται σε επαφή με τον πελάτη
- **Ντράμερ**: κρατά το ρυθμό, χειρίζεται το παρασκήνιο
- **Μπασίστας**: υποστηρίζει όλους, παίρνει δεδομένα
- **Εσείς (διευθυντής της μπάντας)**: συντονίζετε τα πάντα!

Χωρίς συντονισμό; Μόνο θόρυβος. Με ορχήστρωση; Υπέροχη μουσική.

### Γιατί είναι σημαντικό

Ένας μόνο πράκτορας AI που προσπαθεί να κάνει τα πάντα = εξουθένωση. Εξειδικευμένοι πράκτορες που δουλεύουν μαζί = ξεκλείδωμα αποδοτικότητας! 🚀

**Ειλικρινά**: Θυμάστε να προσπαθείτε μόνος σας να ερευνήσετε, να γράψετε και να επιμεληθείτε το podcast σας; Ναι, ήταν άσχημα. Με την ορχήστρωση, κάθε πράκτορας χειρίζεται αυτό που ξέρει καλύτερα. Εσείς παίρνετε μόνο την τελική απόφαση.

**Παραδείγματα πραγματικού κόσμου**: Τα ρομπότ υποστήριξης πελατών ξέρουν πότε να χειριστούν λογαριασμούς, τεχνικά προβλήματα ή πότε να καλέσουν άνθρωπο. Αυτό είναι ορχήστρωση!

## Πράκτορες vs Ροές εργασίας: Ποια είναι η διαφορά;

Σκεφτείτε το έτσι:

### 🤖 Πράκτορας AI = Τζαζ μουσικός
- **Αυθόρμητες αποφάσεις** βασισμένες σε ό,τι ακούει
- **Αυθόρμητη δημιουργία** με τα εργαλεία του
- **Σκέψη** με εγκέφαλο LLM
- **Προσαρμογή** σε ό,τι του δώσετε

### 🎵 Ροή εργασίας = Ορχήστρα που παίζει κλασική μουσική
- **Ακολουθεί παρτιτούρα** (προκαθορισμένα βήματα)
- **Προβλέψιμη** ροή εκτέλεσης
- **Συντονίζει** πολλούς πράκτορες, ανθρώπους, συστήματα
- **Δομημένη** σαν συνταγή

**Η μαγεία**: Η ροή εργασίας *ορχηστρώνει* τους πράκτορες! Δημιουργείτε μια ροή εργασίας και λέτε πότε οι πράκτορες παίζουν το ρόλο τους. Το καλύτερο και των δύο κόσμων. 🎭

## Τρεις τρόποι συντονισμού της ομάδας AI σας

### 1. 🎯 Κεντρικευμένος (εσείς είστε το αφεντικό)

Ένας κύριος πράκτορας δίνει εντολές. Σκεφτείτε το σαν να διαχειρίζεστε μια ομάδα — εσείς αποφασίζετε ποιος κάνει τι και πότε.

**Πλεονεκτήματα**:
- ✅ Ξεκάθαρη ηγεσία (χωρίς σύγχυση)
- ✅ Συνεπείς αποφάσεις
- ✅ Εύκολη αποσφαλμάτωση

**Ιδανικό για**:
- Δρομολόγηση υποστήριξης πελατών («Είναι λογαριασμός ή τεχνική υποστήριξη;»)
- Ροές εργασίας για έγκριση περιεχομένου («Εγκρίνεται αυτό το σενάριο;»)
- Παραγωγή podcast (όπως αυτό που φτιάχνουμε!)

### 2. 🤝 Αποκεντρωμένος (αυτοοργανωμένοι πράκτορες)

Οι πράκτορες μιλούν απευθείας μεταξύ τους και λύνουν προβλήματα σαν ομάδα. Σαν μια ομαδική συνομιλία όπου όλοι συνεργάζονται.

**Πλεονεκτήματα**:
- ✅ Εύκολα επεκτάσιμο (προσθέτετε πράκτορες όποτε θέλετε)
- ✅ Χωρίς μονο σημεία αποτυχίας
- ✅ Φυσική συνεργασία πρακτόρων

**Ιδανικό για**:
- Ερευνητικές ομάδες (κάθε πράκτορας εξερευνά διαφορετικές πηγές)
- Συνεδριάσεις ιδεών
- Κατανεμημένη επίλυση προβλημάτων

### 3. 🔀 Υβριδικό (το καλύτερο και των δύο κόσμων)

Εσείς ορίζετε τη γενική κατεύθυνση, αλλά οι πράκτορες είναι ελεύθεροι να αυτοοργανωθούν στις εργασίες. Σαν ένας CEO που εμπιστεύεται την ομάδα του.

**Τέλειο για**: πολύπλοκα έργα που χρειάζονται έλεγχο και ευελιξία.

## Microsoft Agent Framework: Το εργαλείο ορχήστρωσής σας 🧰

Ήρθε η ώρα να χτίσετε! Εδώ είναι τι θα χρησιμοποιήσετε:

### Δομικά στοιχεία

#### 1. 🧱 Εκτελεστές (οι εργάτες σας)
- **Τι είναι**: Μονάδες επεξεργασίας — μπορεί να είναι πράκτορες ή προσαρμοσμένη λογική
- **Τι κάνουν**: Λαμβάνουν εισροή, εκτελούν εργασία και παράγουν έξοδο
- **Σκεφτείτε τους** σαν σταθμούς στη γραμμή παραγωγής

#### 2. ➡️ Ακμές (συνδέσεις)
- **Τι είναι**: Διαδρομές μεταξύ εκτελεστών
- **Τι κάνουν**: Ελέγχουν τη ροή μηνυμάτων («Μετά το Α, πηγαίνε στο Β»)
- **Σκεφτείτε τις** σαν βέλη σε διαγράμματα ροής

#### 3. 🗺️ Ροές εργασίας (γενικό σχέδιο)
- **Τι είναι**: Ο πλήρης χάρτης εκτελεστών + ακμών
- **Τι κάνουν**: Ορίζουν όλη τη διαδικασία από την αρχή έως το τέλος
- **Σκεφτείτε τις** σαν το πλάνο παραγωγής σας

### Κορυφαίες δυνατότητες που θα αγαπήσετε

**🛡️ Ασφάλεια τύπων**: Τα μηνύματα μεταξύ πρακτόρων ελέγχονται ως προς τους τύπους. Καμία έκπληξη «ω, λάθος τύπος δεδομένων».

**🔀 Ευέλικτη δρομολόγηση**:
- Συνθήκες αν-τότε («αν εγκριθεί, δημοσίευση, αλλιώς επανεγγραφή»)
- Παράλληλη επεξεργασία (πολλοί πράκτορες δουλεύουν ταυτόχρονα)
- Δυναμικές διαδρομές (η ροή εργασίας προσαρμόζεται ανάλογα με τα αποτελέσματα)

**🔌 Εξωτερικές ενσωματώσεις**:
- Σύνδεση σε API
- Προσθήκη σημείων ελέγχου ανθρώπου (εσείς εγκρίνετε πριν τη δημοσίευση)
- Δημιουργία ροών αιτήματος/απόκρισης

**💾 Σημεία ελέγχου**: Αποθηκεύστε την πρόοδο! Αν κάτι καταρρεύσει, συνεχίστε από εκεί που σταματήσατε.

**🤝 Συντονισμός πολλαπλών πρακτόρων**:
- Εκτέλεση πράκτορων κατά σειρά (A → B → C)
- Ταυτόχρονη εκτέλεση τους (A + B + C μαζί)
- Παράδοση μεταξύ πρακτόρων
- Συνεργατική διαχείριση

## Βέλτιστες πρακτικές (επαγγελματικές συμβουλές) 🎯

### 1. Διατηρήστε τον μο듐λο
Κάθε πράκτορας πρέπει να κάνει πραγματικά καλά μία δουλειά. Μην φτιάξετε έναν «σούπερ πράκτορα» που κάνει τα πάντα — θα το μετανιώσετε όταν τον αποσφαλματώσετε.

### 2. Προβλέψτε λάθη
Οι πράκτορες κάνουν λάθη. Το δίκτυο πέφτει. Φτιάξτε χειριστές σφαλμάτων και εναλλακτικά σχέδια. Το μελλοντικό σας εγώ θα σας ευχαριστήσει.

### 3. Παρακολουθήστε τα πάντα
Παρακολουθήστε τι κάνουν οι πράκτορές σας. Χρησιμοποιήστε το DevUI (θα το γνωρίσουμε!) για να δείτε ροές εργασίας σε δράση.

### 4. Βελτιστοποιήστε το μέγεθος των μηνυμάτων
Μην στέλνετε τεράστια αρχεία μεταξύ πρακτόρων. Κρατήστε τα μηνύματα μικρά για ταχύτητα.

### 5. Επιλέξτε το σωστό μοτίβο
Θέλετε έλεγχο; Επιλέξτε κεντρικευμένο. Θέλετε κλίμακα; Επιλέξτε αποκεντρωμένο. Δεν μπορείτε να αποφασίσετε; Επιλέξτε υβριδικό!

## DevUI: Ο αποσφαλματωτής ροών εργασίας σας 🔍

### Τι είναι το DevUI;

Το DevUI είναι σαν παιδική χαρά για να δοκιμάσετε τους πράκτορες και τις ροές εργασίας σας. Είναι ένα web interface όπου μπορείτε:
- 👀 Να παρακολουθήσετε τη ροή εργασίας σας να τρέχει
- 💬 Να συνομιλήσετε απευθείας με τους πράκτορες
- 🔍 Να αποσφαλματώσετε όταν κάτι πάει στραβά
- 📊 Να δείτε ιχνηλασίες και μετρικά απόδοσης

> **Σημαντικό**: Το DevUI προορίζεται μόνο για ανάπτυξη! Μην το χρησιμοποιείτε σε παραγωγή. Σκεφτείτε το σαν το τοπικό σας περιβάλλον δοκιμών.

### Τι το κάνει υπέροχο

- **🖥️ Διαδραστικό Web UI**: κλικ, εισαγωγή, δοκιμές — χωρίς γραμμή εντολών
- **📁 Έτοιμο για drag-and-drop**: ανεβάστε αρχεία, δοκιμάστε με διαφορετικές εισόδους
- **📂 Αυτόματη ανακάλυψη**: δείχνετε ένα φάκελο και βρίσκει αυτόματα όλους τους πράκτορες
- **📋 Χωρίς ανάγκη για setup mode**: καταχωρεί τους πράκτορες στον κώδικα, χωρίς ανάγκη για δομή φακέλων
- **🔌 Συμβατό με OpenAI**: συνεργάζεται με το OpenAI SDK (Ζήτω η συμβατότητα!)
- **👁️ Ενσωματωμένη ιχνηλασία**: δείτε ακριβώς τι κάνουν οι πράκτορές σας

### Πώς λειτουργεί η εισαγωγή

Το DevUI καταλαβαίνει έξυπνα τι χρειάζεστε ως είσοδο:

- **Δοκιμάζετε έναν πράκτορα;** Παίρνετε πεδίο κειμένου και κουμπί για ανέβασμα αρχείων
- **Δοκιμάζετε μια ροή εργασίας;** Η UI δημιουργεί αυτόματα πεδία εισόδου με βάση τι περιμένει η ροή

Είναι σαν μαγεία, αλλά είναι απλά καλός κώδικας. ✨

## Η αποστολή σας: Δημιουργήστε ένα στούντιο podcast 🎬

### Αποστολή 1: Δημιουργήστε έναν μεμονωμένο πράκτορα με DevUI

📂 [01.AgentDevUI](../../../../../WorkshopForAgentic/code/02.Workflow/01.AgentDevUI)

**Πρόκληση**: Πριν φτιάξουμε ολόκληρη την ομάδα, ας δοκιμάσουμε το DevUI με έναν πράκτορα: έναν ειδικό αναζήτησης στο διαδίκτυο.

**Τι φτιάχνετε**:
Έναν πράκτορα που μπορεί να ψάξει το διαδίκτυο για θέματα podcast. Θα τον δοκιμάσετε στο web interface του DevUI `http://localhost:8090`.

**Τι θα μάθετε**:
- 🚀 Πώς να ξεκινήσετε έναν πράκτορα στο DevUI
- 🔍 Πώς να δοκιμάζετε ζωντανά τις απαντήσεις του πράκτορα
- 🛠️ Πώς να κατασκευάσετε προσαρμοσμένα εργαλεία (αναζήτηση στο διαδίκτυο)
- 📊 Πώς να ενεργοποιήσετε την ιχνηλασία για αποσφαλμάτωση
- 🖥️ Πώς να χρησιμοποιήσετε το διαδραστικό web UI

**Κώδικας**:
- `agent.py`: Ο SearchAgent σας με υπερδυνάμεις διαδικτυακής έρευνας
- Χρήση OllamaChatClient για σύνδεση με Qwen
- Υλοποίηση της `web_search()` βοηθητικής συνάρτησης
- Εκκίνηση με `serve()` — ανοίγει αυτόματα το DevUI

**Στόχος**: Ρωτήστε τον πράκτορα «Ποια είναι τα καυτά θέματα στον τομέα της AI;» και δείτε τον να ψάχνει στο διαδίκτυο! 🎉

### Αποστολή 2: Δημιουργήστε μια ροή πολλαπλών πρακτόρων

📂 [02.WorkflowDevUI](../../../../../WorkshopForAgentic/code/02.Workflow/02.WorkflowDevUI)

**Πρόκληση**: Τώρα ξεκινά η διασκέδαση! Δημιουργήστε μια πλήρη ροή εργασίας παραγωγής podcast:
1. 🔍 **Πράκτορας αναζήτησης** → Ερευνά το θέμα σας
2. ✍️ **Πράκτορας σεναρίου** → Γράφει διάλογο μεταξύ δύο παρουσιαστών (στα κινέζικα!)
3. 👤 **Εκτελεστής έγκρισης** → Ζητά την έγκρισή σας ή την απόρριψη
4. 🔄 **Επαναληπτικός κύκλος** → Αν απορριφθεί, ξαναγράψτε με βάση τα σχόλιά σας

**Τι θα μάθετε**:
- 🧱 Δημιουργία εξειδικευμένων πρακτόρων για διαφορετικά καθήκοντα
- 🔗 Σύνδεση πρακτόρων με WorkflowBuilder
- 🔀 Υλοποίηση βρόχου έγκρισης (άνθρωπος εντός του κύκλου!)
- 🚦 Συνθήκες δρομολόγησης (εγκρίνεται vs απορρίπτεται)
- 🔧 Κατασκευή προσαρμοσμένων εκτελεστών για επιχειρησιακή λογική

**Ροή εργασίας**:
```
SearchAgent → ScriptAgent → ReviewExecutor
                             ↑          ↓ (如果被拒绝)
                             ←─────────
```

**Κώδικας**:
- `search_agent/agent.py`: ο ειδικός σας στην έρευνα
- `generate_script_agent/agent.py`: ο σεναριογράφος σας (γράφει στα κινέζικα!)
- `workflow/workflow.py`: εδώ συμβαίνει η μαγεία της ορχήστρωσης
- `main.py`: ξεκινά τα πάντα μέσα στο DevUI

**Στόχος**: Δώστε ένα θέμα, ελέγξτε το σενάριο, απορρίψτε μία φορά για να δοκιμάσετε το βρόχο, και μετά εγκρίνετε! 🎉

### Αποστολή 3: Δημιουργήστε μια εφαρμογή κονσόλας

📂 [03.Application](../../../../../WorkshopForAgentic/code/02.Workflow/03.Application)

**Πρόκληση**: Μετατρέψτε τη ροή εργασίας σας από το DevUI σε μια στιλάτη εφαρμογή τερματικού με χρωματιστή έξοδο, περιστρεφόμενο δείκτη φόρτωσης και αποθήκευση αρχείων. Αυτό είναι έτοιμο για παραγωγή!

**Τι θα μάθετε**:
- ⚡ Πώς να τρέχετε προγραμματιστικά τη ροή εργασίας (χωρίς DevUI)
- 📡 Χρήση ροής γεγονότων (event-driven streaming architecture)
- 🎨 Δημιουργία όμορφου UI τερματικού (χρώματα, περιστρεφόμενοι δείκτες, προοδευτικές μπάρες)
- 💾 Αποθήκευση του τελικού σεναρίου σε αρχείο
- 🔄 Χρήση του asyncio της Python για ασύγχρονη ροή εργασίας

**Τι κάνει**:
1. Σας ρωτά για το θέμα του podcast σας
2. Δείχνει ζωντανή πρόοδο («Ο πράκτορας αναζήτησης δουλεύει...»)
3. Δείχνει το παραγόμενο σενάριο με χρώμα
4. Ζητά την έγκρισή σας
5. Αποθηκεύει το εγκεκριμένο σενάριο στο `podcast.txt`

**Κώδικας**:
- `podcast_app.py`: η κύρια εφαρμογή σας με χειρισμό γεγονότων
- `workflow.py`: επαναχρησιμοποίηση της ροής εργασίας από την Αποστολή 2
- Επεξεργασία γεγονότων: `AgentRunUpdateEvent`, `RequestInfoEvent`, `WorkflowOutputEvent`  
- Χρήση ANSI χρωμάτων για στυλ τερματικού

**Συνθήκη νίκης**: Εκτέλεση εφαρμογής, δημιουργία σεναρίου podcast, και να το βλέπεις να αποθηκεύεται! Έχεις κατασκευάσει ένα πραγματικό εργαλείο.🚀

## Όταν τα πράγματα πάνε στραβά 🔧

### "Η ροή εργασίας μου είναι πολύ περίπλοκη!"  
**Λύση**: Χώρισέ την σε μικρότερες υπο-ροές εργασίας. Κάθε ροή εργασίας πρέπει να κάνει ένα πράγμα καλά. Σύνδεσέ τις αν χρειάζεται.

### "Δεν μπορώ να παρακολουθήσω τι συμβαίνει!"  
**Λύση**: Χρησιμοποίησε σημεία ελέγχου ροής εργασίας για να αποθηκεύεις κατάσταση. Ενεργοποίησε την παρακολούθηση στο DevUI για να βλέπεις κάθε βήμα.

### "Ένα λάθος πράκτορα καταρρίπτει τα πάντα!"  
**Λύση**: Πρόσθεσε όρια σφαλμάτων. Κάθε πράκτορας πρέπει να διαχειρίζεται τις αποτυχίες του και να έχει εναλλακτική συμπεριφορά.

### "Πάει πολύ αργά"  
**Λύση**: Μπορούν οι πράκτορες να τρέχουν παράλληλα; Οι σειριακές ροές είναι εύκολες αλλά αργές. Βρες ευκαιρίες για παραλληλοποίηση!
