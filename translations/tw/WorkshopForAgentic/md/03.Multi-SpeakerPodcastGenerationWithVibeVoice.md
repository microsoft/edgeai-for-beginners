# 第三幕：讓你的播客活起來 🎤

![lab3](../../../../translated_images/zh-TW/lab3.b009c55c4442250b.webp)

## 盛大結局

你已經研究了主題。你已經寫好了腳本。現在是最精彩的部分：把文字變成真實聲音的播客音頻！

引入 **VibeVoice** — 微軟研究院的開源 TTS（文字轉語音）魔法，能製作：
- 🎭 自然的對話聲音
- 👥 多位說話者（最多 4 位！）
- ⏱️ 長時間音頻（最多 90 分鐘！）
- 🎵 富有表現力的朗讀（不是機器人聲！）

這就是合成播客背後的技術。讓我們一起做出你的播客！

## 什麼是 VibeVoice？（厲害的秘訣）

VibeVoice 是微軟研究院獻給世界的禮物。它專為播客風格的對話音頻設計。

### 它有多棒 🔥

* **⏱️ 馬拉松時長**：能持續生成長達90分鐘的連續語音（整集播客！）
* **👥 多說話者魔法**：最多四個具有一致個性的不同聲音
* **⚡ 超高效率**：採用極低 7.5 Hz 幀率，節省運算資源
* **🧠 智能音頻**：結合大型語言模型（理解上下文）與擴散模型（生成真實音頻）
* **🎭 自然流暢**：自動處理輪流講話、停頓和會話節奏

**翻譯**：VibeVoice 不只是念稿子 — 它會像真人交談般 *演繹* 你的內容。

## 開始吧！簡單的方法 🎬

我們把流程簡化到只需一個 Shell 腳本。

### 流程說明

1. **賦予執行權限**：
```bash
chmod +x run_vibe_voice.sh
```

2. **執行腳本**：
```bash
./run_vibe_voice.sh
```

3. **等待魔法發生**（第一次執行可能需要幾分鐘）

### 背後發生了什麼事 🎭

這個腳本就是你的自動化音效工程師：

1. **📥 下載 VibeVoice**：從 GitHub 克隆官方倉庫
2. **📦 安裝依賴**：用 `uv pip` 迅速安裝套件
3. **🎬 生成語音**：運行推理腳本，其中：
   * `--model_path`：指定預訓練的 VibeVoice-7B 模型
   * `--txt_path`：你的 `podcast.txt` 腳本檔
   * `--speaker_names`：分配聲音（預設是 Xinran 和 Anchen）

**結果**：你的腳本變成了真實播客集！🎉

## 你的任務 🎯

讓我們來點有趣的：

### 任務 1：創造內容
編輯 `../03.Application/podcast.txt`，撰寫兩人間的對話。主題可以關於科技、興趣等任何事！只要是對話體即可。

**格式範例**：
```
Speaker 1: Hey! Did you hear about the new AI model?
Speaker 2: No way! Tell me more!
Speaker 1: It's called...
```

### 任務 2：生成音頻
執行腳本並觀看魔法呈現。第一次會花較久時間（模型下載需要時間）。

### 任務 3：聆聽與分析
- 聲音聽起來自然嗎？
- 說話者的聲線是否有區別？
- 輪流講話是否流暢？
- 有沒有機械人聲的感覺？

### 任務 4：嘗試（勇者福利）
修改 `run_vibe_voice.sh` 中的 `--speaker_names`，嘗試不同的聲音組合。VibeVoice 有多個預訓練聲音可選！

**額外挑戰**：嘗試三位說話者的對話！🎆
