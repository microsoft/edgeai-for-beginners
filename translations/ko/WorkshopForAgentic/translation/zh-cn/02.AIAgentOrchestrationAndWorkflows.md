# 第二막：당신의 팟캐스트 제작 팀 구성 🎬

![lab2](../../../../../translated_images/ko/lab2.48444b9d549e7863.webp)

## 이야기 고조

Alex(당신이 첫 번째 막에서 만난 AI 비서)는 훌륭하지만, 한 명의 에이전트만으로 팟캐스트 스튜디오 전체를 운영할 수 없습니다. 당신에게는 *팀*이 필요합니다:
- 🔍 **연구 에이전트**: 인터넷에서 신선한 정보를 검색
- ✍️ **스크립트 에이전트**: 연구 내용을 매력적인 대화로 변환
- 👤 **당신(편집자)**: 스크립트를 승인하거나 다시 작성 요청

AI 에이전트 오케스트레이션에 오신 것을 환영합니다 — 당신이 자신의 AI 팀 감독자가 되는 곳입니다. 어벤져스처럼 생각하세요, 하지만 팟캐스트 제작용으로요.

## 에이전트 오케스트레이션이란? (간단한 버전)

당신이 식당을 운영한다고 상상해보세요. 모든 일을 혼자 다 하진 않죠? 당신에겐:
- 🍳 요리사: 요리 담당
- 👨‍🍳 부주방장: 준비 담당
- 👩‍🍳 서버: 음식 배달 담당

에이전트 오케스트레이션도 같은 개념인데 AI를 활용합니다. 각 에이전트는 전문 분야가 있고, 당신은 이들을 조율해 더 큰 목표를 달성합니다. 한 명의 에이전트에 부담이 쏠리지 않고 작업이 더 빨리 완료됩니다.

### 밴드 비유 🎸

당신의 AI 에이전트는 마치 밴드와 같습니다:
- **보컬**: 고객 대상 주요 업무를 처리하는 주 에이전트
- **드러머**: 리듬을 유지하며 백엔드 작업을 담당
- **베이시스트**: 모두를 지원하며 데이터를 취득
- **당신(밴드 매니저)**: 모두를 조율!

조율이 없으면? 소음일 뿐입니다. 오케스트레이션이 있으면? 아름다운 음악이죠.

### 왜 이게 중요한가

하나의 AI 에이전트가 모든 일을 하려 하면 피로가 쌓입니다. 전문 에이전트들이 함께 일하면 효율성이 극대화됩니다! 🚀

**솔직히 말해보자면**: 혼자서 연구하고, 작성하고, 편집하려 시도했던 기억 있나요? 네, 그건 정말 힘들었습니다. 오케스트레이션을 통해 각 에이전트가 가장 잘하는 일을 맡고, 당신은 최종 결정을 내리기만 하면 됩니다.

**실생활 예**: 고객 지원 챗봇이 청구 문제, 기술 문제, 또는 인간 상담원 연결 등 어떤 상황인지 알고 적절하게 대처하는 것, 그게 오케스트레이션입니다!

## 에이전트 vs 워크플로우: 무슨 차이인가요?

이렇게 생각하세요:

### 🤖 AI 에이전트 = 재즈 뮤지션
- **즉흥적 결정**: 듣고 있는 내용을 바탕으로 즉석에서 판단
- **즉흥 창작**: 자신의 도구를 사용해 문제 해결
- **사고**: LLM 두뇌 사용
- **적응**: 던져진 어떤 것이든 처리

### 🎵 워크플로우 = 클래식 음악을 연주하는 오케스트라
- **악보 따름** (미리 정의된 단계)
- **예측 가능한** 실행 경로
- **조율** 여러 에이전트, 사람, 시스템 간 조직
- **구조화** 요리법처럼

**마법은?** 워크플로우가 *에이전트를 오케스트레이션*하는 것! 워크플로우를 만들어 언제 어떤 에이전트가 역할을 할지 알려줍니다. 두 세계의 장점을 모두 갖췄죠.🎭

## AI 팀을 조율하는 세 가지 방법

### 1. 🎯 중앙집중식 (당신이 보스)
하나의 주 에이전트가 명령을 내립니다. 팀을 관리하는 당신처럼 — 누가 무엇을 언제 할지 결정합니다.

**장점**:
- ✅ 명확한 리더십 (혼란 없음)
- ✅ 일관된 결정
- ✅ 디버깅 용이

**적용 분야**:
- 고객 지원 라우팅 ("이건 청구 문제인가, 기술 지원인가?")
- 콘텐츠 승인 워크플로우 ("이 스크립트 통과인가?")
- 팟캐스트 제작 (바로 우리가 만드는 것!)

### 2. 🤝 분산식 (에이전트들이 스스로 조직)
에이전트들이 직접 소통하며 집단으로 문제 해결. 모두가 채팅방에서 조율하는 것처럼.

**장점**:
- ✅ 확장성 좋음 (새 에이전트를 쉽게 추가)
- ✅ 단일 실패 지점 없음
- ✅ 자연스럽게 협력

**적용 분야**:
- 연구 팀(각각 다른 출처 탐색)
- 브레인스토밍 미팅
- 분산 문제 해결

### 3. 🔀 하이브리드 (두 가지 장점 모두)
당신은 전체 방향을 설정하지만, 에이전트는 작업에 대해 자유롭게 조직. 신뢰하는 팀의 CEO 같음.

**완벽 적용**: 통제와 유연성이 모두 필요한 복잡한 프로젝트.

## Microsoft Agent Framework: 당신의 오케스트레이션 도구 🧰

이제 만들 차례입니다! 이것이 당신이 사용할 것입니다:

### 구성 요소

#### 1. 🧱 실행기 (당신의 작업자)
- **무엇인가?**: 독립된 처리 단위 — 에이전트거나 맞춤 로직일 수 있음
- **무엇을 하나?**: 입력을 받고, 작업을 수행하며, 출력을 생성
- **비유하자면**: 조립 라인의 작업대

#### 2. ➡️ 엣지 (연결)
- **무엇인가?**: 실행기 간 경로
- **무엇을 하나?**: 메시지 흐름 제어 ("A 다음에 B로")
- **비유하자면**: 플로우 차트의 화살표

#### 3. 🗺️ 워크플로우 (전체 계획)
- **무엇인가?**: 실행기와 엣지를 완전히 연결한 그래프
- **무엇을 하나?**: 시작부터 종료까지 전체 프로세스 정의
- **비유하자면**: 제작 프로세스 청사진

### 당신이 좋아할 쿨 기능들

**🛡️ 타입 안정성**: 에이전트 간 메시지는 타입 체크 됨. "앗, 잘못된 데이터 타입" 같은 놀람 없음.

**🔀 유연한 라우팅**:
- If-then 조건 ("승인되면 발행; 아니면 재작성")
- 병렬 처리 (여러 에이전트가 동시에 작업)
- 동적 경로 (워크플로우가 결과에 따라 조정됨)

**🔌 외부 통합**:
- API 연결
- 인간 개입 체크포인트 추가 (당신이 발행 전 승인)
- 요청/응답 흐름 빌딩

**💾 체크포인트**: 진행 상황 저장! 문제가 생기면 중단된 지점부터 재개.

**🤝 다중 에이전트 조율**:
- 순차 실행 (A → B → C)
- 병렬 실행 (A + B + C 동시에)
- 에이전트 간 핸드오프
- 협력 처리

## 베스트 프랙티스 (프로 팁) 🎯

### 1. 모듈화 유지
각 에이전트는 한 가지 일을 제대로 해야 합니다. 모든 걸 하는 "슈퍼 에이전트"를 만들진 마세요 — 디버깅이 불가능해집니다.

### 2. 실패 대비 계획
에이전트는 실수할 수 있습니다. 네트워크도 실패합니다. 오류 처리와 백업 계획 구축하세요. 미래의 당신이 감탄할 겁니다.

### 3. 모든 것을 모니터링
에이전트가 무슨 일을 하는지 추적하세요. DevUI를 사용해(곧 소개합니다!) 실행 중인 워크플로우를 봅니다.

### 4. 메시지 크기 최적화
에이전트 간에 거대한 파일을 주고받지 마세요. 메시지는 가볍고 깔끔하게 하여 속도를 높이세요.

### 5. 올바른 패턴 선택
통제가 필요하면? 중앙집중식 선택. 확장이 필요하면? 분산식 선택. 결정 못하면? 하이브리드 선택!

## DevUI: 당신의 워크플로우 디버거 🔍

### DevUI란?

DevUI는 당신의 에이전트와 워크플로우를 테스트하는 놀이터 같습니다. 웹 인터페이스로:
- 👀 워크플로우 실행 시각화
- 💬 에이전트와 직접 대화
- 🔍 문제 발생 시 디버깅
- 📊 추적 및 성능 지표 확인

> **중요**: DevUI는 개발용입니다! 운영 환경에서는 사용하지 마세요. 로컬 테스트 환경으로 생각하세요.

### 멋진 이유

- **🖥️ 인터랙티브 웹 UI**: 클릭, 입력, 테스트 — 커맨드라인 불필요
- **📁 드래그 앤 드롭 준비됨**: 파일 업로드, 다양한 입력으로 테스트
- **📂 자동 발견**: 폴더를 지정하면 모든 에이전트를 자동으로 찾음
- **📋 설정 불필요**: 코드에서 에이전트 등록만, 폴더 구조 걱정 없음
- **🔌 OpenAI 호환**: OpenAI SDK와 호환(호환성 만세!)
- **👁️ 내장 추적**: 에이전트가 하는 일을 정확히 파악

### 입력 작동 방식

DevUI는 입력 부분에서 똑똑합니다:

- **에이전트 테스트?** 텍스트 박스와 파일 업로드 버튼이 나타납니다
- **워크플로우 테스트?** UI가 워크플로우에서 예상하는 입력 필드를 자동 생성

마법 같지만, 그냥 좋은 코드일 뿐입니다.✨

## 당신의 임무: 팟캐스트 스튜디오 구축 🎬

### 임무 1: DevUI로 단일 에이전트 생성

📂 [01.AgentDevUI](../../../../../WorkshopForAgentic/code/02.Workflow/01.AgentDevUI)

**도전 과제**: 완전한 팀 빌드 전에, 단일 에이전트로 DevUI를 테스트해봅시다: 인터넷 검색 전문가.

**무엇을 만들까요?**: 팟캐스트 주제를 인터넷에서 검색할 수 있는 연구 에이전트. DevUI 웹 인터페이스 `http://localhost:8090`를 사용해 테스트할 예정입니다.

**배울 기술**:
- 🚀 DevUI에서 에이전트 실행
- 🔍 실시간으로 에이전트 응답 테스트
- 🛠️ 사용자 정의 도구(웹 검색) 구축
- 📊 문제 디버깅용 추적 활성화
- 🖥️ 인터랙티브 웹 UI 사용

**코드**:
- `agent.py`: 네트워크 검색 슈퍼파워가 있는 SearchAgent
- OllamaChatClient를 사용해 Qwen에 연결
- `web_search()` 사용자 도구 함수 구현
- `serve()` 로 실행 시작 — DevUI 자동 오픈

**성공 조건**: 에이전트에게 "AI 분야 핫이슈는 무엇인가요?"라고 물어보고 네트워크 검색하는 모습을 확인하세요! 🎉

### 임무 2: 다중 에이전트 워크플로우 구축

📂 [02.WorkflowDevUI](../../../../../WorkshopForAgentic/code/02.Workflow/02.WorkflowDevUI)

**도전 과제**: 진짜 재미가 시작됩니다! 완전한 팟캐스트 제작 워크플로우 구축:
1. 🔍 **검색 에이전트** → 주제 연구
2. ✍️ **스크립트 에이전트** → 두 진행자 간 대화 작성(중국어!)
3. 👤 **검토 실행기** → 당신의 승인 또는 거절 요청
4. 🔄 **순환 피드백** → 거절 시 피드백 기반 재작성

**배울 기술**:
- 🧱 직무별 맞춤 에이전트 생성
- 🔗 WorkflowBuilder로 에이전트 연결
- 🔀 승인 순환 구현 (인간 개입!)
- 🚦 조건 라우팅 (승인 vs 거절)
- 🔧 비즈니스 로직용 맞춤 실행기 구축

**워크플로우**:
```
SearchAgent → ScriptAgent → ReviewExecutor
                             ↑          ↓ (如果被拒绝)
                             ←─────────
```

**코드**:
- `search_agent/agent.py`: 당신의 연구 전문가
- `generate_script_agent/agent.py`: 당신의 스크립트 작성자(중국어)
- `workflow/workflow.py`: 여기서 오케스트레이션 마법 발생
- `main.py`: DevUI에서 시작

**성공 조건**: 주제를 주고 스크립트를 검토한 후, 한 번 거절해 피드백 순환 테스트, 그리고 승인을 진행하세요! 🎉

### 임무 3: 콘솔 애플리케이션 구축

📂 [03.Application](../../../../../WorkshopForAgentic/code/02.Workflow/03.Application)

**도전 과제**: DevUI에서 워크플로우를 세련된 터미널 앱으로 전환하세요. 컬러 출력, 로딩 스피너, 파일 저장 포함. 이제 프로덕션 준비 완료!

**배울 기술**:
- ⚡ 프로그래밍 방식 워크플로우 실행(DevUI 없이)
- 📡 스트리밍 이벤트 기반 아키텍처 사용
- 🎨 아름다운 터미널 UI 만들기(컬러, 스피너, 진행률 표시줄)
- 💾 최종 스크립트를 파일에 저장하기
- 🔄 Python의 asyncio로 비동기 워크플로우 처리

**작동 방식**:
1. 팟캐스트 주제 묻기
2. 실시간 진행 상황 표시 ("검색 에이전트 작업 중…")
3. 생성된 스크립트 컬러 출력
4. 승인 요청
5. 승인된 스크립트를 `podcast.txt`에 저장

**코드**:
- `podcast_app.py`: 이벤트 핸들링 포함 메인 앱
- `workflow.py`: 임무 2 워크플로우 재사용
- 이벤트 처리: `AgentRunUpdateEvent`, `RequestInfoEvent`, `WorkflowOutputEvent`
- 터미널 스타일에 ANSI 색상 사용

**승리 조건**: 애플리케이션을 실행하고 팟캐스트 스크립트를 생성하여 저장되는 것을 확인하세요! 당신은 진짜 도구를 만들었습니다.🚀

## 이미 숙달한 내용 🏆

두 번째 막 이후, 당신은:

- ✅ 보스처럼 여러 AI 에이전트를 오케스트레이션할 수 있습니다
- ✅ 순차적 및 조건부 로직이 있는 워크플로를 구축할 수 있습니다
- ✅ 수동 승인 체크포인트를 추가할 수 있습니다
- ✅ DevUI로 워크플로를 테스트하고 디버깅할 수 있습니다
- ✅ 프로덕션 준비 완료된 콘솔 애플리케이션을 만들 수 있습니다
- ✅ 복잡한 시스템에서 우아하게 오류를 처리할 수 있습니다
- ✅ 모든 프로젝트에 적합한 오케스트레이션 패턴을 선택할 수 있습니다

## 문제가 발생했을 때 🔧

### "내 워크플로가 너무 복잡해요!"
**해결책**: 더 작은 하위 워크플로로 분해하세요. 각 워크플로는 한 가지 작업을 잘해야 합니다. 필요하면 이들을 연결하세요.

### "무슨 일이 일어나고 있는지 추적할 수 없어요!"
**해결책**: 워크플로 체크포인트로 상태를 저장하세요. DevUI에서 추적을 활성화하여 각 단계를 확인하세요.

### "한 에이전트의 오류가 모든 것을 중단시켜요!"
**해결책**: 오류 경계 추가. 각 에이전트는 자신의 실패를 처리하고 대체 행동을 가져야 합니다.

### "너무 느려요"
**해결책**: 에이전트를 병렬로 실행할 수 있나요? 순차적 워크플로는 쉽지만 느립니다. 병렬화할 기회를 찾아보세요!
