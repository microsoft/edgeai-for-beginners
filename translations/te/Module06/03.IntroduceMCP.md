# Section 03 - మోడల్ కాంటెక్స్ట్ ప్రోటోకాల్ (MCP) ఇంటిగ్రేషన్

## MCP (మోడల్ కాంటెక్స్ట్ ప్రోటోకాల్) పరిచయం

మోడల్ కాంటెక్స్ట్ ప్రోటోకాల్ (MCP) అనేది AI అప్లికేషన్లను బాహ్య వ్యవస్థలకు కనెక్ట్ చేయడానికి ఓపెన్-సోర్స్ స్టాండర్డ్. MCP ఉపయోగించి, Claude లేదా ChatGPT వంటి AI అప్లికేషన్లు డేటా సోర్సులు (ఉదా: లోకల్ ఫైళ్లు, డేటాబేసులు), టూల్స్ (ఉదా: సెర్చ్ ఇంజిన్లు, కాలిక్యులేటర్లు), మరియు వర్క్‌ఫ్లోలు (ఉదా: ప్రత్యేక ప్రాంప్ట్లు) కు కనెక్ట్ అవ్వగలవు—దీని ద్వారా అవి ముఖ్యమైన సమాచారాన్ని పొందగలవు మరియు పనులు చేయగలవు.

MCP ని AI అప్లికేషన్ల కోసం **USB-C పోర్ట్** లాగా ఆలోచించండి. USB-C ఎలక్ట్రానిక్ పరికరాలను కనెక్ట్ చేయడానికి ఒక ప్రమాణీకృత మార్గాన్ని అందిస్తే, MCP AI అప్లికేషన్లను బాహ్య వ్యవస్థలకు కనెక్ట్ చేయడానికి ఒక ప్రమాణీకృత మార్గాన్ని అందిస్తుంది.

### MCP ఏమి సాధ్యం చేస్తుంది?

MCP AI అప్లికేషన్లకు శక్తివంతమైన సామర్థ్యాలను అందిస్తుంది:

- **వ్యక్తిగతీకరించిన AI అసిస్టెంట్లు**: ఏజెంట్లు మీ Google క్యాలెండర్ మరియు Notion ను యాక్సెస్ చేసి, మరింత వ్యక్తిగతీకరించిన AI సహాయకుడిగా పనిచేస్తారు
- **అధునాతన కోడ్ జనరేషన్**: Claude Code Figma డిజైన్ ఉపయోగించి పూర్తి వెబ్ యాప్‌ను రూపొందించగలదు
- **ఎంటర్ప్రైజ్ డేటా ఇంటిగ్రేషన్**: ఎంటర్ప్రైజ్ చాట్‌బాట్లు సంస్థలోని అనేక డేటాబేసులకు కనెక్ట్ అవ్వగలవు, వినియోగదారులు చాట్ ద్వారా డేటాను విశ్లేషించగలరు
- **సృజనాత్మక వర్క్‌ఫ్లోలు**: AI మోడల్స్ Blender లో 3D డిజైన్లు సృష్టించి, వాటిని 3D ప్రింటర్ ద్వారా ముద్రించగలవు
- **రియల్-టైమ్ సమాచారం యాక్సెస్**: తాజా సమాచారానికి బాహ్య డేటా సోర్సులకు కనెక్ట్ అవ్వడం
- **సంక్లిష్ట బహుళ-దశ ఆపరేషన్లు**: అనేక టూల్స్ మరియు వ్యవస్థలను కలిపి సాంక్లిష్ట వర్క్‌ఫ్లోలను నిర్వహించడం

### MCP ఎందుకు ముఖ్యం?

MCP ఈకోసిస్టమ్ అంతటా లాభాలను అందిస్తుంది:

**డెవలపర్ల కోసం**: MCP AI అప్లికేషన్ లేదా ఏజెంట్ నిర్మాణం లేదా ఇంటిగ్రేషన్ సమయంలో అభివృద్ధి సమయం మరియు సంక్లిష్టతను తగ్గిస్తుంది.

**AI అప్లికేషన్ల కోసం**: MCP డేటా సోర్సులు, టూల్స్ మరియు యాప్స్ ఈకోసిస్టమ్‌కు యాక్సెస్ అందించి సామర్థ్యాలను పెంచుతుంది మరియు చివరి వినియోగదారుల అనుభవాన్ని మెరుగుపరుస్తుంది.

**చివరి వినియోగదారుల కోసం**: MCP మరింత సామర్థ్యవంతమైన AI అప్లికేషన్లు లేదా ఏజెంట్లను అందిస్తుంది, అవి మీ డేటాను యాక్సెస్ చేసి అవసరమైతే మీ తరఫున చర్యలు తీసుకోగలవు.

## MCP లో చిన్న భాషా మోడల్స్ (SLMs)

చిన్న భాషా మోడల్స్ AI అమలులో సమర్థవంతమైన దృష్టికోణాన్ని ప్రతిబింబిస్తాయి, అనేక లాభాలతో:

### SLM ల లాభాలు
- **వనరుల సమర్థత**: తక్కువ కంప్యూటేషనల్ అవసరాలు
- **వేగవంతమైన ప్రతిస్పందన సమయాలు**: రియల్-టైమ్ అప్లికేషన్ల కోసం తక్కువ ఆలస్యం  
- **ఖర్చు సమర్థత**: కనిష్ట మౌలిక సదుపాయాలు
- **గోప్యత**: డేటా ప్రసారం లేకుండా లోకల్‌గా నడపగలదు
- **అనుకూలీకరణ**: నిర్దిష్ట డొమైన్‌ల కోసం సులభంగా ఫైన్-ట్యూన్ చేయగలదు

### MCP తో SLM లు ఎందుకు బాగా పనిచేస్తాయి

SLMs MCP తో జతకట్టడం ఒక శక్తివంతమైన కలయికను సృష్టిస్తుంది, ఇక్కడ మోడల్ యొక్క తర్క సామర్థ్యాలు బాహ్య టూల్స్ ద్వారా పెంపొందించబడతాయి, వారి తక్కువ పరిమాణాన్ని మెరుగైన ఫంక్షనాలిటీతో పరిహరించుకుంటాయి.

## Python MCP SDK అవలోకనం

Python MCP SDK MCP-సమర్థ అప్లికేషన్ల నిర్మాణానికి పునాది అందిస్తుంది. SDK లో ఉన్నాయి:

- **క్లయింట్ లైబ్రరీలు**: MCP సర్వర్లకు కనెక్ట్ కావడానికి
- **సర్వర్ ఫ్రేమ్‌వర్క్**: కస్టమ్ MCP సర్వర్లు సృష్టించడానికి
- **ప్రోటోకాల్ హ్యాండ్లర్లు**: కమ్యూనికేషన్ నిర్వహణకు
- **టూల్ ఇంటిగ్రేషన్**: బాహ్య ఫంక్షన్లను అమలు చేయడానికి

## ప్రాక్టికల్ అమలు: Phi-4 MCP క్లయింట్

Microsoft యొక్క Phi-4 మినీ మోడల్ MCP సామర్థ్యాలతో ఏకీకృతంగా ఉన్న వాస్తవ ప్రపంచ అమలును పరిశీలిద్దాం.

### MCP ఆర్కిటెక్చర్ అవలోకనం

MCP ఒక **క్లయింట్-సర్వర్ ఆర్కిటెక్చర్** ను అనుసరిస్తుంది, ఇక్కడ MCP హోస్ట్ (Claude Code లేదా Claude Desktop వంటి AI అప్లికేషన్) ఒకటి లేదా ఎక్కువ MCP సర్వర్లకు కనెక్షన్లు ఏర్పాటు చేస్తుంది. MCP హోస్ట్ ప్రతి MCP సర్వర్ కోసం ఒక MCP క్లయింట్ సృష్టించడం ద్వారా ఇది సాధిస్తుంది.

#### ముఖ్య పాత్రధారులు

- **MCP హోస్ట్**: ఒకటి లేదా ఎక్కువ MCP క్లయింట్లను సమన్వయపరచి నిర్వహించే AI అప్లికేషన్
- **MCP క్లయింట్**: MCP సర్వర్‌కు కనెక్షన్‌ను నిర్వహించి MCP హోస్ట్ ఉపయోగించుకునే సందర్భాన్ని పొందే భాగం
- **MCP సర్వర్**: MCP క్లయింట్లకు సందర్భాన్ని అందించే ప్రోగ్రామ్

#### రెండు-పట్టాల ఆర్కిటెక్చర్

MCP రెండు వేర్వేరు లేయర్లతో ఉంటుంది:

**డేటా లేయర్**: క్లయింట్-సర్వర్ కమ్యూనికేషన్ కోసం JSON-RPC ఆధారిత ప్రోటోకాల్‌ను నిర్వచిస్తుంది, ఇందులో:
- లైఫ్‌సైకిల్ నిర్వహణ (కనెక్షన్ ప్రారంభం, సామర్థ్యాల చర్చ)
- కోర్ ప్రిమిటివ్స్ (టూల్స్, వనరులు, ప్రాంప్ట్లు)
- క్లయింట్ ఫీచర్లు (సాంప్లింగ్, ఎలిసిటేషన్, లాగింగ్)
- యుటిలిటీ ఫీచర్లు (నోటిఫికేషన్లు, ప్రోగ్రెస్ ట్రాకింగ్)

**ట్రాన్స్‌పోర్ట్ లేయర్**: కమ్యూనికేషన్ మెకానిజమ్స్ మరియు ఛానల్స్ నిర్వచిస్తుంది:
- **STDIO ట్రాన్స్‌పోర్ట్**: లోకల్ ప్రాసెస్‌ల కోసం స్టాండర్డ్ ఇన్‌పుట్/అవుట్‌పుట్ స్ట్రీమ్స్ ఉపయోగిస్తుంది (ఉత్తమ పనితీరు, నెట్‌వర్క్ ఓవర్‌హెడ్ లేదు)
- **స్ట్రీమబుల్ HTTP ట్రాన్స్‌పోర్ట్**: రిమోట్ సర్వర్ల కోసం HTTP POST మరియు ఐచ్ఛికంగా సర్వర్-సెంట్ ఈవెంట్స్ ఉపయోగిస్తుంది (స్టాండర్డ్ HTTP ఆథెంటికేషన్ మద్దతు)

### MCP కోర్ ప్రిమిటివ్స్

MCP AI అప్లికేషన్లతో పంచుకునే సందర్భ సమాచార రకాలును మరియు చేయగల చర్యల పరిధిని నిర్దేశించే ప్రిమిటివ్స్‌ను నిర్వచిస్తుంది.

#### సర్వర్ ప్రిమిటివ్స్

MCP సర్వర్లు ఎక్స్‌పోజ్ చేయగల మూడు ప్రధాన ప్రిమిటివ్స్:

**టూల్స్**: AI అప్లికేషన్లు చర్యలు చేయడానికి పిలవగల అమలు చేయదగిన ఫంక్షన్లు  
- ఉదాహరణలు: ఫైల్ ఆపరేషన్లు, API కాల్స్, డేటాబేస్ క్వెరీలు  
- పద్ధతులు: `tools/list`, `tools/call`  
- డైనమిక్ డిస్కవరీ మరియు అమలును మద్దతు ఇస్తాయి

**వనరులు**: AI అప్లికేషన్లకు సందర్భ సమాచారాన్ని అందించే డేటా సోర్సులు  
- ఉదాహరణలు: ఫైల్ కంటెంట్లు, డేటాబేస్ రికార్డులు, API ప్రతిస్పందనలు  
- పద్ధతులు: `resources/list`, `resources/read`  
- నిర్మిత డేటాకు యాక్సెస్‌ను సులభతరం చేస్తాయి

**ప్రాంప్ట్లు**: భాషా మోడల్స్‌తో పరస్పర చర్యలను నిర్మించడానికి సహాయపడే పునర్వినియోగ టెంప్లేట్లు  
- ఉదాహరణలు: సిస్టమ్ ప్రాంప్ట్లు, ఫ్యూ-షాట్ ఉదాహరణలు  
- పద్ధతులు: `prompts/list`, `prompts/get`  
- AI పరస్పర చర్య నమూనాలను ప్రమాణీకరిస్తాయి

#### క్లయింట్ ప్రిమిటివ్స్

MCP క్లయింట్లు కూడా సాంప్రదాయాలను నిర్వచిస్తుంది, ఇవి మరింత సంపన్నమైన పరస్పర చర్యలకు అనుమతిస్తాయి:

**సాంప్లింగ్**: సర్వర్లు క్లయింట్ యొక్క AI అప్లికేషన్ నుండి భాషా మోడల్ పూర్తి చేయింపులను అభ్యర్థించగలవు  
- పద్ధతి: `sampling/complete`  
- మోడల్-స్వతంత్ర సర్వర్ అభివృద్ధిని సాధ్యం చేస్తుంది  
- హోస్ట్ యొక్క భాషా మోడల్ యాక్సెస్‌ను అందిస్తుంది

**ఎలిసిటేషన్**: సర్వర్లు వినియోగదారుల నుండి అదనపు సమాచారాన్ని అభ్యర్థించగలవు  
- పద్ధతి: `elicitation/request`  
- వినియోగదారుల పరస్పర చర్య మరియు నిర్ధారణను సాధ్యం చేస్తుంది  
- డైనమిక్ సమాచార సేకరణకు మద్దతు ఇస్తుంది

**లాగింగ్**: సర్వర్లు క్లయింట్లకు లాగ్ సందేశాలను పంపగలవు  
- డీబగ్గింగ్ మరియు మానిటరింగ్ కోసం ఉపయోగిస్తారు  
- సర్వర్ ఆపరేషన్లపై దృష్టిని అందిస్తుంది

### MCP ప్రోటోకాల్ లైఫ్‌సైకిల్

#### ప్రారంభం మరియు సామర్థ్య చర్చ

MCP ఒక స్థితిస్థాపక ప్రోటోకాల్, ఇది లైఫ్‌సైకిల్ నిర్వహణ అవసరం. ప్రారంభ ప్రక్రియ అనేక ముఖ్యమైన ప్రయోజనాలను అందిస్తుంది:

1. **ప్రోటోకాల్ వెర్షన్ చర్చ**: క్లయింట్ మరియు సర్వర్ రెండూ అనుకూల ప్రోటోకాల్ వెర్షన్లను ఉపయోగిస్తున్నాయో లేదో నిర్ధారిస్తుంది (ఉదా: "2025-06-18")  
2. **సామర్థ్యాల కనుగొనడం**: ప్రతి పక్షం మద్దతు ఇచ్చే ఫీచర్లు మరియు ప్రిమిటివ్స్‌ను ప్రకటిస్తుంది  
3. **గుర్తింపు మార్పిడి**: గుర్తింపు మరియు వెర్షనింగ్ సమాచారాన్ని అందిస్తుంది

```python
# ఉదాహరణ ప్రారంభ అభ్యర్థన
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "initialize",
  "params": {
    "protocolVersion": "2025-06-18",
    "capabilities": {
      "elicitation": {},  # క్లయింట్ వినియోగదారు పరస్పర చర్యను మద్దతు ఇస్తుంది
      "sampling": {}      # క్లయింట్ LLM పూర్తి వివరాలను అందించగలదు
    },
    "clientInfo": {
      "name": "edge-ai-client",
      "version": "1.0.0"
    }
  }
}
```

```python
# అందుబాటులో ఉన్న సాధనాలను కనుగొనండి

        "location": "San Francisco",

    }
```


### దశ 4: ప్రోగ్రామాటిక్ ఉపయోగం

```python
import asyncio
from ghmodel_mcp_demo import OllamaClient, Phi4MiniMCPClient

async def automated_interaction():
    # MCP సర్వర్ పరామితులను కాన్ఫిగర్ చేయండి
    server_params = StdioServerParameters(
        command="npx",
        args=["@playwright/mcp@latest"],
        env=None,
    )
    
    # MCP క్లయింట్ మరియు ప్రాసెస్ టూల్స్ సృష్టించండి
    async with Phi4MiniMCPClient(server_params) as mcp_client:
        tools = await process_mcp_tools(mcp_client)
        llm_client = OllamaClient()
        
        # టూల్ సామర్థ్యాలతో ప్రతిస్పందనను ఉత్పత్తి చేయండి
        response, messages = await llm_client.generate_response(
            "Help me automate a web task",
            tools
        )
        return response

# ఆటోమేషన్‌ను అమలు చేయండి
result = asyncio.run(automated_interaction())
print(result)
```

## అధునాతన ఫీచర్లు

### బహుళ-బ్యాక్‌ఎండ్ మద్దతు

అమలు Ollama మరియు vLLM రెండింటినీ మద్దతు ఇస్తుంది, మీ అవసరాల ఆధారంగా ఎంచుకోవచ్చు:

- **Ollama**: లోకల్ అభివృద్ధి మరియు పరీక్షలకు ఉత్తమం  
- **vLLM**: ఉత్పత్తి మరియు అధిక-థ్రూపుట్ పరిస్థితులకు ఆప్టిమైజ్ చేయబడింది

### సౌకర్యవంతమైన కనెక్షన్ ప్రోటోకాల్‌లు

రెండు కనెక్షన్ మోడ్‌లు మద్దతు పొందుతాయి:

**STDIO మోడ్**: ప్రత్యక్ష ప్రాసెస్ కమ్యూనికేషన్  
- తక్కువ ఆలస్యం  
- లోకల్ టూల్స్‌కు అనుకూలం  
- సులభమైన సెటప్

**SSE మోడ్**: HTTP ఆధారిత స్ట్రీమింగ్  
- నెట్‌వర్క్-సమర్థవంతం  
- పంపిణీ వ్యవస్థలకు ఉత్తమం  
- రియల్-టైమ్ అప్‌డేట్లు

### టూల్ ఇంటిగ్రేషన్ సామర్థ్యాలు

సిస్టమ్ వివిధ టూల్స్‌తో ఇంటిగ్రేట్ చేయగలదు:  
- వెబ్ ఆటోమేషన్ (Playwright)  
- ఫైల్ ఆపరేషన్లు  
- API పరస్పర చర్యలు  
- సిస్టమ్ కమాండ్లు  
- కస్టమ్ ఫంక్షన్లు

## లోపాల నిర్వహణ మరియు ఉత్తమ ఆచారాలు

### సమగ్ర లోపాల నిర్వహణ

అమలు లోపాల నిర్వహణ కోసం బలమైన వ్యవస్థను కలిగి ఉంది:

**కనెక్షన్ లోపాలు:**  
- MCP సర్వర్ వైఫల్యాలు  
- నెట్‌వర్క్ టైమ్‌ఔట్‌లు  
- కనెక్టివిటీ సమస్యలు

**టూల్ అమలు లోపాలు:**  
- టూల్స్ లేమి  
- పారామీటర్ ధృవీకరణ  
- అమలు వైఫల్యాలు

**ప్రతిస్పందన ప్రాసెసింగ్ లోపాలు:**  
- JSON పార్సింగ్ సమస్యలు  
- ఫార్మాట్ అసమతుల్యతలు  
- LLM ప్రతిస్పందన అసాధారణతలు

### ఉత్తమ ఆచారాలు

1. **వనరు నిర్వహణ**: async context managers ఉపయోగించండి  
2. **లోపాల నిర్వహణ**: సమగ్ర try-catch బ్లాక్స్ అమలు చేయండి  
3. **లాగింగ్**: సరైన లాగింగ్ స్థాయిలను ఎనేబుల్ చేయండి  
4. **భద్రత**: ఇన్‌పుట్‌లను ధృవీకరించి అవుట్‌పుట్‌లను శుభ్రపరచండి  
5. **పనితీరు**: కనెక్షన్ పూలింగ్ మరియు క్యాచింగ్ ఉపయోగించండి

## MCP ఈకోసిస్టమ్ మరియు అభివృద్ధి

### MCP ప్రాజెక్ట్ పరిధి

మోడల్ కాంటెక్స్ట్ ప్రోటోకాల్ ఈకోసిస్టమ్‌లో కొన్ని ముఖ్య భాగాలు ఉన్నాయి:

- **[MCP స్పెసిఫికేషన్](https://modelcontextprotocol.io/specification/latest)**: క్లయింట్లు మరియు సర్వర్ల కోసం అమలు అవసరాలను వివరించే అధికారిక స్పెసిఫికేషన్  
- **[MCP SDKs](https://modelcontextprotocol.io/docs/sdk)**: MCP అమలు చేసే వివిధ ప్రోగ్రామింగ్ భాషల SDKలు  
- MCP అభివృద్ధి టూల్స్: MCP సర్వర్లు మరియు క్లయింట్లను అభివృద్ధి చేయడానికి టూల్స్, అందులో [MCP ఇన్స్పెక్టర్](https://github.com/modelcontextprotocol/inspector) కూడా ఉంది  
- **[MCP రిఫరెన్స్ సర్వర్ అమలులు](https://github.com/modelcontextprotocol/servers)**: MCP సర్వర్ల రిఫరెన్స్ అమలులు

### MCP అభివృద్ధి ప్రారంభం

MCP తో నిర్మాణం ప్రారంభించడానికి:

**సర్వర్లు నిర్మించండి**: [మీ డేటా మరియు టూల్స్‌ను ఎక్స్‌పోజ్ చేయడానికి MCP సర్వర్లు సృష్టించండి](https://modelcontextprotocol.io/docs/develop/build-server)

**క్లయింట్లు నిర్మించండి**: [MCP సర్వర్లకు కనెక్ట్ అయ్యే అప్లికేషన్లు అభివృద్ధి చేయండి](https://modelcontextprotocol.io/docs/develop/build-client)

**సిద్ధాంతాలు నేర్చుకోండి**: [MCP యొక్క కోర్ సిద్ధాంతాలు మరియు ఆర్కిటెక్చర్ అర్థం చేసుకోండి](https://modelcontextprotocol.io/docs/learn/architecture)

## ముగింపు

MCPతో ఏకీకృతమైన SLMలు AI అప్లికేషన్ అభివృద్ధిలో ఒక పరివర్తనాత్మక మార్పును సూచిస్తాయి. చిన్న మోడల్స్ సమర్థతను బాహ్య టూల్స్ శక్తితో కలిపి, డెవలపర్లు వనరుల సమర్థతతో పాటు అత్యంత సామర్థ్యవంతమైన తెలివైన వ్యవస్థలను సృష్టించగలరు.

మోడల్ కాంటెక్స్ట్ ప్రోటోకాల్ AI అప్లికేషన్లను బాహ్య వ్యవస్థలకు కనెక్ట్ చేయడానికి ఒక ప్రమాణీకృత మార్గాన్ని అందిస్తుంది, USB-C ఎలక్ట్రానిక్ పరికరాలకు యూనివర్సల్ కనెక్షన్ ప్రమాణాన్ని అందించే విధంగా. ఈ ప్రమాణీకరణ:

- **సులభమైన ఇంటిగ్రేషన్**: AI మోడల్స్‌ను విభిన్న డేటా సోర్సులు మరియు టూల్స్‌కు కనెక్ట్ చేయండి  
- **ఈకోసిస్టమ్ వృద్ధి**: ఒకసారి నిర్మించి, అనేక AI అప్లికేషన్లలో ఉపయోగించండి  
- **పెరిగిన సామర్థ్యాలు**: SLMలను బాహ్య ఫంక్షనాలిటీతో పెంపొందించండి  
- **రియల్-టైమ్ అప్‌డేట్లు**: డైనమిక్, స్పందనాత్మక AI అప్లికేషన్లకు మద్దతు

ప్రధాన అంశాలు:  
- MCP AI అప్లికేషన్లు మరియు బాహ్య వ్యవస్థల మధ్య బ్రిడ్జ్‌గా ఓపెన్ స్టాండర్డ్  
- ప్రోటోకాల్ టూల్స్, వనరులు, ప్రాంప్ట్లను కోర్ ప్రిమిటివ్స్‌గా మద్దతు ఇస్తుంది  
- రియల్-టైమ్ నోటిఫికేషన్లు డైనమిక్, స్పందనాత్మక అప్లికేషన్లను సాధ్యం చేస్తాయి  
- సరైన లైఫ్‌సైకిల్ నిర్వహణ మరియు లోపాల నిర్వహణ ఉత్పత్తి వాడుకకు అవసరం  
- ఈకోసిస్టమ్ సమగ్ర SDKలు మరియు అభివృద్ధి టూల్స్ అందిస్తుంది
