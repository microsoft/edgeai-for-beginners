# WebGPU + ONNX Runtime డెమో

ఈ డెమో బ్రౌజర్‌లో నేరుగా AI మోడల్స్‌ను WebGPU హార్డ్‌వేర్ వేగవంతీకరణ కోసం మరియు ONNX Runtime Web ఉపయోగించి ఎలా నడపాలో చూపిస్తుంది.

## ఇది ఏమి చూపిస్తుంది

- **బ్రౌజర్-ఆధారిత AI**: మోడల్స్‌ను పూర్తిగా బ్రౌజర్‌లో నడపండి  
- **WebGPU వేగవంతీకరణ**: అందుబాటులో ఉన్నప్పుడు హార్డ్‌వేర్ వేగవంతీకరణ  
- **గోప్యత-ముందస్తు**: మీ డివైస్ నుండి డేటా బయటకు వెళ్లదు  
- **జీరో ఇన్‌స్టాల్**: ఏ అనుకూల బ్రౌజర్‌లోనైనా పనిచేస్తుంది  
- **సున్నితమైనFallback**: WebGPU అందుబాటులో లేకపోతే CPUకిFallback అవుతుంది  

## అవసరాలు

**బ్రౌజర్ అనుకూలత:**  
- Chrome/Edge 113+ WebGPU ఎనేబుల్ చేయబడింది  
- WebGPU స్థితి తనిఖీ చేయండి: `chrome://gpu`  
- WebGPU ఎనేబుల్ చేయండి: `chrome://flags/#enable-unsafe-webgpu`  

## డెమో నడపడం

### ఎంపిక 1: లోకల్ సర్వర్ (సిఫార్సు చేయబడింది)

```cmd
# Navigate to the demo directory
cd Module08\samples\04\webgpu-demo

# Start a local server
python -m http.server 5173

# Open browser to http://localhost:5173
```
  
### ఎంపిక 2: VS కోడ్ లైవ్ సర్వర్

1. VS కోడ్‌లో "Live Server" ఎక్స్‌టెన్షన్ ఇన్‌స్టాల్ చేయండి  
2. `index.html` పై రైట్-క్లిక్ → "Open with Live Server"  
3. డెమో ఆటోమేటిక్‌గా బ్రౌజర్‌లో తెరుస్తుంది  

## మీరు ఏమి చూడబోతున్నారు

1. **WebGPU గుర్తింపు**: బ్రౌజర్ అనుకూలత తనిఖీ  
2. **మోడల్ లోడింగ్**: MNIST క్లాసిఫయర్ డౌన్లోడ్ చేసి ప్రారంభిస్తుంది  
3. **ఇన్ఫరెన్స్ అమలు**: నమూనా డేటాపై అంచనా నడుపుతుంది  
4. **పర్ఫార్మెన్స్ మెట్రిక్స్**: లోడ్ సమయం మరియు ఇన్ఫరెన్స్ వేగం చూపిస్తుంది  
5. **ఫలితాల ప్రదర్శన**: అంచనా నమ్మకంతో పాటు రా అవుట్పుట్లు  

## అంచనా పనితీరు

| ఎగ్జిక్యూషన్ ప్రొవైడర్ | మోడల్ లోడ్ | ఇన్ఫరెన్స్ | గమనికలు |
|-------------------|------------|-----------|-------|
| **WebGPU** | సుమారు 2-5 సెకన్లు | సుమారు 10-50 మిల్లీసెకన్లు | హార్డ్‌వేర్ వేగవంతీకరణ |
| **CPU (WASM)** | సుమారు 2-5 సెకన్లు | సుమారు 50-200 మిల్లీసెకన్లు | సాఫ్ట్‌వేర్Fallback |

## సమస్య పరిష్కారం

**WebGPU అందుబాటులో లేదు:**  
- Chrome/Edge 113+కి అప్‌డేట్ చేయండి  
- `chrome://flags`లో WebGPU ఎనేబుల్ చేయండి  
- GPU డ్రైవర్లు తాజా ఉన్నాయో తనిఖీ చేయండి  
- డెమో ఆటోమేటిక్‌గా CPUకిFallback అవుతుంది  

**లోడింగ్ లోపాలు:**  
- మీరు HTTP ద్వారా సర్వ్ చేస్తున్నారా (file:// కాదు) అని నిర్ధారించుకోండి  
- మోడల్ డౌన్లోడ్ కోసం నెట్‌వర్క్ కనెక్షన్ తనిఖీ చేయండి  
- ONNX మోడల్ CORS బ్లాక్ చేయడం లేదో తనిఖీ చేయండి  

**పర్ఫార్మెన్స్ సమస్యలు:**  
- WebGPU CPU కంటే గణనీయమైన వేగవంతీకరణ ఇస్తుంది  
- మొదటి రన్ మోడల్ డౌన్లోడ్ కారణంగా మందగిస్తుంది  
- తరువాతి రన్స్ బ్రౌజర్ క్యాష్ ఉపయోగిస్తాయి  

## Foundry Localతో ఇంటిగ్రేషన్

ఈ WebGPU డెమో Foundry Localను ఈ విధంగా పూర్తి చేస్తుంది:

- **క్లయింట్-సైడ్ ఇన్ఫరెన్స్** గోప్యతకు పరమ  
- **ఆఫ్‌లైన్ సామర్థ్యాలు** ఇంటర్నెట్ లేనప్పుడు  
- **ఎడ్జ్ డిప్లాయ్‌మెంట్** వనరుల పరిమిత వాతావరణాలకు  
- **హైబ్రిడ్ ఆర్కిటెక్చర్స్** లోకల్ మరియు సర్వర్ ఇన్ఫరెన్స్ కలిపి  

ఉత్పత్తి అప్లికేషన్ల కోసం, పరిగణించండి:  
- సర్వర్-సైడ్ ఇన్ఫరెన్స్ కోసం Foundry Local ఉపయోగించండి  
- క్లయింట్-సైడ్ ప్రీప్రాసెసింగ్/పోస్ట్‌ప్రాసెసింగ్ కోసం WebGPU ఉపయోగించండి  
- లోకల్/రిమోట్ ఇన్ఫరెన్స్ మధ్య తెలివైన రౌటింగ్ అమలు చేయండి  

## సాంకేతిక వివరాలు

**ఉపయోగించిన మోడల్:**  
- MNIST అంకెల క్లాసిఫయర్ (ONNX ఫార్మాట్)  
- ఇన్‌పుట్: 28x28 గ్రేస్కేల్ చిత్రాలు  
- అవుట్పుట్: 10-వర్గాల ప్రాబబిలిటీ పంపిణీ  
- పరిమాణం: సుమారు 500KB (త్వరిత డౌన్లోడ్)  

**ONNX Runtime Web:**  
- GPU వేగవంతీకరణ కోసం WebGPU ఎగ్జిక్యూషన్ ప్రొవైడర్  
- CPUFallback కోసం WASM ఎగ్జిక్యూషన్ ప్రొవైడర్  
- ఆటోమేటిక్ ఆప్టిమైజేషన్ మరియు గ్రాఫ్ ఆప్టిమైజేషన్  

**బ్రౌజర్ APIs:**  
- హార్డ్‌వేర్ యాక్సెస్ కోసం WebGPU  
- బ్యాక్‌గ్రౌండ్ ప్రాసెసింగ్ కోసం Web Workers (భవిష్యత్ అభివృద్ధి)  
- సమర్థవంతమైన గణన కోసం WebAssembly  

## తదుపరి దశలు

- కస్టమ్ ONNX మోడల్స్‌తో ప్రయత్నించండి  
- నిజమైన చిత్రం అప్‌లోడ్ మరియు వర్గీకరణ అమలు చేయండి  
- పెద్ద మోడల్స్ కోసం స్ట్రీమింగ్ ఇన్ఫరెన్స్ జోడించండి  
- కెమెరా/మైక్రోఫోన్ ఇన్‌పుట్‌తో ఇంటిగ్రేట్ చేయండి
