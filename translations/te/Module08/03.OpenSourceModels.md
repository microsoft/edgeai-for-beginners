# సెషన్ 3: ఓపెన్-సోర్స్ మోడల్ డిస్కవరీ మరియు మేనేజ్‌మెంట్

## అవలోకనం

ఈ సెషన్ ఫౌండ్రీ లోకల్‌తో ప్రాక్టికల్ మోడల్ డిస్కవరీ మరియు మేనేజ్‌మెంట్‌పై కేంద్రీకృతమైంది. మీరు అందుబాటులో ఉన్న మోడల్స్‌ను ఎలా జాబితా చేయాలో, వివిధ ఎంపికలను పరీక్షించాలో, మరియు ప్రాథమిక పనితీరు లక్షణాలను ఎలా అర్థం చేసుకోవాలో నేర్చుకుంటారు. ఈ విధానం ఫౌండ్రీ CLIతో హ్యాండ్స్-ఆన్ అన్వేషణను ప్రాధాన్యం ఇస్తుంది, తద్వారా మీ ఉపయోగాల కోసం సరైన మోడల్స్‌ను ఎంచుకోవడంలో సహాయపడుతుంది.

## నేర్చుకునే లక్ష్యాలు

- మోడల్ డిస్కవరీ మరియు మేనేజ్‌మెంట్ కోసం ఫౌండ్రీ CLI కమాండ్లలో నైపుణ్యం పొందడం  
- మోడల్ క్యాష్ మరియు లోకల్ స్టోరేజ్ నమూనాలను అర్థం చేసుకోవడం  
- వివిధ మోడల్స్‌ను త్వరగా పరీక్షించి పోల్చడం నేర్చుకోవడం  
- మోడల్ ఎంపిక మరియు బెంచ్‌మార్కింగ్ కోసం ప్రాక్టికల్ వర్క్‌ఫ్లోలను స్థాపించడం  
- ఫౌండ్రీ లోకల్ ద్వారా అందుబాటులో ఉన్న పెరుగుతున్న మోడల్ ఎకోసిస్టమ్‌ను అన్వేషించడం  

## ముందస్తు అవసరాలు

- సెషన్ 1: ఫౌండ్రీ లోకల్‌తో ప్రారంభించడం పూర్తి చేయడం  
- ఫౌండ్రీ లోకల్ CLI ఇన్‌స్టాల్ చేసి యాక్సెస్ చేయగలగడం  
- మోడల్ డౌన్‌లోడ్ల కోసం సరిపడా నిల్వ స్థలం (మోడల్స్ 1GB నుండి 20GB+ వరకు ఉండవచ్చు)  
- మోడల్ రకాల మరియు ఉపయోగాల ప్రాథమిక అవగాహన-ఫౌండ్రీ లోకల్‌తో సోర్స్ మోడల్స్  

## అవలోకనం

ఈ సెషన్ ఓపెన్-సోర్స్ మోడల్స్‌ను ఫౌండ్రీ L## భాగం 6: హ్యాండ్స్-ఆన్ వ్యాయామం

### వ్యాయామం: మోడల్ డిస్కవరీ మరియు పోలిక

సాంపిల్ 03 ఆధారంగా మీ స్వంత మోడల్ మూల్యాంకన స్క్రిప్ట్ సృష్టించండి:

```cmd
REM create_model_test.cmd
@echo off
echo Model Discovery and Testing Script
echo =====================================

echo.
echo Step 1: List available models
foundry model list

echo.
echo Step 2: Check what's cached
foundry cache list

echo.
echo Step 3: Start phi-4-mini for testing
foundry model run phi-4-mini --verbose

echo.
echo Step 4: Test with a simple prompt
curl -X POST http://localhost:8000/v1/chat/completions ^
  -H "Content-Type: application/json" ^
  -d "{\"model\":\"phi-4-mini\",\"messages\":[{\"role\":\"user\",\"content\":\"Hello, please introduce yourself.\"}],\"max_tokens\":100}"

echo.
echo Model test complete!
```

### మీ పని

1. **సాంపిల్ 03 స్క్రిప్ట్‌ను నడపండి**: `samples\03\list_and_bench.cmd`  
2. **వివిధ మోడల్స్‌ను ప్రయత్నించండి**: కనీసం 3 వేర్వేరు మోడల్స్‌ను పరీక్షించండి  
3. **పనితీరు పోల్చండి**: వేగం మరియు ప్రతిస్పందన నాణ్యతలో తేడాలను గమనించండి  
4. **ఫలితాలను డాక్యుమెంట్ చేయండి**: ఒక సులభమైన పోలిక చార్ట్ సృష్టించండి  

### ఉదాహరణ పోలిక ఫార్మాట్

## భాగం 7: సమస్య పరిష్కారం మరియు ఉత్తమ ఆచారాలు

### సాధారణ సమస్యలు మరియు పరిష్కారాలు

**మోడల్ ప్రారంభం కావడం లేదు:**  
```cmd
REM Check service status
foundry service status

REM Restart service if needed
foundry service stop
foundry service start

REM Try with verbose output
foundry model run phi-4-mini --verbose
```
  
**స్మృతి తక్కువగా ఉంది:**  
- చిన్న మోడల్స్‌తో ప్రారంభించండి (`phi-4-mini`)  
- ఇతర అప్లికేషన్లను మూసివేయండి  
- తరచుగా పరిమితులను ఎదుర్కొంటే RAM ను అప్‌గ్రేడ్ చేయండి  

**మందగతి పనితీరు:**  
- మోడల్ పూర్తిగా లోడ్ అయ్యిందని నిర్ధారించుకోండి (వివరణాత్మక అవుట్‌పుట్‌ను తనిఖీ చేయండి)  
- అవసరం లేని బ్యాక్‌గ్రౌండ్ అప్లికేషన్లను మూసివేయండి  
- వేగవంతమైన నిల్వ (SSD) పరిగణించండి  

### ఉత్తమ ఆచారాలు

1. **చిన్నదిగా ప్రారంభించండి**: సెటప్‌ను ధృవీకరించడానికి `phi-4-mini` తో ప్రారంభించండి  
2. **ఒకేసారి ఒక మోడల్**: కొత్త మోడల్స్ ప్రారంభించే ముందు పాత మోడల్స్‌ను ఆపండి  
3. **వనరులను పర్యవేక్షించండి**: స్మృతి వినియోగాన్ని గమనించండి  
4. **నిరంతరం పరీక్షించండి**: సమానమైన ప్రాంప్ట్‌లను ఉపయోగించి న్యాయమైన పోలికలు చేయండి  
5. **ఫలితాలను డాక్యుమెంట్ చేయండి**: మీ ఉపయోగాల కోసం మోడల్ పనితీరు గురించి గమనికలు ఉంచండి

## భాగం 1: ఫౌండ్రీ CLIతో మోడల్ డిస్కవరీ

### ప్రాథమిక మోడల్ మేనేజ్‌మెంట్ కమాండ్లు

ఫౌండ్రీ CLI మోడల్ డిస్కవరీ మరియు మేనేజ్‌మెంట్ కోసం సూటిగా కమాండ్లను అందిస్తుంది:

```cmd
REM List all available models in the catalog
foundry model list

REM List cached (downloaded) models
foundry cache list

REM Check cache directory location
foundry cache ls
```
  
### మీ మొదటి మోడల్స్ నడపడం

పనితీరు లక్షణాలను అర్థం చేసుకోవడానికి ప్రాచుర్యం పొందిన, బాగా పరీక్షించిన మోడల్స్‌తో ప్రారంభించండి:

```cmd
REM Run Phi-4-Mini (lightweight, fast)
foundry model run phi-4-mini --verbose

REM Run Qwen 2.5 7B (larger, more capable)
foundry model run qwen2.5-7b --verbose

REM Run DeepSeek (specialized for coding)
foundry model run deepseek-r1-7b --verbose
```
  
**గమనిక:** `--verbose` ఫ్లాగ్ ప్రారంభ సమాచారం వివరాలను అందిస్తుంది, అందులో:  
- మోడల్ డౌన్‌లోడ్ పురోగతి (మొదటి నడుపులో)  
- స్మృతి కేటాయింపు వివరాలు  
- సర్వీస్ బైండింగ్ సమాచారం  
- పనితీరు ప్రారంభీకరణ గణాంకాలు  

### మోడల్ వర్గాల అర్థం చేసుకోవడం

**చిన్న భాషా మోడల్స్ (SLMs):**  
- `phi-4-mini`: వేగవంతమైనది, సమర్థవంతమైనది, సాధారణ చాట్ కోసం గొప్పది  
- `phi-4`: మెరుగైన తర్కంతో మరింత సామర్థ్యం కలిగిన వెర్షన్  

**మధ్యస్థ మోడల్స్:**  
- `qwen2.5-7b`: అద్భుతమైన తర్కం మరియు పొడవైన సందర్భం  
- `deepseek-r1-7b`: కోడ్ జనరేషన్ కోసం ఆప్టిమైజ్ చేయబడింది  

**పెద్ద మోడల్స్:**

## భాగం 5: సులభమైన మోడల్ బెంచ్‌మార్కింగ్

### ప్రాథమిక పనితీరు పరీక్ష

మోడల్ పనితీరు పోల్చడానికి సూటిగా ఈ విధానం ఉంది:

```python
# simple_bench.py - నమూనా 03 నమూనాల ఆధారంగా
import time
import requests
import json

def test_model_response(model_name, prompt="Explain edge AI in one sentence."):
    """Test a single model with a prompt and measure response time."""
    start_time = time.time()
    
    try:
        response = requests.post(
            "http://localhost:8000/v1/chat/completions",
            headers={"Content-Type": "application/json"},
            json={
                "model": model_name,
                "messages": [{"role": "user", "content": prompt}],
                "max_tokens": 64
            },
            timeout=30
        )
        
        elapsed = time.time() - start_time
        
        if response.status_code == 200:
            result = response.json()
            return {
                "model": model_name,
                "latency_sec": round(elapsed, 3),
                "response": result["choices"][0]["message"]["content"],
                "status": "success"
            }
        else:
            return {
                "model": model_name,
                "status": "error",
                "error": f"HTTP {response.status_code}"
            }
            
    except Exception as e:
        return {
            "model": model_name,
            "status": "error", 
            "error": str(e)
        }

# ప్రస్తుతం నడుస్తున్న మోడల్‌ను పరీక్షించండి
if __name__ == "__main__":
    # వేర్వేరు మోడల్స్‌తో పరీక్షించండి (ప్రతి మోడల్‌ను ముందుగా ప్రారంభించండి)
    test_models = ["phi-4-mini", "qwen2.5-7b", "deepseek-r1-7b"]
    
    print("Model Performance Test")
    print("=" * 50)
    
    for model in test_models:
        print(f"\nTesting {model}...")
        print("Note: Make sure this model is running first with 'foundry model run {model}'")
        
        result = test_model_response(model)
        
        if result["status"] == "success":
            print(f"✅ {model}: {result['latency_sec']}s")
            print(f"   Response: {result['response'][:100]}...")
        else:
            print(f"❌ {model}: {result['error']}")
```
