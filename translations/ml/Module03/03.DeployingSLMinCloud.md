## Phi-4-mini കണ്ടെയ്‌നർ ഡിപ്ലോയ്മെന്റിലേക്ക് പരിചയം

സ്മോൾ ലാംഗ്വേജ് മോഡലുകൾ (SLMs) എഡ്ജ് AI-യിൽ ഒരു നിർണായക പുരോഗതിയാണ്, സ്രോതസ്സ്-പരിമിത ഉപകരണങ്ങളിൽ സങ്കീർണ്ണമായ നാചുറൽ ലാംഗ്വേജ് പ്രോസസ്സിംഗ് കഴിവുകൾ സാധ്യമാക്കുന്നു. മൈക്രോസോഫ്റ്റിന്റെ Phi-4-mini-instruct എന്ന സ്റ്റേറ്റ്-ഓഫ്-ദി-ആർട്ട് റീസണിംഗ് മോഡലിന്റെ കണ്ടെയ്‌നറൈസ്ഡ് ഡിപ്ലോയ്മെന്റ് തന്ത്രങ്ങളിലാണ് ഈ ട്യൂട്ടോറിയൽ കേന്ദ്രീകരിക്കുന്നത്, കഴിവും കാര്യക്ഷമതയും തമ്മിൽ സമതുല്യം പുലർത്തുന്നു.

### ഫീച്ചർ ചെയ്ത മോഡൽ: Phi-4-mini-instruct

**Phi-4-mini-instruct (3.8B പാരാമീറ്ററുകൾ)**: മെമ്മറി/കമ്പ്യൂട്ട് പരിമിത പരിസ്ഥിതികൾക്കായി രൂപകൽപ്പന ചെയ്ത മൈക്രോസോഫ്റ്റിന്റെ ഏറ്റവും പുതിയ ലൈറ്റ്‌വെയിറ്റ് ഇൻസ്ട്രക്ഷൻ-ട്യൂൺഡ് മോഡൽ, അതുല്യ കഴിവുകളോടെ:
- **ഗണിതീയ റീസണിംഗ്, സങ്കീർണ്ണ കണക്കുകൾ**
- **കോഡ് ജനറേഷൻ, ഡീബഗ്ഗിംഗ്, വിശകലനം**
- **ലോജിക്കൽ പ്രശ്ന പരിഹാരം, ഘട്ടം ഘട്ടമായി റീസണിംഗ്**
- **വിശദമായ വിശദീകരണങ്ങൾ ആവശ്യമായ വിദ്യാഭ്യാസ പ്രയോഗങ്ങൾ**
- **ഫംഗ്ഷൻ കോളിംഗ്, ടൂൾ ഇന്റഗ്രേഷൻ**

"Small SLMs" വിഭാഗത്തിൽ (1.5B - 13.9B പാരാമീറ്ററുകൾ) Phi-4-mini റീസണിംഗ് കഴിവും സ്രോതസ്സ് കാര്യക്ഷമതയും തമ്മിൽ മികച്ച സമതുല്യം പുലർത്തുന്നു.

### Phi-4-mini കണ്ടെയ്‌നറൈസ്ഡ് ഡിപ്ലോയ്മെന്റിന്റെ ഗുണങ്ങൾ

- **ഓപ്പറേഷണൽ കാര്യക്ഷമത**: കുറഞ്ഞ കംപ്യൂട്ടേഷൻ ആവശ്യകതയോടെ റീസണിംഗ് ടാസ്കുകൾക്ക് വേഗത്തിലുള്ള ഇൻഫറൻസ്
- **ഡിപ്ലോയ്മെന്റ് ഫ്ലെക്സിബിലിറ്റി**: പ്രൈവസി മെച്ചപ്പെടുത്തുന്ന ലോക്കൽ പ്രോസസ്സിങ്ങിലൂടെ ഓൺ-ഡിവൈസ് AI കഴിവുകൾ
- **ചെലവ് കാര്യക്ഷമത**: വലിയ മോഡലുകളെ അപേക്ഷിച്ച് കുറഞ്ഞ ഓപ്പറേഷണൽ ചെലവിൽ ഗുണമേന്മ നിലനിർത്തുന്നു
- **ഐസൊലേഷൻ**: മോഡൽ ഇൻസ്റ്റൻസുകൾക്കിടയിൽ ശുദ്ധമായ വേർതിരിവും സുരക്ഷിത എക്സിക്യൂഷൻ പരിസ്ഥിതികളും
- **സ്കെയിലബിലിറ്റി**: വർദ്ധിച്ച റീസണിംഗ് ത്രൂപുട്ടിനായി എളുപ്പത്തിൽ ഹോറിസോണ്ടൽ സ്കെയിലിംഗ്

## പഠന ലക്ഷ്യങ്ങൾ

ഈ ട്യൂട്ടോറിയൽ അവസാനിപ്പിക്കുമ്പോൾ, നിങ്ങൾക്ക് കഴിയും:

- വിവിധ കണ്ടെയ്‌നറൈസ്ഡ് പരിസ്ഥിതികളിൽ Phi-4-mini-instruct ഡിപ്ലോയ് ചെയ്യാനും ഓപ്റ്റിമൈസ് ചെയ്യാനും
- വ്യത്യസ്ത ഡിപ്ലോയ്മെന്റ് സാഹചര്യങ്ങൾക്ക് അനുയോജ്യമായ ക്വാണ്ടൈസേഷൻ, കംപ്രഷൻ തന്ത്രങ്ങൾ നടപ്പിലാക്കാനും
- റീസണിംഗ് വർക്ക്‌ലോഡുകൾക്കായി പ്രൊഡക്ഷൻ-റെഡി കണ്ടെയ്‌നർ ഓർക്കസ്ട്രേഷൻ കോൺഫിഗർ ചെയ്യാനും
- പ്രത്യേക ഉപയോഗ കേസുകളുടെ ആവശ്യകതകൾ അടിസ്ഥാനമാക്കി അനുയോജ്യമായ ഡിപ്ലോയ്മെന്റ് ഫ്രെയിംവർക്ക് തിരഞ്ഞെടുക്കാനും
- കണ്ടെയ്‌നറൈസ്ഡ് SLM ഡിപ്ലോയ്മെന്റുകൾക്കായി സുരക്ഷ, മോണിറ്ററിംഗ്, സ്കെയിലിംഗ് മികച്ച പ്രാക്ടീസുകൾ പ്രയോഗിക്കാനും

## Phi-4-mini ക്ലാസിഫിക്കേഷൻ മനസ്സിലാക്കൽ

### മോഡൽ സവിശേഷതകൾ

**സാങ്കേതിക വിശദാംശങ്ങൾ:**
- **പാരാമീറ്ററുകൾ**: 3.8 ബില്യൺ (Small SLM വിഭാഗം)
- **ആർക്കിടെക്ചർ**: ഗ്രൂപ്പ്ഡ്-ക്വറി അറ്റൻഷൻ ഉള്ള ഡെൻസും ഡികോഡർ-ഓൺലി ട്രാൻസ്ഫോർമർ
- **കോൺടെക്സ്റ്റ് നീളം**: 128K ടോക്കൺസ് (മികച്ച പ്രകടനത്തിനായി 32K ശുപാർശ)
- **വോകാബുലറി**: 200K ടോക്കൺസ്, ബഹുഭാഷാ പിന്തുണയോടെ
- **ട്രെയിനിംഗ് ഡാറ്റ**: 5T ടോക്കൺസ് ഉയർന്ന ഗുണമേന്മയുള്ള റീസണിംഗ്-സാന്ദ്ര ഉള്ളടക്കം

### സ്രോതസ്സ് ആവശ്യകതകൾ

| ഡിപ്ലോയ്മെന്റ് തരം | കുറഞ്ഞ RAM | ശുപാർശ ചെയ്ത RAM | VRAM (GPU) | സ്റ്റോറേജ് | സാധാരണ ഉപയോഗ കേസുകൾ |
|-----------------|---------|-----------------|------------|---------|-------------------|
| **ഡെവലപ്പ്മെന്റ്** | 6GB | 8GB | - | 8GB | ലോക്കൽ ടെസ്റ്റിംഗ്, പ്രോട്ടോടൈപ്പിംഗ് |
| **പ്രൊഡക്ഷൻ CPU** | 8GB | 12GB | - | 10GB | എഡ്ജ് സർവറുകൾ, ചെലവ്-ഓപ്റ്റിമൈസ്ഡ് ഡിപ്ലോയ്മെന്റ് |
| **പ്രൊഡക്ഷൻ GPU** | 6GB | 8GB | 4-6GB | 8GB | ഉയർന്ന ത്രൂപുട്ട് റീസണിംഗ് സർവീസുകൾ |
| **എഡ്ജ് ഓപ്റ്റിമൈസ്ഡ്** | 4GB | 6GB | - | 6GB | ക്വാണ്ടൈസ്ഡ് ഡിപ്ലോയ്മെന്റ്, IoT ഗേറ്റ്വേകൾ |

### Phi-4-mini കഴിവുകൾ

- **ഗണിതീയ മികവ്**: പുരോഗമിച്ച അങ്കഗണിതം, ആൽജിബ്ര, കാൽക്കുലസ് പ്രശ്ന പരിഹാരം
- **കോഡ് ഇന്റലിജൻസ്**: പൈത്തൺ, ജാവാസ്ക്രിപ്റ്റ്, ബഹുഭാഷാ കോഡ് ജനറേഷൻ, ഡീബഗ്ഗിംഗ്
- **ലോജിക്കൽ റീസണിംഗ്**: ഘട്ടം ഘട്ടമായി പ്രശ്ന വിഭജനം, പരിഹാര നിർമ്മാണം
- **വിദ്യാഭ്യാസ പിന്തുണ**: പഠനത്തിനും പഠിപ്പിക്കലിനും അനുയോജ്യമായ വിശദമായ വിശദീകരണങ്ങൾ
- **ഫംഗ്ഷൻ കോളിംഗ്**: ടൂൾ ഇന്റഗ്രേഷൻ, API ഇടപെടലുകൾക്ക് നേറ്റീവ് പിന്തുണ

## vLLM കണ്ടെയ്‌നർ ഡിപ്ലോയ്മെന്റ്

vLLM Phi-4-mini-instruct-ന് മികച്ച പിന്തുണ നൽകുന്നു, ഓപ്റ്റിമൈസ്ഡ് ഇൻഫറൻസ് പ്രകടനവും OpenAI-സമാനമായ API-കളും ഉണ്ട്, പ്രൊഡക്ഷൻ റീസണിംഗ് സർവീസുകൾക്കായി ഇത് അനുയോജ്യമാണ്.

### ക്വിക്ക് സ്റ്റാർട്ട് ഉദാഹരണങ്ങൾ

#### അടിസ്ഥാന CPU ഡിപ്ലോയ്മെന്റ് (ഡെവലപ്പ്മെന്റ്)
```bash
# വികസനത്തിനും പരിശോധനയ്ക്കുമായി CPU-ഓപ്റ്റിമൈസ്ഡ് ഡിപ്ലോയ്മെന്റ്
docker run --name phi4-mini-dev \
  -e HUGGING_FACE_HUB_TOKEN=${HF_TOKEN} \
  -p 8000:8000 \
  --memory="8g" --cpus="4" \
  vllm/vllm-openai:latest \
  --model microsoft/Phi-4-mini-instruct \
  --max-model-len 4096 \
  --max-num-seqs 4 \
  --trust-remote-code
```

#### GPU-അക്സിലറേറ്റഡ് പ്രൊഡക്ഷൻ ഡിപ്ലോയ്മെന്റ്
```bash
# ഉയർന്ന പ്രകടനമുള്ള നിരൂപണത്തിനായി GPU വിന്യാസം
docker run --runtime nvidia --gpus all \
  --name phi4-mini-prod \
  -e HUGGING_FACE_HUB_TOKEN=${HF_TOKEN} \
  -p 8000:8000 \
  vllm/vllm-openai:latest \
  --model microsoft/Phi-4-mini-instruct \
  --max-model-len 8192 \
  --gpu-memory-utilization 0.8 \
  --enable-auto-tool-choice \
  --trust-remote-code
```

### പ്രൊഡക്ഷൻ കോൺഫിഗറേഷൻ

```yaml
version: '3.8'
services:
  phi4-mini-reasoning:
    image: vllm/vllm-openai:latest
    container_name: phi4-mini-production
    ports:
      - "8000:8000"
    volumes:
      - ~/.cache/huggingface:/root/.cache/huggingface
      - ./logs:/app/logs
    environment:
      - HUGGING_FACE_HUB_TOKEN=${HF_TOKEN}
      - CUDA_VISIBLE_DEVICES=0
    command: >
      --model microsoft/Phi-4-mini-instruct
      --host 0.0.0.0
      --port 8000
      --max-model-len 4096
      --max-num-seqs 8
      --gpu-memory-utilization 0.8
      --trust-remote-code
      --enable-auto-tool-choice
      --quantization awq
    deploy:
      resources:
        limits:
          memory: 12G
        reservations:
          devices:
            - driver: nvidia
              count: 1
              capabilities: [gpu]
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 30s
      timeout: 10s
      retries: 3
```

### Phi-4-mini റീസണിംഗ് കഴിവുകൾ ടെസ്റ്റ് ചെയ്യൽ

```bash
# ഗണിത തർക്കം പരീക്ഷിക്കുക
curl http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "microsoft/Phi-4-mini-instruct",
    "messages": [
      {"role": "system", "content": "You are a mathematical reasoning assistant. Show your work step by step."},
      {"role": "user", "content": "A train travels 240 km in 3 hours. If it increases its speed by 20 km/h, how long would the same journey take?"}
    ],
    "max_tokens": 200,
    "temperature": 0.3
  }'

# കോഡ് സൃഷ്ടി പരീക്ഷിക്കുക
curl http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "microsoft/Phi-4-mini-instruct",
    "messages": [
```

### ലളിതമാക്കിയ സർവർ ഇംപ്ലിമെന്റേഷൻ

```python
# app/server.py - Phi-4-mini വിവേക പ്രവർത്തനങ്ങൾക്കായി മെച്ചപ്പെടുത്തിയിരിക്കുന്നു
import os
import time
import onnxruntime as ort
from transformers import AutoTokenizer
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI(title="Phi-4-mini ONNX Engine")

class ReasoningRequest(BaseModel):
    prompt: str
    task_type: str = "reasoning"  # വിവേകം, കോഡിംഗ്, ഗണിതം
    max_length: int = 200
    temperature: float = 0.3

class Phi4MiniEngine:
    def __init__(self):
        self.model = None
        self.tokenizer = None
        self.load_model()
    
    def load_model(self):
        model_path = "/app/models/phi4-mini-instruct.onnx"
        if os.path.exists(model_path):
            # വിവേക പ്രവർത്തനങ്ങൾക്കായി മെച്ചപ്പെടുത്തിയ പ്രൊവൈഡറുകൾ
            providers = [
                ('CUDAExecutionProvider', {
                    'arena_extend_strategy': 'kSameAsRequested',
                    'cudnn_conv_algo_search': 'HEURISTIC',
                }),
                ('CPUExecutionProvider', {
                    'intra_op_num_threads': 4,
                    'inter_op_num_threads': 2,
                })
            ]
            
            self.model = ort.InferenceSession(model_path, providers=providers)
            self.tokenizer = AutoTokenizer.from_pretrained("microsoft/Phi-4-mini-instruct")
            print("✓ Phi-4-mini model loaded successfully")
        else:
            print("✗ Model file not found. Please convert the model first.")
    
    def generate_reasoning(self, request: ReasoningRequest):
        if not self.model:
            raise ValueError("Model not loaded")
        
        # മെച്ചപ്പെട്ട വിവേകത്തിനായി ടാസ്ക്-നിർദ്ദിഷ്ട പ്രോംപ്റ്റിംഗ്
        task_prompts = {
            "reasoning": "Think step by step and show your reasoning clearly:",
            "math": "Solve this mathematical problem step by step:",
            "coding": "Write clean, well-commented code for this task:"
        }
        
        system_prompt = task_prompts.get(request.task_type, "")
        full_prompt = f"{system_prompt}\n{request.prompt}"
        
        # ടോക്കൺ ചെയ്യുക, ഇൻഫറൻസ് നടത്തുക
        inputs = self.tokenizer.encode(full_prompt, return_tensors="np", max_length=2048, truncation=True)
        
        start_time = time.time()
        outputs = self.model.run(None, {"input_ids": inputs})
        inference_time = time.time() - start_time
        
        # പ്രതികരണം ഡീകോഡ് ചെയ്യുക
        generated_text = self.tokenizer.decode(outputs[0][0], skip_special_tokens=True)
        
        return {
            "generated_text": generated_text,
            "task_type": request.task_type,
            "inference_time": inference_time,
            "model": "phi4-mini-instruct-onnx"
        }

# എഞ്ചിൻ ആരംഭിക്കുക
engine = Phi4MiniEngine()

@app.post("/reasoning")
async def generate_reasoning(request: ReasoningRequest):
    try:
        return engine.generate_reasoning(request)
    except Exception as e:
        return {"error": str(e)}

@app.get("/health")
async def health():
    return {
        "status": "healthy" if engine.model else "model_not_loaded",
        "model": "phi4-mini-instruct",
        "capabilities": ["reasoning", "math", "coding"]
    }

@app.get("/")
async def root():
    return {
        "name": "Phi-4-mini ONNX Engine",
        "model": "microsoft/Phi-4-mini-instruct",
        "endpoints": ["/reasoning", "/health"],
        "capabilities": ["mathematical reasoning", "code generation", "logical problem solving"]
    }
```

### മോഡൽ കൺവർഷൻ സ്ക്രിപ്റ്റ്

```python
# convert_phi4_mini.py - Phi-4-mini നെ ഓപ്റ്റിമൈസ്ഡ് ONNX ആയി മാറ്റുക
import os
from pathlib import Path
from optimum.onnxruntime import ORTModelForCausalLM, ORTOptimizer, ORTQuantizer
from optimum.onnxruntime.configuration import AutoQuantizationConfig, AutoOptimizationConfig
from transformers import AutoTokenizer

def convert_phi4_mini():
    print("Converting Phi-4-mini-instruct to optimized ONNX...")
    
    model_name = "microsoft/Phi-4-mini-instruct"
    output_dir = Path("./models/phi4-mini-onnx")
    output_dir.mkdir(parents=True, exist_ok=True)
    
    # ഘട്ടം 1: ONNX ആയി മാറ്റുക
    print("Step 1: Converting to ONNX format...")
    model = ORTModelForCausalLM.from_pretrained(
        model_name,
        export=True,
        provider="CPUExecutionProvider",
        use_cache=True
    )
    
    # ഘട്ടം 2: റീസണിംഗ് ടാസ്കുകൾക്കായി ഓപ്റ്റിമൈസേഷനുകൾ പ്രയോഗിക്കുക
    print("Step 2: Applying reasoning-specific optimizations...")
    optimization_config = AutoOptimizationConfig.with_optimization_level(
        optimization_level="O3",
        optimize_for_gpu=True,
        fp16=True
    )
    
    optimizer = ORTOptimizer.from_pretrained(model)
    optimizer.optimize(save_dir=output_dir, optimization_config=optimization_config)
    
    # ഘട്ടം 3: എഡ്ജ് ഡിപ്ലോയ്മെന്റിനായി ക്വാണ്ടൈസേഷൻ പ്രയോഗിക്കുക
    print("Step 3: Applying quantization...")
    quantization_config = AutoQuantizationConfig.avx512_vnni(
        is_static=False,
        per_channel=True
    )
    
    quantizer = ORTQuantizer.from_pretrained(output_dir)
    quantizer.quantize(
        save_dir=output_dir / "quantized",
        quantization_config=quantization_config
    )
    
    # ഘട്ടം 4: ടോക്കനൈസർ സേവ് ചെയ്യുക
    tokenizer = AutoTokenizer.from_pretrained(model_name)
    tokenizer.save_pretrained(output_dir)
    
    # ഘട്ടം 5: അന്തിമ ഓപ്റ്റിമൈസ്ഡ് മോഡൽ സൃഷ്ടിക്കുക
    final_model_path = Path("./models/phi4-mini-instruct.onnx")
    quantized_files = list((output_dir / "quantized").glob("*.onnx"))
    if quantized_files:
        import shutil
        shutil.copy2(quantized_files[0], final_model_path)
        print(f"✓ Phi-4-mini converted and optimized: {final_model_path}")
    
    return final_model_path

if __name__ == "__main__":
    convert_phi4_mini()
```

- **OpenAI API സ്പെസിഫിക്കേഷൻ**: vLLM അനുയോജ്യത പരിശോധനയ്ക്ക്
- **Docker മികച്ച പ്രാക്ടീസുകൾ**: കണ്ടെയ്‌നർ സുരക്ഷയും ഓപ്റ്റിമൈസേഷനും
- **Kubernetes ഡിപ്ലോയ്മെന്റ്**: പ്രൊഡക്ഷൻ സ്കെയിലിംഗിനുള്ള ഓർക്കസ്ട്രേഷൻ പാറ്റേണുകൾ

### പഠന സ്രോതസ്സ്
- **SLM പ്രകടന ബെഞ്ച്മാർക്കിംഗ്**: താരതമ്യ വിശകലന രീതികൾ
- **എഡ്ജ് AI ഡിപ്ലോയ്മെന്റ്**: സ്രോതസ്സ്-പരിമിത പരിസ്ഥിതികൾക്കുള്ള മികച്ച പ്രാക്ടീസുകൾ
- **റീസണിംഗ് ടാസ്‌ക് ഓപ്റ്റിമൈസേഷൻ**: ഗണിത, ലോജിക്കൽ പ്രശ്നങ്ങൾക്ക് പ്രോംപ്റ്റിംഗ് തന്ത്രങ്ങൾ
- **കണ്ടെയ്‌നർ സുരക്ഷ**: AI മോഡൽ ഡിപ്ലോയ്മെന്റുകൾക്കുള്ള ഹാർഡനിംഗ് പ്രാക്ടീസുകൾ

## പഠന ഫലങ്ങൾ

ഈ മോഡ്യൂൾ പൂർത്തിയാക്കിയ ശേഷം, നിങ്ങൾക്ക് കഴിയും:

1. വിവിധ ഫ്രെയിംവർക്കുകൾ ഉപയോഗിച്ച് Phi-4-mini-instruct മോഡൽ കണ്ടെയ്‌നറൈസ്ഡ് പരിസ്ഥിതികളിൽ ഡിപ്ലോയ് ചെയ്യാൻ
2. വ്യത്യസ്ത ഹാർഡ്‌വെയർ പരിസ്ഥിതികൾക്കായി SLM ഡിപ്ലോയ്മെന്റുകൾ കോൺഫിഗർ ചെയ്യാനും ഓപ്റ്റിമൈസ് ചെയ്യാനും
3. കണ്ടെയ്‌നറൈസ്ഡ് AI ഡിപ്ലോയ്മെന്റുകൾക്കായി സുരക്ഷ മികച്ച പ്രാക്ടീസുകൾ നടപ്പിലാക്കാനും
4. പ്രത്യേക ഉപയോഗ കേസുകളുടെ ആവശ്യകതകൾ അടിസ്ഥാനമാക്കി അനുയോജ്യമായ ഡിപ്ലോയ്മെന്റ് ഫ്രെയിംവർക്ക് താരതമ്യം ചെയ്ത് തിരഞ്ഞെടുക്കാനും
5. പ്രൊഡക്ഷൻ-ഗ്രേഡ് SLM സർവീസുകൾക്കായി മോണിറ്ററിംഗ്, സ്കെയിലിംഗ് തന്ത്രങ്ങൾ പ്രയോഗിക്കാനും
