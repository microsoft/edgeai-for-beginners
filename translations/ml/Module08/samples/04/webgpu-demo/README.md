# WebGPU + ONNX Runtime ഡെമോ

ഈ ഡെമോ WebGPU ഉപയോഗിച്ച് ഹാർഡ്‌വെയർ ആക്സിലറേഷനുമായി ബ്രൗസറിൽ നേരിട്ട് AI മോഡലുകൾ എങ്ങനെ ഓടിക്കാമെന്ന് കാണിക്കുന്നു, കൂടാതെ ONNX Runtime Web ഉപയോഗിക്കുന്നു.

## ഇത് എന്താണ് പ്രദർശിപ്പിക്കുന്നത്

- **ബ്രൗസർ അടിസ്ഥാനമാക്കിയ AI**: മോഡലുകൾ പൂർണ്ണമായും ബ്രൗസറിൽ ഓടിക്കുക  
- **WebGPU ആക്സിലറേഷൻ**: ലഭ്യമായപ്പോൾ ഹാർഡ്‌വെയർ ആക്സിലറേറ്റഡ് ഇൻഫറൻസ്  
- **സ്വകാര്യത മുൻഗണന**: നിങ്ങളുടെ ഉപകരണത്തിൽ നിന്ന് ഡാറ്റ പുറത്തേക്ക് പോവില്ല  
- **സീറോ ഇൻസ്റ്റാൾ**: ഏതെങ്കിലും അനുയോജ്യമായ ബ്രൗസറിൽ പ്രവർത്തിക്കുന്നു  
- **ഗ്രേസ്‌ഫുൾ ഫാൾബാക്ക്**: WebGPU ലഭ്യമല്ലെങ്കിൽ CPU-യിലേക്ക് തിരിച്ചുപോകും  

## ആവശ്യകതകൾ

**ബ്രൗസർ അനുയോജ്യത:**  
- WebGPU സജ്ജമാക്കിയ Chrome/Edge 113+  
- WebGPU നില പരിശോധിക്കുക: `chrome://gpu`  
- WebGPU സജ്ജമാക്കുക: `chrome://flags/#enable-unsafe-webgpu`  

## ഡെമോ ഓടിക്കൽ

### ഓപ്ഷൻ 1: ലോക്കൽ സർവർ (ശുപാർശ ചെയ്യുന്നു)

```cmd
# Navigate to the demo directory
cd Module08\samples\04\webgpu-demo

# Start a local server
python -m http.server 5173

# Open browser to http://localhost:5173
```
  
### ഓപ്ഷൻ 2: VS കോഡ് ലൈവ് സർവർ

1. VS കോഡിൽ "Live Server" എക്സ്റ്റൻഷൻ ഇൻസ്റ്റാൾ ചെയ്യുക  
2. `index.html`-ൽ റൈറ്റ്-ക്ലിക്ക് → "Open with Live Server"  
3. ഡെമോ സ്വയം ബ്രൗസറിൽ തുറക്കും  

## നിങ്ങൾ കാണുന്നത്

1. **WebGPU കണ്ടെത്തൽ**: ബ്രൗസർ അനുയോജ്യത പരിശോധിക്കുന്നു  
2. **മോഡൽ ലോഡിംഗ്**: MNIST ക്ലാസിഫയർ ഡൗൺലോഡ് ചെയ്ത് ഇൻഷിയലൈസ് ചെയ്യുന്നു  
3. **ഇൻഫറൻസ് എക്സിക്യൂഷൻ**: സാമ്പിൾ ഡാറ്റയിൽ പ്രവചനങ്ങൾ നടത്തുന്നു  
4. **പ്രകടന മെട്രിക്‌സ്**: ലോഡ് സമയം, ഇൻഫറൻസ് വേഗത കാണിക്കുന്നു  
5. **ഫലങ്ങൾ പ്രദർശനം**: പ്രവചന വിശ്വാസ്യതയും റോ ഔട്ട്പുട്ടുകളും  

## പ്രതീക്ഷിക്കുന്ന പ്രകടനം

| എക്സിക്യൂഷൻ പ്രൊവൈഡർ | മോഡൽ ലോഡ് | ഇൻഫറൻസ് | കുറിപ്പുകൾ |
|-------------------|------------|-----------|-------|
| **WebGPU** | ~2-5 സെക്കൻഡ് | ~10-50 മില്ലിസെക്കൻഡ് | ഹാർഡ്‌വെയർ ആക്സിലറേറ്റഡ് |
| **CPU (WASM)** | ~2-5 സെക്കൻഡ് | ~50-200 മില്ലിസെക്കൻഡ് | സോഫ്റ്റ്‌വെയർ ഫാൾബാക്ക് |

## പ്രശ്നപരിഹാരം

**WebGPU ലഭ്യമല്ലെങ്കിൽ:**  
- Chrome/Edge 113+ ലേക്ക് അപ്ഡേറ്റ് ചെയ്യുക  
- `chrome://flags`-ൽ WebGPU സജ്ജമാക്കുക  
- GPU ഡ്രൈവർസ് പുതുക്കിയിട്ടുണ്ടെന്ന് ഉറപ്പാക്കുക  
- ഡെമോ സ്വയം CPU-യിലേക്ക് ഫാൾബാക്ക് ചെയ്യും  

**ലോഡിംഗ് പിശകുകൾ:**  
- HTTP വഴി സർവ് ചെയ്യുന്നതാണെന്ന് ഉറപ്പാക്കുക (file:// അല്ല)  
- മോഡൽ ഡൗൺലോഡിനായി നെറ്റ്‌വർക്ക് കണക്ഷൻ പരിശോധിക്കുക  
- ONNX മോഡൽ CORS തടയുന്നില്ലെന്ന് ഉറപ്പാക്കുക  

**പ്രകടന പ്രശ്നങ്ങൾ:**  
- WebGPU CPU-യെക്കാൾ വേഗതയേകുന്നു  
- ആദ്യ ഓട്ടം മോഡൽ ഡൗൺലോഡിനാൽ മന്ദഗതിയാകും  
- പിന്നീട് ഓട്ടങ്ങൾ ബ്രൗസർ കാഷ് ഉപയോഗിക്കും  

## Foundry Local-നൊപ്പം ഇന്റഗ്രേഷൻ

ഈ WebGPU ഡെമോ Foundry Local-നെ പൂരിപ്പിക്കുന്നു, കാണിക്കുന്നത്:  

- **ക്ലയന്റ്-സൈഡ് ഇൻഫറൻസ്** പരമസ്വകാര്യതയ്ക്ക്  
- **ഓഫ്‌ലൈൻ കഴിവുകൾ** ഇന്റർനെറ്റ് ലഭ്യമല്ലാത്തപ്പോൾ  
- **എഡ്ജ് ഡിപ്ലോയ്മെന്റ്** സ്രോതസ്സ്-പരിമിത പരിസരങ്ങളിൽ  
- **ഹൈബ്രിഡ് ആർക്കിടെക്ചറുകൾ** ലോക്കൽ, സർവർ ഇൻഫറൻസ് സംയോജിപ്പിച്ച്  

പ്രൊഡക്ഷൻ ആപ്ലിക്കേഷനുകൾക്കായി പരിഗണിക്കുക:  
- സർവർ-സൈഡ് ഇൻഫറൻസിനായി Foundry Local ഉപയോഗിക്കുക  
- ക്ലയന്റ്-സൈഡ് പ്രീപ്രോസസ്സിംഗ്/പോസ്റ്റ്‌പ്രോസസ്സിംഗിനായി WebGPU ഉപയോഗിക്കുക  
- ലോക്കൽ/റിമോട്ട് ഇൻഫറൻസ് ഇടയിൽ ബുദ്ധിമുട്ടുള്ള റൂട്ടിംഗ് നടപ്പിലാക്കുക  

## സാങ്കേതിക വിശദാംശങ്ങൾ

**ഉപയോഗിച്ച മോഡൽ:**  
- MNIST ഡിജിറ്റ് ക്ലാസിഫയർ (ONNX ഫോർമാറ്റ്)  
- ഇൻപുട്ട്: 28x28 ഗ്രേസ്‌കെയിൽ ചിത്രങ്ങൾ  
- ഔട്ട്പുട്ട്: 10-ക്ലാസ് പ്രൊബബിലിറ്റി വിതരണങ്ങൾ  
- വലുപ്പം: ~500KB (വേഗത്തിലുള്ള ഡൗൺലോഡ്)  

**ONNX Runtime Web:**  
- GPU ആക്സിലറേഷനായി WebGPU എക്സിക്യൂഷൻ പ്രൊവൈഡർ  
- CPU ഫാൾബാക്കിനായി WASM എക്സിക്യൂഷൻ പ്രൊവൈഡർ  
- സ്വയം ഓപ്റ്റിമൈസേഷൻ, ഗ്രാഫ് ഓപ്റ്റിമൈസേഷൻ  

**ബ്രൗസർ APIകൾ:**  
- ഹാർഡ്‌വെയർ ആക്സസ്‌ക്കായി WebGPU  
- പശ്ചാത്തല പ്രോസസ്സിംഗിനായി Web Workers (ഭാവി മെച്ചപ്പെടുത്തൽ)  
- കാര്യക്ഷമ കംപ്യൂട്ടേഷനായി WebAssembly  

## അടുത്ത ഘട്ടങ്ങൾ

- കസ്റ്റം ONNX മോഡലുകൾ ഉപയോഗിച്ച് പരീക്ഷിക്കുക  
- യഥാർത്ഥ ചിത്രം അപ്‌ലോഡ് ചെയ്ത് ക്ലാസിഫിക്കേഷൻ നടപ്പിലാക്കുക  
- വലിയ മോഡലുകൾക്കായി സ്ട്രീമിംഗ് ഇൻഫറൻസ് ചേർക്കുക  
- ക്യാമറ/മൈക്രോഫോൺ ഇൻപുട്ടുമായി ഇന്റഗ്രേറ്റ് ചെയ്യുക
