# Module 08 സാമ്പിളുകൾ: Foundry ലൊക്കൽ ഡെവലപ്പ്മെന്റ് ഗൈഡ്

## അവലോകനം

ഈ സമഗ്ര ശേഖരം **Foundry Local SDK** ഉം **Shell Command** സമീപനങ്ങളും ഉപയോഗിച്ച് പ്രൊഡക്ഷൻ-റെഡി AI ആപ്ലിക്കേഷനുകൾ നിർമ്മിക്കുന്നതിനുള്ള ഉദാഹരണങ്ങൾ കാണിക്കുന്നു. ഓരോ സാമ്പിളും എഡ്ജ് AI ഡെവലപ്പ്മെന്റിന്റെ വ്യത്യസ്ത വശങ്ങൾ പ്രദർശിപ്പിക്കുന്നു, അടിസ്ഥാന REST ഇന്റഗ്രേഷനിൽ നിന്ന് പുരോഗമിച്ച മൾട്ടി-ഏജന്റ് സിസ്റ്റങ്ങൾ വരെ.

## ഡെവലപ്പ്മെന്റ് സമീപനം: SDK vs Shell Commands

### Foundry Local SDK ഉപയോഗിക്കേണ്ടത്:

- **പ്രോഗ്രാമാറ്റിക് നിയന്ത്രണം**: ഏജന്റ് ലൈഫ്‌സൈക്കിൾ, മൂല്യനിർണ്ണയം, അല്ലെങ്കിൽ ഡിപ്ലോയ്മെന്റ് വർക്ക്‌ഫ്ലോകൾ പൂർണ്ണമായി നിയന്ത്രിക്കേണ്ടത്
- **കസ്റ്റം ടൂളിംഗ്**: Foundry Local ചുറ്റുപാടിൽ ഓട്ടോമേഷൻ നിർമ്മിക്കൽ (CI/CD ഇന്റഗ്രേഷൻ, മൾട്ടി-ഏജന്റ് ഓർക്കസ്ട്രേഷൻ)
- **സൂക്ഷ്മ ആക്‌സസ്**: വിശദമായ ഏജന്റ് മെറ്റാഡേറ്റ, വേർഷനിംഗ്, അല്ലെങ്കിൽ മൂല്യനിർണ്ണയ റണ്ണർ നിയന്ത്രണം ആവശ്യമായപ്പോൾ
- **Python ഇന്റഗ്രേഷൻ**: Python-ഭാരമുള്ള പരിസരങ്ങളിൽ പ്രവർത്തിക്കുമ്പോൾ അല്ലെങ്കിൽ Foundry ലജിക് വ്യാപക ആപ്ലിക്കേഷനുകളിൽ ഉൾപ്പെടുത്തുമ്പോൾ
- **എന്റർപ്രൈസ് വർക്ക്‌ഫ്ലോകൾ**: Microsoft റഫറൻസ് ആർക്കിടെക്ചറുകളുമായി പൊരുത്തപ്പെടുന്ന മോഡുലാർ വർക്ക്‌ഫ്ലോകളും പുനരുപയോഗയോഗ്യമായ മൂല്യനിർണ്ണയ പൈപ്പ്ലൈനുകളും നടപ്പിലാക്കുമ്പോൾ

### Shell Commands ഉപയോഗിക്കേണ്ടത്:

- **വേഗത്തിലുള്ള ടെസ്റ്റിംഗ്**: വേഗത്തിൽ ലൊക്കൽ ടെസ്റ്റിംഗ്, മാനുവൽ ഏജന്റ് ലോഞ്ചുകൾ, അല്ലെങ്കിൽ സെറ്റപ്പ് പരിശോധന നടത്തുമ്പോൾ
- **CLI ലളിതത്വം**: ഏജന്റുകൾ ആരംഭിക്കൽ/നിർത്തൽ, ലോഗുകൾ പരിശോധിക്കൽ, അല്ലെങ്കിൽ അടിസ്ഥാന മൂല്യനിർണ്ണയങ്ങൾക്കായി ലളിതമായ CLI പ്രവർത്തനങ്ങൾ ആവശ്യമുള്ളപ്പോൾ
- **ലഘുവായ ഓട്ടോമേഷൻ**: പൂർണ്ണ SDK ഇന്റഗ്രേഷൻ ആവശ്യമില്ലാത്ത ലളിതമായ ഓട്ടോമേഷൻ സ്ക്രിപ്റ്റിംഗ്
- **വേഗത്തിലുള്ള ആവർത്തനം**: ഡീബഗ്ഗിംഗ്, ഡെവലപ്പ്മെന്റ് ചക്രങ്ങൾ, പ്രത്യേകിച്ച് പരിമിത പരിസരങ്ങളിൽ അല്ലെങ്കിൽ റിസോഴ്‌സ് ഗ്രൂപ്പ്-നിലവാര ഡിപ്ലോയ്മെന്റുകളിൽ
- **സെറ്റപ്പ് & പരിശോധന**: പ്രാഥമിക പരിസ്ഥിതി ക്രമീകരണവും വേഗത്തിലുള്ള പരിശോധനാ പ്രവർത്തനങ്ങളും

## മികച്ച പ്രാക്ടീസുകളും ശുപാർശ ചെയ്ത വർക്ക്‌ഫ്ലോയും

ഏജന്റ് ലൈഫ്‌സൈക്കിൾ മാനേജ്മെന്റ്, ആശ്രിതത്വം ട്രാക്കിംഗ്, കുറഞ്ഞ അവകാശമുള്ള പുനരുപയോഗയോഗ്യമായ സിദ്ധാന്തങ്ങൾ അടിസ്ഥാനമാക്കി:

### ഘട്ടം 1: അടിസ്ഥാനവും സെറ്റപ്പും
1. പ്രാഥമിക സെറ്റപ്പിനും വേഗത്തിലുള്ള പരിശോധനയ്ക്കും **Shell Commands** ഉപയോഗിക്കുക
2. CLI ടൂളുകളും അടിസ്ഥാന മോഡൽ ഡിപ്ലോയ്മെന്റും ഉപയോഗിച്ച് **പരിസ്ഥിതി സ്ഥിരീകരിക്കുക**
3. ലളിതമായ REST കോൾസും ഹെൽത്ത് ചെക്കുകളും ഉപയോഗിച്ച് **കണക്ടിവിറ്റി ടെസ്റ്റ് ചെയ്യുക**

### ഘട്ടം 2: ഡെവലപ്പ്മെന്റും ഇന്റഗ്രേഷനും
1. സ്കെയിലബിൾ, ട്രേസബിൾ വർക്ക്‌ഫ്ലോകൾക്കായി **SDK-യിലേക്ക് മാറുക**
2. സങ്കീർണ്ണ ഏജന്റ് ഇടപെടലുകൾക്കായി **പ്രോഗ്രാമാറ്റിക് നിയന്ത്രണം നടപ്പിലാക്കുക**
3. കമ്മ്യൂണിറ്റി-റെഡി ടെംപ്ലേറ്റുകളും Azure OpenAI ഇന്റഗ്രേഷനും വേണ്ടി **കസ്റ്റം ടൂളുകൾ നിർമ്മിക്കുക**

### ഘട്ടം 3: പ്രൊഡക്ഷനും സ്കെയിലും
1. ഓപ്പറേഷനുകൾക്കായി CLIയും ആപ്ലിക്കേഷൻ ലജിക്കിനായി SDKയും സംയോജിപ്പിക്കുന്ന **ഹൈബ്രിഡ് സമീപനം**
2. മോണിറ്ററിംഗ്, ലോഗിംഗ്, ഡിപ്ലോയ്മെന്റ് പൈപ്പ്ലൈനുകൾ ഉൾപ്പെടുന്ന **എന്റർപ്രൈസ് ഇന്റഗ്രേഷൻ**
3. പുനരുപയോഗയോഗ്യമായ ടെംപ്ലേറ്റുകളും മികച്ച പ്രാക്ടീസുകളും വഴി **കമ്മ്യൂണിറ്റി സംഭാവന**
