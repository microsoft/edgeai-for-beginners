# വർക്‌ഷോപ്പ് സ്ക്രിപ്റ്റുകൾ

വർക്‌ഷോപ്പ് സാമഗ്രികളുടെ ഗുണനിലവാരവും ഏകരൂപതയും നിലനിർത്താൻ ഉപയോഗിക്കുന്ന ഓട്ടോമേഷൻ සහ പിന്തുണാ സ്ക്രിപ്റ്റുകൾ ഈ ഡയറക്ടറിയിൽ ഉൾക്കൊള്ളുന്നു.

## ഉള്ളടക്കം

| ഫയൽ | ഉദ്ദേശ്യം |
|------|---------|
| `lint_markdown_cli.py` | പഴയതായ Foundry Local CLI കമാൻഡ് പാറ്റേണുകൾ ബ്ലോക്ക് ചെയ്യാൻ മാർക്ക്ഡൗൺ കോഡ് ഫെൻസുകൾ ലിന്റ് ചെയ്യുന്നു. |
| `export_benchmark_markdown.py` | മൾട്ടി-മോഡൽ ലാറ്റൻസി ബെഞ്ച്മാർക്ക് നടത്തുകയും മാർക്ക്ഡൗൺ + JSON റിപ്പോർട്ടുകൾ പുറപ്പെടുവിക്കുകയും ചെയ്യുന്നു. |

## 1. മാർക്ക്ഡൗൺ CLI പാറ്റേൺ ലിന്റർ

`lint_markdown_cli.py` എല്ലാ വിവർത്തനമല്ലാത്ത `.md` ഫയലുകളിലും **കോഡ് ഫെൻസുകൾക്കുള്ളിൽ** (``` ... ```) അനുവദനീയമല്ലാത്ത Foundry Local CLI പാറ്റേണുകൾ സ്കാൻ ചെയ്യുന്നു. ചരിത്രപരമായ സാഹചര്യത്തിനായി വിവരാത്മക പ്രോസിൽ പഴയ കമാൻഡുകൾ പരാമർശിക്കാം.

### പഴയ പാറ്റേണുകൾ (കോഡ് ഫെൻസുകൾക്കുള്ളിൽ ബ്ലോക്ക് ചെയ്യുന്നു)

ലിന്റർ പഴയ CLI പാറ്റേണുകൾ ബ്ലോക്ക് ചെയ്യുന്നു. പകരം ആധുനിക ഓപ്ഷനുകൾ ഉപയോഗിക്കുക.

### ആവശ്യമായ മാറ്റങ്ങൾ
| പഴയത് | പകരം ഉപയോഗിക്കുക |
|------------|-------------|
| `foundry model chat <a> "..."` | `foundry model run <a> --prompt "..."` |
| `foundry model list --running` | `foundry model list` |
| `foundry model list --cached` | `foundry cache list` |
| `foundry model stats` | ബെഞ്ച്മാർക്ക് സ്ക്രിപ്റ്റ് + സിസ്റ്റം ടൂളുകൾ (`Task Manager`, `nvidia-smi`) |
| `foundry model benchmark` | `samples/session03/benchmark_oss_models.py` |
| `foundry model list --available` | `foundry model list` |

### എക്സിറ്റ് കോഡുകൾ
| കോഡ് | അർത്ഥം |
|------|---------|
| 0 | ലംഘനങ്ങൾ കണ്ടെത്തിയില്ല |
| 1 | ഒരു അല്ലെങ്കിൽ കൂടുതൽ പഴയ പാറ്റേണുകൾ കണ്ടെത്തി |

### ലോക്കലായി പ്രവർത്തിപ്പിക്കൽ
റിപ്പോസിറ്ററി റൂട്ടിൽ നിന്ന് (ശുപാർശ ചെയ്യുന്നു):

Windows (PowerShell):
```powershell
python Workshop\scripts\lint_markdown_cli.py --verbose
```

macOS / Linux:
```bash
python Workshop/scripts/lint_markdown_cli.py --verbose
```

### പ്രീ-കമ്മിറ്റ് ഹുക്ക് (ഐച്ഛികം)
```bash
echo "python Workshop/scripts/lint_markdown_cli.py" > .git/hooks/pre-commit
chmod +x .git/hooks/pre-commit
```
ഇത് പഴയ പാറ്റേണുകൾ ഉൾപ്പെടുത്തുന്ന കമ്മിറ്റുകൾ തടയുന്നു.

### CI ഇന്റഗ്രേഷൻ
GitHub Action വർക്‌ഫ്ലോ (`.github/workflows/markdown-cli-lint.yml`) ഓരോ പുഷ്, പുൾ റിക്വസ്റ്റ് എന്നിവയ്ക്കും `main` ഉം `Reactor` ബ്രാഞ്ചുകളിലുമുള്ള ലിന്റർ പ്രവർത്തിപ്പിക്കുന്നു. പരാജയപ്പെട്ട ജോലികൾ മർജ് ചെയ്യുന്നതിന് മുമ്പ് പരിഹരിക്കണം.

### പുതിയ പഴയ പാറ്റേണുകൾ ചേർക്കൽ
1. `lint_markdown_cli.py` തുറക്കുക.
2. `DEPRECATED` ലിസ്റ്റിൽ `(regex, suggestion)` എന്ന ട്യൂപ്പിൾ ചേർക്കുക. റോ സ്ട്രിംഗ് ഉപയോഗിച്ച് ആവശ്യമായിടത്ത് `\b` വാക്ക് അതിരുകൾ ഉൾപ്പെടുത്തുക.
3. ലോക്കലായി ലിന്റർ ഓടിച്ച് കണ്ടെത്തൽ സ്ഥിരീകരിക്കുക.
4. കമ്മിറ്റ് ചെയ്ത് പുഷ് ചെയ്യുക; CI പുതിയ നിയമം പ്രാബല്യത്തിൽ വരുത്തും.

ഉദാഹരണ ചേർക്കൽ:
```python
DEPRECATED.append((r"\\bfoundry\\s+experimental\\s+foo\\b", "Remove experimental foo usage"))
```

### വിശദീകരണ പരാമർശങ്ങൾ അനുവദിക്കൽ
കോഡ് ഫെൻസുകൾ മാത്രമാണ് നിർബന്ധിതമായത്, അതിനാൽ വിവരണാത്മക എഴുത്തിൽ പഴയ കമാൻഡുകൾ സുരക്ഷിതമായി വിവരിക്കാം. വ്യത്യാസം കാണിക്കാൻ അവ ഫെൻസിനുള്ളിൽ കാണിക്കേണ്ടി വന്നാൽ, ട്രിപ്പിൾ ബാക്ക്‌ടിക്കുകൾ ഇല്ലാത്ത ഫെൻസ്ഡ് ബ്ലോക്ക് (ഉദാ: ഇൻഡെന്റ് ചെയ്യുക അല്ലെങ്കിൽ ഉദ്ധരിക്കുക) ചേർക്കുക അല്ലെങ്കിൽ പseudo രൂപത്തിലേക്ക് പുനഃരാഖ്യാനം ചെയ്യുക.

### പ്രത്യേക ഫയലുകൾ ഒഴിവാക്കൽ (അഡ്വാൻസ്ഡ്)
ആവശ്യമായാൽ, പാരമ്പര്യ ഉദാഹരണങ്ങൾ റിപോയ്ക്ക് പുറത്തുള്ള വേറെ ഫയലിൽ വയ്ക്കുക അല്ലെങ്കിൽ ഡ്രാഫ്റ്റ് ചെയ്യുമ്പോൾ വ്യത്യസ്ത എക്സ്റ്റൻഷൻ ഉപയോഗിച്ച് പേരിടുക. വിവർത്തന പകർപ്പുകൾക്ക് സ്വയം പ്രവർത്തിക്കുന്ന ഒഴിവാക്കലുകൾ (പാതയിൽ `translations` ഉള്ളവ) ഉണ്ട്.

### പ്രശ്നപരിഹാരം
| പ്രശ്നം | കാരണം | പരിഹാരം |
|-------|-------|-----------|
| ലിന്റർ നിങ്ങൾ അപ്ഡേറ്റ് ചെയ്ത ലൈനിൽ ഫ്ലാഗ് ചെയ്യുന്നു | Regex വളരെ വ്യാപകമാണ് | അധിക വാക്ക് അതിരുകൾ (`\b`) അല്ലെങ്കിൽ ആങ്കറുകൾ ഉപയോഗിച്ച് പാറ്റേൺ കുറയ്ക്കുക |
| CI പരാജയപ്പെടുന്നു പക്ഷേ ലോക്കലായി പാസ്സാണ് | വ്യത്യസ്ത Python പതിപ്പ് അല്ലെങ്കിൽ കമ്മിറ്റ് ചെയ്യാത്ത മാറ്റങ്ങൾ | ലോക്കലായി വീണ്ടും ഓടിക്കുക, ക്ലീൻ വർക്ക് ട്രീ ഉറപ്പാക്കുക, വർക്‌ഫ്ലോ Python പതിപ്പ് (3.11) പരിശോധിക്കുക |
| താൽക്കാലികമായി ഒഴിവാക്കേണ്ടത് | അടിയന്തര ഹോട്ട്‌ഫിക്സ് | ഉടൻ പരിഹാരം പ്രയോഗിക്കുക; താൽക്കാലിക ബ്രാഞ്ച് ഉപയോഗിച്ച് ഫോളോ-അപ്പ് PR പരിഗണിക്കുക (ബൈപാസ് സ്വിച്ച് ചേർക്കുന്നത് ഒഴിവാക്കുക) |

### കാരണവിവരണം
*നിലവിലെ* സ്ഥിരമായ CLI ഉപരിതലവുമായി ഡോക്യുമെന്റേഷൻ പൊരുത്തപ്പെടുത്തുന്നത് വർക്‌ഷോപ്പ് തടസ്സം ഒഴിവാക്കുന്നു, പഠനാർത്ഥികളുടെ ആശയക്കുഴപ്പം കുറയ്ക്കുന്നു, CLI സബ്‌കമാൻഡുകൾ ഒഴിഞ്ഞുപോകുന്നതിന് പകരം പരിപാലിത Python സ്ക്രിപ്റ്റുകൾ വഴി പ്രകടന അളവുകൾ കേന്ദ്രികൃതമാക്കുന്നു.

വർക്‌ഷോപ്പ് ഗുണനിലവാര ഉപകരണങ്ങളുടെ ഭാഗമായാണ് പരിപാലനം. മെച്ചപ്പെടുത്തലുകൾക്കായി (ഉദാ: സ്വയം പരിഹാര നിർദ്ദേശങ്ങൾ അല്ലെങ്കിൽ HTML റിപ്പോർട്ട് സൃഷ്ടിക്കൽ) ഒരു ഇഷ്യൂ തുറക്കുക അല്ലെങ്കിൽ PR സമർപ്പിക്കുക.

## 2. സാമ്പിൾ സാധുത പരിശോധന സ്ക്രിപ്റ്റ്

`validate_samples.py` എല്ലാ Python സാമ്പിൾ ഫയലുകളും സിന്റാക്സ്, ഇംപോർട്ടുകൾ, മികച്ച പ്രാക്ടീസുകൾ പാലിക്കുന്നുണ്ടോ എന്ന് പരിശോധിക്കുന്നു.

### ഉപയോഗം
```bash
# എല്ലാ സാമ്പിളുകളും സാധുത പരിശോധിക്കുക
python scripts/validate_samples.py

# പ്രത്യേക സെഷൻ സാധുത പരിശോധിക്കുക
python scripts/validate_samples.py --session 01

# വിശദമായ മോഡ് (മികച്ച പ്രാക്ടീസ് മുന്നറിയിപ്പുകൾ ഉൾപ്പെടുന്നു)
python scripts/validate_samples.py --verbose

# സംക്ഷിപ്തം മാത്രം
python scripts/validate_samples.py --summary
```

### എന്താണ് പരിശോധിക്കുന്നത്
- ✅ Python സിന്റാക്സ് സാധുത
- ✅ ആവശ്യമായ ഇംപോർട്ടുകൾ ഉള്ളത്
- ✅ പിശക് കൈകാര്യം ചെയ്യൽ നടപ്പിലാക്കൽ (വെർബോസ് മോഡ്)
- ✅ ടൈപ്പ് ഹിന്റുകൾ ഉപയോഗം (വെർബോസ് മോഡ്)
- ✅ ഫംഗ്ഷൻ ഡോക്സ്ട്രിംഗ്‌സ് (വെർബോസ് മോഡ്)
- ✅ SDK റഫറൻസ് ലിങ്കുകൾ (വെർബോസ് മോഡ്)

### പരിസ്ഥിതി വ്യത്യാസങ്ങൾ
- `SKIP_IMPORT_CHECK=1` - ഇംപോർട്ട് പരിശോധന ഒഴിവാക്കുക
- `SKIP_SYNTAX_CHECK=1` - സിന്റാക്സ് പരിശോധന ഒഴിവാക്കുക

### എക്സിറ്റ് കോഡുകൾ
- `0` - എല്ലാ സാമ്പിളുകളും സാധുത പരിശോധന പാസ്സായി
- `1` - ഒരു അല്ലെങ്കിൽ കൂടുതൽ സാമ്പിളുകൾ പരാജയപ്പെട്ടു

## 3. സാമ്പിൾ ടെസ്റ്റ് റണ്ണർ

`test_samples.py` എല്ലാ സാമ്പിളുകളിലും സ്മോക്ക് ടെസ്റ്റുകൾ നടത്തുന്നു, പിശകുകൾ കൂടാതെ പ്രവർത്തിക്കുന്നുണ്ടോ എന്ന് പരിശോധിക്കുന്നു.

### ഉപയോഗം
```bash
# എല്ലാ സാമ്പിളുകളും പരീക്ഷിക്കുക
python scripts/test_samples.py

# പ്രത്യേക സെഷൻ പരീക്ഷിക്കുക
python scripts/test_samples.py --session 01

# വേഗത്തിലുള്ള മോഡ് (കുറഞ്ഞ ടൈംഔട്ടുകൾ)
python scripts/test_samples.py --quick

# വിശദമായ മോഡ് (ഔട്ട്പുട്ട് കാണിക്കുക)
python scripts/test_samples.py --verbose
```

### മുൻ‌അവശ്യങ്ങൾ
- Foundry Local സർവീസ് പ്രവർത്തനത്തിൽ: `foundry service start`
- മോഡലുകൾ ലോഡ് ചെയ്തിട്ടുണ്ട്: `foundry model run phi-4-mini`
- ആശ്രിതങ്ങൾ ഇൻസ്റ്റാൾ ചെയ്തിട്ടുണ്ട്: `pip install -r requirements.txt`

### എന്താണ് പരിശോധിക്കുന്നത്
- ✅ സാമ്പിൾ റൺടൈം പിശകുകൾ കൂടാതെ പ്രവർത്തിക്കുമോ
- ✅ ആവശ്യമായ ഔട്ട്പുട്ട് സൃഷ്ടിക്കപ്പെടുന്നുണ്ടോ
- ✅ പരാജയത്തിൽ ശരിയായ പിശക് കൈകാര്യം
- ✅ പ്രകടനം (പ്രവർത്തന സമയം)

### പരിസ്ഥിതി വ്യത്യാസങ്ങൾ
- `FOUNDRY_LOCAL_ALIAS=phi-4-mini` - ടെസ്റ്റിംഗിനായി ഉപയോഗിക്കുന്ന മോഡൽ
- `TEST_TIMEOUT=30` - ഓരോ സാമ്പിളിനും സെക്കൻഡുകളിൽ ടൈംഔട്ട്

### പ്രതീക്ഷിക്കുന്ന പരാജയങ്ങൾ
ചില ടെസ്റ്റുകൾ ഓപ്ഷണൽ ആശ്രിതങ്ങൾ ഇൻസ്റ്റാൾ ചെയ്തിട്ടില്ലെങ്കിൽ പരാജയപ്പെടാം (ഉദാ: `ragas`, `sentence-transformers`). ഇൻസ്റ്റാൾ ചെയ്യുക:
```bash
pip install sentence-transformers ragas datasets
```

### എക്സിറ്റ് കോഡുകൾ
- `0` - എല്ലാ ടെസ്റ്റുകളും പാസ്സായി
- `1` - ഒരു അല്ലെങ്കിൽ കൂടുതൽ ടെസ്റ്റുകൾ പരാജയപ്പെട്ടു

## 4. ബെഞ്ച്മാർക്ക് മാർക്ക്ഡൗൺ എക്സ്പോർട്ടർ

സ്ക്രിപ്റ്റ്: `export_benchmark_markdown.py`

ഒരു മോഡൽ സെറ്റിനുള്ള പ്രകടന പട്ടിക പുനരുത്പാദനയോഗ്യമാക്കുന്നു.

### ഉപയോഗം
```powershell
python Workshop\scripts\export_benchmark_markdown.py --models "qwen2.5-0.5b" --prompt "Explain retrieval augmented generation briefly." --rounds 3 --output benchmark_report.md
```

### ഔട്ട്പുട്ടുകൾ
| ഫയൽ | വിവരണം |
|------|-------------|
| `benchmark_report.md` | മാർക്ക്ഡൗൺ പട്ടിക (ശരാശരി, p95, ടോക്കൺ/സെക്കൻഡ്, ഐച്ഛികം ആദ്യ ടോക്കൺ) |
| `benchmark_report.json` | വ്യത്യാസം & ചരിത്രത്തിനുള്ള കച്ചവട മെട്രിക്‌സ് അറേ |

### ഓപ്ഷനുകൾ
| ഫ്ലാഗ് | വിവരണം | ഡിഫോൾട്ട് |
|------|-------------|---------|
| `--models` | കോമയാൽ വേർതിരിച്ച മോഡൽ അലിയാസുകൾ | (ആവശ്യമാണ്) |
| `--prompt` | ഓരോ റൗണ്ടിലും ഉപയോഗിക്കുന്ന പ്രോംപ്റ്റ് | (ആവശ്യമാണ്) |
| `--rounds` | ഓരോ മോഡലിനും റൗണ്ടുകൾ | 3 |
| `--output` | മാർക്ക്ഡൗൺ ഔട്ട്പുട്ട് ഫയൽ | `benchmark_report.md` |
| `--json` | JSON ഔട്ട്പുട്ട് ഫയൽ | `benchmark_report.json` |
| `--fail-on-empty` | എല്ലാ ബെഞ്ച്മാർക്കുകളും പരാജയപ്പെട്ടാൽ നോൺ-സീറോ എക്സിറ്റ് | ഓഫ് |

പരിസ്ഥിതി വ്യത്യാസം `BENCH_STREAM=1` ആദ്യ ടോക്കൺ ലാറ്റൻസി അളവ് ചേർക്കുന്നു.

### കുറിപ്പുകൾ
- സ്ഥിരതയുള്ള മോഡൽ ബൂട്ട്‌സ്റ്റ്രാപ്പും കാഷിംഗുംക്കായി `workshop_utils` പുനരുപയോഗിക്കുന്നു.
- വേറെ വർക്ക് ഡയറക്ടറിയിൽ നിന്ന് ഓടിച്ചാൽ, `workshop_utils` കണ്ടെത്താൻ പാത ഫാൾബാക്കുകൾ ശ്രമിക്കും.
- GPU താരതമ്യത്തിനായി: ഒരിക്കൽ ഓടിക്കുക, CLI കോൺഫിഗിലൂടെ ആക്സിലറേഷൻ സജീവമാക്കുക, വീണ്ടും ഓടിച്ച് JSON വ്യത്യാസം പരിശോധിക്കുക.
