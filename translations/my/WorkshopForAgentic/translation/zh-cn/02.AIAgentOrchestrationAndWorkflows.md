# ဒုတိယအပိုင်း- သင့် Podcast ထုတ်လုပ်မှုအဖွဲ့ကို ဖွဲ့စည်းခြင်း 🎬

![lab2](../../../../../translated_images/my/lab2.48444b9d549e7863.webp)

## ဇာတ်အိမ် ပိုမိုဆန်းစစ်ခြင်း

Alex(သင်၏ ပထမအပိုင်းမှ AI အကူအညီ) သည် မကောင်းခြင်းမရှိပါ၊ သို့ရာတွင် AI တစ်ဦးတည်းဖြင့် podcast စွန့်ခွာမှုလုပ်ငန်းစဉ်အစုစည်းကို စီမံခန့်ခွဲ၍ မဖြစ်နိုင်ပါ။ သင်အား *အဖွဲ့* တစ်ခုလိုအပ်သည်:
- 🔍 **သုတေသန AI**: အင်တာနက်တွင် သတင်းအချက်အလက် အသစ်များကို ရှာဖွေခြင်း
- ✍️ **စာသားရေး AI**: သုတေသနမှ ရလာသော အချက်အလက်များကို စိတ်ဝင်စားဖွယ် စကားပြောများသို့ ပြောင်းလဲခြင်း
- 👤 **သင်(အယ်ဒီတာ)**: စာသားရေးများကို အတည်ပြုခြင်း သို့မဟုတ် ပြန်လည်ရေးသားရန် ပေးပို့ခြင်း

သင့်အား ကြိုဆိုပါသည် **AI Agents 编排** မှာ —— သင်သည် သင့် AI အဖွဲ့ရဲ့ ဒါရိုက်တာ ဖြစ်လာမည်။ Avengers အဖွဲ့လို ထင်မြင်ပါ၊ ဒါပေမယ့် podcast ထုတ်လုပ်မှုပိုင်းအတွက်သာဖြစ်သည်။

## အာဂျင့်စီ编排 ဆိုတာဘာလဲ? (ရိုးရှင်းသော ဗားရှင်း)

သင်သည် မုန့်ဟင်းခါးဆိုင်တစ်ခုကို စီမံထိန်းချုပ်နေသည်ကို စဉ်းစားပါ။ သင်သည် ငါးဂျုံတစ္ခုတည်းကပဲ လုပ်မည် မဟုတ်ပါ၊ သင့်မှာ:
- 🍳 ဟင်းချက်သူ (Chef) က အစားအစာတွေ ချက်စရာ
- 👨‍🍳 အထောက်အကူ ဟင်းချက်သူ သည် စီစဉ်ပေးခြင်း
- 👩‍🍳 ဝန်ဆောင်မှု လုပ်သူများသည် အစားအစာ ပို့ဆောင်ခြင်း

AI Agents 编排 သည် တူညီသော အတွေးအခေါ်ဖြစ်သည်၊ ဒါပေမယ့် AI ဖြင့်။ အဘယ်သူမဆို အထူးကျွမ်းကျင်မှုရှိပြီး သင်သည် သူတို့အား ပူးပေါင်းညှိနှိုင်းသည်။ ထိန်းချုပ်မှုကို AI Agent တစ်ဦးတည်း မစိတ်ပင်ပန်းခြင်း၊ အလုပ် မြန်ဆန်စွာ ပြီးမြောက်သည်။

### အတူတူအဖွဲ့လိုသဘောတူညီချက် 🎸

သင့် AI အေးဂျင့်သည် တေးဂိုဏ်းတစ်ခု ကြောင့်ဖြစ်သည်။  
- **မင်းသမီး**: ဖောက်သည်နဲ့ ရိုက်နှက်စေသော အဓိက AI Agent  
- **ဒေါက်တာ**: တေးသံနှင့် ချိန်ညှိသူ  
- **ဘေ့စ်ဂစ်တာ**: အားလုံးအတွက် ပံ့ပိုးပေးသူ၊ အချက်အလက်ကို ရယူသူ  
- **သင်(အဖွဲ့ကိုင်တွယ်သူ)**: အားလုံးကိုညှိနှိုင်းပါ။

ညှိနှိုင်းမှုမရှိရင်? သတင်းအညိုအစပ်နဲ့လူနာဘယ်ဘယ်ကိုမျှ ့စေလိုက်တယ်။ 编排 တွေ ရှိရင်? အလွမ်းဖြာသောတေးဂီတ။

### အဘယ်ကြောင့် အရေးကြီးသနည်း

AI Agent တစ်ခုက အားလုံး လုပ်ပေးတယ် ဆိုသည် အလုပ်ရှုပ်ခြင်းဖြစ်သည်။ အထူးကျွမ်းကျင် AI Agent များပူးပေါင်းပြီး အကျိုးရှိမှု တိုးတက်လာသည်!🚀

**မှန်ကန်သောအတိုင်း**: သင့် podcast ကို သီးသန့် သုတေသန၊ စာတမ်းရေးခြင်း၊ တည်းဖြတ်တင်ပြခြင်း လုပ်ရာ ခက်ခဲခဲ့လား? ဟုတ်ကဲ့၊ ဒါဟာ မကောင်းပါတယ်။ 编排 ဖြင့် AI Agent တစ်ဦးစီမှာ သူတို့ ကျွမ်းကျင်ရာအလုပ်ကိုသာ စီမံပေးမည်။ သင်သည် နောက်ဆုံးဆုံးဖြတ်ချက်ပဲ ချမှတ်ပါ။

**ကမ္ဘာ့အမှန်တကယ် ဥပမာ**: Customer support chatbot က ဘယ်အခါ ဘယ်အရာကို စီမံရမယ်ဆိုတာ သိတယ်- ငွေတောင်းခံတာလား၊ နည်းပညာပြသနာလား၊ လူကို ခေါ်တာလား။ အဲဒါလို 编排  သည်။

## AI Agent နဲ့ Workflow:ဘာကွာခြားသလဲ?

ဒီလိုစဉ်းစားပါ:

### 🤖 AI Agent = Jazz musician
- **စိတ်အားထက်သန်စွာ ဆုံးဖြတ်ချက် ယခုနားထောင်ထိုင်မှုအပေါ် အခြေခံသည်**  
- **အျဖစ္အပ်က္ဖန်တီးမှု**: ၎င်း၏ကိရိယာနှင့် ဖြေရှင်းချက်များကို အသုံးပြုသည်  
- **စဉ်းစားခြင်း**: LLM အတွင်းပြဿနာကို ကိုင်တွယ်ခြင်း  
- **လိုအပ်ချက်နှင့် ကိုက်ညီမှု**: သင်ပေးသော အရာတိုင်းကို လိုက်လျောညီထွေစွာ လုပ်ဆောင်သည်  

### 🎵 Workflow = Orchestra playing classical music
- **ပြဇာတ်စာအုပ်နဲ့ လိုက်လျောညီထွေ ကျင့်သုံးသည်** (ကြိုတင်သတ်မှတ်ထားသော အဆင့်များ)  
- **ခန့်မှန်းနိုင်သော** လုပ်ထုံးလုပ်နည်း စနစ်  
- **ညှိနှိုင်းမှု**: AI Agent များ၊ လူများနှင့် စနစ်များကို ပူးပေါင်းစည်းခြင်း  
- **ဖွဲ့စည်းထားသော** စားဖိုချက်တစ်ခုလို သဘောတူညီမှု  

**မောဂျစ်**: Workflow *编排* AI Agents! သင်သည် Workflow တစ်ခုတည်ဆောက်ပြီး AI Agents များ ဘယ်အချိန် အလုပ်လုပ်ရမည်ကို ပြောပြသည်။ နှစ်ခုလုံး ပေါင်းတာ ဖြစ်တာပေါ့။ 🎭

## သုံးမျိုးရှိသော သင်၏ AI အဖွဲ့ ကိုညှိနှိုင်းနည်း

### 1. 🎯 စုစည်းမှု (သင်မှာ ဥက္ကဋ္ဌ)

AI Agent တစ်ဦး အားအော်ဒါပေးခြင်း။ သင့်ကို အသင်းတစ်ခု စီမံခန့်ခွဲသူလို စဉ်းစားပါ– သင်ဆုံးဖြတ်သည် ဘယ်သူ ဘာလုပ်မလဲ နှင့် ဘယ်အချိန်လုပ်မလဲ။

**အားသာချက်**:
- ✅ ခြေငုံချဲ့သော ဦးဆောင်မှု ( ရှုပ်ထွေးမှု မရှိ)  
- ✅ အတူတကြ ဆုံးဖြတ်ချက်ရရှိမှု  
- ✅ သော့ချက် မရှုပ်ထွေးမှု  

**သင့်တော်သည်**:
- ဖောက်သည်ဝန်ဆောင်မှု လမ်းကြောင်းချခြင်း (" ငွေတွက်ခြင်းလား နည်းပညာ ကူညီမှုလား?")  
- အကြောင်းအရာ အတည်ပြုလုပ်ငန်းစဉ် (" ဒီစာသားရေးကို သဘောတူသွားလား?")  
- Podcast ထုတ်လုပ်ခြင်း (ဒီဟာ လက်ရှိ ဖန်တီးနေသည်!)  

### 2. 🤝 ခွဲဝေပြီးစည်းရုံးခြင်း (AI Agent များကို ကိုယ်တိုင် စည်းစုခြင်း)

AI Agents များ တိုက်ရိုက် ဆက်သွယ်ပြီး အဖွဲ့တစ်ခုလို ပြဿနာများကို ဖြေရှင်းသည်။ လူတိုင်း ပူးပေါင်းညှိနှိုင်းနေတဲ့ စည်းလုံးညီညွတ်မှု လို Chat Group လိုပဲ။

**အားသာချက်**:
- ✅ တိုးချဲ့ရလွယ် (AI Agent များ ပိုထည့်နိုင်သည်)  
- ✅ Single point failure မရှိခြင်း  
- ✅ AI Agents များ သဘာဝအတိုင်း ပူးပေါင်းဆောင်ရွက်ခြင်း  

**သင့်တော်သည်**:
- သုတေသနအဖွဲ့ ( AI Agent တစ်ဦးစီ ကွဲပြားသော ရင်းမြစ်များ ကို တူးဖော်စူးစမ်း)  
- ကြံဆ ယူညီဖို့ ဇုန်များ  
- ပျံ့နှံ့ စီမံခန့်ခွဲမှု  

### 3. 🔀 ပေါင်းစပ် (နှစ်ခုစလုံး အကျိုးရှိ)

သင် စုစည်းရာနေရာကို သတ်မှတ်၊ AI Agents များကို လွတ်လပ်စွာ စည်းရုံးရေးကို ခွင့်ပြုသော CEO တစ်ဦးလို ဖြစ်သည်။

**အလွန်သင့်တော်သည်**: စွမ်းဆောင်မှု နှင့် လွတ်လပ်မှု ရှိတဲ့ အခက်အခဲများစွာပါဝင်သော စီမံကိန်းများ။

## Microsoft Agent Framework: သင့် 编排 ကိရိယာပုံး 🧰

အခုတည်ဆောက်ချိန်ပါ! သင် အသုံးပြုမည့်ကိရိယာများမှာ:

### ဖွဲ့စည်းပစ္စည်းများ

#### ၁။ 🧱 အကောင်ထည် (သင်၏ အလုပ်သမား)
- **အတိုင်းအတာ**: မတူညီသော လုပ်ငန်းတစ်ခုချင်းစီ — AI Agent သို့မဟုတ် စိတ်တိုင်းကျ လုပ်ဆောင်ချက် Logic  
- **လုပ်ဆောင်ချက်**: ဦးနှောက်ဝင်အချက်အလက် လက်ခံ၊ အလုပ် ပြီးမြောက်စေ၊ ထွက်ပေါက် ထုတ်ပေးသည်  
- **စဉ်းစားချက်**: အလုပ်စီးလိုင်းရှိ ဇယားခင္း  

#### ၂။ ➡️ လမ်းသစ်များ (ချိတ်ဆက်ခြင်း)
- **အတိုင်းအတာ**: အကောင်ထည်များကြား လမ်းကြောင်းများ  
- **လုပ်ဆောင်ချက်**: သတင်းအချက်အလက် လှိုင်းကြောင်းကို ထိန်းချုပ်သည် ("A ပြီးတဲ့နောက် B သို့ သွားပါ")  
- **စဉ်းစားချက်**: လုပ်ထုံးလုပ်နည်းဇယားရှိ မျဉ်းသားများ  

#### ၃။ 🗺️ Workflow (အထွတ်အနစ်စီမံချက်)
- **အတိုင်းအတာ**: အကောင်ထည်များနှင့် လမ်းကြောင်းများ အပြည့်အစုံဇယား  
- **လုပ်ဆောင်ချက်**: စတင်မှ အဆုံးအထိ အစီအစဉ် ဖေါ်ပြသည်  
- **စဉ်းစားချက်**: သင့်ထုတ်လုပ်မှု လမ်းညွှန်ကတ်ကြမ်း  

### သင့်အတွက် စိတ်ချရသည့် အင်္ဂါရပ်များ

**🛡️ အမျိုးအစားလုံခြုံမှု**: AI Agents ကြား သတင်းအချက်အလက်များသည် အမျိုးအစား စစ်ဆေးပြီးမှ လုပ်ဆောင်သည်။ "အို၊ မမှန်တဲ့ အမျိုးအစား" လျှင် စိုးရိမ်စရာ မရှိ။

**🔀 လိုက်လျောညီထွေ လမ်းညွှန်မှု**:
- If-then အခြေအနေ ("ဆိုင်းငံ့ရင် ထုတ်ဝေ၊ မဆိုင်းငံ့ရင် ပြန်ရေး" )  
- 병렬 처리 ( အများကြီး AI Agent တပြိုင်နက် လုပ်ဆောင်)  
- လည်ပတ်လမ်းကြောင်းအသစ် (Workflow အတွင်း ရလဒ်များအပေါ် မူတည်ပြီး ချိန်ညှိ)  

**🔌 အပြင်ကိရိယာ ပေါင်းသင်းခြင်း**:
- API နှင့် ဆက်သွယ်ရန်  
- လူကျင့်တင် စစ်ဆေးပုံများ ထည့်သွင်းရန် (ထုတ်ဝေမီ သင်အတည်ပြုသည်)  
- တောင်းဆိုမှု/တုံ့ပြန်မှု လှိုင်းကြောင်း တည်ဆောက်ခြင်း  

**💾 စစ်ဆေးမွမ်းမံနေရာ**: အဆင်မပြေခဲ့ရင် ရပ်နေရာမှ စတင်လုပ်ဆောင်နိုင်ရန်။

**🤝 အများ AI Agent ပူးပေါင်းညှိနှိုင်းခြင်း**:
- Agent များကို တစီးတည်း အဆက်လိုက် ပြုလုပ်ခြင်း (A → B → C)  
- 병렬 처리 (A + B + C လက်တွဲ အလုပ်လုပ်)  
- Agent များကြား လက်ခံချိတ်ဆက်  
- ပူးပေါင်းလုပ်ဆောင်မှု  

## အကောင်းဆုံး အလေ့အထများ (အတွေ့အကြုံ တီကွတ်များ)🎯

### ၁။ ပိုင်းခြားထားပါ
AI Agent တစ်ခုစီသည် တစ်ခုသော လုပ်ငန်းကိုသာ ကောင်းကောင်း ပြုလုပ်ရမည်။ အားလုံးလုပ်သော "super AI Agent" တစ်ခု မလုပ်ပါနှင့် — ဂျုံတော့ ပြန်ပြီး တည်းဖြတ်ရာတွင် စိတ်မကောင်းဖြစ်မည်။

### ၂။ ပျက်ကွက်မှုအတွက် ပြင်ဆင်ထားပါ
AI Agent များသည် ပြောင်းလဲမှုတတ်သည်။ အင်တာနက် ချိတ်ဆက်မှု ကွဲပြားနိုင်သည်။ မှားယွင်းမှု ကိုင်တွယ်မှု နှင့် အ_backup စီမံချက် တည်ဆောက်ပါ။ မနက်ဖြန် ရင်ဆိုင်ရမည့် သင်သည် ကျေးဇူးတင်လိမ့်မည်။

### ၃။ အားလုံးကို အကဲဖြတ်ခြင်း
သင်၏ AI Agents များ ဘာလုပ်နေလဲ ကြည့်ရှုပါ။ DevUI( ကျနော်တို့ နောက်ပိုင်း စတင်မိတ်ဆက်မယ်!) ဖြင့် စစ်ဆေး နေ့စဉ် Workflow များကို ကြည့်ပါ။

### ၄။ သတင်းများအရွယ်အစား ပိုမိုတိကျစွာ လျော့ချပါ
AI Agents အကြား မကြီးမားသော ဖိုင်များ ဖြန့်ဝေခြင်း မလုပ်ပါနှင့်။ သတင်းကောင်းများအားလုံးကို တိုတောင်းစွာ ထားပါ၊ အမြန်နှုန်း တိုးစေမည်။

### ၅။ သင့်လျော်သော ပုံစံ ရွေးချယ်ပါ
ထိန်းချုပ်မှုလိုအပ်သလား? စုစည်းမှုရွေးပါ။ ပမာဏလိုအပ်သလား? ခွဲဝေရေးရွေးပါ။ ဆုံးဖြတ်၍မရနိုင်ပါက? ပေါင်းစပ်မှုနှင့် ပတ်သက်သောကို ရွေးပါ။

## DevUI: သင့် Workflow ကို စမ်းသပ်ခြင်း 🔍

### DevUI ဆိုတာဘာလဲ?

DevUI သင်၏ AI Agents နှင့် Workflow များ စမ်းသပ်ခန်း အဖွဲ့လို ဖြစ်သည်။ Web မျက်နှာပြင်တစ်ခုဖြစ်၍ သင်သည်:
- 👀 သင့် Workflow များ လုပ်ဆောင်မှုကို ကြည့်ရှုနိုင်သည်  
- 💬 AI Agents များနှင့် တိုက်ရိုက် စကားပြောနိုင်သည်  
- 🔍 မှားယွင်းမှုများ စစ်ဆေးနိုင်သည်  
- 📊 လမ်းကြောင်းနဲ့ စွမ်းဆောင်ရည် စာရင်းများ ကြည့်ရှုနိုင်သည်  

> **အရေးကြီး**: DevUI သည် ဖွံ့ဖြိုးရေးအတွက်သာ ဖြစ်သည်! လက်ရှိ မူလ ထုတ်လုပ်မှုတွင် အသုံးမပြုနှင့်။ ဒေသိယ စမ်းသပ်ပတ်ဝန်းကျင်တစ်ခုလိုစဉ်းစားပါ။

### အကြောင်းပြချက်များ အမိုက်စားချည်း

- **🖥️ ကြိုက်နှစ်သက်ဖွယ် Web UI**: နှိပ်ပြီး၊ ထည့်ပြီး၊ စမ်းသပ် — command line မလိုပါ  
- **📁 ဖိုင်လိုက်နာ**: upload ဖိုင်၊ ကွဲပြားသော input များဖြင့် စမ်းသပ်  
- **📂 အလိုအလျောက် ရှာဖွေသည်**: ဖိုင်ဖိုဒါ တစ်ခုသို့ ဦးတည်၍ AI Agents အားလုံးကို ပြသ  
- **📋 mode မလိုပါ**: နောက်ခံတွင် AI Agent များကို register လုပ်ပြီး၊ ဖိုင်ဖိုဒါ လုပ်ထုံးလုပ်နည်း လိုအပ်မည်မဟုတ်  
- **🔌 OpenAI မျက်နှာချင်းဆိုင်**: OpenAI SDK နှင့် ကိုက်ညီမှုရှိ (ကိုက်ညီမှု ရနိုင်သောအရာ!)  
- **👁️ ထဲက tracking**: သင်၏ AI Agents မည်သို့ လုပ်ဆောင်နေသည်ကို တိကျစွာ ကြည့်ရှုနိုင်သည်  

### Input မည်သို့ လုပ်ဆောင်သနည်း

DevUI သည် Input များကို ဉာဏ်တော်ပါးစွာ ကိုင်တွယ်သည်။

- **AI Agent စမ်းသပ် တယ်လား?** သင်သည် စာသား box နှင့် ဖိုင် upload ခလုတ်ရရှိသည်  
- **Workflow စမ်းသပ်တယ်လား?** သင့် Workflow  မျှော်မှန်းထားသည့် အချက်အလက်အရ UI မှ input နေရာများ ဖန်တီးပေးသည်  

အဲဒါက မာဂျစ်လိုပဲ၊ ဒါပေမယ့် အဆင်ပြေသော code ပါ။✨

## သင်၏ လုပ်ငန်း: Podcast စွန့်ခွာမှု အဖွဲ့ ဖန်တီးခြင်း 🎬

### တာဝန် ၁: DevUI အသုံးပြု၍ Single Agent တည်ဆောက်ခြင်း

📂 [01.AgentDevUI](../../../../../WorkshopForAgentic/code/02.Workflow/01.AgentDevUI)

**စိန်ခေါ်မှု**: အားလုံးကို ဖန်တီးခင် ကိုယ်တစ်ယောက် DevUI သုံး၍ စမ်းသပ်ကြမည်- အင်တာနက်ရှာဖွေရေး ကဏ္ဍအတွက် AI Agent တစ်ခု

**သင် ဖန်တီးမယ့် အရာ**:  
Podcast ခေါင်းစဉ် ရှာဖွေရေး များပြုလုပ်နိုင်သော သုတေသန AI Agent တစ်ခု။ DevUI ရဲ့ Web UI `http://localhost:8090` ဖြင့် စမ်းသပ်မည်။

**သင် သိရှိမယ့် ကျွမ်းကျင်မှုများ**:  
- 🚀 DevUI မှ AI Agent စတင်ခြင်း  
- 🔍 အချက်အလက် တုံ့ပြန်မှု တိုက်ရိုက် စမ်းသပ်ခြင်း  
- 🛠️ ထူးခြားသော ကိရိယာများ တည်ဆောက်ခြင်း (အင်တာနက်ရှာဖွေခြင်း)  
- 📊 ပြဿနာများလိုအပ်များကို Debug အောင် ဖွင့်ခြင်း  
- 🖥️ မျက်နှာပြင် interactive Web UI အသုံးပြုခြင်း  

**ကုဒ်**:  
- `agent.py` : သင့် SearchAgent လူမူကွန်ယက်ရှာဖွေရေး လုပ်ဆောင်ချက် ပါဝင်သည်  
- OllamaChatClient ကို Qwen နှင့် ချိတ်ဆက်ရန် အသုံးပြုသည်  
- `web_search()` ကိရိယာ function ဖြင့် တည်ဆောက်သည်  
- `serve()` အသုံးပြု၍ စတင် သောအခါ DevUI ကို အလိုအလျောက် ဖွင့်သည်  

**အောင်မြင်မှုအချက်**:  
သင့် AI Agent ကို "AI နယ်ပယ်မှာ ဘာ hot စိတ်ဝင်စားစရာတွေ ရှိလဲ?" ဟု မေး၍ အင်တာနက်ရိုက်နှိပ်မှုကို ကြည့်ရှုခြင်း!🎉

### တာဝန် ၂: Multi-Agent Workflow တည်ဆောက်ခြင်း

📂 [02.WorkflowDevUI](../../../../../WorkshopForAgentic/code/02.Workflow/02.WorkflowDevUI)

**စိန်ခေါ်မှု**: အမှန်တကယ် ပျော်စရာ စတင်ပါပြီ! Podcast ထုတ်လုပ်မှု အလုပ်စီးတစ်ခု အပြည့်အစုံ တည်ဆောက်ပါ:  
1. 🔍 **သုတေသန AI Agent** → သင်၏ ခေါင်းစဉ်ကို စူးစမ်း  
2. ✍️ **စာသားရေး AI Agent** → အိမ်သာ နှစ်ဦးအကြား စကားပြော စာသားများ ရေးသားပါ (တရုတ်ဘာသာဖြင့်!)  
3. 👤 **အတည်ပြုပေးသူ** → သင်၏ အတည်ပြုခြင်း သို့မဟုတ် ငြင်းပယ်ခြင်း လိုအပ်မှု  
4. 🔄 **ဆက်လက်ပြန်သွားမှု** → ငြင်းပယ်လျှင် သင့်တုံ့ပြန်ချက်အရ ပြန်ရေးသားချက် ပြုလုပ်ခြင်း  

**သင် ရရှိနိုင်သော ကျွမ်းကျင်မှုများ**:  
- 🧱 အလုပ်အမျိုးမျိုး အတွက် အထူး AI Agent များတည်ဆောက်ခြင်း  
- 🔗 WorkflowBuilder ကို အသုံးပြု၍ AI Agents ချိတ်ဆက်ခြင်း  
- 🔀 အတည်ပြု မွန်းကြပ်ချိန် (လူကျင့်ယ့် နှင့် လက်ဆောင် များပါ!)  
- 🚦 အခြေအနေ အရ လမ်းညွှန်မှု (အတည်ပြု vs ငြင်းပယ်)  
- 🔧 စီးပွားရေး Logic အတွက် ထူးခြားသော အကောင်ထည်များ ဖန်တီးခြင်း  

**Workflow**:  
```
SearchAgent → ScriptAgent → ReviewExecutor
                             ↑          ↓ (如果被拒绝)
                             ←─────────
```
  
**ကုဒ်**:  
- `search_agent/agent.py` : သင့် သုတေသန အထူးကု  
- `generate_script_agent/agent.py` : သင့်စာသားရေး ဆရာ (တရုတ်ဘာသာဖြင့်!)  
- `workflow/workflow.py` : 编排 magic ဖြစ်သောနေရာ  
- `main.py` : DevUI တွင် စတင် ချိတ်ဆက်ထားသည်  

**အောင်မြင်မှုအချက်**:  
ခေါင်းစဉ် တစ်ခုပေးပြီး စာသားရေး ကို စိစစ်၊ တစ်ကြိမ် ငြင်းပယ်၍ မူလ လည်ပတ်မှု စမ်းသပ်ပြီး သဘောတူ/အတည်ပြုရန်!🎉

### တာဝန် ၃: ကွန်ဆေါ့ အက်ပလီကေးရှင်း ဖန်တီးခြင်း

📂 [03.Application](../../../../../WorkshopForAgentic/code/02.Workflow/03.Application)

**စိန်ခေါ်မှု**: သင့် Workflow ကို DevUI မှ စတင်၍ အရောင်အလန်းရှိ ထွက်ရှိချက်များ၊ loading spinner နှင့် ဖိုင်သိမ်းဆည်းခြင်းပါ သင့် terminal အက်ပ်သို့ ပြောင်းလဲပါ။ ဒီဟာ ပြုတ်လုပ်မှုအသင့် ဖြစ်တယ်!

**သင် သိရှိမယ့် ကျွမ်းကျင်မှုများ**:  
- ⚡ Workflow ကို ပရိုဂရမ်ပြုလုပ် လည်ပတ်နည်း (DevUI မလိုအပ်)  
- 📡 သတင်းရရှိခြင်း အခြေခံ event-driven architecture အသုံးပြုခြင်း  
- 🎨 ကောင်းမွန်သော terminal UI ဖန်တီးခြင်း (အရောင်၊ spinner, progress bar)  
- 💾 နောက်ဆုံး စာသားကို ဖိုင်ထဲ သိမ်းဆည်းခြင်း  
- 🔄 Python ၏ asyncio ဖြင့် asynchronous Workflow ကို ကိုင်တွယ်ခြင်း  

**လုပ်ဆောင်ချက်**:  
1. သင့် Podcast အကြောင်း မေးမြန်းခြင်း  
2. သတင်းအချက်အလက် တုံ့ပြန်မှုများ real-time ပြသသည် ("Search Agent လုပ်ဆောင်နေသည်...")  
3. အရောင်ဖြင့် စာသားရေး စာသားကို ပြသပါ  
4. သင့် သဘောတူခြင်း မေးစရာ  
5. အတည်ပြုသော စာသားကို `podcast.txt` တွင် သိမ်းဆည်းခြင်း  

**ကုဒ်**:  
- `podcast_app.py` : event handlers ပါဝင်သော သင့် အဓိက app  
- `workflow.py` : တာဝန် ၂ ၏ Workflow ကို ထပ်မံ အသုံးပြုသည်  
- အဖြစ်ပျက်များကို ကိုင်တွယ်ခြင်း - `AgentRunUpdateEvent`၊ `RequestInfoEvent`၊ `WorkflowOutputEvent`
- ANSI အရောင်များနဲ့ terminal ပုံစံများကို အသုံးပြုခြင်း

**အောင်မြင်မှု စံနှုန်း** - အက်ပ်ကို အလုပ်လုပ်ပေးလိုက်ပါ၊ Podcast စာသားကို ဖန်တီး ပေးပါ၊ သိမ်းဆည်းထားတာကို မြင်ရပါပြီ! သင်ဟာ အမှန်တကယ် အသုံးဝင်တဲ့ကိရိယာတစ်ခုကို တည်ဆောက်ပြီး ဖြစ်ပြီ။🚀
