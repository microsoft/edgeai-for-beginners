# ផ្នែក ០៣ - ការតភ្ជាប់ប្រព័ន្ធគម្រោងសេចក្ដី (MCP)

## ការណែនាំអំពី MCP (ប្រព័ន្ធគម្រោងសេចក្ដី)

ប្រព័ន្ធគម្រោងសេចក្ដី (MCP) គឺជាមាត្រដ្ឋានបើកចំហសម្រាប់ភ្ជាប់កម្មវិធី AI ទៅប្រព័ន្ធខាងក្រៅ។ ដោយប្រើ MCP កម្មវិធី AI ដូចជា Claude ឬ ChatGPT អាចភ្ជាប់ទៅប្រភពទិន្នន័យ (ឧ. ឯកសារប្រព័ន្ធក្នុង, ឃ្លាំងទិន្នន័យ), ឧបករណ៍ (ឧ. ម៉ាស៊ីនស្វែងរក, ម៉ាស៊ីនគណនា) និងលំហូរងារ (ឧ. បង្ហាញពិសេស) — អនុញ្ញាតឱ្យពួកវាស្វែងរកព័ត៌មានសំខាន់ និងអនុវត្តភារកិច្ច។

គិតថា MCP ជាស្រទាប់ **បំពាន់ USB-C សម្រាប់កម្មវិធី AI**។ ដូចដែល USB-C ផ្តល់វិធីសាស្ត្រតភ្ជាប់ឧបករណ៍អេឡិចត្រូនិចដែលមានស្តង់ដារ MCP ផ្តល់វិធីសាស្ត្រតភ្ជាប់កម្មវិធី AI ទៅប្រព័ន្ធខាងក្រៅដែលមានស្តង់ដា។

### MCP អាចអោយអ្វី?

MCP បង្ហាញពីសមត្ថភាពខ្លាំងសម្រាប់កម្មវិធី AI៖

- **ជំនួយកររបស់ AI បុគ្គលិក**: អ្នកប្រើអាចចូលដំណើរការ Google Calendar និង Notion របស់អ្នក ដោយអង្គចងក្រង AI ដែលមានបុគ្គលិកលក្ខណៈផ្ទាល់ខ្លួន
- **ការបង្កើតកូដកម្រិតខ្ពស់**: Claude Code អាចបង្កើតកម្មវិធីវែបឯកសារមួយទាំងមូលដោយប្រើការរចនារបស់ Figma
- **ការតភ្ជាប់ទិន្នន័យសហគ្រាស**: chatbot សហគ្រាសអាចភ្ជាប់ទៅឃ្លាំងទិន្នន័យជាច្រើនក្នុងអង្គភាព សម្រួលអ្នកប្រើភ្ជាប់និងវិភាគទិន្នន័យតាមរយៈការជជែកសារព័ត៌មាន
- **លំហូរងារច្នៃប្រឌិត**: ម៉ូដែល AI អាចបង្កើតការរចនាទ្រង់ទ្រាយ 3D លើ Blender ហើយបោះពុម្ពដោយម៉ាស៊ីនបោះពុម្ព 3D
- **ការចូលដំណើរការព័ត៌មានពេលពិត**: ភ្ជាប់ទៅប្រភពទិន្នន័យខាងក្រៅសម្រាប់ព័ត៌មានទាន់សម័យ
- **ប្រតិបត្តិការលំដាប់ច្រើនស្មុគស្មាញ**: អនុវត្តលំហូរងារដ៏ស្មុគស្មាញដែលបញ្ចូលឧបករណ៍ និងប្រព័ន្ធជាច្រើន

### ហេតុអ្វីបានជា MCP សំខាន់?

MCP ផ្តល់អត្ថប្រយោជន៍ទូទាំងប្រព័ន្ធបរិយាកាស៖

**សម្រាប់អ្នកអភិវឌ្ឍន៍**: MCP កាត់បន្ថយពេលវេលា និងភាពស្មុគស្មាញនៅពេលបង្កើត ឬភ្ជាប់កម្មវិធី AI ឬភ្នាក់ងារឲ្យមានសោភ័ណភាព។

**សម្រាប់កម្មវិធី AI**: MCP ផ្តល់ការចូលដំណើរការទៅប្រព័ន្ធធនធានទិន្នន័យ ឧបករណ៍ និងកម្មវិធីផ្សេងៗដែលបង្កើនសមត្ថភាព និងធ្វើឲ្យបទពិសោធន៍អ្នកប្រើល្អប្រសើរ។

**សម្រាប់អ្នកប្រើចុងក្រោយ**: MCP បង្កើតកម្មវិធី AI ឬភ្នាក់ងារជាច្រើនដែលអាចចូលដំណើរការទិន្នន័យរបស់អ្នក និងអនុវត្តន៍សកម្មភាពជំនួសពេលចាំបាច់។

## ម៉ូដែលភាសាតូច (SLMs) ក្នុង MCP

ម៉ូដែលភាសាតូចតំណាងឱ្យវិធីសាស្ត្រប្រើប្រាស់ AI ដែលមានប្រសិទ្ធភាព និងផ្តល់អត្ថប្រយោជន៍ជាច្រើន៖

### អត្ថប្រយោជន៍របស់ SLMs
- **ប្រើប្រាស់ធនធានតិច**: ការទាមទារជំនួយលើកុំព្យូទ័រតិច
- **ពេលតបឆ្លើយរហ័ស**: កាត់បន្ថយពេលរងចាំសម្រាប់កម្មវិធីពេលពិត  
- **ថ្លៃក្រោម**: មិនតម្រូវឧបករណ៍ធំ
- **ភាពឯកជន**: អាចរត់បានក្នុងមូលដ្ឋាន ដោយគ្មានការបញ្ជូនទិន្នន័យ
- **ការប្តូរតាមតម្រូវការ**: ងាយស្រួលបង្ហាត់បន្ថែមសម្រាប់ដែនកំណត់ជាក់លាក់

### ហេតុអ្វីបានជា SLMs ដំណើរការល្អជាមួយ MCP

SLMs ប្រើជាមួយ MCP បង្កើតបទដំណើរតំណាងខ្លាំងជាមួយសមត្ថភាពគំនិតរបស់ម៉ូដែល ដោយបន្ថែមសមត្ថភាពតាមឧបករណ៍ខាងក្រៅ ដើម្បីបំពេញភាពតូចរបស់ប៉ារ៉ាម៉ែត្រ។

## គ្រឿងតភ្ជាប់ MCP Python SDK

Python MCP SDK ផ្តល់មូលដ្ឋានសម្រាប់បង្កើតកម្មវិធីដែលអាចប្រើ MCP បាន។ SDK រួមមានៈ

- **បណ្ណាល័យអតិថិជន**: សម្រាប់ភ្ជាប់ទៅម៉ាស៊ីនមេ MCP
- **ស៊ែរស្ថាបត្យកម្ម**: សម្រាប់បង្កើតម៉ាស៊ីនមេ MCP ផ្ទាល់ខ្លួន
- **អ្នកគ្រប់គ្រងប្រព័ន្ធ**: សម្រាប់គ្រប់គ្រងការប្រាស្រ័យទាក់ទង
- **ការសម្ពាធឧបករណ៍**: សម្រាប់អនុវត្តមុខងារខាងក្រៅ

## ការអនុវត្តប្រាក់ពិត: អតិថិជន Phi-4 MCP

ចូររុករកការអនុវត្តប្រាក់ពិតជាមួយម៉ូដែល Phi-4 តូចរបស់ Microsoft ដែលមានការតភ្ជាប់ជាមួយ MCP។

### ទិដ្ឋភាពផ្នែកស្ថាបត្យកម្ម MCP

MCP តាមរយៈ **ស្ថាបត្យកម្មអតិថិជន-ម៉ាស៊ីនមេ** ដែលម្ចាស់ MCP (កម្មវិធី AI ដូចជា Claude Code ឬ Claude Desktop) បង្កើតការតភ្ជាប់ទៅម៉ាស៊ីនមេ MCP មួយឬច្រើន។ ម្ចាស់ MCP សម្រេចបានដោយបង្កើតអតិថិជន MCP មួយសម្រាប់ម៉ាស៊ីនមេ MCP នីមួយៗ។

#### អ្នកចូលរួមសំខាន់ៗ

- **ម្ចាស់ MCP**: កម្មវិធី AI ដែលសហការយ និងគ្រប់គ្រងអតិថិជន MCP មួយឬច្រើន
- **អតិថិជន MCP**: ផ្នែកមួយដែលថែរក្សាការតភ្ជាប់ទៅម៉ាស៊ីនមេ MCP ហើយទទួលព័ត៌មានបរិបទពីម៉ាស៊ីនមេខណៈដែលម្ចាស់ MCP ប្រើប្រាស់
- **ម៉ាស៊ីនមេ MCP**: កម្មវិធីដែលផ្តល់ព័ត៌មានបរិបទទៅអតិថិជន MCP

#### ស្ថាបត្យកម្មពីរជាន់

MCP មានពីរជាន់ដាច់ដោយឡែក៖

**ជាន់ទិន្នន័យ**: កំណត់ប្រព័ន្ធ JSON-RPC សម្រាប់ការប្រាស្រ័យទាក់ទងអតិថិជន-ម៉ាស៊ីនមេ ដូចជាៈ
- ការគ្រប់គ្រងជីវិត (ការចាប់ផ្តើមតភ្ជាប់, ចរចាសមត្ថភាព)
- គ្រឹះសំខាន់ (ឧបករណ៍, ធនធាន, បង្ហាញ)
- លក្ខណៈអតិថិជន (ការគំរូ, ការទទួលបាន, ការចុះបញ្ជី)
- លក្ខណៈជំនួយ (ការជូនដំណឹង, ការតាមដានវឌ្ឍនភាព)

**ជាន់ដឹកជញ្ជូន**: កំណត់មេកានិច និងប៉ុស្តិ៍តផ្លូវការប្រាស្រ័យទាក់ទង៖
- **ដឹកជញ្ជូន STDIO**: ប្រើស្ទ្រីមបញ្ចូល/ចេញស្តង់ដាសម្រាប់ដំណើរការផ្ទាល់ (អប្បបរមាទំហំល្បឿន ធ្វើការលើកុំព្យូទ័រនៅក្នុង)
- **ដឹកជញ្ជូន HTTP អាចបញ្ជូនបន្ត**: ប្រើ HTTP POST ជាមួយ Server-Sent Events ជាជម្រើសសម្រាប់ម៉ាស៊ីនមេចម្ងាយ (គាំទ្រការផ្ទៀងផ្ទាត់ HTTP ស្តង់ដា)

### គ្រឹះសំខាន់ MCP

MCP កំណត់គ្រឹះសំខាន់ដែលបញ្ជាក់ប្រភេទព័ត៌មានបរិបទដែលអាចចែករំលែកជាមួយកម្មវិធី AI និងប្រភេទសកម្មភាពដែលអាចអនុវត្តបាន។

#### គ្រឹះម៉ាស៊ីនមេ

MCP កំណត់គ្រឹះសំខាន់បីដែលម៉ាស៊ីនមេអាចបង្ហាញ៖

**ឧបករណ៍**: មុខងារអាចអនុវត្តបាន ដែលកម្មវិធី AI អាចហៅដើម្បីអនុវត្តសកម្មភាព  
- ឧទាហរណ៍: ប្រតិបត្តិការឯកសារ, ហៅ API, សំណួរសំណុំទិន្នន័យ  
- វិធីសាស្ត្រ៖ `tools/list`, `tools/call`  
- គាំទ្រការស្វែងរក និងអនុវត្តសកម្មភាពឆាប់រហ័ស

**ធនធាន**: ប្រភពទិន្នន័យផ្តល់ព័ត៌មានបរិបទទៅកម្មវិធី AI  
- ឧទាហរណ៍: គ contenu des fichiers, enregistrements de bases de données, réponses API  
- វិធីសាស្ត្រ៖ `resources/list`, `resources/read`  
- អនុញ្ញាតឲ្យចូលដំណើរការទិន្នន័យរចនាសម្ព័ន្ធ

**បង្ហាញ**: ទំរង់ដែលអាចប្រើម្ដងទៀត ដើម្បីជួយរៀបចំការទំនាក់ទំនងជាមួយម៉ូដែលភាសា  
- ឧទាហរណ៍៖ បង្ហាញប្រព័ន្ធ, ឧទាហរណ៍បិទបើក  
- វិធីសាស្ត្រ៖ `prompts/list`, `prompts/get`  
- ស្តង់ដារទំរង់ទំនាក់ទំនង AI

#### គ្រឹះអតិថិជន

MCP ក៏កំណត់គ្រឹះដែលអតិថិជនអាចបង្ហាញ ដើម្បីធ្វើការទំនាក់ទំនងលម្អិត៖

**ការគំរូ**: អនុញ្ញាតម៉ាស៊ីនមេសុំការបញ្ចប់ម៉ូដែលភាសាពីកម្មវិធី AI របស់អតិថិជន  
- វិធីសាស្ត្រ៖ `sampling/complete`  
- អនុញ្ញាតអភិវឌ្ឍន៍ម៉ាស៊ីនមេធ្វើការលឿនដែលមិនពឹងផ្អែកលើម៉ូដែល  
- ឲ្យចូលដំណើរការម៉ូដែលភាសារបស់ម្ចាស់

**ការស្នើសុំព័ត៌មានបន្ថែម**: អនុញ្ញាតម៉ាស៊ីនមេលើកស្នើឲ្យមានព័ត៌មានបន្ថែមពីអ្នកប្រើ  
- វិធីសាស្ត្រ៖ `elicitation/request`  
- អនុញ្ញាតការប្រាស្រ័យផ្ទាល់ និងការបញ្ជាក់ពីអ្នកប្រើ  
- គាំទ្រការប្រមូលព័ត៌មានឆាប់រហ័ស

**កំណត់ហេតុ**: អនុញ្ញាតម៉ាស៊ីនមេស្នើសុំផ្ញើសារកំណត់ហេតុទៅអតិថិជន  
- ប្រើសម្រាប់ដោះស្រាយ និងត្រួតពិនិត្យ  
- ផ្តល់ភាពច្បាស់លាស់ចំពោះប្រតិបត្តិការ

### ជីវិតវិធីប្រព័ន្ធ MCP

#### ចាប់ផ្តើម និងចរចាសមត្ថភាព

MCP គឺជាប្រព័ន្ធមានស្ថានភាព ដែលតម្រូវឲ្យគ្រប់គ្រងជីវិតវិធី។ ដំណើរការចាប់ផ្តើមមានគោលបំណងសំខាន់៖

1. **ចរចារកំណែព្រឹត្តិការណ៍**: ប្រាកដថាអតិថិជន និងម៉ាស៊ីនមេទាំងពីរប្រើកំណែព្រឹត្តិការណ៍ដែលសមរម្យ (ឧ. "2025-06-18")  
2. **រកឃើញសមត្ថភាព**: អ្នកផ្ទាល់ខ្លួននីមួយផ្សាយលក្ខណៈ និងគ្រឹះសំខាន់ដែលគាំទ្រ  
3. **បម្លែងអត្តសញ្ញាណ**: ផ្តល់ព័ត៌មានអត្តសញ្ញាណ និងកំណែ

```python
# សំណើការចាប់ផ្តើមគំរូ
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "initialize",
  "params": {
    "protocolVersion": "2025-06-18",
    "capabilities": {
      "elicitation": {},  # អតិថិជនគាំទ្រការប្រតិបត្តិការជាមួយអ្នកប្រើ
      "sampling": {}      # អតិថិជនអាចផ្តល់ការបញ្ចប់ LLM បាន
    },
    "clientInfo": {
      "name": "edge-ai-client",
      "version": "1.0.0"
    }
  }
}
```

#### រកឃើញឧបករណ៍ និងអនុវត្ត

បន្ទាប់ពីចាប់ផ្តើម អតិថិជនអាចរកឃើញ និងអនុវត្តឧបករណ៍៖

```python
# រកឃើញឧបករណ៍ដែលមាន
tools_response = await session.list_tools()

# ប្រតិបត្តិឧបករណ៍មួយ

    "weather_current",
    {
        "location": "San Francisco",
        "units": "imperial"
    }
)
```

```python
import asyncio
from ghmodel_mcp_demo import OllamaClient, Phi4MiniMCPClient

async def automated_interaction():
    # កំណត់ប៉ារ៉ាម៉ែត្រ​ម៉ាស៊ីន​បម្រើ MCP

        command="npx",
        args=["@playwright/mcp@latest"],
        env=None,
    )
    
    # បង្កើតអតិថិជន MCP និង​ឧបករណ៍​ដំណើរការ
    async with Phi4MiniMCPClient(server_params) as mcp_client:
        tools = await process_mcp_tools(mcp_client)
        llm_client = OllamaClient()
        
        # បង្កើតចម្លើយជាមួយសមត្ថភាពឧបករណ៍
        response, messages = await llm_client.generate_response(
            "Help me automate a web task",
            tools
        )
        return response

# ដំណើរការ​ស្វ័យប្រវត្តិការ
result = asyncio.run(automated_interaction())
print(result)
```

### អនុវត្តល្អៗ

1. **គ្រប់គ្រងធនធាន**: ប្រើ async context managers

## សេចក្តីសន្និដ្ឋាន

SLMs ដែលបញ្ចូលជាមួយ MCP បំលែងការអភិវឌ្ឍកម្មវិធី AI ជាជំហានថ្មី។ ដោយបញ្ចូលប្រសិទ្ធភាពនៃម៉ូដែលតូចជាមួយសមត្ថភាពឧបករណ៍ខាងក្រៅ អ្នកអភិវឌ្ឍន៍អាចបង្កើតប្រព័ន្ធឆ្លាតវៃ ដែលប្រើប្រាស់ធនធានយ៉ាងមានប្រសិទ្ធភាព និងមានសមត្ថភាពខ្ពស់។

ប្រព័ន្ធគម្រោងសេចក្ដីផ្តល់វិធីសាស្ត្រស្តង់ដារសម្រាប់ភ្ជាប់កម្មវិធី AI ទៅប្រព័ន្ធខាងក្រៅ ដូចដែល USB-C ផ្តល់ស្តង់ដារតភ្ជាប់ជាសកលសម្រាប់ឧបករណ៍អេឡិចត្រូនិច។ ស្តង់ដារនេះអនុញ្ញាត៖

- **ការរួមបញ្ចូលគ្នា យ៉ាងដោយសោភ័ណភាព**: ភ្ជាប់ម៉ូដែល AI ទៅប្រភពទិន្នន័យ និងឧបករណ៍ជាច្រើន  
- **ការលូតលាស់ប្រព័ន្ធបរិយាស្ថាន**: បង្កើតម្តងមួយ ដើម្បីប្រើប្រាស់ក្នុងកម្មវិធី AI ច្រើន  
- **សមត្ថភាពត្រឹមត្រូវ**: បន្ថែមសមត្ថភាពខាងក្រៅទៅ SLMs  
- **ការអាប់ដេតពេលពិត**: គាំទ្រកម្មវិធី AI ឆ្លាតវៃ និងឆាប់ឆន្ទៈ

ចំណុចសំខាន់៖  
- MCP គឺជាមាត្រដ្ឋានបើកចំហរ ដែលជាស្ពានភ្ជាប់កម្មវិធី AI និងប្រព័ន្ធខាងក្រៅ  
- ព្រឹត្តិការណ៍គាំទ្រឧបករណ៍, ធនធាន និងបង្ហាញជាគ្រឹះសំខាន់  
- ជូនដំណឹងពេលពិតអនុញ្ញាតកម្មវិធីឆ្លាតវៃឆាប់ឆន្ទៈ  
- ការគ្រប់គ្រងជីវិតវិធី និងការគ្រប់គ្រងកំហុសត្រឹមត្រូវសំខាន់សម្រាប់បរិបទផលិតកម្ម  
- ប្រព័ន្ធមាន SDK និងឧបករណ៍អភិវឌ្ឍបំពេញលំហ
