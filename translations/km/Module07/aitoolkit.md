# មិគ្ខៃ AI សម្រាប់ Visual Studio Code - មគ្គុទេសក៍អភិវឌ្ឍន៍ Edge AI

## ការណែនាំ

ស្វាគមន៍មកកាន់មគ្គុទេសក៍​ពេញលេញសម្រាប់ការប្រើប្រាស់ មិគ្ខៃ AI សម្រាប់ Visual Studio Code ក្នុងការអភិវឌ្ឍ Edge AI។ ពេលដែលបច្ចេកវិទ្យាបញ្ញាសិប្បនិម្មិតផ្លាស់ប្តូរពីកុំព្យូទ័រខ្មែរប្រភេទ cloud ដែលកណ្តាល ទៅកាន់ឧបករណ៍ edge ចែកចាយ អ្នកអភិវឌ្ឍត្រូវការឧបករណ៍ដែលមានភាពពលកម្មខ្ពស់ និងបញ្ចូលគ្នា ដែលអាចដោះស្រាយបញ្ហាពិសេសនៃការដាក់បញ្ចូលនៅតំបន់ edge - ពីការបញ្ចាក់ធនធានកំណត់រហូតដល់តម្រូវការប្រតិបត្តិការផ្លូវការ។

មិគ្ខៃ AI សម្រាប់ Visual Studio Code ផ្ដល់ជំហរអភិវឌ្ឍពេញលេញដែលបានរចនាឡើងសម្រាប់ការសាងសង់ ពិនិត្យ និងធ្វើអុបទីម៉ាយសកម្មភាពកម្មវិធី AI ដែលដំណើរការដោយមានប្រសិទ្ធិភាពលើឧបករណ៍ edge។ មិនថាអ្នកកំពុងអភិវឌ្ឍសម្រាប់កម្តៅ IoT សម្រាប់ឧបករណ៍ចល័ត ប្រព័ន្ធបង្កប់ ឬម៉ាស៊ីនបម្រើ edge ក៏ដោយ មិគ្ខៃនេះជួយអនុវត្តសកម្មភាពអភិវឌ្ឍរបស់អ្នកឲ្យរហ័សនៅក្នុងបរិយាកាស VS Code ដែលអ្នកស្គាល់។

មគ្គុទេសក៍នេះនឹងផ្លាស់ប្តូរអ្នកឆ្លងកាត់គំនិតសំខាន់ៗ ឧបករណ៍ និងអនុពលល្អបំផុតសម្រាប់ប្រើប្រាស់ មិគ្ខៃ AI ក្នុងគម្រោង Edge AI របស់អ្នក ពីការជ្រើសរើសម៉ូឌែលដំបូងរហូតដល់ការដាក់បញ្ចូលផលិតកម្ម។

## សង្ខេប

មិគ្ខៃ AI សម្រាប់ Visual Studio Code ជាការពង្រីកកម្រិតខ្ពស់ដែលធ្វើឲ្យការអភិវឌ្ឍភាគីផ្សេង និងការបង្កើតកម្មវិធី AI មានភាពរហ័ស។ មិគ្ខៃនេះផ្ដល់នូវសមត្ថភាពពេញលេញសម្រាប់ស្វែងរក ប៉ាន់ប្រមាណ និងដាក់បញ្ចូលម៉ូឌែល AI ពីអ្នកផ្គត់ផ្គង់ជាច្រើន ដូចជា Anthropic, OpenAI, GitHub, Google — ជាមួយការគាំទ្រការរត់ម៉ូឌែលក្នុងតំបន់ដោយប្រើ ONNX និង Ollama។

អ្វីដែលខុសគ្នានៃ មិគ្ខៃ AI គឺចំណុចពហុវិធីសាស្រ្តសម្រាប់ជីវចិញ្ចឹមអភិវឌ្ឍ AI ទាំងមូល។ មិនដូចឧបករណ៍អភិវឌ្ឍ AI ប្រពៃណីដែលផ្តោតលើផ្នែកតែមួយ ប៉ុន្តែ មិគ្ខៃ AI ផ្តល់បរិយាកាសបញ្ចូលគ្នាដែលគ្របដណ្តប់ការស្វែងរកម៉ូឌែល ការសាកល្បង អភិវឌ្ឍភាគីផ្សេង ការប៉ាន់ប្រមាណ និងការដាក់បញ្ចូល ក្រោមបរិយាកាស VS Code ដែលអ្នកស្គាល់។

វេទិកានេះរចនាគ្នា សម្រាប់ការសាងសង់ប្រតិបត្តិការ លឿន និងការដាក់បញ្ចូលផលិតកម្ម ជាមួយលក្ខណៈពិសេសដូចជា ការបង្កើតបញ្ញាក់ ការចាប់ផ្តើមរហ័ស ការចូលរួមឧបករណ៍ MCP (Model Context Protocol) និងសមត្ថភាពប៉ាន់ប្រមាណយ៉ាងទូលំទូលាយ។ សម្រាប់ការអភិវឌ្ឍ Edge AI វានិយាយថា អ្នកអាចអភិវឌ្ឍ សាកល្បង និងធ្វើអុបទីម៉ាយកម្មវិធី AI សម្រាប់ស្ថានการณ์ដាក់បញ្ចូល edge បានយ៉ាងមានប្រសិទ្ធភាព ខណៈរក្សាប្រតិបត្តិការអភិវឌ្ឍន៍ពេញលេញនៅក្នុង VS Code។

## គោលបំណងសិក្សា

នៅចប់មគ្គុទេសក៍នេះ អ្នកនឹងអាច៖

### ជំនាញមូលដ្ឋាន
- **ដំឡើង និងកំណត់រចនាសម្ព័ន្ធ** មិគ្ខៃ AI សម្រាប់ Visual Studio Code សម្រាប់លំហូរការងារអភិវឌ្ឍ Edge AI
- **រុករក និងប្រើប្រាស់** ចំណុចប្រទាក់ មិគ្ខៃ AI រួមមាន បញ្ជីម៉ូឌែល វាយលេង និងអ្នកបង្កើតភាគីផ្សេង
- **ជ្រើសរើស និងប៉ាន់ប្រមាណ** ម៉ូឌែល AI ដែលសមស្របសម្រាប់ការដាក់បញ្ចូល edge ដោយផ្អែកលើសមត្ថភាព និងដែនកំណត់ធនធាន
- **ផ្លាស់ប្ដូរ និងធ្វើអុបទីម៉ាយ** ម៉ូឌែលដោយប្រើរបៀប ONNX និងបច្ចេកទេសបញ្ចុះកម្រិតគុណភាពសម្រាប់ឧបករណ៍ edge

### ជំនាញអភិវឌ្ឍ Edge AI
- **រចនា និងអនុវត្ត** កម្មវិធី Edge AI ដោយប្រើបរិយាកាសអភិវឌ្ឍរួម
- **ធ្វើតេស្តម៉ូឌែល** ក្នុងលក្ខខណ្ឌដូចជា edge ដោយប្រើការព្យាករណ៍ក្នុងតំបន់ និងតាមដានធនធាន
- **បង្កើត និងប្ដូរតម្រូវ** អ្នកភាគីផ្សេង AI ដែលបានអុបទីម៉ាយសម្រាប់ស្ថានการณ์ edge
- **ប៉ាន់ប្រមាណសមត្ថភាពម៉ូឌែល** ដោយប្រើគណនាតម្លៃដែលពាក់ព័ន្ធ edge computing (រយៈពេលឆ្លើយ តម្រូវការចងចាំ, ភាពត្រឹមត្រូវ)

### ការធ្វើអុបទីម៉ាយ និងការដាក់បញ្ចូល
- **អនុវត្តបច្ចេកទេសបញ្ចុះកម្រិត និងកាត់បន្ថយ** ដើម្បីកាត់បន្ថយទំហំម៉ូឌែល ខណៈរក្សាសមត្ថភាពទទួលយកបាន
- **ធ្វើអុបទីម៉ាយម៉ូឌែល** សម្រាប់វេទិកាឧបករណ៍ edge ជាក់លាក់ រួមមាន CPU, GPU, និងអ៊ិនធឺក NPU
- **អនុវត្តអនុពលល្អបំផុត** សម្រាប់អភិវឌ្ឍ Edge AI រួមមានការគ្រប់គ្រងធនធាន និងយុទ្ធសាស្ត្រត្រឡប់ក្រោយ
- **រៀបចំម៉ូឌែល និងកម្មវិធី** សម្រាប់ដាក់បញ្ចូលផលិតកម្មនៅលើឧបករណ៍ edge

### គំនិត Edge AI ជាន់ខ្ពស់
- **បញ្ចូលគ្នាជាមួយក្រចេះ Edge AI** រួមមាន ONNX Runtime, Windows ML និង TensorFlow Lite
- **អនុវត្តសំណុំប្លង់ម៉ូឌែលច្រើន** និងស្ថានការណ៍ហ្វែតាទដែរ សម្រាប់បរិយាកាស edge
- **ដោះស្រាយបញ្ហាទូទៅរបស់ Edge AI** រួមមានកំណត់ចងចាំ ដំណើរការឆ្លើយបន្ទាន់ និងភាពឆបាគ្នា ឧបករណ៍
- **រចនាយុទ្ធសាស្ត្រត្រួតពិនិត្យ និងកំណត់ហេតុ** សម្រាប់កម្មវិធី Edge AI នៅក្នុងផលិតកម្ម

### ការអនុវត្តអនុភាព
- **សាងសង់ដំណោះស្រាយ Edge AI ពីដើមដល់ចប់** ចាប់ពីជ្រើសរើសម៉ូឌែលរហូតដល់ការដាក់បញ្ចូល
- **បង្ហាញជំនាញ** ក្នុងលំហូរការអភិវឌ្ឍដែលពាក់ព័ន្ធ edge និងបច្ចេកទេសអុបទីម៉ាយ
- **អនុវត្តគំនិតដែលបានរៀន** ទៅកាន់ករណីប្រើប្រាស់ edge AI មួយចំនួនដូចជា IoT, ចល័ត និងកម្មវិធីបង្កប់
- **ប៉ាន់ប្រមាណ និងប្រៀបធៀប**យុទ្ធសាស្ត្រដាក់បញ្ចូល edge AI ផ្សេងៗ និងការបញ្ជួស

## លក្ខណៈសម្បត្តិសំខាន់សម្រាប់ការអភិវឌ្ឍ Edge AI

### 1. បញ្ជីម៉ូឌែល និងការស្វែងរក
- **គាំទ្រតាមអ្នកផ្គត់ផ្គង់ច្រើន**៖ រុករក និងចូលដំណើរការម៉ូឌែល AI ពី Anthropic, OpenAI, GitHub, Google និងអ្នកផ្គត់ផ្គង់ផ្សេងទៀត
- **បញ្ចូលម៉ូឌែលក្នុងតំបន់**៖ ស្វែងរកម៉ូឌែល ONNX និង Ollama យ៉ាងងាយស្រួលសម្រាប់ដាក់បញ្ចូល edge
- **ម៉ូឌែល GitHub**៖ ដំណើរការចូលដំណើរការដោយផ្ទាល់ជាមួយម៉ូឌែល GitHub
- **ប្រៀបធៀបម៉ូឌែល**៖ ប្រៀបធៀបម៉ូឌែលជាប់គ្នាដើម្បីស្វែងរកតុល្យភាពល្អបំផុតសម្រាប់ដែនកំណត់ឧបករណ៍ edge

### 2. ពហុបទលេងអន្តរជាតិ
- **បរិយាកាសសាកល្បងអន្តរកម្ម**៖ សាកល្បងយ៉ាងលឿនជាមួយសមត្ថភាពម៉ូឌែលនៅក្នុងបរិយាកាសគ្រប់គ្រង
- **គាំទ្រពហុម៉ូដ**៖ សាកល្បងជាមួយរូបភាព អត្ថបទ និងទិន្នន័យបញ្ចូលផ្សេងៗ ដែលគេប្រើប្រាស់នៅ edge
- **សាកល្បងពេលវេលាចំពោះពេល**៖ ទទួលបានមតិយោបល់ភ្លាមៗពីការឆ្លើយតប និងសមត្ថភាពម៉ូឌែល
- **ធ្វើអុបទីម៉ាយប៉ារ៉ាម៉ែត្រ**៖ ធ្វើតំណល់ប៉ារ៉ាម៉ែត្រម៉ូឌែលសម្រាប់តម្រូវការដាក់បញ្ចូល edge

### 3. អ្នកបង្កើតបញ្ញាក់ (Agent)  
- **បង្កើតភាសាធម្មជាតិ**៖ បង្កើតបញ្ញាក់ដើមដោយប្រើការពិពណ៌នាភាសាធម្មជាតិ
- **ដំណើរការផ្ទេរពីរនិច**៖ ធ្វើការកែលម្អបញ្ញាក់ទៅតាមការឆ្លើយតប និងសមត្ថភាពម៉ូឌែល
- **បំបែកភារកិច្ច**៖ បំបែកភារកិច្ចស្មុគស្មាញដោយខ្សែបញ្ញាក់ និងលទ្ធផលរៀបចំ
- **គាំទ្រផ្លាស់ប្ដូរប្រភេទបំរែបំរួល**៖ ប្រើប្រាស់អថេរនៅក្នុងបញ្ញាក់សម្រាប់អរិយធម៌ភាគីផ្សេងឌីណាមិក
- **បង្កើតកូដផលិតកម្ម**៖ បង្កើតកូដត្រៀមផលិតកម្មសម្រាប់អភិវឌ្ឍកម្មវិធីបានយ៉ាងលឿន

### 4. ទប់រត់ និងប៉ាន់ប្រមាណច្រើនជួរ
- **សាកល្បងម៉ូឌែលប.multiple**៖ ដំណើរការបញ្ញាក់ច្រើនជាមួយម៉ូឌែលជ្រើសរើសក្នុងពេលតែមួយ
- **សាកល្បងមានប្រសិទ្ធភាពនៅលំនាំធំ**៖ សាកល្បងទិន្នន័យបញ្ចូល និងការកំណត់ដោយមានប្រសិទ្ធភាព
- **ករណីតេស្តផ្ទាល់ខ្លួន**៖ បើករត់ភាគីផ្សេងជាមួយករណីតេស្តសម្រាប់ធ្វើតេស្តមុខងារ
- **ប្រៀបធៀបសមត្ថភាព**៖ ប្រៀបធៀបលទ្ធផលជាច្រើនម៉ូឌែល និងកំណត់រចនាសម្ព័ន្ធផ្សេងៗ

### 5. ប៉ាន់ប្រមាណម៉ូឌែលជាមួយឯកទិន្នន័យ
- **គណនាតម្លៃស្ដង់ដារ**៖ សាកល្បងម៉ូឌែល AI ដោយប្រើអ្នកប៉ាន់ប្រមាណក្នុងសំណុំ (ពិន្ទុ F1, សមាសភាព, ស្រដៀង, សមត្ថភាពផ្ទាល់)
- **អ្នកប៉ាន់ប្រមាណផ្ទាល់ខ្លួន**៖ បង្កើតវិធានប៉ាន់ប្រមាណផ្ទាល់ខ្លួនសម្រាប់ករណីប្រើប្រាស់ជាក់លាក់
- **បញ្ចូលឯកទិន្នន័យ**៖ សាកល្បងម៉ូឌែលជាមួយឯកទិន្នន័យពេញលេញ
- **វាស់សមត្ថភាព**៖ បរិមាណសមត្ថភាពម៉ូឌែលសម្រាប់សេចក្ដីសម្រេចចិត្តដាក់បញ្ចូល edge

### 6. សមត្ថភាពបង្កើតឡើងវិញ (Fine-tuning)
- **ប្ដូរតម្រូវម៉ូឌែល**៖ ប្ដូរតម្រូវម៉ូឌែលសម្រាប់ករណីប្រើប្រាស់ និងដែនកំណត់ជាក់លាក់
- **ការរៀបចំពិសេស**៖ ប្ដូរតម្រូវម៉ូឌែលទៅគោលដៅពិសេសៗ
- **អុបទីម៉ាយសម្រាប់ edge**៖ បង្កើតឡើងវិញម៉ូឌែលសម្រាប់ដែនកំណត់ edge ជាក់លាក់
- **បណ្តុះបណ្តាលវិស័យជាក់លាក់**៖ បង្កើតម៉ូឌែលគោលបំណងសម្រាប់ករណីប្រើប្រាស់ edge ជាក់លាក់

### 7. ការបញ្ចូលឧបករណ៍ MCP
- **ការតភ្ជាប់ឧបករណ៍ខាងក្រៅ**៖ តភ្ជាប់ភាគីផ្សេងទៅឧបករណ៍ខាងក្រៅតាមរយៈម៉ាស៊ីនបម្រើ Model Context Protocol
- **សកម្មភាពជាក់ស្តែង**៖ អនុញ្ញាតឱ្យភាគីផ្សេង ស្វែងរកទិន្នន័យ បើកប្រើ API ឬដំណើរការគោលលទ្ធ់ផ្ទាល់ខ្លួន
- **ម៉ាស៊ីនបម្រើ MCP មានស្រាប់**៖ ប្រើឧបករណ៍ពី command (stdio) ឬ HTTP (server-sent event)
- **អភិវឌ្ឍ MCP ផ្ទាល់ខ្លួន**៖ សាងសង់ និងជំនួយបង្កើតម៉ាស៊ីនបម្រើ MCP ថ្មីជាមួយការសាកល្បងក្នុងAgent Builder

### 8. អភិវឌ្ឍ និង សាកល្បងភាគីផ្សេង
- **គាំទ្រហៅមុខងារ**៖ អនុញ្ញាតឱ្យភាគីផ្សេងហៅមុខងារខាងក្រៅយ៉ាងឌីណាមិក
- **សាកល្បងតភ្ជាប់ពេលវេលាពិត**៖ សាកល្បងការតភ្ជាប់ជាមួយការដំណើរការពេលវេលាពិត និងការប្រើឧបករណ៍
- **Versioning ភាគីផ្សេង**៖ គ្រប់គ្រងកំណែភាគីផ្សេងជាមួយសមត្ថភាពប្រៀបធៀបលទ្ធផល
- **ការធ្វើបញ្ហា និងតាមដាន**៖ លទ្ធភាពតាមដាន និងដោះស្រាយកំហុសក្នុងតំបន់សម្រាប់អភិវឌ្ឍភាគីផ្សេង

## លំហូរការងារអភិវឌ្ឍ Edge AI

### ជំរាបទី ១៖ ស្វែងរក និងជ្រើសរើសម៉ូឌែល
1. **រុករកបញ្ជីម៉ូឌែល**៖ ប្រើបញ្ជីម៉ូឌែលសម្រាប់រកម៉ូឌែលសមស្របសម្រាប់ដាក់បញ្ចូល edge
2. **ប្រៀបធៀបសមត្ថភាព**៖ ប៉ាន់ប្រមាណម៉ូឌែលដោយផ្អែកលើទំហំ ភាពត្រឹមត្រូវ និងល្បឿនព្យាករណ៍
3. **សាកល្បងក្នុងតំបន់**៖ ប្រើម៉ូឌែល Ollama ឬ ONNX ដើម្បីសាកល្បងក្នុងតំបន់មុនដាក់បញ្ចូល edge
4. **វាយតម្លៃតម្រូវការធនធាន**៖ កំណត់ចំណាំចងចាំ និងតម្រូវការគណនា សម្រាប់ឧបករណ៍ edge គោលដៅ

### ជំរាបទី ២៖ អុបទីម៉ាយម៉ូឌែល
1. **ផ្លាស់ប្តូរទៅ ONNX**៖ បម្លែងម៉ូឌែលជ្រើសរើសទៅទ្រង់ទ្រាយ ONNX សម្រាប់ភាពឆបគ្នា edge
2. **អនុវត្តបច្ចេកទេសបញ្ចុះកម្រិត**៖ កាត់បន្ថយទំហំម៉ូឌែលតាមរយៈការបញ្ចុះកម្រិត INT8 ឬ INT4
3. **អុបទីម៉ាយឧបករណ៍**៖ ធ្វើអុបទីម៉ាយសម្រាប់ឧបករណ៍ edge គោលដៅ (ARM, x86, ឧបករណ៍ជំនួយពិសេស)
4. **ផ្ទៀងផ្ទាត់សមត្ថភាព**៖ ពិនិត្យថាម៉ូឌែលដែលបានអុបទីម៉ាយរក្សាសមត្ថភាពទទួលយកបាន

### ជំរាបទី ៣៖ អភិវឌ្ឍកម្មវិធី
1. **រចនាភាគីផ្សេង**៖ ប្រើ Agent Builder ដើម្បីបង្កើតភាគីផ្សេង AI អុបទីម៉ាយសម្រាប់ edge
2. **បច្ចេកទេសបញ្ញាក់**៖ បង្កើតបញ្ញាក់ដែលធ្វើការយ៉ាងមានប្រសិទ្ធភាពជាមួយម៉ូឌែល edge ទំហំតូច
3. **សាកល្បងតភ្ជាប់**៖ សាកល្បងភាគីផ្សេងនៅក្នុងលក្ខខណ្ឌយោង edge
4. **បង្កើតកូដ**៖ បង្កើតកូដផលិតកម្មដែលអុបទីម៉ាយសម្រាប់ដាក់បញ្ចូល edge

### ជំរាបទី ៤៖ ប៉ាន់ប្រមាណ និងសាកល្បង
1. **ប៉ាន់ប្រមាណជាច្រក**៖ សាកល្បងការកំណត់ជាច្រើនដើម្បីរកលក្ខខណ្ឌល្អបំផុតសម្រាប់ edge
2. **វិភាគសមត្ថភាព**៖ វិភាគល្បឿនព្យាករណ៍ ការប្រើប្រាស់ចងចាំ និងភាពត្រឹមត្រូវ
3. **សមមាត្រវីជាមួយ edge**៖ សាកល្បងក្នុងលក្ខខណ្ឌ​ដូច​គោលដៅ​ដាក់​បញ្ចូល edge
4. **សាកល្បងកំណត់បន្ទុក**៖ ប៉ាន់ប្រមាណសមត្ថភាពក្រោមលក្ខខណ្ឌស្ទេសនានា

### ជំរាបទី ៥៖ រៀបចំដាក់បញ្ចូល
1. **អុបទីម៉ាយចុងក្រោយ**៖ អនុវត្តអុបទីម៉ាយចុងក្រោយពីលទ្ធផលសាកល្បង
2. **វេចខ្ចប់ដាក់បញ្ចូល**៖ វេចខ្ចប់ម៉ូឌែល និងកូដសម្រាប់ដាក់បញ្ចូល edge
3. **ឯកសារយោង**៖ សរសេរឯកសារអំពីតម្រូវការ និងការកំណត់រចនាសម្ព័ន្ធដាក់បញ្ចូល
4. **រៀបចំការត្រួតពិនិត្យ**៖ រៀបចំការត្រួតពិនិត្យ និងកំណត់ហេតុសម្រាប់ដាក់បញ្ចូល edge

## គោលដៅអ្នកប្រើប្រាស់សម្រាប់អភិវឌ្ឍ Edge AI

### អ្នកអភិវឌ្ឍ Edge AI
- អ្នកអភិវឌ្ឍកម្មវិធីសម្រាប់ឧបករណ៍ edge និងដំណោះស្រាយ IoT
- អ្នកអភិវឌ្ឍប្រព័ន្ធបង្កប់ចូល AI ក្នុងឧបករណ៍មានកំណត់ធនធាន
- អ្នកអភិវឌ្ឍចល័តបង្កើតកម្មវិធី AI លើឧបករណ៍ចល័ត និងថេប្លេត

- វិស្វករ AI អុបទីម៉ាយម៉ូឌែលសម្រាប់ដាក់បញ្ចូល edge និងគ្រប់គ្រងប៉ាយប៊្លាញ
- វិស្វករ DevOps ដាក់បញ្ចូល និងគ្រប់គ្រងម៉ូឌែល AI ក្រោមបណ្តាប្រព័ន្ធ edge ចែកចាយ
- វិស្វករសមត្ថភាពអុបទីម៉ាយបន្ទុក AI សម្រាប់ដែនកំណត់ឧបករណ៍ edge

### អ្នកស្រាវជ្រាវ និងអ្នកបង្រៀន
- អ្នកស្រាវជ្រាវ AI អភិវឌ្ឍម៉ូឌែល និងអាល់ហ្គរីធម៍ប្រសើរសម្រាប់ edge computing
- អ្នកបង្រៀនបង្រៀនគំនិត Edge AI និងបង្ហាញបច្ចេកទេសអុបទីម៉ាយ
- និស្សិតរៀនអំពីបញ្ហា និងដំណោះស្រាយក្នុងការដាក់បញ្ចូល Edge AI

## ករណីប្រើប្រាស់ Edge AI

### ឧបករណ៍ IoT ចំណេះដឹង
- **ការទំណាក់ទិន្នន័យរូបភាពពេលវេលាពិត**៖ ដាក់បញ្ចូលម៉ូឌែលកុំព្យូទ័រមើលផ្ទាល់ឧបករណ៍កាមេរ៉ា និងឧបករណ៍បញ្ចូល
- **បំលែងសំលេង**៖ អនុវត្តការបញ្ជាក់សំលេង និងបំលែងភាសាធម្មជាតិសម្រាប់បំពងសំលេងឆ្លាតវៃ
- **ថែទាំមុនពេលខូចខាត**៖ ដំណើរការម៉ូឌែលរកឃើញករណីមិនធម្មតា លើឧបករណ៍ edge ឧស្សាហកម្ម
- **ត្រួតពិនិត្យបរិស្ថាន**៖ ដាក់បញ្ចូលម៉ូឌែលវិភាគទិន្នន័យអង្គភាពបរិស្ថាន

### កម្មវិធីចល័ត និងបង្កប់
- **បកប្រែភាសានៅលើឧបករណ៍**៖ អនុវត្តម៉ូឌែលបកប្រែភាសាដោយអាចដំណើរការលើឧបករណ៍ក្រៅបណ្តាញ
- **រៀបចំរូបភាពបន្ថែម**៖ ដាក់បញ្ចូលការទំនាក់ទំនងត្តឯកតា និងតាមដានវត្ថុសម្រាប់កម្មវិធី AR
- **ត្រួតពិនិត្យសុខភាព**៖ ដំណើរការម៉ូឌែលវិភាគសុខភាពលើឧបករណ៍ពាក់ និងឧបករណ៍វេជ្ជសាស្រ្ត
- **ប្រព័ន្ធអាគុយម៉ាទិក**៖ អនុវត្តម៉ូឌែលជ្រើសរើសសម្រាប់ឧបករណ៍ដ្រង់ បង្គោល និងយានយន្ត

### រចនាសម្ព័ន្ធកុំព្យូទ័រនៅ Edge
- **មជ្ឈមណ្ឌលទិន្នន័យ Edge**៖ ដាក់បញ្ចូលម៉ូឌែល AI នៅមជ្ឈមណ្ឌលទិន្នន័យ edge សម្រាប់កម្មវិធីមានភាពយឺតទាប
- **ការបញ្ចូល CDN**៖ រួមបញ្ចូលសមត្ថភាពបង្កើត AI ក្នុងបណ្ដាញផ្តល់មាតិកា
- **5G Edge**៖ ប្រើប្រាស់ Edge computing 5G សម្រាប់កម្មវិធីអាចរកកម្លាំងដោយ AI
- **Fog Computing**៖ អនុវត្តបង្កើត AI នៅក្នុងបរិយាកាស Fog computing

### មគ្គុទេសក៍ចាប់ផ្តើមរហ័ស

យើងសូមណែនាំឲ្យចាប់ផ្តើមជាមួយម៉ូឌែលដែលផ្ទុកដោយ GitHub ដើម្បីទទួលបានបទពិសោធន៍ដ៏រលូន៖

1. **ដំឡើង**៖ អនុវត្តតាម មគ្គុទេសក៍ដំឡើង [installation guide](https://code.visualstudio.com/docs/intelligentapps/overview#_install-and-setup) ដើម្បីកំណត់មិគ្ខៃ AI សម្រាប់ឧបករណ៍របស់អ្នក
2. **ស្វែងរកម៉ូឌែល**៖ ពីមើលពង្រីកចំណាត់ថ្នាក់ជ្រើសរើស **CATALOG > Models** ដើម្បីស្វែងរកម៉ូឌែលដែលមាន
3. **ម៉ូឌែល GitHub**៖ ចាប់ផ្តើមជាមួយម៉ូឌែលដែលផ្ទុកនៅ GitHub សម្រាប់ការចូលរួមល្អបំផុត
4. **សាកល្បងនៅក្នុង Playground**៖ ពីកាតម៉ូឌែលណាមួយ ជ្រើស **Try in Playground** ដើម្បីចាប់ផ្តើមសាកល្បងសមត្ថភាពម៉ូឌែល

### ជំហានដល់ជំហានសម្រាប់អភិវឌ្ឍ Edge AI

#### ជំហាន ១៖ ស្វែងរក និងជ្រើសរើសម៉ូឌែល  
1. បើកមើលមិគ្ខៃ AI នៅក្នុង Activity Bar របស់ VS Code  
2. រុករកបញ្ជីម៉ូឌែលសម្រាប់ម៉ូឌែលដែលសាកសមសម្រាប់ដាក់បញ្ចូល edge  
3. ផ្ទុកតាមអ្នកផ្គត់ផ្គង់ (GitHub, ONNX, Ollama) ដោយផ្អែកលើតម្រូវការរបស់អ្នក  
4. ប្រើ **Try in Playground** ដើម្បីសាកល្បងសមត្ថភាពម៉ូឌែលភ្លាមៗ  

#### ជំហាន ២៖ អភិវឌ្ឍភាគីផ្សេង  
1. ប្រើ **Prompt (Agent) Builder** ដើម្បីបង្កើតភាគីផ្សេង AI អុបទីម៉ាយសម្រាប់ edge  
2. បង្កើតបញ្ញាក់ដើមដោយការពិពណ៌នាភាសាធម្មជាតិ  
3. កែប្រែ និងបង្កើនបញ្ញាក់ ដោយផ្អែកលើការឆ្លើយតបម៉ូឌែល  
4. បញ្ចូលឧបករណ៍ MCP ដើម្បីពង្រីកសមត្ថភាពភាគីផ្សេង
#### ជំហានទី 3៖ ការប្រឡងនិងវាយតម្លៃ  
1. ប្រើ **ការរត់បង្គូល** ដើម្បីតេស្តបច្ចោមបាច់ច្រើនលើម៉ូដែលដែលបានជ្រើស  
2. រត់ភ្នាក់ងារជាមួយករណីតេស្តដើម្បីផ្ទៀងផ្ទាត់មុខងារ  
3. វាយតម្លៃភាពគ្រឹះនិងកម្រិតសមត្ថភាពដោយប្រើកម្រិតមេត្រីចដែលមានរួចឬក៏បុគ្គលិក  
4. ប្រៀបធៀបម៉ូដែលនិងការកំណត់រចនាសម្ព័ន្ធខុសៗគ្នា  

#### ជំហានទី 4៖ ការស្វែងយល់និងបម្រុងទុក  
1. ប្តូរតម្រូវម៉ូដែលសម្រាប់ករណីវែងជាប់បន្ទាត់ពិសេស  
2. អនុវត្តការស្វែងយល់ពិសេសសម្រាប់ផ្នែកជាក់លាក់ទៀងតែលោក  
3. បង្កើនប្រសិទ្ធភាពដោយគោលការណ៍ដាក់តម្រូវសម្រាប់បរិយាកាសមុខមាត់  
4. កំណែដំណើរការនិងប្រៀបធៀបការកំណត់រចនាសម្ព័ន្ធភ្នាក់ងារ  

#### ជំហានទី 5៖ ការរៀបចំដាក់បញ្ចាំង  
1. បង្កើតកូដធ្វើការផលិតរួចជាស្រេចដោយប្រើ Agent Builder  
2. បំពាក់ការតភ្ជាប់ម៉ាស៊ីនបម្រើ MCP សម្រាប់ការប្រើប្រាស់ផលិតផល  
3. រៀបចំកញ្ចប់ដាក់បញ្ចាំងសម្រាប់ឧបករណ៍ខ្នាតតូច  
4. កំណត់ការត្រួតពិនិត្យនិងវាយតម្លៃកម្រិតមេត្រីច

## ពិនិត្យនិងអនុវត្តល្អបំផុតសម្រាប់ការអភិវឌ្ឍ Edge AI  

### ជម្រើសម៉ូដែល  
- **ការរឹតបន្តឹងទំហំ**៖ ជ្រើសម៉ូដែលឲ្យសមស្របនឹងកំណត់ចំណុចផ្ទុកក្នុងឧបករណ៍គោលដៅ  
- **ល្បឿនប្រើការសន្មត់**៖ ផ្តល់អាទិភាពម៉ូដែលដែលមានល្បឿនប្រើការសន្មត់លឿនសម្រាប់កម្មវិធីពេលវេលាពិត  
- **ការជជែកលើភាពត្រឹមត្រូវ**៖ តុល្យភាពភាពត្រឹមត្រូវម៉ូដែលជាមួយកំណត់ធនធាន  
- **ការភ្ជាប់ទ្រង់ទ្រាយ**៖ ជ្រើសទ្រង់ទ្រាយ ONNX ឬទ្រង់ទ្រាយដែលបានធ្វើឲ្យសមរម្យសម្រាប់ថេបជាមុខមាត់  

### បច្ចេកទេសបង្កើនប្រសិទ្ធភាព  
- **Quantization**៖ ប្រើ Quantization INT8 ឬ INT4 ដើម្បីកាត់បន្ថយទំហំម៉ូដែលនិងបង្កើនល្បឿន  
- **Pruning**៖ ដកបំបាត់ប៉ារ៉ាម៉ែត្រម៉ូដែលមិនចាំបាច់ដើម្បីកាត់បន្ថយតម្រូវការគណនា  
- **Knowledge Distillation**៖ បង្កើតម៉ូដែលតូចៗដែលថែមទាំងរក្សាប្រសិទ្ធភាពនៃម៉ូដែលធំៗ  
- **Hardware Acceleration**៖ ប្រើ NPUs, GPUs ឬកម្មវិធីបំពាក់ពិសេសពេលដែលមាន  
 
### ស្ទ្រីមការអភិវឌ្ឍ  
- **ការតេស្តប្រេកង់**៖ តេស្តជាញឹកញាប់នៅក្នុងលក្ខខណ្ឌដូច Edge ពេលអភិវឌ្ឍ  
- **ការត្រួតពិនិត្យសមត្ថភាព**៖ ត្រួតពិនិត្យការប្រើធនធាននិងល្បឿនប្រើសន្មត់ជាប់ជាបន្ត  
- **ការគ្រប់គ្រងកំណែ**៖ តាមដានកំណែម៉ូដែលនិងការកំណត់បង្កើនប្រសិទ្ធភាព  
- **ឯកសារ**៖ ចងក្រងឯកសារ ពីសេចក្តីសម្រេចការបង្កើនប្រសិទ្ធភាពទាំងអស់ និងការជំរុញលទ្ធផល  

### គោលការណ៍ដាក់បញ្ចាំង  
- **ការត្រួតពិនិត្យធនធាន**៖ ត្រួតពិនិត្យអង្គចងចាំ CPU និងថាមពលក្នុងការផលិត  
- **យុទ្ធសាស្ត្រត្រឡប់ក្រោយ**៖ ធ្វើបច្ចេកទេសត្រឡប់ក្រោយសម្រាប់ករណីម៉ូដែលបរាជ័យ  
- **គ្រោងធ្វើបច្ចុប្បន្នភាព**៖ ដំឡើងផែនការសម្រាប់បច្ចុប្បន្នភាពម៉ូដែលនិងគ្រប់គ្រងកំណែ  
- **សុវត្ថិភាព**៖ អនុវត្តវិធានសុវត្ថិភាពសមរម្យសម្រាប់កម្មវិធី Edge AI  

## ការតភ្ជាប់ជាមួយស៊ុមរបស់ Edge AI  

### ONNX Runtime  
- **ការដាក់គ្រប់ផ្លាតហ្វូម**៖ ដាក់ម៉ូដែល ONNX ទៅលើវេទិកាខ្នាតតូចផ្សេងៗ  
- **បង្កើនប្រសិទ្ធភាពតាមស៊ុមរឹង**៖ ប្រើប្រសិទ្ធភាពពិសេស ONNX Runtime ទៅលើស៊ុមរឹង  
- **គាំទ្រជំនួយលើទូរស័ព្ទ**៖ ប្រើ ONNX Runtime Mobile សម្រាប់កម្មវិធីទូរស័ព្ទដៃនិងថេប្លែត  
- **ការបញ្ចូល IoT**៖ ដាក់លើឧបករណ៍ IoT ដោយប្រើការចែកចាយទំងន់ស្រាលរបស់ ONNX Runtime  

### Windows ML  
- **ឧបករណ៍ Windows**៖ បង្កើនប្រសិទ្ធភាពសម្រាប់ឧបករណ៍ Edge ដែលដំណើរការលើវីនដូហ៍និងកុំព្យូទ័រ  
- **ល្បឿនលឿនដោយ NPU**៖ ប្រើ Neural Processing Units លើឧបករណ៍វីនដូ  
- **DirectML**៖ ប្រើ DirectML សម្រាប់បង្កើនល្បឿន GPU នៅលើវីនដូម  
- **ការតភ្ជាប់ UWP**៖ បញ្ចូលជាមួយកម្មវិធី Universal Windows Platform  

### TensorFlow Lite  
- **បង្កើនប្រសិទ្ធភាពលើទូរស័ព្ទ**៖ ដាក់ម៉ូដែល TensorFlow Lite លើឧបករណ៍ទូរស័ព្ទនិងឧបករណ៍ចម្លាប់ទិន្នន័យ  
- **Hardware Delegates**៖ ប្រើយន្តការរឹងពិសេសសម្រាប់បង្កើនល្បឿន  
- **Micro Controllers**៖ ដាក់លើមីក្រូកុងត្រូឡ័រដោយប្រើ TensorFlow Lite Micro  
- **គាំទ្រច្រើនផ្លាតហ្វូម**៖ ដាក់លើ Android, iOS និងប្រព័ន្ធ Linux ជាស៊ុមចម្លងភ្ជាប់  

### Azure IoT Edge  
- **លាយបញ្ចូលពពកនិង Edge**៖ បង្កើតការបណ្តុះបណ្តាលក្នុងពពកជាមួយការសន្មត់នៅលើ Edge  
- **ដាក់ជំរិត**៖ ដាក់ម៉ូដែល AI ជារូបមន្ត IoT Edge  
- **គ្រប់គ្រងឧបករណ៍**៖ គ្រប់គ្រងឧបករណ៍ Edge និងអាប់ដេតម៉ូដែលពីចម្ងាយ  
- **ទិន្នន័យត្រួតពិនិត្យ**៖ ប្រមូលទិន្នន័យប្រសិទ្ធភាពនិងមេត្រីចម៉ូដែលពីការដាក់បញ្ចាំង Edge  

## សញ្ញាបញ្ហា Edge AI កម្រិតខ្ពស់  

### ការដាក់ជ្រើសម៉ូដែលច្រើន  
- **ការប្រមូលម៉ូដែល**៖ ដាក់ម៉ូដែលច្រើនសម្រាប់កែលម្អភាពត្រឹមត្រូវ ឬការស្តារឡើងវិញ  
- **ការតេស្ត A/B**៖ តេស្តម៉ូដែលផ្សេងៗតែមួយពេលលើឧបករណ៍ Edge  
- **ជ្រើសរើសឌីណាមិក**៖ ជ្រើសម៉ូដែលផ្អែកលើលក្ខខណ្ឌឧបករណ៍បច្ចុប្បន្ន  
- **ចែករំលែកធនធាន**៖ បង្កើនប្រសិទ្ធភាពកំណត់ធនធានចែករំលែករវាងម៉ូដែលជាច្រើនដែលបានដាក់បញ្ចាំង  

### ការសិក្សាសហគ្រិន  
- **ការបណ្តុះបណ្តាលចែកចាយ**៖ បណ្តុះបណ្តាលម៉ូដែលលើឧបករណ៍ Edge ច្រើន  
- **ការពារជួញដូរបញ្ហាឯកជន**៖ រក្សារទិន្នន័យបណ្តុះបណ្តាលនៅក្នុងស្រុក ខណៈដែលចែករំលែកការកែលម្អម៉ូដែល  
- **ការសិក្សារួមគ្នា**៖ ធ្វើឲ្យឧបករណ៍អាចរៀនពីបទពិសោធរួមគ្នា  
- **ការសម្រុះសម្រួល Edge-Cloud**៖ សម្រុះសម្រួលការសិក្សារវ่างឧបករណ៍ Edge និងហើយពពក  

### ការដំណើរការពេលវេលាពិត  
- **ការដំណើរការស្ទ្រីម**៖ ដំណើរការទិន្នន័យចរាចរបន្តបន្តលើឧបករណ៍ Edge  
- **ការសន្មត់យឺតទាប**៖ បង្កើនប្រសិទ្ធភាពសម្រាប់ការសន្មត់យឺតបំផុត  
- **ការដំណើរការបណ្ដុំទិន្នន័យ**៖ ដំណើរការបណ្ដុំទិន្នន័យដោយមានប្រសិទ្ធភាពលើឧបករណ៍ Edge  
- **ការដំណើរការបត់បែន**៖ កែសម្រួលការដំណើរការតាមលក្ខខណ្ឌឧបករណ៍បច្ចុប្បន្ន

- **ការភ្ជាប់ស៊ុមរឹង**៖ ម៉ូដែលមិនស័ក្តិសមនឹងស៊ុមរឹងគោលដៅ

- **ការតេស្តរាលដាល**៖ តេស្តកំណែបន្ថែមមួយៗសម្រាប់បំបែកបញ្ហា  
- **ការសុភាពស្មើស្មាញស៊ុមរឹង**៖ ប្រើឧបករណ៍អភិវឌ្ឍន៍ដើម្បីស្រមោលស៊ុមរឹងគោលដៅ  

### ដំណោះស្រាយបង្កើនប្រសិទ្ធភាព

- **រចនាសម្ព័ន្ធម៉ូដែល**៖ ពិចារណារចនាសម្ព័ន្ធម៉ូដែលខុសៗគ្នាដែលបានបង្កើនសមរម្យសម្រាប់ Edge  
- **បង្កើនប្រសិទ្ធភាពកំពូលព្រឹត្តិកម្ម**៖ បង្កើតប្រសិទ្ធភាពការបង្ហាញទិន្នន័យមុន សម្រាប់ការរឹតបន្តឹង Edge  
- **បង្កើនប្រសិទ្ធភាពប្រើសន្មត់**៖ ប្រើកម្រិតល្បឿនប្រើសន្មត់ពិសេសទៅតាមស៊ុមរឹង

## សេចក្ដីសន្និដ្ឋាន  

AI Toolkit សម្រាប់ Visual Studio Code ជាវេទិកាទូលំទូលាយសម្រាប់ការអភិវឌ្ឍ AI រយៈពេលទាន់សម័យ ផ្តល់សមត្ថភាពអភិវឌ្ឍភ្នាក់ងារដែលមានប្រយោជន៍ពិសេសសម្រាប់កម្មវិធី Edge AI។ ជាមួយសៀវភៅម៉ូដែលធំរបស់វាដែលគាំទ្រអ្នកផ្គត់ផ្គង់ដូចជា Anthropic, OpenAI, GitHub, និង Google ផ្តល់ជាមួយសមត្ថភាពបង្កើតនៅក្នុងក្នុងប្រព័ន្ធ ONNX និង Ollama វាផ្តល់សេវាកម្មបត់បែនដែលចាំបាច់សម្រាប់ស្ថានភាពដាក់បញ្ចាំង Edge ផ្សេងៗ។  

កម្លាំងរបស់ ToolKit រួមមាន៖ ពីការស្វែងរកម៉ូដែលនិងការប្រមាញ់សាកល្បងនៅក្នុង Playground ទៅការអភិវឌ្ឍភ្នាក់ងារដែលស្មុគស្មាញជាមួយ Prompt Builder, សមត្ថភាពវាយតម្លៃពេញលេញ និងការតភ្ជាប់ឧបករណ៍ MCP ដោយរលូន។ សម្រាប់អ្នកអភិវឌ្ឍ Edge AI វានាំមកនូវការបង្កើតសំណុំគំរូនិងការតេស្តភ្នាក់ងារយ៉ាងលឿន មុនការដាក់បញ្ចាំង Edge ជាមួយសមត្ថភាពបង្វែរយ៉ាងរហ័សនិងការបង្កើនប្រសិទ្ធភាពសម្រាប់បរិយាកាសមានកំណត់ធនធាន។  

អត្ថប្រយោជន៍សំខាន់សម្រាប់ការអភិវឌ្ឍ Edge AI រួមមាន៖  
- **សាកល្បងយ៉ាងលឿន**៖ តេស្តម៉ូដែលនិងភ្នាក់ងារយ៉ាងលឿន មុនការដាក់បញ្ចាំង Edge  
- **បត់បែនលើអ្នកផ្គត់ផ្គង់ច្រើន**៖ ទទួលបានម៉ូដែលពីប្រភពផ្សេងៗ ដើម្បីរកដំណោះស្រាយ Edge ដ៏ល្អបំផុត  
- **អភិវឌ្ឍក្នុងស្រុក**៖ តេស្តជាមួយ ONNX និង Ollama សម្រាប់អភិវឌ្ឍក្រៅបណ្ដាញនិងការការពារភាពឯកជន  
- **រួចរាល់សម្រាប់ផលិតផល**៖ បង្កើតកូដរួចរាល់ ដើម្បីប្រើប្រាស់ផលិតផល និងភ្ជាប់ជាមួយឧបករណ៍ផ្សេងៗតាម MCP  
- **វាយតម្លៃពេញលេញ**៖ ប្រើអនុគមន៍ built-in និង custom ដើម្បីផ្ទៀងផ្ទាត់សមត្ថភាព Edge AI  

នៅពេល AI កំពុងរំកិលទៅរកស្ថានភាពដាក់បញ្ចាំង Edge AI, AI Toolkit សម្រាប់ VS Code ផ្តល់បរិយាកាសអភិវឌ្ឍន៍ និងស្ទ្រីមការងារដែលចាំបាច់សម្រាប់បង្កើត, តេស្ត និងបង្កើនប្រសិទ្ធភាពកម្មវិធីឆ្លាតវ័យសម្រាប់បរិយាកាសដែលមានកំណត់ធនធាន។ មិនថាលោកអ្នកកំពុងអភិវឌ្ឍដំណោះស្រាយ IoT, កម្មវិធី AI លើទូរស័ព្ទឬប្រព័ន្ធបញ្ចូលឆ្លាតវ័យ ToolKit នេះផ្តល់ជូនលក្ខណៈពិសេសទូលំទូលាយ និងតំបន់ការងារស្ទ្រីមការងារ ដែលគាំទ្រជីវិតរបរអភិវឌ្ឍ Edge AI ទាំងមូល។  

ជាមួយការអភិវឌ្ឍជាប់លាប់និងសហគមន៍ដ៏សកម្ម (ផ្កាយ GitHub 1.8k+), AI Toolkit រក្សាថា មានទីតាំងនៅខ្លួនមុខនៃឧបករណ៍អភិវឌ្ឍ AI ផ្សេងៗ ក្នុងការបន្តខិតខំប្រឹងប្រែងឆ្លើយតបតម្រូវការ អ្នកអភិវឌ្ឍ AI សម័យថ្មី ដែលកំពុងសាងសង់សម្រាប់ស្ថានភាពដាក់បញ្ចាំង Edge។
