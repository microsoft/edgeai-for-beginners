## ការណែនាំស្តីពីការដាក់ Phi-4-mini ក្នុងខុងតឺន័រ

ម៉ូឌែលភាសាតូច (Small Language Models - SLMs) ជាការរីកចម្រើនសំខាន់ចំពោះ EdgeAI ដែលអនុញ្ញាតឱ្យមានសមត្ថភាពដើម្បីដោះស្រាយភាសាធម្មជាតិស្មុគស្មាញលើឧបករណ៍ដែលមានធនធានកំណត់។ មេរៀននេះផ្ដោតលើយុទ្ធសាស្ត្រដាក់ប្រើក្នុងខុងតឺន័រសម្រាប់ Phi-4-mini-instruct របស់ Microsoft ដែលជាម៉ូឌែលដឹកនាំការយោលដែលផ្គូផ្គងសមត្ថភាពនិងប្រសិទ្ធភាព។

### ម៉ូឌែលលេចធ្លោ៖ Phi-4-mini-instruct

**Phi-4-mini-instruct (3.8B parameters)**: ម៉ូឌែលកែទ្រង់ទ្រាយស្រាលដែលបានលៃតម្រូវជាប្រភេទ instruction-tuned របស់ Microsoft សម្រាប់បរិបទដែលមានការកំណត់អំពីអង្គចងចាំ/កម្រិតកុំព្យូទ័រ និងមានសមត្ថភាពកម្ពស់ក្នុង៖
- **ការគិតយោលគណិត និងការគណនាស្មុគស្មាញ**
- **ការបង្កើតកូដ ការរំកិលកំហុស និងការវិភាគ**
- **ការដោះស្រាយបញ្ហាលក្ខណៈយុតិធម៌ និងការគិតជាជំហានៗ**
- **កម្មវិធីអប់រំដែលត្រូវការពន្យល់លម្អិត**
- **ការហៅមុខងារ និងការរួមបញ្ចូលឧបករណ៍**

សមាជិកនៃប្រភេទ "Small SLMs" (1.5B - 13.9B parameters) ធ្វើឱ្យ Phi-4-mini មានតុល្យភាពដ៏ល្អពីសមត្ថភាពយោល និងប្រសិទ្ធភាពធនធាន។

### អត្ថប្រយោជន៍នៃការដាក់ Phi-4-mini ក្នុងខុងតឺន័រ

- **ប្រសិទ្ធភាពប្រតិបត្តិការ**: វេចខ្ចប់ឱ្យបានទេនទាន់សម្រាប់ភារកិច្ចរៀបចំយោលដោយការទាមទារគណនា ទាបជាង
- **ភាពបត់បែនក្នុងការដាក់ប្រើ**: សមត្ថភាព AI លើឧបករណ៍ជាមួយភាពឯកជនកើនឡើងតាមរយៈការក-processing ជាលokal
- **ថ្លៃដើមសម្រួល**: កាត់បន្ថយការចំណាយប្រតិបត្តិទំនងដូចជាម៉ូឌែលធំៗ ពីព្រោះរក្សាគុណភាព
- **ការបំបែកដោយပៃស្រាល**: ការបំបែកស្អាតរវាងអត្ថបទម៉ូឌែល និងបរិវេណប្រតិបត្តិការ មានសុវត្ថិភាព
- **ការចែកចាយតាមទង់ទាយ**: អាចពង្រីកជារូបមន្តទ្រង់ទ្រាយស្រួលសម្រាប់បំពេញ throughput រៀបចំយោល

## ការយល់ដឹងអំពីចំណាត់ថ្នាក់ Phi-4-mini

### បញ្ជាក់ម៉ូឌែល

**ព័ត៌មានបច្ចេកទេស:**
- **Parameters**: 3.8 billion (ចំណាត់ថ្នាក់ Small SLM)
- **Architecture**: Dense decoder-only Transformer ជាមួយ grouped-query attention
- **Context Length**: 128K tokens (128K tokens; 32K ត្រូវបានផ្តល់អនុសាសន៍សម្រាប់សមត្ថភាពល្អបំផុត)
- **Vocabulary**: 200K tokens ជាមួយការគាំទ្រអភូមិសាស្ត្រភាសាច្រើន
- **Training Data**: 5T tokens នៃមាតិកាគុណភាពខ្ពស់ផ្ទុកការគិតយោល

### សមត្ថភាពរបស់ Phi-4-mini

- **លទ្ធភាពគណិតវិទ្យា**: អាចដោះស្រាយអង្គវិភាគ គណិត ឧបករណ៍ និងប្រធានបទ calculus ខ្ពស់
- **ឆ្លើយតបកូដ**: បង្កើតកូដ Python, JavaScript និងភាសាច្រើន ជាមួយការរំកិលកំហុស
- **ការគិតយោលត្រឹមត្រូវ**: បំបែកបញ្ហាជាជំហាន និងសាងសង់ដំណោះស្រាយ
- **គាំទ្រអប់រំ**: ពន្យល់លម្អិតសម្រាប់ស្ថានការ រៀន និងបង្រៀន
- **ការហៅមុខងារ**: គាំទ្រដល់ការរួមបញ្ចូលឧបករណ៍ និង API

## ការដាក់ប្រើ vLLM ក្នុងខុងតឺន័រ

vLLM ផ្ដល់ការគាំទ្រល្អសម្រាប់ Phi-4-mini-instruct ជាមួយសមត្ថភាព inference ដែលបានបង្កើននិង API ដែលអាចសមរម្យជាមួយ OpenAI, ធ្វើឱ្យវាเหมาะសម្រាប់សេវាកម្មរៀបចំយោលក្នុងផលិតកម្ម។

### ឧទាហរណៃចាប់ផ្តើមរហ័ស

#### ការដាក់ប្រើ CPU មូលដ្ឋាន (ការអភិវឌ្ឍ)
```bash
# ការដាក់ប្រើដែលបង្កើនប្រសិទ្ធភាពលើ CPU សម្រាប់ការអភិវឌ្ឍន៍ និងការសាកល្បង
docker run --name phi4-mini-dev \
  -e HUGGING_FACE_HUB_TOKEN=${HF_TOKEN} \
  -p 8000:8000 \
  --memory="8g" --cpus="4" \
  vllm/vllm-openai:latest \
  --model microsoft/Phi-4-mini-instruct \
  --max-model-len 4096 \
  --max-num-seqs 4 \
  --trust-remote-code
```

#### ការដាក់ប្រើផលិតកម្មដែលបង្កើនដោយ GPU
```bash
# ការដាក់ប្រើ GPU សម្រាប់ការសន្មតដែលមានប្រសិទ្ធភាពខ្ពស់
docker run --runtime nvidia --gpus all \
  --name phi4-mini-prod \
  -e HUGGING_FACE_HUB_TOKEN=${HF_TOKEN} \
  -p 8000:8000 \
  vllm/vllm-openai:latest \
  --model microsoft/Phi-4-mini-instruct \
  --max-model-len 8192 \
  --gpu-memory-utilization 0.8 \
  --enable-auto-tool-choice \
  --trust-remote-code
```

### ការកំណត់សម្រាប់ផលិតកម្ម

```yaml
version: '3.8'
services:
  phi4-mini-reasoning:
    image: vllm/vllm-openai:latest
    container_name: phi4-mini-production
    ports:
      - "8000:8000"
    volumes:
      - ~/.cache/huggingface:/root/.cache/huggingface
      - ./logs:/app/logs
    environment:
      - HUGGING_FACE_HUB_TOKEN=${HF_TOKEN}
      - CUDA_VISIBLE_DEVICES=0
    command: >
      --model microsoft/Phi-4-mini-instruct
      --host 0.0.0.0
      --port 8000
      --max-model-len 4096
      --max-num-seqs 8
      --gpu-memory-utilization 0.8
      --trust-remote-code
      --enable-auto-tool-choice
      --quantization awq
    deploy:
      resources:
        limits:
          memory: 12G
        reservations:
          devices:
            - driver: nvidia
              count: 1
              capabilities: [gpu]
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 30s
      timeout: 10s
      retries: 3
```

### ការអនុវត្តម៉ាស៊ីនបម្រើដែលបានសម្រួល

```python
# app/server.py - បានបង្កើតឲ្យល្អប្រសើរសម្រាប់ភារកិច្ចការគិតវិភាគរបស់ Phi-4-mini
import os
import time
import onnxruntime as ort
from transformers import AutoTokenizer
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI(title="Phi-4-mini ONNX Engine")

class ReasoningRequest(BaseModel):
    prompt: str
    task_type: str = "reasoning"  # ការគិតវិភាគ, ការសរសេរកូដ, គណិតវិទ្យា
    max_length: int = 200
    temperature: float = 0.3

class Phi4MiniEngine:
    def __init__(self):
        self.model = None
        self.tokenizer = None
        self.load_model()
    
    def load_model(self):
        model_path = "/app/models/phi4-mini-instruct.onnx"
        if os.path.exists(model_path):
            # អ្នកផ្គត់ផ្គង់ដែលបានបង្កើតឲ្យល្អប្រសើរសម្រាប់ភារកិច្ចការគិតវិភាគ
            providers = [
                ('CUDAExecutionProvider', {
                    'arena_extend_strategy': 'kSameAsRequested',
                    'cudnn_conv_algo_search': 'HEURISTIC',
                }),
                ('CPUExecutionProvider', {
                    'intra_op_num_threads': 4,
                    'inter_op_num_threads': 2,
                })
            ]
            
            self.model = ort.InferenceSession(model_path, providers=providers)
            self.tokenizer = AutoTokenizer.from_pretrained("microsoft/Phi-4-mini-instruct")
            print("✓ Phi-4-mini model loaded successfully")
        else:
            print("✗ Model file not found. Please convert the model first.")
    
    def generate_reasoning(self, request: ReasoningRequest):
        if not self.model:
            raise ValueError("Model not loaded")
        
        # ការបញ្ជាក់តាមភារកិច្ចសម្រាប់ធ្វើឲ្យការគិតវិភាគប្រសើរឡើង
        task_prompts = {
            "reasoning": "Think step by step and show your reasoning clearly:",
            "math": "Solve this mathematical problem step by step:",
            "coding": "Write clean, well-commented code for this task:"
        }
        
        system_prompt = task_prompts.get(request.task_type, "")
        full_prompt = f"{system_prompt}\n{request.prompt}"
        
        # បំបែកជា token និងដំណើរការព្យាករ
        inputs = self.tokenizer.encode(full_prompt, return_tensors="np", max_length=2048, truncation=True)
        
        start_time = time.time()
        outputs = self.model.run(None, {"input_ids": inputs})
        inference_time = time.time() - start_time
        
        # បកប្រែចម្លើយ
        generated_text = self.tokenizer.decode(outputs[0][0], skip_special_tokens=True)
        
        return {
            "generated_text": generated_text,
            "task_type": request.task_type,
            "inference_time": inference_time,
            "model": "phi4-mini-instruct-onnx"
        }

# ចាប់ផ្តើមម៉ូទ័រ
engine = Phi4MiniEngine()

@app.post("/reasoning")
async def generate_reasoning(request: ReasoningRequest):
    try:
        return engine.generate_reasoning(request)
    except Exception as e:
        return {"error": str(e)}

@app.get("/health")
async def health():
    return {
        "status": "healthy" if engine.model else "model_not_loaded",
        "model": "phi4-mini-instruct",
        "capabilities": ["reasoning", "math", "coding"]
    }

@app.get("/")
async def root():
    return {
        "name": "Phi-4-mini ONNX Engine",
        "model": "microsoft/Phi-4-mini-instruct",
        "endpoints": ["/reasoning", "/health"],
        "capabilities": ["mathematical reasoning", "code generation", "logical problem solving"]
    }
```

### ស្គ្រីបបម្លែងម៉ូឌែល

```python
# convert_phi4_mini.py - បម្លែង Phi-4-mini ទៅជា ONNX ដែលបានបង្កើតឲ្យមានប្រសិទ្ធភាព
import os
from pathlib import Path
from optimum.onnxruntime import ORTModelForCausalLM, ORTOptimizer, ORTQuantizer
from optimum.onnxruntime.configuration import AutoQuantizationConfig, AutoOptimizationConfig
from transformers import AutoTokenizer

def convert_phi4_mini():
    print("Converting Phi-4-mini-instruct to optimized ONNX...")
    
    model_name = "microsoft/Phi-4-mini-instruct"
    output_dir = Path("./models/phi4-mini-onnx")
    output_dir.mkdir(parents=True, exist_ok=True)
    
    # ជំហាន 1: បម្លែងទៅជា ONNX
    print("Step 1: Converting to ONNX format...")
    model = ORTModelForCausalLM.from_pretrained(
        model_name,
        export=True,
        provider="CPUExecutionProvider",
        use_cache=True
    )
    
    # ជំហាន 2: អនុវត្តការបង្កើតប្រសិទ្ធភាពសម្រាប់កិច្ចការធ្វើហេតុផល
    print("Step 2: Applying reasoning-specific optimizations...")
    optimization_config = AutoOptimizationConfig.with_optimization_level(
        optimization_level="O3",
        optimize_for_gpu=True,
        fp16=True
    )
    
    optimizer = ORTOptimizer.from_pretrained(model)
    optimizer.optimize(save_dir=output_dir, optimization_config=optimization_config)
    
    # ជំហាន 3: អនុវត្តការបង្រួមចំនួន (quantization) សម្រាប់ដាក់ប្រើលើឧបករណ៍ព្រំ
    print("Step 3: Applying quantization...")
    quantization_config = AutoQuantizationConfig.avx512_vnni(
        is_static=False,
        per_channel=True
    )
    
    quantizer = ORTQuantizer.from_pretrained(output_dir)
    quantizer.quantize(
        save_dir=output_dir / "quantized",
        quantization_config=quantization_config
    )
    
    # ជំហាន 4: រក្សាទុកកម្មវិធីបំបែកពាក្យ (tokenizer)
    tokenizer = AutoTokenizer.from_pretrained(model_name)
    tokenizer.save_pretrained(output_dir)
    
    # ជំហាន 5: បង្កើតម៉ូដែលចុងក្រោយដែលបានធ្វើឲ្យមានប្រសិទ្ធភាព
    final_model_path = Path("./models/phi4-mini-instruct.onnx")
    quantized_files = list((output_dir / "quantized").glob("*.onnx"))
    if quantized_files:
        import shutil
        shutil.copy2(quantized_files[0], final_model_path)
        print(f"✓ Phi-4-mini converted and optimized: {final_model_path}")
    
    return final_model_path

if __name__ == "__main__":
    convert_phi4_mini()
```

## លទ្ធផលការរៀន

បន្ទាប់ពីបញ្ចប់មូឌុលនេះ អ្នកនឹងអាច៖

1. ដាក់ទម្រង់ម៉ូឌែល Phi-4-mini-instruct ក្នុងបរិបទខុងតឺន័រប្រើស៊ុមជាច្រើន
2. កំណត់និងបង្កើនប្រសិទ្ធភាពការដាក់ SLM សម្រាប់បរិក្ខារថេរៗផ្សេងគ្នា
3. អនុវត្តវិធានសុវត្ថិភាពល្អបំផុតសម្រាប់ដាក់ AI ក្នុងខុងតឺន័រ
4. ប្រៀបធៀបទំនាក់ទំនង និងជ្រើសរើសស៊ុមដាក់ប្រើសមរម្យទៅតាមតម្រូវការ
5. អនុវត្តយុទ្ធសាស្ត្រតាមដាន និងការពង្រីកសម្រាប់សេវាកម្ម SLM កម្រិតផលិតកម្ម
