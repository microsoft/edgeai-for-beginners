## Introduction

OpenVINO (Open Visual Inference and Neural Network Optimization) គឺជាបណ្ណាល័យខ្ទង់បើករបស់ Intel សម្រាប់ដាក់បញ្ចូល ដំណោះស្រាយ AI ដែលមានសមត្ថភាពល្អនៅលើ cloud, លើម៉ាស៊ីនក្នុងក្រុមហ៊ុន (on-premises) និងនៅលើឧបករណ៍ edge។ មិនថាអ្នកគោលដៅទៅលើ CPU, GPU, VPU ឬឧបករណ៍បង្កើតល្បឿន AI ជាក់លាក់ណា OpenVINO ផ្តល់នូវសមត្ថភាពអុបទីម៉ាយពេញលេញ ខណៈដែលរក្សានូវភាពត្រឹមត្រូវនៃម៉ូឌែល និងអនុញ្ញាតឱ្យដាក់បញ្ចូលបានលើបណ្តាប្រព័ន្ធផ្សេងៗ។

## What is OpenVINO?

OpenVINO គឺជាឧបករណ៍ប្រភពបើកដែលអនុញ្ញាតឱ្យអ្នកអភិវឌ្ឍន៍អាចធ្វើអុបទីម៉ាយ បម្លែង និងដាក់បញ្ចូលម៉ូឌែល AI យ៉ាងមានប្រសិទ្ធភាពលើឧបករណ៍រួមផ្សំផ្សេងៗ។ វាទំនាក់ទំនងពីរតួអង្គសំខាន់ៗ: OpenVINO Runtime សម្រាប់ inference, Neural Network Compression Framework (NNCF) សម្រាប់អុបទីម៉ាយម៉ូឌែល, និង OpenVINO Model Server សម្រាប់ការដាក់បញ្ចូលដែលអាចពង្រីកបាន។

### លក្ខណៈសំខាន់ៗ

- **ការដាក់ជាមួយគាំទ្រ Platform ខុសៗគ្នា**: គាំទ្រលើ Linux, Windows, និង macOS ជាមួយ API Python, C++, និង C
- **ការបង្កើនល្បឿនដោយឧបករណ៍ហេដ្វែរ**: ការស្វែងរកឧបករណ៍ស្វ័យប្រវត្តិ និងអុបទីម៉ាយសម្រាប់ CPU, GPU, VPU និងអាក់ទាលីយ៉ាត AI
- **ស៊ុមសម្រាប់កាត់បន្ថយទំហំម៉ូឌែល**: វិធីសាស្ត្រកាំបិទ, កាត់បន្ថយសម្ភារៈ និងបច្ចេកវិទ្យា​អុបទីម៉ាយជាសំខាន់ តាមរយៈ NNCF
- **ភាពសមរម្យជាមួយ Framework**: គាំទ្រម៉ូឌែលពី TensorFlow, ONNX, PaddlePaddle, និង PyTorch ដោយផ្ទាល់
- **គាំទ្រ Generative AI**: OpenVINO GenAI លក្ខណៈពិសេសសម្រាប់ដាក់បញ្ចូលម៉ូឌែលភាសាធំ និងកម្មវិធី Generative AI

### អត្ថប្រយោជន៍

- **អុបទីម៉ាយកម្រិតផ្នែកសមត្ថភាព**: ការកែលម្អល្បឿនយ៉ាងច្រើនដោយសេចក្ដីបាត់តិចនៃភាពត្រឹមត្រូវ
- **កាត់បន្ថយចំនួនអាស្រ័យភាពក្នុងការដាក់បញ្ចូល**: គម្រោងក្រៅតិចធ្វើឲ្យការដំឡើង និងការដាក់បញ្ចូលកាន់តែងាយស្រួល
- **កំណត់ពេលចាប់ផ្តើមបានលឿនជាងមុន**: ការផ្ដល់លក្ខណៈអុបទីម៉ាយក្នុងការបញ្ចូលម៉ូឌែល និង caching ដើម្បីចាប់ផ្តើមកម្មវិធីបានរហ័ស
- **អាចពង្រីកបាន**: ពីឧបករណ៍ edge ទៅហាប់ឃ្លោដដោយប្រើ API ស្រដៀងគ្នា
- **ស្រេចសម្រាប់ផលិតកម្ម**: មានភាពទៀងទាត់នៅកម្រិតសហគ្រាស ជាមួយឯកសារពេញលេញ និងសហគមន៍គាំទ្រ

## Installation

### តម្រូវការ​មុន

- Python 3.8 ឬខ្ពស់ជាងនេះ
- មេនេជឺរបណ្ណារា pip
- បរិយាកាស virtual environment (ផ្តល់អនុសាសន៍)
- ឧបករណ៍ដែលសមរម្យ (ផ្តល់អនុសាសន៍ Intel CPUs ប៉ុន្តែគាំទ្រអាគីតិចទៀត)

### ការដំឡើងមូលដ្ឋាន

Create and activate a virtual environment:

```bash
# បង្កើតបរិយាកាសស្មើ
python -m venv openvino-env

# បើកបរិយាកាសស្មើ
# លើ Windows:
openvino-env\Scripts\activate
# លើ macOS/Linux:
source openvino-env/bin/activate
```

Install OpenVINO Runtime:

```bash
pip install openvino
```

Install NNCF for model optimization:

```bash
pip install nncf
```

### OpenVINO GenAI Installation

សម្រាប់កម្មវិធី Generative AI:

```bash
pip install openvino-genai
```

### ការពឹងផ្អែកជាជម្រើស

កញ្ចប់បន្ថែមសម្រាប់ករណីប្រើប្រាស់ជាក់លាក់:

```bash
# សម្រាប់ Jupyter notebooks និងឧបករណ៍អភិវឌ្ឍន៍
pip install openvino[dev]

# សម្រាប់ការគាំទ្រម៉ូឌែល TensorFlow
pip install openvino[tensorflow]

# សម្រាប់ការគាំទ្រម៉ូឌែល PyTorch
pip install openvino[pytorch]

# សម្រាប់ការគាំទ្រម៉ូឌែល ONNX
pip install openvino[onnx]
```

### ផ្ទៀងផ្ទាត់ការដំឡើង

```bash
python -c "from openvino import Core; print('OpenVINO version:', Core().get_versions())"
```

ប្រសិនបើជោគជ័យ អ្នកគួរតែឃើញព័ត៌មានកំណែ OpenVINO។

## Quick Start Guide

### ការធ្វើអុបទីម៉ាយម៉ូឌែលដំបូងរបស់អ្នក

យើងនឹងបម្លែង និងធ្វើអុបទីម៉ាយម៉ូឌែលពី Hugging Face ដោយប្រើ OpenVINO:

```python
from optimum.intel import OVModelForCausalLM
from transformers import AutoTokenizer, pipeline

# ផ្ទុក និងបម្លែងម៉ូដែលទៅទ្រង់ទ្រាយ OpenVINO IR
model_id = "microsoft/DialoGPT-small"
ov_model = OVModelForCausalLM.from_pretrained(
    model_id, 
    export=True,
    compile=False
)

# ផ្ទុក tokenizer
tokenizer = AutoTokenizer.from_pretrained(model_id)

# រក្សាទុកម៉ូដែលដែលបានបម្លែង
save_directory = "models/dialogpt-openvino"
ov_model.save_pretrained(save_directory)
tokenizer.save_pretrained(save_directory)

# ផ្ទុក និង compile សម្រាប់ inference
ov_model = OVModelForCausalLM.from_pretrained(
    save_directory,
    device="CPU"  # ឬ "GPU", "AUTO"
)

# បង្កើត pipeline សម្រាប់ inference
pipe = pipeline("text-generation", model=ov_model, tokenizer=tokenizer)
result = pipe("Hello, how are you?", max_length=50)
print(result)
```

### អ្វីដែលដំណើរការនេះធ្វើ

វដ្តអុបទីម៉ាយរួមមាន: រៀបចំម៉ូឌែលដើមពី Hugging Face, បម្លែងទៅទ្រង់ទ្រាយ OpenVINO Intermediate Representation (IR), អនុវត្តអុបទីម៉ាយលំនាំដើម, និងកំណត់បំពេញសម្រាប់ឧបករណ៍គោលដៅ។

### ប៉ារ៉ាម៉ែត្រសំខាន់ៗពន្យល់

- `export=True`: បម្លែងម៉ូឌែលទៅទ្រង់ទ្រាយ OpenVINO IR
- `compile=False`: ឈប់ការបង្កប់រហូតដល់ពេលរត់ត្រូវបានទាមទារ សម្រាប់ភាពបត់បែន
- `device`: ឧបករណ៍គោលដៅ ("CPU", "GPU", "AUTO" សម្រាប់ការជ្រើសរើសដោយស្វ័យប្រវត្តិ)
- `save_pretrained()`: រក្សាទុកម៉ូឌែលដែលបានអុបទីម៉ាយសម្រាប់ការប្រើឡើងវិញ

## Example: Converting and Optimizing Models with OpenVINO

### ជំហាន 1: ការបម្លែងម៉ូឌែលជាមួយ NNCF Quantization

នេះគឺជារបៀបអនុវត្ត quantization បន្ទាប់ពីបង្រៀនដោយប្រើ NNCF:

```python
import nncf
from openvino import Core
from optimum.intel import OVModelForCausalLM
import torch
from transformers import AutoTokenizer

# ចាប់ផ្តើម NNCF សម្រាប់បម្លែងទៅជាតម្លៃចំនួន
model_id = "microsoft/DialoGPT-small"

# ផ្ទុកម៉ូឌែលក្នុងទ្រង់ទ្រាយ OpenVINO
ov_model = OVModelForCausalLM.from_pretrained(
    model_id, 
    export=True,
    compile=False
)

# បង្កើតសំណុំទិន្នន័យសម្រាប់ការកែតម្រូវក្នុងការបម្លែងទៅជាតម្លៃចំនួន
tokenizer = AutoTokenizer.from_pretrained(model_id)
calibration_data = [
    "Hello, how are you today?",
    "What is artificial intelligence?",
    "Tell me about machine learning.",
    "How does deep learning work?",
    "Explain neural networks."
]

def create_calibration_dataset():
    for text in calibration_data:
        tokens = tokenizer.encode(text, return_tensors="pt")
        yield {"input_ids": tokens}

# អនុវត្តការបម្លែងទៅជាតម្លៃចំនួនបន្ទាប់ពីការបណ្តុះបណ្តាល
core = Core()
model = core.read_model(ov_model.model_path)

# កំណត់រចនាសម្ព័ន្ធសម្រាប់ការបម្លែងទៅជាតម្លៃចំនួន
quantization_config = nncf.QuantizationConfig(
    input_info=nncf.InputInfo(
        sample_size=(1, 10),  # ទំហំក្រុម (batch_size), ប្រវែងលំដាប់ (sequence_length)
        type="long"
    )
)

# បង្កើតម៉ូឌែលដែលបានបម្លែងទៅជាតម្លៃចំនួន
quantized_model = nncf.quantize_with_tune_runner(
    model,
    create_calibration_dataset(),
    quantization_config
)

# រក្សាទុកម៉ូឌែលដែលបានបម្លែងទៅជាតម្លៃចំនួន
import openvino as ov
ov.save_model(quantized_model, "models/dialogpt-quantized.xml")
```

### ជំហាន 2: អុបទីម៉ាយកម្រិតខ្ពស់ជាមួយការបង្រួមទម្ងន់ (Weight Compression)

សម្រាប់ម៉ូឌែលប្រភេទ transformer អនុវត្តការ compress ទម្ងន់:

```python
import nncf
from openvino import Core

# ផ្ទុកម៉ូឌែល
core = Core()
model = core.read_model("models/dialogpt-openvino")

# អនុវត្តការបង្រួមទម្ងន់សម្រាប់ LLMs
compressed_model = nncf.compress_weights(
    model,
    mode=nncf.CompressWeightsMode.INT4_SYM,  # ឬ INT4_ASYM, INT8
    ratio=0.8,  # អត្រាបង្រួម
    group_size=128  # ទំហំក្រុមសម្រាប់ quantization
)

# រក្សាទុកម៉ូឌែលដែលបានបង្រួម
import openvino as ov
ov.save_model(compressed_model, "models/dialogpt-compressed.xml")
```

### ជំហាន 3: ការធ្វើ Inference ជាមួយម៉ូឌែលដែលបានអុបទីម៉ាយ

```python
from openvino import Core
import numpy as np

# ចាប់ផ្តើម OpenVINO Core
core = Core()

# ផ្ទុកម៉ូឌែលដែលបានបង្កើតឱ្យមានប្រសិទ្ធភាព
model = core.read_model("models/dialogpt-compressed.xml")

# បង្កើតទ្រង់ទ្រាយម៉ូឌែលសម្រាប់ឧបករណ៍គោលដៅ
compiled_model = core.compile_model(model, "CPU")

# យកព័ត៌មានអំពីបញ្ចូល និងបញ្ចេញ
input_layer = compiled_model.input(0)
output_layer = compiled_model.output(0)

# រៀបចំទិន្នន័យបញ្ចូល
input_text = "Hello, how are you?"
tokens = tokenizer.encode(input_text, return_tensors="np")

# រត់ការព្យាករណ៍
result = compiled_model([tokens])[output_layer]

# បកស្រាយលទ្ធផល
output_tokens = np.argmax(result, axis=-1)
generated_text = tokenizer.decode(output_tokens[0], skip_special_tokens=True)
print(f"Generated: {generated_text}")
```

models/dialogpt-compressed/

## Advanced Usage

### កំណត់រចនាសម្ព័ន្ធជាមួយ NNCF YAML

សម្រាប់វដ្តអុបទីម៉ាយស្មុគស្មាញ ប្រើឯកសារ​កំណត់រចនាសម្ព័ន្ធ NNCF:

```yaml
# nncf_config.yaml
input_info:
  sample_size: [1, 512]
  type: "long"

compression:
  algorithm: quantization
  initializer:
    precision:
      bitwidth_per_scope: [[8, 'default']]
    range:
      num_init_samples: 300
    batchnorm_adaptation:
      num_bn_adaptation_samples: 2000

target_device: CPU
```

អនុវត្តកំណត់រចនាសម្ព័ន្ធ:

```python
import nncf
from openvino import Core

# ផ្ទុកម៉ូឌែល និងការកំណត់
core = Core()
model = core.read_model("model.xml")
nncf_config = nncf.NNCFConfig.from_json("nncf_config.yaml")

# អនុវត្តការបង្រួម
compressed_model = nncf.create_compressed_model(model, nncf_config)
```

### អុបទីម៉ាយសម្រាប់ GPU

សម្រាប់សម្រួលកម្មវិធី GPU:

```python
from optimum.intel import OVModelForCausalLM

# ផ្ទុកម៉ូឌែលជាមួយឧបករណ៍ GPU
ov_model = OVModelForCausalLM.from_pretrained(
    "models/dialogpt-openvino",
    device="GPU",
    ov_config={"PERFORMANCE_HINT": "THROUGHPUT"}
)

# កំណត់រចនាសម្ព័ន្ធសម្រាប់អត្រាប្រតិបត្តិខ្ពស់
ov_model.ov_config.update({
    "NUM_STREAMS": "AUTO",
    "INFERENCE_NUM_THREADS": "AUTO"
})
```

### អុបទីម៉ាយសម្រាប់ដំណើរការកញ្ចប់ (Batch Processing)

```python
from openvino import Core

core = Core()
model = core.read_model("model.xml")

# កំណត់សម្រាប់ដំណើរការជាក្រុម
config = {
    "PERFORMANCE_HINT": "THROUGHPUT",
    "INFERENCE_NUM_THREADS": "AUTO",
    "NUM_STREAMS": "AUTO"
}

compiled_model = core.compile_model(model, "CPU", config)

# ដំណើរការបញ្ចូលច្រើន
batch_inputs = [tokens1, tokens2, tokens3]
results = compiled_model(batch_inputs)
```

### ដាក់បញ្ចូលជាមួយ Model Server

ដាក់បញ្ចូលម៉ូឌែលដែលបានអុបទីម៉ាយជាមួយ OpenVINO Model Server:

```bash
# ដំឡើង OpenVINO Model Server
pip install ovms

# ចាប់ផ្ដើមម៉ាស៊ីនបម្រើម៉ូឌែល
ovms --model_name dialogpt --model_path models/dialogpt-compressed --port 9000
```

កូដClient សម្រាប់ម៉ូឌែលស៊ែវើ:

```python
import requests
import json

# រៀបចំសំណើ
data = {
    "inputs": {
        "input_ids": [[1, 2, 3, 4, 5]]  # លេខសម្គាល់តូកិន
    }
}

# ផ្ញើសំណើទៅម៉ាស៊ីនបម្រើម៉ូឌែល
response = requests.post(
    "http://localhost:9000/v1/models/dialogpt:predict",
    json=data
)

result = response.json()
print(result["outputs"])
```

## Best Practices

### 1. ជ្រើសរើសម៉ូឌែល និងរៀបចំ
- ប្រើម៉ូឌែលពី framework ដែលគាំទ្រ (PyTorch, TensorFlow, ONNX)
- ត្រួតពិនិត្យឲ្យ input នៃម៉ូឌែលមានទ្រង់ទ្រាយថេរឬទំហំមានទីដឹង
- សាកល្បងជាមួយសំណុំទិន្នន័យតំណាងសម្រាប់ calibration

### 2. ជ្រើសយុទ្ធសាស្ត្រអុបទីម៉ាយ
- **Post-training Quantization**: ចាប់ផ្តើមពីនេះសម្រាប់អុបទីម៉ាយរហ័ស
- **Weight Compression**: អចលនាសម្រាប់ម៉ូឌែលភាសាធំ និង transformers
- **Quantization-aware Training**: ប្រើពេលដែលភាពត្រឹមត្រូវមានសារៈសំខាន់

### 3. អុបទីម៉ាយឧបករណ៍ជាក់លាក់
- **CPU**: ប្រើ INT8 quantization សម្រាប់ការសមមាញឹកល្អ
- **GPU**: ប្រើ FP16 precision និងការកែប្ដូរកញ្ចប់
- **VPU**: ផ្តោតលើការរំលាយម៉ូឌែល និងការរួមប្លង់ស្រទាប់

### 4. កែលម្អប្រសិទ្ធភាព
- **Throughput Mode**: សម្រាប់ដំណើរការជាច្រើនក្នុងកញ្ចប់
- **Latency Mode**: សម្រាប់កម្មវិធីពេលវេលាពិតប្រាកដឆ្លើយតប
- **AUTO Device**: ដាក់ឱ្យ OpenVINO ជ្រើសឧបករណ៍ល្អបំផុត

### 5. ការគ្រប់គ្រងអង្គចងចាំ
- ប្រើទ្រង់ទ្រាយឌីណាមិចយ៉ាងម៉ត់ចត់ ដើម្បីជៀសវាងការកើនអង្គចងចាំ
- អនុវត្ត caching ម៉ូឌែលសម្រាប់ការផ្ទុកលឿនបន្ទាប់មក
- តាមដានការប្រើប្រាស់អង្គចងចាំក្នុងពេលប្រតិបត្តិការ

### 6. ផ្ទៀងផ្ទាត់ភាពត្រឹមត្រូវ
- តែងតែផ្ទៀងផ្ទាត់ម៉ូឌែលដែលបានអុបទីម៉ាយជាមួយលទ្ធផលដើម
- ប្រើសំណុំទិន្នន័យតំណាងសម្រាប់ការវាយតម្លៃ
- ស្រាវជ្រាវការអុបទីម៉ាយ تدريجي (ចាប់ពីការកំណត់ការកាន់តែសូម្បី)

## Troubleshooting

### បញ្ហាទូទៅ

#### 1. បញ្ហាក្នុងការដំឡើង
```bash
# សម្អាត cache របស់ pip និងតម្លើងឡើងវិញ
pip cache purge
pip uninstall openvino nncf
pip install openvino nncf --no-cache-dir
```

#### 2. បញ្ហាក្នុងការបម្លែងម៉ូឌែល
```python
# ពិនិត្យភាពសមរម្យរបស់ម៉ូដែល
from openvino.tools.mo import convert_model

try:
    ov_model = convert_model("model.onnx")
    print("Conversion successful")
except Exception as e:
    print(f"Conversion failed: {e}")
```

#### 3. បញ្ហាប្រសិទ្ធភាព
```python
# បើកសូចនាការ​សមត្ថភាព
config = {
    "PERFORMANCE_HINT": "LATENCY",  # ឬ "THROUGHPUT"
    "INFERENCE_PRECISION_HINT": "f32"  # ឬ "f16"
}
compiled_model = core.compile_model(model, "CPU", config)
```

#### 4. បញ្ហាអង្គចងចាំ
- កាត់បន្ថយទំហំកញ្ចប់ម៉ូឌែលក្នុងពេលអុបទីម៉ាយ
- ប្រើការបញ្ចូនទិន្នន័យស្វាគមន៍សម្រាប់សំណុំទិន្នន័យធំ
- បើក model caching: `core.set_property("CPU", {"CACHE_DIR": "./cache"})`

#### 5. ការធ្លាក់ចុះនៃភាពត្រឹមត្រូវ
- ប្រើភាពប្រាកដខ្ពស់ជាង (INT8 ជំនួស INT4)
- បង្កើនទំហំសំណុំទិន្នន័យសម្រាប់ calibration
- អនុវត្តអុបទីម៉ាយ Mixed Precision

### ការត្រួតពិនិត្យប្រសិទ្ធភាព

```python
# តាមដានប្រសិទ្ធភាពនៃការសន្មត
import time

start_time = time.time()
result = compiled_model([input_data])
inference_time = time.time() - start_time

print(f"Inference time: {inference_time:.4f} seconds")
```
