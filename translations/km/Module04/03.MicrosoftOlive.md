## ការណែនាំ

Microsoft Olive គឺជាឧបករណ៍កែ​សម្រួល​ម៉ូដែល​ដែល​មាន​សមត្ថភាពខ្លាំង និង​យល់ពី​ឧបករណ៍ (hardware-aware) ដែលងាយស្រួលប្រើ ដើម្បីសម្រួលដំណើរការកែសម្រួលម៉ូដែលម៉ាស៊ីនរៀនសម្រាប់ដាក់ប្រើលើវេទិកាឧបករណ៍ផ្សេងៗ។ មិនថាអ្នកមានគោលដៅលើ CPU, GPU ឬឧបករណ៍ជំនួយ AI ពិសេស ក៏ដោយ Olive ជួយឱ្យទទួលបានប្រសិទ្ធភាពល្អបំផុត ខណៈដែលថែទាំតុល្យភាពនៃភាពត្រឹមត្រូវនៃម៉ូដែល។

## Microsoft Olive ជាអ្វី?

Olive គឺជាឧបករណ៍​កែសម្រួលម៉ូដែល​ដែលយល់ពីឧបករណ៍ និងងាយស្រួលប្រើ ដែលផ្សំបច្ចេកទេសដឹកនាំឧស្សាហកម្មនៅផ្នែកសង្កត់ម៉ូដែល (compression), ការ​បង្កើនប្រសិទ្ធភាព (optimization), និងកា​របកប្រែ (compilation)។ វាធ្វើការជាមួយ ONNX Runtime ដើម្បីសម្រួលដំណោះស្រាយ E2E សម្រាប់ការប៉ាន់ស្មាន។

### លក្ខណៈពិសេសសំខាន់ៗ

- **Hardware-Aware Optimization**: ជ្រើសយកបច្ចេកទេសកែសម្រួលល្អបំផុតសម្រាប់ឧបករណ៍គោលដៅរបស់អ្នកដោយស្វ័យប្រវត្តិ
- **40+ Built-in Optimization Components**: គ្របដណ្តប់ចំពោះការសង្កត់ម៉ូដែល, ការquantization, ការ​បង្កើន​ប្រសិទ្ធភាពក្រាប និងផ្សេងៗទៀត
- **Easy CLI Interface**: ពាក្យបញ្ជាងាយស្រួលសម្រាប់ភារកិច្ចកែសម្រួលទូទៅ
- **Multi-Framework Support**: ធ្វើការជាមួយ PyTorch, ម៉ូដែល Hugging Face និង ONNX
- **Popular Model Support**: Olive អាចបង្កើនប្រសិទ្ធភាពស្វ័យប្រវត្តិតទៅលើស្ថាបត្យកម្មម៉ូដែលពេញនិយមដូចជា Llama, Phi, Qwen, Gemma, និងផ្សេងទៀតពីប្រអប់ចេញ

### អត្ថប្រយោជន៍

- **កាត់បន្ថយ​ពេលអភិវឌ្ឍន៍**: មិនចាំបាច់សាកល្បងដោយដៃជាមួយបច្ចេកទេសកែសម្រួលជាច្រើន
- **ការកើនប្រសិទ្ធភាព**: ការវិវឌ្ឍន៍ល្បឿនយល់ស្រួល (អាចដល់ 6x ក្នុងករណីខ្លះ)
- **ដាក់ប្រើឆ្លងវេទិកា**: ម៉ូដែលដែលបានបង្កើនប្រសិទ្ធភាពធ្វើការ​បានលើឧបករណ៍ និងប្រព័ន្ធប្រតិបត្តិការផ្សេងៗ
- **ថែទាំភាពត្រឹមត្រូវ**: ការកែសម្រួលរក្សាតម្លៃគុណភាពម៉ូដែល ខណៈពេលបង្កើនប្រសិទ្ធភាព

- អ្នកគ្រប់គ្រងកញ្ចប់ pip

### ការតំឡើងមូលដ្ឋាន

Create and activate a virtual environment:

```bash
# បង្កើតបរិយាកាសស្មើ
python -m venv olive-env

# ចាប់ផ្ដើមប្រើបរិយាកាសស្មើ
# លើ Windows:
olive-env\Scripts\activate
# លើ macOS/Linux:
source olive-env/bin/activate
```

Install Olive with auto-optimization features:

```bash
pip install olive-ai[auto-opt]
pip install transformers onnxruntime-genai
```

## មគ្គុទេសក៍ចាប់ផ្តើមរហ័ស

### ការOptimize ជាលើកដំបូងរបស់អ្នក

Let's optimize a small language model using Olive's auto-optimization feature:

```bash
olive auto-opt \
  --model_name_or_path HuggingFaceTB/SmolLM2-135M-Instruct \
  --output_path models/smolm2-optimized \
  --device cpu \
  --provider CPUExecutionProvider \
  --use_ort_genai \
  --precision int4 \
  --log_level 1
```

### អ្វីដែលពាក្យបញ្ជានេះធ្វើ

ដំណើរការកែសម្រួលរួមមាន៖ ទទួលយកម៉ូដែលពី cache ក្នុងរឹងកុំព្យូទ័រ, ចាប់យក ONNX Graph និងរក្សាទុកទម្ងន់ក្នុងឯកសារ ONNX data, កែសម្រួល ONNX Graph, និងបកសម្ពាធម៉ូដែលទៅ int4 ដោយប្រើវិធីសាស្ត្រ RTN។

### ការ​ពណ៌នា​អំពីប៉ារ៉ាម៉ែត្រ​នៃពាក្យ​បញ្ជា

- `--model_name_or_path`: អំពីសម្គាល់ម៉ូដែល Hugging Face ឬផ្លូវក្នុងមូលដ្ឋានទិន្នន័យក្នុងលក្ខណៈមូលដ្ឋាន
- `--output_path`: ថតដែលម៉ូដែលដែលបានបង្កើនប្រសិទ្ធភាពនឹងត្រូវរក្សា
- `--device`: ឧបករណ៍គោលដៅ (cpu, gpu)
- `--provider`: សផ្ដាកម្មប្រតិបត្តិការ (CPUExecutionProvider, CUDAExecutionProvider, DmlExecutionProvider)
- `--use_ort_genai`: ប្រើ ONNX Runtime Generate AI សម្រាប់ការប៉ាន់ស្មាន
- `--precision`: ភាពទ្រង់ទ្រាយ quantization (int4, int8, fp16)
- `--log_level`: កម្រិតការចុះសារ (0=តិចបំផុត, 1=លម្អិត)

## ឧទាហរណ៍: កែប្រែ Qwen3 ទៅ ONNX INT4

ផ្អែកលើឧទាហរណ៍ Hugging Face ដែលផ្ដល់នៅ [lokinfey/Qwen3-8B-ONNX-INT4-CPU](https://huggingface.co/lokinfey/Qwen3-8B-ONNX-INT4-CPU), នេះជាវិធីកែសម្រួលម៉ូដែល Qwen3៖

### ជំហ៊ាន 1: ទាញយកម៉ូដែល (ជាជម្រើស)

ដើម្បីកាត់បន្ថយពេលទាញយក, cache ចំពោះឯកសារសាកចាំតែកម្រិតចាំបាច់៖

```bash

```

### ជំហ៊ាន 2: កែសម្រួលម៉ូដែល Qwen3

```bash
olive auto-opt \
  --model_name_or_path Qwen/Qwen2.5-0.5B-Instruct \
  --output_path models/qwen3-onnx-int4 \
  --device cpu \
  --provider CPUExecutionProvider \
  --use_ort_genai \
  --precision int4 \
  --log_level 1
```

### ជំហ៊ាន 3: ពិនិត្យម៉ូដែលដែលបានបង្កើនប្រសិទ្ធភាព

Create a simple Python script to test your optimized model:

```python
import onnxruntime_genai as og

# ផ្ទុកម៉ូឌែលដែលបានបង្កើនប្រសិទ្ធភាព
model = og.Model('models/qwen3-onnx-int4')
tokenizer = og.Tokenizer(model)

# បង្កើតគំរូសន្ទនា
chat_template = '<|im_start|>user\n{input}<|im_end|>\n<|im_start|>assistant\n'

# បង្កើតអត្ថបទ
prompt = "What is machine learning?"
input_tokens = tokenizer.encode(chat_template.format(input=prompt))

params = og.GeneratorParams(model)
params.set_search_options(max_length=200)
params.input_ids = input_tokens

generator = og.Generator(model, params)

print("Generated response:")
while not generator.is_done():
    generator.compute_logits()
    generator.generate_next_token()
    
    new_token = generator.get_next_tokens()[0]
    print(tokenizer.decode([new_token]), end='', flush=True)

print()
```

## អនុវត្តិល្អបំផុត

### 1. ការ​ជ្រើសរើស​ម៉ូដែល
- ចាប់ផ្តើមជាមួយម៉ូដែលតូចសម្រាប់សាកល្បង (ឧ. 0.5B-7B parameters)
- ធានាថាស្ថាបត្យកម្មម៉ូដែលគោលដៅរបស់អ្នកត្រូវបានគាំទ្រីដោយ Olive

### 2. ការពិចារណាអំពីឧបករណ៍
- តម្រូវឲ្យការគោលដៅកែសម្រួលសមនឹងឧបករណ៍ដែលអ្នកនឹងដាក់ប្រើ
- ប្រើកែសម្រួល GPU ប្រសិនបើមានឧបករណ៍ដែលគាំទ្រ CUDA
- ពិចារណា DirectML សម្រាប់ម៉ាស៊ីន Windows ដែលមានក្រាហ្វិករួម

### 3. ការ​ជ្រើសរើស​ភាព​ច្បាស់ (Precision)
- **INT4**: សម្ពាធយ៉ាងខ្លាំង ប៉ះពាល់តិចលើភាពត្រឹមត្រូវ
- **INT8**: សមតុល្យល្អរវាងទំហំ និងភាពត្រឹមត្រូវ
- **FP16**: ការបាត់បង់ភាពត្រឹមត្រូវតិច តែកាត់បន្ថយទំហំមធ្យម

### 4. ការ​ធ្វើតេស្ត និង ធ្វើអះអាង
- តែងតែធ្វើតេស្តម៉ូដែលដែលបានកែសម្រួលដោយប្រើករណីការប្រើប្រាស់ជាក់លាក់របស់អ្នក
- ប្រៀបធៀបទិន្នន័យសមត្ថភាព (latency, throughput, accuracy)
- ប្រើទិន្នន័យបញ្ចូលដែលតំណាងឱ្យលក្ខណៈពិសេសសម្រាប់ការប៉ាន់ប្រមាណ

### 5. ការ​កែសម្រួលជាដំណាក់កាល
- ចាប់ផ្តើមដោយការកែសម្រួលស្វ័យប្រវត្តិសម្រាប់លទ្ធផលលឿន
- ប្រើឯកសារកំណត់រចនាសម្ព័ន្ធសម្រាប់ការគ្រប់គ្រងចំណុចតូចៗ
- សាកល្បងជាមួយច្រកកែសម្រួលផ្សេងៗ

រេសីពីជាច្រើនរួមបញ្ចូលសន្ទស្សន៍សមត្ថភាពបង្ហាញ:
- **Latency Improvements**: ល្បឿនឡើងរៀងរាល់ប្រព័ន្ធ 2-6x កាលពីមុន
- **Memory Reduction**: កាត់បន្ថយការប្រើប្រាស់អង្គចងចាំ 50-75% ជាមួយការបង្កើនប្រសិទ្ធភាព
- **Accuracy Retention**: រក្សាបានភាពត្រឹមត្រូវ 95-99%
