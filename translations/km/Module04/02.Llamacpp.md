## Introduction

មេរៀននេះគ្រប់គ្រងនឹងណែនាំអ្នកពីអ្វីដែលអ្នកត្រូវដឹងអំពី Llama.cpp ទាំងពីការ​ដំឡើងមូលដ្ឋាន ដល់សេចក្តីប្រើប្រាស់ជាន់ខ្ពស់។ Llama.cpp គឺជាការអនុវត្ត C++ ខ្លាំងមួយដែលអនុញ្ញាតឱ្យធ្វើ inference របស់ ម៉ូឌែលភាសាធំ (LLMs) បានយ៉ាងមានប្រសិទ្ធភាព ជាមួយការកំណត់តិច និងប្រសិទ្ធភាពល្អលើឧបករណ៍ផ្សេងៗ។

## What is Llama.cpp?

Llama.cpp ជាស៊ុមផ្នែក inference LLM សរសេរជា C/C++ ដែលអនុញ្ញាតឱ្យដំណើរការម៉ូឌែលភាសាធំក្នុងកន្លែងកុំព្យូទ័រផ្ទាល់បានដោយការកំណត់តិច និងមានប្រសិទ្ធភាពខ្ពស់លើវេទិកាច្រើន។ លក្ខណៈសំខាន់រួមមាន៖

### លក្ខណៈស្នូល
- **ការអនុវត្តជា C/C++ ផ្ទាល់** គ្មានការពឹងផ្អែក
- **ការគាំទ្រឆ្លងវេទិកា** (Windows, macOS, Linux)
- **ការអុបទីម៉ាយសម្រាប់ឧបករណ៍** សម្រាប់ស្ថាបត្យកម្មផ្សេងៗ
- **គាំទ្រការកំណត់បរិមាណប៊ីត (quantization)** (1.5-bit ទៅ 8-bit integer)
- **គាំទ្រការល្បឿនលឿន CPU និង GPU**
- **ប្រើប្រាស់ម៉េមូរីយ៉ាងមានប្រសិទ្ធភាព** សម្រាប់បរិបទដែលមានកម្រិតកំណត់

### អត្ថប្រយោជន៍
- ដំណើរការយ៉ាងមានប្រសិទ្ធភាពលើ CPU ដោយមិនទាមទារ​ឧបករណ៍ពិសេស
- គាំទ្រច្រើន GPU backends (CUDA, Metal, OpenCL, Vulkan)
- ទ្រេត និងអាចដំណើរការ portability
- Apple silicon បានទទួលជាទីគោរព - បានអុបទីម៉ាយស៍តាម ARM NEON, Accelerate និង Metal frameworks
- គាំទ្រកម្រិត quantization ផ្សេងៗ ដើម្បីកាត់បន្ថយការប្រើប្រាស់ម៉េមូរី

## Installation

### Method 1: Pre-built Binaries (Recommended for Beginners)

#### Download from GitHub Releases
1. ចូលទៅកាន់ [Llama.cpp GitHub Releases](https://github.com/ggml-org/llama.cpp/releases)
2. ទាញយកប៊ីណារីសមរម្យសម្រាប់ប្រព័ន្ធរបស់អ្នក៖
   - `llama-<version>-bin-win-<feature>-<arch>.zip` សម្រាប់ Windows
   - `llama-<version>-bin-macos-<feature>-<arch>.zip` សម្រាប់ macOS
   - `llama-<version>-bin-linux-<feature>-<arch>.zip` សម្រាប់ Linux

3. ដកសាររួចបញ្ចូលថតទៅក្នុង PATH នៃប្រព័ន្ធរបស់អ្នក

#### Using Package Managers

**macOS (Homebrew):**
```bash
brew install llama.cpp
```

**Linux (Various distributions):**
```bash
# អ៊ុបុនទូ/ដេប៊ីយ៉ាន់
sudo apt install llama.cpp

# អាចលីនុច
sudo pacman -S llama.cpp
```

### Method 2: Python Package (llama-cpp-python)

#### Basic Installation
```bash
pip install llama-cpp-python
```

#### With Hardware Acceleration
```bash
# សម្រាប់ CUDA (GPU របស់ NVIDIA)
CMAKE_ARGS="-DGGML_CUDA=on" pip install llama-cpp-python

# សម្រាប់ Metal (ស៊ីលីកុនរបស់ Apple)
CMAKE_ARGS="-DGGML_METAL=on" pip install llama-cpp-python

# សម្រាប់ OpenBLAS (ការបង្កើនប្រសិទ្ធភាព CPU)
CMAKE_ARGS="-DGGML_BLAS=ON -DGGML_BLAS_VENDOR=OpenBLAS" pip install llama-cpp-python
```

## Building from Source

### Prerequisites

**System Requirements:**
- C++ compiler (GCC, Clang, or MSVC)
- CMake (version 3.14 or higher)
- Git
- Build tools សម្រាប់វេទិការបស់អ្នក

**Installing Prerequisites:**

**macOS:**
```bash
xcode-select --install
```

**Ubuntu/Debian:**
```bash
sudo apt update
sudo apt install build-essential cmake git
```

**Windows:**
- ដំឡើង Visual Studio 2022 ជាមួយឧបករណ៍អភិវឌ្ឍន៍ C++
- ដំឡើង CMake ពីគេហទំព័រផ្លូវការ
- ដំឡើង Git

### Basic Build Process

1. **Clone the repository:**
```bash
git clone https://github.com/ggml-org/llama.cpp
cd llama.cpp
```

2. **Configure the build:**
```bash
cmake -B build
```

3. **Build the project:**
```bash
cmake --build build --config Release
```

For faster compilation, use parallel jobs:
```bash
cmake --build build --config Release -j 8
```

### Hardware-Specific Builds

#### CUDA Support (NVIDIA GPUs)
```bash
cmake -B build -DGGML_CUDA=ON
cmake --build build --config Release
```

#### Metal Support (Apple Silicon)
```bash
cmake -B build -DGGML_METAL=ON
cmake --build build --config Release
```

#### OpenBLAS Support (CPU Optimization)
```bash
cmake -B build -DGGML_BLAS=ON -DGGML_BLAS_VENDOR=OpenBLAS
cmake --build build --config Release
```

#### Vulkan Support
```bash
cmake -B build -DGGML_VULKAN=1
cmake --build build --config Release
```

### Advanced Build Options

#### Debug Build
```bash
cmake -B build -DCMAKE_BUILD_TYPE=Debug
cmake --build build
```

#### With Additional Features
```bash
cmake -B build \
    -DGGML_CUDA=ON \
    -DGGML_BLAS=ON \
    -DGGML_BLAS_VENDOR=OpenBLAS \
    -DBUILD_SHARED_LIBS=ON
```

## Model Quantization

### Understanding GGUF Format

GGUF (Generalized GGML Unified Format) ជាទ្រង់ទ្រាយឯកសារសម្រួលដែលបានបង្កើតឡើងសម្រាប់ដំណើរការម៉ូឌែលភាសាធំបានយ៉ាងមានប្រសិទ្ធភាព ប្រើជាមួយ Llama.cpp និងស៊ុមផ្សេងៗ។ វាផ្តល់៖

- ការផ្ទុកទំងន់ម៉ូឌែលដែលបានវិចារណាមានស្តង់ដារ
- គាំទ្រលើវេទិកាច្រើនបានកាន់តែប្រសើរ
- ប្រសិទ្ធភាពលើកំណត់កាន់តែឡើង
- ការគ្រប់គ្រង metadata ដែលមានប្រសិទ្ធភាព

### Quantization Types

Llama.cpp គាំទ្រកម្រិត quantization ផ្សេងៗ៖

| ប្រភេទ | ប៊ីត | ការពិពណ៌នា | ករណីប្រើប្រាស់ |
|------|------|-------------|----------|
| F16 | 16 | ភាពពាក់កណ្តាល (half precision) | គុណភាពខ្ពស់, តម្រូវម៉េមូរីច្រើន |
| Q8_0 | 8 | ការកំណត់ 8 ប៊ីត | តុល្យភាពល្អ |
| Q4_0 | 4 | ការកំណត់ 4 ប៊ីត | គុណភាពមធ្យម, ទំហំតូច |
| Q2_K | 2 | ការកំណត់ 2 ប៊ីត | ទំហំតូចបំផុត, គុណភាពទាប |

### Converting Models

#### From PyTorch to GGUF
```bash
# បម្លែងម៉ូដែល Hugging Face
python convert_hf_to_gguf.py path/to/model --outdir ./models

# បម្លែងម៉ូដែលទៅជាកម្រិតចំនួន
./llama-quantize ./models/model.gguf ./models/model-q4_0.gguf q4_0
```

#### Direct Download from Hugging Face
ម៉ូឌែលជាច្រើនមាននៅក្នុងទ្រង់ទ្រាយ GGUF លើ Hugging Face៖
- ស្វែងរកម៉ូឌែលដែលមាន "GGUF" ក្នុងឈ្មោះ
- ទាញយកកម្រិត quantization ដែលសមរម្យ
- ប្រើដោយផ្ទាល់ជាមួយ llama.cpp

## Basic Usage

### Command Line Interface

#### Simple Text Generation
```bash
# ការបំពេញអត្ថបទមូលដ្ឋាន
./llama-cli -m model.gguf -p "Hello, my name is" -n 50

# ម៉ូដជជែកអន្តរកម្ម
./llama-cli -m model.gguf -cnv
```

#### Using Models from Hugging Face
```bash
# ទាញយក ហើយរត់ដោយផ្ទាល់
./llama-cli -hf microsoft/DialoGPT-medium
```

#### Server Mode
```bash
# ចាប់ផ្តើមម៉ាស៊ីនមេ
./llama-server -m model.gguf --host 0.0.0.0 --port 8080

# ជាមួយការបង្កើនល្បឿនដោយ GPU
./llama-server -m model.gguf --n-gpu-layers 32
```

### Common Parameters

| ព៉ារាម៉ែត្រ | ការពិពណ៌នា | ឧទាហរណ៍ |
|-----------|-------------|---------|
| `-m` | ផ្លូវឯកសារម៉ូឌែល | `-m model.gguf` |
| `-p` | អត្ថបទ prompt | `-p "Hello world"` |
| `-n` | ចំនួន tokens ត្រូវផលិត | `-n 100` |
| `-c` | ទំហំ context | `-c 4096` |
| `-t` | ចំនួន threads | `-t 8` |
| `-ngl` | ស្រទាប់ GPU | `-ngl 32` |
| `-temp` | សីតុណ្ហភាព | `-temp 0.7` |

# > អ្នកអាចជួយខ្ញុំបានអ្វីខ្លះ?
# ខ្ញុំអាចជួយក្នុងភារកិច្ចនានាដូចជា...

## Python Integration

### Basic Usage with llama-cpp-python

```python
from llama_cpp import Llama

# រៀបចំម៉ូឌែល
llm = Llama(
    model_path="./models/model.gguf",
    n_ctx=2048,
    n_threads=8,
    n_gpu_layers=32
)

# បង្កើតអត្ថបទ
output = llm("Hello, my name is", max_tokens=50)
print(output['choices'][0]['text'])
```

### Chat Interface

```python
from llama_cpp import Llama

llm = Llama(model_path="./models/chat-model.gguf")

# ការបំពេញការជជែក
response = llm.create_chat_completion(
    messages=[
        {"role": "user", "content": "Hello!"}
    ],
    temperature=0.7,
    max_tokens=100
)

print(response['choices'][0]['message']['content'])
```

### Streaming Responses

```python
# ការបង្កើតអត្ថបទជាស្ទ្រីម
stream = llm("Tell me a story", max_tokens=200, stream=True)

for output in stream:
    print(output['choices'][0]['text'], end='', flush=True)
```

### Integration with LangChain

```python
from langchain.llms import LlamaCpp
from langchain.prompts import PromptTemplate
from langchain.chains import LLMChain

# ចាប់ផ្តើម LLM
llm = LlamaCpp(
    model_path="./models/model.gguf",
    n_ctx=2048,
    n_threads=8
)

# បង្កើតទម្រង់ពាក្យបញ្ជា
template = "Question: {question}\nAnswer:"
prompt = PromptTemplate(template=template, input_variables=["question"])

# បង្កើតខ្សែ
chain = LLMChain(llm=llm, prompt=prompt)

# ប្រើខ្សែ
result = chain.run(question="What is artificial intelligence?")
print(result)
```

- ពិនិត្យសិទ្ធិឯកសារ
- ធានាថាមាន RAM គ្រប់គ្រាន់
- សាកល្បងកម្រិត quantization ផ្សេងៗ

**បញ្ហា៖ ប្រសិទ្ធភាពអន់**
- បើកការលឿនស្វ័យប្រវត្តិក្រាហ្វិក (hardware acceleration)
- បន្ថែមចំនួន thread
- ប្រើកម្រិត quantization ដែលសមរម្យ
- ពិនិត្យការប្រើប្រាស់ម៉េមូរី GPU

- ផ្ទៀងផ្ទាត់ការដំឡើង CUDA toolkit

## Best Practices

### Model Selection
1. **ជ្រើសកម្រិត quantization ដែលសមរម្យ** មូលដ្ឋានលើ hardware របស់អ្នក
2. **គិតពីទំហំម៉ូឌែល និងការជួញដូរ គុណភាព** 
3. **សាកល្បងម៉ូឌែលផ្សេងៗ** សម្រាប់ករណីប្រើប្រាស់ជាក់លាក់របស់អ្នក

### Performance Optimization
1. **ប្រើ GPU acceleration** ប្រសិនបើមាន
2. **បង្កើន/បន្ថយចំនួន thread** ដើម្បីឲ្យសមរម្យសម្រាប់ CPU របស់អ្នក
3. **កំណត់ទំហំ context ឱ្យសមរម្យ** សម្រាប់ករណីប្រើប្រាស់របស់អ្នក
4. **បើក memory mapping** សម្រាប់ម៉ូឌែលធំៗ

### Production Deployment
1. **ប្រើ server mode** សម្រាប់ចូលប្រើ API
2. **អនុវត្តការគ្រប់គ្រងកំហុសដែលល្អ**
3. **តាមដានកំណត់ប្រើធនធាន**
4. **តម្លើង logging និង monitoring**

### Development Workflow
1. **ចាប់ផ្តើមជាមួយម៉ូឌែលតូច** សម្រាប់ការធ្វើតេស្ត
2. **ប្រើ version control** សម្រាប់កំណត់រចនាសម្ព័ន្ធម៉ូឌែល
3. **សរសេរឯកសារកំណត់រចនាសម្ព័ន្ធរបស់អ្នក**
4. **ធ្វើតេស្តលើវេទិកាប្រភេទផ្សេងៗ**

### Security Considerations
1. **ផ្ទៀងផ្ទាត់ prompt ទាញចូល**
2. **អនុវត្តការកំណត់រង្វាស់អត្រា (rate limiting)**
3. **សន្តិសុខចំណុចចូល API**
4. **តាមដានគំរូប្រើប្រាស់ជាដំណេក**

## Conclusion

Llama.cpp ផ្តល់នូវវិធីដែលមានឥទ្ធិពល និងប្រសិទ្ធភាពក្នុងការប្រើម៉ូឌែលភាសាធំនៅលើឧបករណ៍ផ្ទាល់ក្នុងបរិបទឧបករណ៍ផ្សេងៗ។ មិនថាអ្នកកំពុងអភិវឌ្ឍកម្មវិធី AI, ធ្វើការស្រាវជ្រាវ ឬសាកល្បងជាមួយ LLMs, ស៊ុមនេះផ្តល់នូវចលនានិងប្រសិទ្ធភាពដែលត្រូវការសម្រាប់ករណីប្រើប្រាស់ជាមួួយ។

ចំណុចសំខាន់ៗដែលត្រូវយកចិត្តទុកដាក់៖
- ជ្រើសវិធីដំឡើងដែលសមរម្យសម្រាប់តម្រូវការរបស់អ្នក
- អុបទីម៉ាយសម្រាប់កំណត់ហេដ្ឋារ (hardware) របស់អ្នក
- ចាប់ផ្តើមពីការប្រើប្រាស់មូលដ្ឋាន ហើយបន្តទៅលក្ខណៈជាន់ខ្ពស់逐步
- ពិចារណាប្រើ Python bindings សម្រាប់ការរួមបញ្ចូលបានងាយស្រួល
- បន្សល់តាមអនុវត្តិល្អសម្រាប់ការដាក់ចេញផលិតកម្ម

សម្រាប់ព័ត៌មានបន្ថែម និងការអាប់ដេត សូមចូលទៅកាន់ [ឃ្លាំងផ្ទះផ្លូវការរបស់ Llama.cpp](https://github.com/ggml-org/llama.cpp) និងយោងទៅកាន់ឯកសារយោងពេញលេញ និងធនធានសហគមន៍ដែលមាន។
