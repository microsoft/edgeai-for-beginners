## Introduction

Qualcomm QNN (Qualcomm Neural Network) គឺជា សម្ព័ន្ធការងារសម្រាប់ inference AI រួមមានគ្រប់ទិស ដែលរចនាឡើងដើម្បីបញ្ចេញសមត្ថភាពពេញលេញនៃឧបករណ៍អ៉ាហ្វីញាង AI របស់ Qualcomm រួមទាំង Hexagon NPU, Adreno GPU, និង Kryo CPU។ មិនថាអ្នកកំពុងគោលដៅលើឧបករណ៍ចល័ត ស្ថាប័ន Edge ឬប្រព័ន្ធយានយន្តទេ QNN ផ្តល់នូវសមត្ថភាព inference ដែលបានអុបទិម៉ាយសម្រាប់ប្រើប្រាស់ឧបករណ៍ពិសេសរបស់ Qualcomm ដើម្បីទទួលបានសមត្ថភាព និងប្រសិទ្ធភាពថាមពលអតិបរមា។

## What is Qualcomm QNN?

Qualcomm QNN គឺជាសម្ព័ន្ធការងារជាមួយនឹង interface រួមមួយសម្រាប់ inference AI ដែលអនុញ្ញាតឱ្យអ្នកអភិវឌ្ឍន៍ដាក់ហូឡូម៉ូដែល AI បានយ៉ាងមានប្រសិទ្ធភាពលើស្ថាបត្យកម្មគណនា heterogeneous របស់ Qualcomm។ វាប្រើប្រាស់ interface កម្មProgramming រោមតែមួយសម្រាប់ចូលប្រើ Hexagon NPU (Neural Processing Unit), Adreno GPU, និង Kryo CPU ហើយជ្រើសរើសឧបករណ៍ប.processing ត្រូវគ្នាស្វ័យប្រវត្តិសម្រាប់ស្រទាប់ម៉ូដែល និងប្រតិបត្តការផ្សេងៗ ។

### Key Features

- **Heterogeneous Computing**: ការចូលប្រើរួមទៅកាន់ NPU, GPU, និង CPU ជាមួយការចែកចាយបន្ទុកស្វ័យប្រវត្តិ
- **Hardware-Aware Optimization**: ការអុបទិម៉ាយពិសេសសម្រាប់វេទិកា Qualcomm Snapdragon
- **Quantization Support**: បច្ចេកវិទ្យា Quantization កម្រិតខ្ពស់ INT8, INT16 និង mixed-precision
- **Model Conversion Tools**: គាំទ្រដោយផ្ទាល់សម្រាប់ម៉ូដែល TensorFlow, PyTorch, ONNX, និង Caffe
- **Edge AI Optimized**: រចនាឡើងពិសេសសម្រាប់ការដាក់ចេញលើឧបករណ៍ចល័ត និង Edge ដែលផ្តោតលើប្រសិទ្ធភាពថាមពល

### Benefits

- **Low Latency**: Inference ដែលរួមបញ្ចូល hardware acceleration មានកម្រាស់ពេលពេកតិចសម្រាប់កម្មវិធីពេលពិត
- **Scalable Deployment**: ចាប់ពីស្មាតហ្វូនដល់វេទិកាយានយន្តក្នុងប្រព័ន្ធ Qualcomm
- **Production Ready**: សម្ព័ន្ធដែលបានសាកល្បងក្នុងឧបករណ៍លើកលែងរាប់លានម៉ឺនឧបករណ៍

## Installation

### Prerequisites

- Qualcomm QNN SDK (ទាមទារការចុះឈ្មោះជាមួយ Qualcomm)
- Python 3.7 ឬខ្ពស់ជាង
- ឧបករណ៍ Qualcomm ឬកម្មវិធីសមមាត្រដែលឆមាស
- Android NDK (សម្រាប់ការដាក់ចេញលើឧបករណ៍ចល័ត)
- សេវាកម្មអភិវឌ្ឍលើ Linux ឬ Windows

### QNN SDK Setup

1. **Register and Download**: ចូលទស្សនាផ្លូវការនៅ Qualcomm Developer Network ដើម្បីចុះឈ្មោះ និងទាញយក QNN SDK
2. **Extract SDK**: ដកឯកសារ QNN SDK ទៅក្នុងថត development របស់អ្នក
3. **Set Environment Variables**: កំណត់ផ្លូវសម្រាប់ឧបករណ៍និងបណ្ណាល័យ QNN

```bash
# កំណត់អថេរបរិយាកាសសម្រាប់ QNN
export QNN_SDK_ROOT=/path/to/qnn-sdk
export PATH=$QNN_SDK_ROOT/bin:$PATH
export LD_LIBRARY_PATH=$QNN_SDK_ROOT/lib:$LD_LIBRARY_PATH
```

### Python Environment Setup

Create and activate a virtual environment:

```bash
# បង្កើតបរិយាកាសនិម្មិត
python -m venv qnn-env

# បើកបរិយាកាសនិម្មិត
# នៅលើ Windows:
qnn-env\Scripts\activate
# នៅលើ Linux:
source qnn-env/bin/activate
```

Install required Python packages:

```bash
pip install numpy tensorflow torch onnx
```

### Verify Installation

```bash
# ពិនិត្យភាពអាចប្រើបាននៃឧបករណ៍ QNN
qnn-model-lib-generator --help
qnn-context-binary-generator --help
qnn-net-run --help
```

If successful, you should see help information for each QNN tool.

## Quick Start Guide

### Your First Model Conversion

Let's convert a simple PyTorch model to run on Qualcomm hardware:

```python
import torch
import torch.nn as nn
import numpy as np

# កំណត់ម៉ូឌែលសាមញ្ញ
class SimpleModel(nn.Module):
    def __init__(self):
        super(SimpleModel, self).__init__()
        self.conv1 = nn.Conv2d(3, 32, 3, padding=1)
        self.relu = nn.ReLU()
        self.conv2 = nn.Conv2d(32, 64, 3, padding=1)
        self.pool = nn.AdaptiveAvgPool2d((1, 1))
        self.fc = nn.Linear(64, 10)
    
    def forward(self, x):
        x = self.relu(self.conv1(x))
        x = self.relu(self.conv2(x))
        x = self.pool(x)
        x = x.view(x.size(0), -1)
        x = self.fc(x)
        return x

# បង្កើត និងនាំចេញម៉ូឌែល
model = SimpleModel()
model.eval()

# បង្កើតបញ្ចូលគំរូសម្រាប់ការតាមដាន
dummy_input = torch.randn(1, 3, 224, 224)

# នាំចេញទៅ ONNX
torch.onnx.export(
    model,
    dummy_input,
    "simple_model.onnx",
    export_params=True,
    opset_version=11,
    do_constant_folding=True,
    input_names=['input'],
    output_names=['output'],
    dynamic_axes={'input': {0: 'batch_size'},
                  'output': {0: 'batch_size'}}
)
```

### Convert ONNX to QNN Format

```bash
# បម្លែងម៉ូដែល ONNX ទៅជាបណ្ណាល័យម៉ូដែល QNN
qnn-onnx-converter \
    --input_network simple_model.onnx \
    --output_path simple_model.cpp \
    --input_dim input 1,3,224,224 \
    --quantization_overrides quantization_config.json
```

### Generate QNN Model Library

```bash
# បង្កើតបណ្ណាល័យម៉ូដែល
qnn-model-lib-generator \
    -c simple_model.cpp \
    -b simple_model.bin \
    -t x86_64-linux-clang \
    -l simple_model \
    -o simple_model_qnn.so
```

### What This Process Does

ដំណើរការអុបទិម៉ាយនេះរួមមាន: បម្លែងម៉ូដែលដើមទៅទ្រង់ទ្រាយ ONNX, បម្លែង ONNX ទៅតំណាងមធ្យម QNN, អនុវត្តការអុបទិម៉ាយដែលពាក់ព័ន្ធនឹង hardware ពិសេស, ហើយបង្កើតបណ្ណាល័យម៉ូដែលដែលបានបង្ហប់សម្រាប់ការចែកចាយ។

### Key Parameters Explained

- `--input_network`: ឯកសារ ម៉ូដែល ONNX ប្រភព
- `--output_path`: ឯកសារ C++ មួយដែលបានផលិត
- `--input_dim`: មាត្រដ្ឋាន tensor នៃការបញ្ចូលសម្រាប់ការអុបទិម៉ាយ
- `--quantization_overrides`: កំណត់កំណត់ Quantization ផ្ទាល់ខ្លួន
- `-t x86_64-linux-clang`: ស្ថាបត្យកម្មគោលដៅ និងកម្មឧបករណ៍បង្កើត

### Step 4: Inference with QNN Runtime

```python
import ctypes
import numpy as np

# ផ្ទុកបណ្ណាល័យ QNN
qnn_lib = ctypes.CDLL('./libQnn.so')

class QNNInference:
    def __init__(self, model_path, backend='htp'):
        self.model_path = model_path
        self.backend = backend
        self.context = None
        self._initialize()
    
    def _initialize(self):
        # ចាប់ផ្តើម runtime របស់ QNN
        # ផ្ទុកម៉ូឌែល និងបង្កើត context សម្រាប់ inference
        pass
    
    def preprocess_input(self, data):
        # បម្លែងទិន្នន័យបញ្ចូលទៅជា quantized ប្រសិនបើចាំបាច់
        if self.is_quantized:
            # អនុវត្តព៉ារ៉ាម៉ែត្រ quantization
            scale = self.input_scale
            offset = self.input_offset
            quantized = np.clip(
                np.round(data / scale + offset), 
                0, 255
            ).astype(np.uint8)
            return quantized
        return data.astype(np.float32)
    
    def inference(self, input_data):
        # ដំណើរការមុនលើទិន្នន័យបញ្ចូល
        processed_input = self.preprocess_input(input_data)
        
        # ដំណើរការ inference លើឧបករណ៍រឹង Qualcomm
        # នេះនឹងហៅទៅកាន់ QNN C++ API
        output = self._run_inference(processed_input)
        
        # ដំណើរការបន្ទាប់លើលទ្ធផល
        return self.postprocess_output(output)
    
    def postprocess_output(self, output):
        # ដកការបរិមាណពីលទ្ធផល ប្រសិនបើចាំបាច់
        if self.is_quantized:
            scale = self.output_scale
            offset = self.output_offset
            dequantized = (output.astype(np.float32) - offset) * scale
            return dequantized
        return output

# ការប្រើប្រាស់
inference_engine = QNNInference("model_optimized.so", backend="htp")
result = inference_engine.inference(input_tensor)
print(f"Inference result: {result}")
```

### Dynamic Quantization

Apply quantization at runtime for better accuracy:

```python
class DynamicQuantization:
    def __init__(self, model_path):
        self.model_path = model_path
        self.calibration_data = []
    
    def collect_statistics(self, calibration_dataset):
        """Collect activation statistics for quantization"""
        for data in calibration_dataset:
            # រត់ការទស្សន៍ទាយ (inference) និងប្រមូលជួរតម្លៃរបស់ activation
            activations = self.forward_hooks(data)
            self.calibration_data.append(activations)
    
    def compute_quantization_params(self):
        """Compute optimal quantization parameters"""
        params = {}
        for layer_name, activations in self.calibration_data:
            min_val = np.min(activations)
            max_val = np.max(activations)
            
            # គណនាស្កែល និងអ៊ុបសែតសម្រាប់ INT8 quantization
            scale = (max_val - min_val) / 255.0
            offset = -min_val / scale
            
            params[layer_name] = {
                "scale": scale,
                "offset": int(offset),
                "min": min_val,
                "max": max_val
            }
        
        return params
    
    def apply_quantization(self, quantization_params):
        """Apply computed quantization parameters"""
        config = {
            "activation_encodings": {},
            "param_encodings": {}
        }
        
        for layer, params in quantization_params.items():
            config["activation_encodings"][layer] = {
                "bitwidth": 8,
                "scale": params["scale"],
                "offset": params["offset"],
                "min": params["min"],
                "max": params["max"]
            }
        
        return config
```

### Performance Profiling

Monitor performance across different backends:

```python
import time
import psutil

class QNNProfiler:
    def __init__(self):
        self.metrics = {}
    
    def profile_inference(self, inference_func, input_data, num_runs=100):
        """Profile inference performance"""
        latencies = []
        cpu_usage = []
        memory_usage = []
        
        for i in range(num_runs):
            # តាមដានធនធានប្រព័ន្ធ
            process = psutil.Process()
            cpu_before = process.cpu_percent()
            memory_before = process.memory_info().rss
            
            # វាស់ពេលវេលាការទាយលទ្ធផល
            start_time = time.perf_counter()
            result = inference_func(input_data)
            end_time = time.perf_counter()
            
            latency = (end_time - start_time) * 1000  # បម្លែងទៅជា ms
            latencies.append(latency)
            
            # ប្រមូលការប្រើប្រាស់ធនធាន
            cpu_after = process.cpu_percent()
            memory_after = process.memory_info().rss
            
            cpu_usage.append(cpu_after - cpu_before)
            memory_usage.append(memory_after - memory_before)
        
        return {
            "avg_latency_ms": np.mean(latencies),
            "p95_latency_ms": np.percentile(latencies, 95),
            "p99_latency_ms": np.percentile(latencies, 99),
            "throughput_fps": 1000 / np.mean(latencies),
            "avg_cpu_usage": np.mean(cpu_usage),
            "avg_memory_delta_mb": np.mean(memory_usage) / (1024 * 1024)
        }

# ការប្រើប្រាស់
profiler = QNNProfiler()
htp_metrics = profiler.profile_inference(htp_inference, test_data)
gpu_metrics = profiler.profile_inference(gpu_inference, test_data)
cpu_metrics = profiler.profile_inference(cpu_inference, test_data)

print("HTP Performance:", htp_metrics)
print("GPU Performance:", gpu_metrics)
print("CPU Performance:", cpu_metrics)
```

### Automated Backend Selection

Implement intelligent backend selection based on model characteristics:

```python
class BackendSelector:
    def __init__(self):
        self.backend_capabilities = {
            "htp": {
                "supported_ops": ["Conv2d", "Dense", "BatchNorm", "ReLU"],
                "max_tensor_size": 8 * 1024 * 1024,  # 8MB
                "preferred_precision": "int8",
                "power_efficiency": 0.9
            },
            "gpu": {
                "supported_ops": ["Conv2d", "Dense", "ReLU", "Softmax"],
                "max_tensor_size": 64 * 1024 * 1024,  # 64MB
                "preferred_precision": "fp16",
                "power_efficiency": 0.6
            },
            "cpu": {
                "supported_ops": ["*"],  # ប្រតិបត្តិការទាំងអស់
                "max_tensor_size": 512 * 1024 * 1024,  # 512MB
                "preferred_precision": "fp32",
                "power_efficiency": 0.4
            }
        }
    
    def select_optimal_backend(self, model_info, constraints):
        """Select optimal backend based on model and constraints"""
        scores = {}
        
        for backend, caps in self.backend_capabilities.items():
            score = 0
            
            # ពិនិត្យការគាំទ្រប្រតិបត្តិការ
            if all(op in caps["supported_ops"] or "*" in caps["supported_ops"] 
                   for op in model_info["operations"]):
                score += 30
            
            # ពិនិត្យភាពសមរម្យនៃទំហំតង់ស័រ
            if model_info["max_tensor_size"] <= caps["max_tensor_size"]:
                score += 25
            
            # ការពិចារណាអំពីប្រសិទ្ធភាពថាមពល
            if constraints.get("power_critical", False):
                score += caps["power_efficiency"] * 25
            
            # អាទិភាពនៃសមត្ថភាព
            if constraints.get("performance_critical", False):
                if backend == "htp":
                    score += 20
            
            scores[backend] = score
        
        return max(scores, key=scores.get)

# ការប្រើប្រាស់
selector = BackendSelector()
model_info = {
    "operations": ["Conv2d", "ReLU", "Dense"],
    "max_tensor_size": 4 * 1024 * 1024,
    "precision": "int8"
}
constraints = {
    "power_critical": True,
    "performance_critical": True
}

optimal_backend = selector.select_optimal_backend(model_info, constraints)
print(f"Recommended backend: {optimal_backend}")
```

## Best Practices

### 1. Model Architecture Optimization
- **Layer Fusion**: បញ្ចូលប្រតិបត្តិការ ដូចជា Conv+BatchNorm+ReLU ដើម្បីប្រើប្រាស់ NPU បានល្អជាង
- **Depth-wise Separable Convolutions**: ជ្រើសរើសប្រភេទទាំងនេះជំនួសការបញ្ជាទូទៅសម្រាប់ការដាក់ចេញលើឧបករណ៍ចល័ត
- **Quantization-Friendly Designs**: ប្រើ activation ReLU និងជៀសវាងប្រតិបត្តិការ​ដែលមិនសមនឹងការចុះកម្រិតបានល្អ

### 2. Quantization Strategy
- **Post-Training Quantization**: ចាប់ផ្តើមជាមួយនេះសម្រាប់ការចេញផ្សាយរហ័ស
- **Calibration Dataset**: ប្រើទិន្នន័យតំណាងដែលគ្របដណ្តប់លើការប្រែប្រួលទាំងអស់នៃការបញ្ចូល
- **Mixed Precision**: ប្រើ INT8 សម្រាប់ស្រទាប់ភាគច្រើន រក្សាស្រទាប់សំខាន់ក្នុងកម្រិតដែលខ្ពស់ជាង

### 3. Backend Selection Guidelines
- **NPU (HTP)**: ល្អបំផុតសម្រាប់កិច្ចការជាមួយ CNN, ម៉ូដែលបានចុះកម្រិត, និងកម្មវិធីដែលសាមញ្ញផ្នែកថាមពល
- **GPU**: ឈានដល់ប្រសិទ្ធភាពល្អសម្រាប់ប្រតិបត្តិការគណនា ខុសៗ មូដែលធំ និង FP16 precision
- **CPU**: ជាជម្រើសមុនសម្រាប់ប្រតិបត្តិការ​ដែលមិនបានគាំទ្រ និងការដាប់ bug

### 4. Performance Optimization
- **Batch Size**: ប្រើទំហំ batch 1 សម្រាប់កម្មវិធីពេលពិត, batch ធំសម្រាប់ throughput
- **Input Preprocessing**: កាត់បន្ថយការចម្លងទិន្នន័យ និងការបម្លែង
- **Context Reuse**: កំណត់ context មុនដើម្បីជៀសវាងការបង្កើតកូដនៅ runtime

### 5. Memory Management
- **Tensor Allocation**: ប្រើការចំណាយអង្គចងចាំស្ថិតិ​នៅពេលអាចធ្វើទៅបាន ដើម្បីជៀសវាងចំណាយ runtime
- **Memory Pools**: អនុវត្ត memory pools ផ្ទាល់ខ្លួនសម្រាប់ tensors ដែលចំណាយញឹកញាប់
- **Buffer Reuse**: ប្រើឡើងវិញប៊ុហ្វเฟอร์បញ្ចូល/បញ្ចេញរវាងការហៅ inference

### 6. Power Optimization
- **Performance Modes**: ប្រើរបៀបប្រសិទ្ធភាពសមរម្យដោយផ្អែកលើកំដៅ
- **Dynamic Frequency Scaling**: អនុញ្ញាតឲ្យប្រព័ន្ធបង្កើន/កាត់សមត្ថភាពប័ណ្ណហៅតាមបន្ទុក
- **Idle State Management**: ចេញធនធាន​ឲ្យត្រឹមត្រូវនៅពេលមិនប្រើ

### Performance Debugging

```python
# បង្កើតឧបករណ៍វិភាគសមត្ថភាព
class QNNDebugger:
    def __init__(self, model_path):
        self.model_path = model_path
        self.layer_timings = {}
    
    def profile_layers(self, input_data):
        """Profile individual layer performance"""
        # នេះនឹងទាមទារ​ការ​តភ្ជាប់​ជាមួយ API ការ​ពិនិត្យសមត្ថភាព​របស់ QNN
        for layer_name in self.get_layer_names():
            start = time.perf_counter()
            # អនុវត្តស្រទាប់
            end = time.perf_counter()
            self.layer_timings[layer_name] = (end - start) * 1000
    
    def analyze_bottlenecks(self):
        """Identify performance bottlenecks"""
        sorted_layers = sorted(
            self.layer_timings.items(), 
            key=lambda x: x[1], 
            reverse=True
        )
        
        print("Top 5 slowest layers:")
        for layer, time_ms in sorted_layers[:5]:
            print(f"  {layer}: {time_ms:.2f} ms")
    
    def suggest_optimizations(self):
        """Suggest optimization strategies"""
        suggestions = []
        
        for layer, time_ms in self.layer_timings.items():
            if time_ms > 10:  # ស្រទាប់ចំណាយពេលលើសពី 10ms
                if "conv" in layer.lower():
                    suggestions.append(f"Consider depthwise separable conv for {layer}")
                elif "dense" in layer.lower():
                    suggestions.append(f"Consider quantization for {layer}")
        
        return suggestions
```
