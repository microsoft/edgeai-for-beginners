# ផ្នែកទី ៤៖ ការដាក់ឲ្យដំណើរការ - ការអនុវត្តម៉ូដែលដែលត្រៀមរួចសម្រាប់ផលិតកម្ម

## គ្រឹះមូល

មេរៀនទាំងមូលនេះនឹងណែនាំអ្នកតាមដំណើរការពេញលេញនៃការដាក់ម៉ូដែលដែលបានបញ្ចូលបន្ថែម (fine-tuned) និងបានកំណត់តម្លៃខ្ទង់ (quantized) ដោយប្រើ Foundry Local។ យើងនឹងគ្របដណ្តប់ពីការបំលែងម៉ូដែល ការបង្កើតតម្លៃខ្ទង់ និងកំណត់ការកម្មវិធីដំណើរការ ចាប់ពីដំណាក់កាលដំបូងដល់ចុងក្រោយ។

## អ្វីដែលត្រូវមានជាមុន

មុនពេលចាប់ផ្តើម សូមធានាថាអ្នកមានរបស់ដូចខាងក្រោម៖

- ✅ ម៉ូដែល onnx ដែលបានបញ្ចូលបន្ថែមរួចស្រ្តាស់សម្រាប់ដាក់ប្រាប់អោយដំណើរការ
- ✅ គណនីយន្តរយៈ Windows ឬ Mac
- ✅ Python 3.10 ឬខ្ពស់ជាងនេះ
- ✅ អាទិភាពមាន RAM ខ្ពស់មិនតិចជាង 8GB
- ✅ Foundry Local បានដំឡើងលើប្រព័ន្ធរបស់អ្នក

## ផ្នែកទី ១៖ ការរៀបចំបរិស្ថាន

### ការដំឡើងឧបករណ៍ដែលត្រូវការ

បើក Terminal របស់អ្នក (Command Prompt នៅលើ Windows, Terminal នៅលើ Mac) ហើយរត់ពាក្យបញ្ជាខាងក្រោមជាក្រោមជួរ៖

```bash
# អាប់ដេតបណ្ណាល័យ transformers ទៅជាវើស្យុងថ្មីបំផុត
pip install transformers -U

# តម្លើង Microsoft Olive (ឧបករណ៍បម្លែងម៉ូដែល)
pip install git+https://github.com/microsoft/Olive.git

# តម្លើង ONNX Runtime GenAI
git clone https://github.com/microsoft/onnxruntime-genai
cd onnxruntime-genai && python build.py --config Release
pip install {Your build release path}/onnxruntime_genai-0.9.0.dev0-cp311-cp311-linux_x86_64.whl

# តម្លើងរាល់ការទាមទារបន្ថែមផ្សេងទៀត
pip install onnx onnxruntime numpy
```
  
⚠️ **សម្គាល់សំខាន់**៖ អ្នកត្រូវការកំណែ CMake 3.31 ឬថ្មីជាងនេះដែលអាចទាញយកបានពី [cmake.org](https://cmake.org/download/)។

## ផ្នែកទី ២៖ ការបំលែងម៉ូដែល និងការកំណត់តម្លៃខ្ទង់

### ជ្រើសរើសទ្រង់ទ្រាយដែលត្រឹមត្រូវ

សម្រាប់ម៉ូដែលភាសាខ្នាតតូចដែលបានបញ្ចូលបន្ថែម យើងណែនាំឲ្យប្រើ **ទ្រង់ទ្រាយ ONNX** ព្រោះវារួមបញ្ចូល៖

- 🚀 ការបង្កើតប្រសិទ្ធភាពល្អប្រសើរ
- 🔧 ការដាក់អនុវត្តដោយមិនផ្អែកលើឧបករណ៍ជាក់លាក់
- 🏭 សមត្ថភាពសម្រាប់ផលិតកម្ម
- 📱 សមត្ថភាពឆ្លងកាត់វេទិកាប្រភេទផ្សេងៗ

### វិធីសាស្រ្តទី ១៖ ការបំលែងតាមពាក្យបញ្ជា​ដូច​មួយ (ណែនាំ)

ប្រើពាក្យបញ្ជារខាងក្រោម ដើម្បីបំលែងម៉ូដែលដែលបានបញ្ចូលបន្ថែមរបស់អ្នកដោយផ្ទាល់៖

```bash
olive auto-opt \
    --model_name_or_path /path/to/your/finetuned/model \
    --device cpu \
    --provider CPUExecutionProvider \
    --use_model_builder \
    --precision int4 \
    --output_path models/your-model-name/onnx \
    --log_level 1
```
  
**សេចក្ដីពន្យល់អំពីប៉ារ៉ាមេត្រ៖**  
- `--model_name_or_path`: ផ្លូវទៅម៉ូដែលដែលបានបញ្ចូលបន្ថែមរបស់អ្នក  
- `--device cpu`: ប្រើ CPU សម្រាប់បង្កើតប្រសិទ្ធភាព  
- `--precision int4`: ប្រើការកំណត់តម្លៃខ្ទង់ INT4 (កាត់បន្ថយទំហំប្រហែល ៥០%)  
- `--output_path`: ផ្លូវចេញសម្រាប់ម៉ូដែលដែលបានបំលែង

### វិធីសាស្រ្តទី ២៖ វិធីសាស្រ្តឯកសារកំណត់ការកំណត់ (សំរាប់អ្នកមានជំនាញ)

បង្កើតឯកសារកំណត់ឈ្មោះ `finetuned_conversion_config.json`៖

```json
{
    "input_model": {
        "type": "HfModel",
        "model_path": "/path/to/your/finetuned/model",
        "task": "text-generation"
    },
    "systems": {
        "local_system": {
            "type": "LocalSystem",
            "accelerators": [
                {
                    "execution_providers": [
                        "CPUExecutionProvider"
                    ]
                }
            ]
        }
    },
    "passes": {
        "builder": {
            "type": "ModelBuilder",
            "config": {
                "precision": "int4",
                "quantization_config": {
                    "block_size": 128,
                    "is_symmetric": false
                }
            }
        }
    },
    "host": "local_system",
    "target": "local_system",
    "cache_dir": "cache",
    "output_dir": "models/output/your-finetuned-model-onnx"
}
```
  
បន្ទាប់មករត់៖

```bash
olive run --config ./finetuned_conversion_config.json
```
  
### ប្រៀបធៀបជម្រើសកំណត់តម្លៃខ្ទង់

| កម្រិតត្រឹមត្រូវ | ទំហំឯកសារ | ល្បឿនវិភាគ | គុណភាពម៉ូដែល | ការប្រើប្រាស់ដែលណែនាំ |
|-----------|-----------|-----------------|---------------|-----------------|
| FP16      | មូលដ្ឋាន × 0.5 | លឿន | ល្អបំផុត | កំពូលឧបករណ៍ |
| INT8      | មូលដ្ឋាន × 0.25 | លឿនណាស់ | ល្អ | ជម្រើសត្រូវសម |
| INT4      | មូលដ្ឋាន × 0.125 | លឿនជាងគេ | ដែលអាចទទួលបាន | មានកំណត់ធនធាន |

💡 **ការប្រកាន់ប្រជា**៖ ចាប់ផ្តើមជាមួយការកំណត់តម្លៃខ្ទង់ INT4 សម្រាប់ការដាក់ដំណើរការដំបូង។ ប្រសិនប្រសេរ គុណភាពមិនសូវល្អ សាកល្បង INT8 ឬ FP16។

## ផ្នែកទី ៣៖ ការកំណត់ Foundry Local សម្រាប់ដាក់ដំណើរការ

### បង្កើតការកំណត់ម៉ូដែល

ចូលទៅកាន់ថតម៉ូដែលរបស់ Foundry Local៖

```bash
foundry cache cd ./models/
```
  
បង្កើតរចនាសម្ព័ន្ធថតម៉ូដែលរបស់អ្នក៖

```bash
mkdir -p ./models/custom/your-finetuned-model
```
  
បង្កើតឯកសារ `inference_model.json` ក្នុងថតម៉ូដែលរបស់អ្នក៖

```json
{
  "Name": "your-finetuned-model-int4",
  "Description": "Fine-tuned quantized model",
  "Version": "1.0",
  "PromptTemplate": {
    "system": "<|im_start|>system\n{Content}<|im_end|>",
    "user": "<|im_start|>user\n{Content}<|im_end|>",
    "assistant": "<|im_start|>assistant\n{Content}<|im_end|>",
    "prompt": "<|im_start|>user\n{Content}<|im_end|>\n<|im_start|>assistant"
  },
  "ModelConfig": {
    "max_length": 2048,
    "temperature": 0.7,
    "top_p": 0.9,
    "repetition_penalty": 1.1
  }
}
```

## ផ្នែកទី ៤៖ សាកល្បងម៉ូដែល និងបង្កើតប្រសិទ្ធភាព

### ពិនិត្យមើលការដំឡើងម៉ូដែល

ពិនិត្យមើលថា Foundry Local អាចស្គាល់ម៉ូដែលរបស់អ្នកបានទេ៖

```bash
foundry cache ls
```
  
អ្នកគួរតែលឿនឃើញ `your-finetuned-model-int4` នៅក្នុងបញ្ជី។

### ចាប់ផ្តើមសាកល្បងម៉ូដែល

```bash
foundry model run your-finetuned-model-int4
```
  
### វិភាគតម្លៃប្រសិទ្ធភាព

តាមដានកាតព្វកិច្ចសំខាន់នៅពេលសាកល្បង៖

1. **ពេលវេលាលយោល**៖ វាស់ពេលវេលាមធ្យមក្នុងការឆ្លើយតប  
2. **ការប្រើប្រាស់អង្គចងចាំ**៖ តាមដានការប្រើប្រាស់ RAM  
3. **ការប្រើប្រសិទ្ធភាព CPU**៖ ពិនិត្យផ្ទុកCPU  
4. **គុណភាពលទ្ធផល**៖ បា្រមាញ់ភាពទាក់ទងនិងការសម្របសម្រួលនៃការឆ្លើយតប

### បញ្ជីត្រួតពិនិត្យការផ្ទៀងផ្ទាត់គុណភាព

- ✅ ម៉ូដែលឆ្លើយតបបានត្រឹមត្រូវចំពោះសំណួរផ្នែកជាក់លាក់ដែលបានបញ្ចូលបន្ថែម  
- ✅ ទ្រង់ទ្រាយការឆ្លើយតបត្រូវគ្នានឹងរចនាបថលទ្ធផលដែលរំពឹងទុក  
- ✅ មិនមានរន្ធងារអង្គចងចាំពេលប្រើប្រាស់រយៈពេលវែង  
- ✅ ប្រសិទ្ធភាពស្ថាពរនៅមួយចំនួនវែងនៃបញ្ចូល  
- ✅ ការគ្រប់គ្រងម៉ូដែលត្រឹមត្រូវលើករណីជ្រៀតចូលចាញ់ និងបញ្ចូលមិនត្រឹមត្រូវ
