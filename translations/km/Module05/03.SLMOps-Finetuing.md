## ការណែនាំអំពីការបូកបន្ថែម

**ការបូកបន្ថែម** គឺជបច្ចេកទេសរៀនម៉ាស៊ីនដ៏មានអំណាច ដែលពាក់ព័ន្ធនឹងការបំលែងម៉ូឌែលដែលបានហ្វឹកហាត់រួចមកហើយ ដើម្បីធ្វើភារកិច្ចជាក់លាក់ ឬធ្វើការជាមួយសំណុំទិន្នន័យជាក់លាក់។ ជំនួសការបណ្តុះម៉ូឌែលពីដើម ការបូកបន្ថែមប្រើប្រាស់ចំណេះដឹងដែលបានរៀនរួចហើយពីម៉ូឌែលដែលបានហ្វឹកហាត់ និងកែសម្រួលវាសម្រាប់ករណីប្រើប្រាស់របស់អ្នក។

### តើការបូកបន្ថែមជាអ្វី?

ការបូកបន្ថែមគឺជារាងមួយនៃ**ការរៀនបញ្ជូន** ដែលអ្នក៖
- ចាប់ផ្តើមពីម៉ូឌែលដែលបានហ្វឹកហាត់រួច ហើយបានរៀនលំនាំទូទៅពីសំណុំទិន្នន័យធំៗ
- កែតម្រូវប៉ារ៉ាម៉ែត្រចំហំពេញរបស់ម៉ូឌែលដោយប្រើសំណុំទិន្នន័យជាក់លាក់របស់អ្នក
- រក្សាទុកចំណេះដឹងមានតម្លៃ ខណៈពេលបំលែងម៉ូឌែលឲ្យមានជំនាញល្អក្នុងភារកិច្ចរបស់អ្នក

គិតវាដូចជាការបង្រៀនម្ហូបស្វែងយល់ពីឆេហ្វដែលមានជំនាញក្នុងការធ្វើម្ហូបថ្មីៗ - គាត់បានយល់ដឹងពីមូលដ្ឋានធ្វើម្ហូបរួចហើយ ប៉ុន្តែត្រូវរៀនបច្ចេកទេស និងរសជាតិជាក់លាក់ទៀតសម្រាប់រចនាបទថ្មី។

### អត្ថប្រយោជន៍សំខាន់ៗ

- **ប្រសិទ្ធភាពពេលវេលា**៖ លឿនជាងការបណ្តុះពីដើមយ៉ាងខ្លាំង
- **ប្រសិទ្ធភាពទិន្នន័យ**៖ ត្រូវការសំណុំទិន្នន័យតូចជាងដើម្បីទទួលបានប្រសិទ្ធិភាពល្អ
- **មានតម្លៃថោក**៖ តម្រូវការគណនា​ក្រោម
- **ការទទួលលទ្ធផលល្អជាង**៖ ជាញឹកញាប់ទទួលបានលទ្ធផលល្អជាងការបណ្តុះពីដើម
- **បច្ចេកវិទ្យាអ្នកធ្វើការរស់រវើក**៖ ធ្វើឲ្យ AI មានអំពើរហូតទៅក្រុមតូច និងអង្គភាពតូចៗអាចចូលប្រើបាន

## ហេតុអ្វីបានជា ការបូកបន្ថែម មានសារៈសំខាន់

### ករណីប្រើប្រាស់ពិតប្រាកដ

ការបូកបន្ថែមចាំបាច់នៅក្នុងស្ថានការណ៍ជាច្រើន៖

**1. ការបន្សំសមាសភាពដែនទំនិញ**
- AI វេជ្ជសាស្ត្រ​៖ ប្ដូរម៉ូឌែលភាសាទូទៅសម្រាប់ពាក្យបញ្ជាក់វេជ្ជសាស្ត្រ និងកំណត់ត្រាគ្លីនីក
- បច្ចេកវិទ្យាមួយច្បាប់​៖ ប្តូរម៉ូឌែលឲ្យមានជំនាញក្នុងវិភាគឯកសារច្បាប់ និងពិនិត្យខ្ចីសេវា
- សេវាកម្មហិរញ្ញវត្ថុ៖ ការប្ដូរម៉ូឌែលសម្រាប់វិភាគរបាយការណ៍ហិរញ្ញវត្ថុ និងការវាយតម្លៃហានិភ័យ

**2. ការស្វែងយល់ជាប្រភេទភារកិច្ច**
- ការចេញផ្សាយមាតិកា៖ បូកបន្ថែមសម្រាប់រចនាបទសរសេរច្បាស់លាស់ ឬស្ទីលមានសំឡេងជាក់លាក់
- ការបង្កើតកូដ៖ កែសម្រួលម៉ូឌែលសម្រាប់ភាសាកម្មវិធី ឬស៊ុមផ្នែកបណ្តើរ
- ការបកប្រែ៖ ធ្វើឲ្យរលូនសម្រាប់គ្រាប់ភាសា ឬដែនបច្ចេកទេសជាក់លាក់

**3. ការអនុវត្តនៅក្រុមហ៊ុន**
- សេវាកម្មអតិថិជន៖ បង្កើតបច្ចេកវិទ្យាផ្សារាសន្ទនាឲ្យយល់ពាក្យពេញនិយមរបស់ក្រុមហ៊ុន
- ឯកសារមួយខាងក្នុង​៖ បង្កើតជំនួយកាក់ទឹកដៃ AI ដែលស្គាល់ដំណើរការអង្គភាព
- ដំណោះស្រាយបច្ចេកទេសជាក់លាក់៖ អភិវឌ្ឍម៉ូឌែលដែលយល់ពាក្យឈ្មោះ និងដំណើរការផ្នែកសេដ្ឋកិច្ច

## ប្រភេទនៃការបូកបន្ថែម

### 1. ការបូកបន្ថែមពេញលេញ (ការបូកបន្ថែមលើការណែនាំ)

នៅក្នុងការបូកបន្ថែមពេញលេញ ប៉ារ៉ាម៉ែត្រម៉ូឌែលទាំងអស់ត្រូវបានធ្វើបច្ចុប្បន្នភាពពេលហ្វឹកហាត់។ វិធីសាស្រ្តនេះ៖
- ផ្តល់នូវភាពបត់បែន និងសមត្ថភាពល្អបំផុត
- តម្រូវធនធានគណនាច្រើន
- ផលិតជាម៉ូឌែលថ្មីទាំងស្រុង
- ល្អសម្រាប់ករណីមានទិន្នន័យហ្វឹកហាត់ច្រើន និងធនធានគណនាច្រើន

### 2. ការបូកបន្ថែមប្រសិទ្ធភាពប៉ារ៉ាម៉ែត្រ (PEFT)

វិធីសាស្រ្ត PEFT បែរជាបានធ្វើបច្ចុប្បន្នភាពតែប៉ារ៉ាម៉ែត្រតិចតួចបង្ហាញពីប្រសិទ្ធភាពខ្ពស់៖

#### ការបង់តម្លៃទាប (LoRA)
- បន្ថែមម៉ាទ្រីសបែងចែកជាតំណំតិចតួចដែលអាចហ្វឹកហាត់ទៅលើបំពង់ករណីមានដើម
- បន្ថយចំនួនប៉ារ៉ាម៉ែត្រអាចហ្វឹកហាត់បានយ៉ាងខ្លាំង
- រក្សាប្រសិទ្ធភាពជិតស្និតជាមួយការបូកបន្ថែមពេញលេញ
- អនុញ្ញាតឲ្យប្ដូរបរិមាណតាមលំនាំរបៀបបានយ៉ាងងាយស្រួល

#### QLoRA (LoRA ចំនួនកំណត់)
- ប្រើផ្សំ LoRA ជាមួយបច្ចេកទេសបញ្ចាក់ចំនួន
- បន្ថយតម្រូវការចងចាំបន្ថែម
- អនុញ្ញាតឲ្យបូកបន្ថែមម៉ូឌែលធំៗនៅលើឧបករណ៍ផ្ទាល់ខ្លួន
- សម្របសម្រួលរវាងប្រសិទ្ធភាព និងលទ្ធផល

#### អាដាប់ធ័រ
- បញ្ចូលបណ្តាញប្រសាទតូចនៅរវាងស្រទាប់ដែលមានរួចហើយ
- អនុញ្ញាតឲ្យបូកបន្ថែមជាក់លាក់បណ្តោះអាសននា ខណៈដែលរក្សាម៉ូឌែលមូលដ្ឋាន់ទឹចត្រជាក់
- អនុញ្ញាតដល់វិធីសាស្ត្រម៉ូឌែលកំណត់របៀបជាច្រើន

### 3. ការបូកបន្ថែមលើភារកិច្ចជាក់លាក់

ផ្តោតការលើបង្រួមម៉ូឌែលសម្រាប់ភារកិច្ច៖
- **ការបែងចែក**៖ កែប្រែម៉ូឌែលសម្រាប់ភារកិច្ចចាត់ជំពូក
- **ការបង្កើត**៖ ប្រសិទ្ធភាពសម្រាប់ការបង្កើតមាតិកា និងការបង្កើតអត្ថបទ
- **ការដកស្រង់**៖ បូកបន្ថែមសម្រាប់ការដកស្រង់ព័ត៌មាន និងការទទួលស្គាល់ឈ្មោះអង្គភាព
- **ការរំពឹងទុក**៖ បំលែងម៉ូឌែលសម្រាប់ការរំពឹងឯកសារ

## ការបូកបន្ថែមជាមួយ Microsoft Olive

Microsoft Olive គឺជាឧបករណ៍សម្រាប់បង្កើនប្រសិទ្ធភាពម៉ូឌែល ដែលបង្កប់ជាកញ្ចប់ពេញលេញ ដែលបង្រួមបង្រួមដំណើរការបូកបន្ថែម ខណៈផ្តល់នូវលក្ខណៈពិសេសដែលសមស្របសម្រាប់សហគ្រាស។

### តើ Microsoft Olive ជាអ្វី?

Microsoft Olive គឺជាឧបករណ៍បើកប្រភពសម្រាប់បង្កើនប្រសិទ្ធភាពម៉ូឌែល ដែល៖
- ផ្តល់ភាពងាយស្រួលក្នុងដំណើរការបូកបន្ថែមសម្រាប់ឧបករណ៍បំណងផ្សេងៗ
- មានសំណូមពរខាងក្នុងសម្រាប់រចនាសម្ព័ន្ធម៉ូឌែលពេញនិយម (Llama, Phi, Qwen, Gemma)
- ផ្តល់ជម្រើសផ្សាយបង្ហោះទាំងមេឃនិងក្នុងស្រុក
- សម្រុះសម្រួលជាមួយ Azure ML និងសេវាកម្ម AI របស់ Microsoft ផ្សេងទៀត
- គាំទ្រការបង្កើនប្រសិទ្ធភាព និងការបញ្ចាក់ទឹកចំនួនដោយស្វ័យប្រវត្តិ

### លក្ខណៈពិសេសសំខាន់ៗ

- **បង្កើនប្រសិទ្ធភាពតាមឧបករណ៍**៖ បង្កើនប្រសិទ្ធភាពម៉ូឌែលសម្រាប់ឧបករណ៍ជាក់លាក់ (CPU, GPU, NPU)
- **គាំទ្រផ្លូវប្លែកៗ**៖ ធ្វើការជាមួយម៉ូឌែល PyTorch, Hugging Face និង ONNX
- **ដំណើរការដោយស្វ័យប្រវត្តិ**៖ កាត់បន្ថយការកំណត់ដៃ និងការសាកល្បងកំហុស
- **ការសម្រុះទៅកាន់សហគ្រាស**៖ ជំនួយបង្កើតសម្រាប់ Azure ML និងការផ្សាយបង្ហោះបណ្ដាញ
- **រចនាសម្ព័ន្ធដែលអាចពង្រីកបាន**៖ អាចបន្ថែមបច្ចេកទេសបង្កើនប្រសិទ្ធភាពផ្ទាល់ខ្លួន

### ការដំឡើងនិងការតំឡើង

#### ការដំឡើងមូលដ្ឋាន

```bash
# បង្កើតបរិស្ថានវីចចुअល
python -m venv olive-env
source olive-env/bin/activate  # លើ Windows: olive-env\Scripts\activate

# ដំឡើង Olive ជាមួយមុខងារធ្វើប្រសើរឯក automática
pip install olive-ai[auto-opt]

# ដំឡើងការពឹងផ្អែកបន្ថែម
pip install transformers onnxruntime-genai
```

#### អាស្រ័យភាពជាជម្រើស

```bash
# សម្រាប់បង្កើនប្រសិទ្ធិភាព CPU
pip install olive-ai[cpu]

# សម្រាប់បង្កើនប្រសិទ្ធិភាព GPU
pip install olive-ai[gpu]

# សម្រាប់ DirectML (វិនដូ)
pip install olive-ai[directml]

# សម្រាប់ការរួមបញ្ចូល Azure ML
pip install olive-ai[azureml]
```

#### ពិនិត្យការដំឡើង

```bash
# ស្វែងរកថា Olive CLI មានស្រាប់ទេ
olive --help

# ពិនិត្យការតំឡើង
python -c "import olive; print('Olive installed successfully')"
```

## ឧទាហរណ៍អនុវត្ត

### ឧទាហរណ៍ 1៖ ការបូកបន្ថែមមូលដ្ឋានជាមួយ Olive CLI

ឧទាហរណ៍នេះបង្ហាញពីការបូកបន្ថែមម៉ូឌែលភាសាតូចសម្រាប់ចាត់ថ្នាក់ពាក្យបន្ទាត់៖

#### ជំហាន 1: រៀបចំបរិយាកាសរបស់អ្នក

```bash
# កំណត់បរិយាកាស
mkdir fine-tuning-project
cd fine-tuning-project

# ទាញយកទិន្នន័យគំរូ (ជាចំរើន - Olive អាចទាញយកទិន្នន័យដោយស្វ័យប្រវត្តិ)
huggingface-cli login  # ប្រសិនបើប្រើឯកសារទិន្នន័យបញ្ញ_privileged
```

#### ជំហាន 2: បូកបន្ថែមម៉ូឌែល

```bash
# គោលបំណងប៉ះពាល់គឺបញ្ជាការទល់នឹងមូលដ្ឋាន
olive finetune \
  --model_name_or_path meta-llama/Llama-3.2-1B-Instruct \
  --trust_remote_code \
  --output_path models/llama/ft \
  --data_name xxyyzzz/phrase_classification \
  --text_template "<|start_header_id|>user<|end_header_id|>\n{phrase}<|eot_id|><|start_header_id|>assistant<|end_header_id|>\n{tone}" \
  --method lora \
  --max_steps 100 \
  --log_level 1
```

#### ជំហាន 3: បង្កើនប្រសិទ្ធភាពសម្រាប់ផ្សាយបន្ត

```bash
# បំលែងទៅទ្រង់ទ្រាយ ONNX សម្រាប់ការវិភាគប្រសើរជាងមុន
olive auto-opt \
  --model_name_or_path models/llama/ft/model \
  --adapter_path models/llama/ft/adapter \
  --device cpu \
  --provider CPUExecutionProvider \
  --use_ort_genai \
  --output_path models/llama/onnx \
  --log_level 1
```

### ឧទាហរណ៍ 2៖ ការកំណត់រចនាសម្ព័ន្ធកម្រិតខ្ពស់ជាមួយសំណុំទិន្នន័យផ្ទាល់ខ្លួន

#### ជំហាន 1: រៀបចំសំណុំទិន្នន័យផ្ទាល់ខ្លួន

បង្កើតឯកសារ JSON ជាមួយទិន្នន័យហ្វឹកហាត់របស់អ្នក៖

```json
[
  {
    "input": "What is machine learning?",
    "output": "Machine learning is a subset of artificial intelligence that enables computers to learn and improve from experience without being explicitly programmed."
  },
  {
    "input": "Explain neural networks",
    "output": "Neural networks are computing systems inspired by biological neural networks that learn from data through interconnected nodes or neurons."
  }
]
```

#### ជំហាន 2: បង្កើតឯកសារកំណត់រចនាសម្ព័ន្ធ

```yaml
# olive-config.yaml
model:
  type: PyTorchModel
  config:
    model_path: "microsoft/DialoGPT-medium"
    task: "text-generation"

data_configs:
  - name: "custom_dataset"
    type: "HuggingfaceContainer"
    load_dataset_config:
      data_files: "path/to/your/dataset.json"
      split: "train"
    pre_process_data_config:
      text_template: "User: {input}\nAssistant: {output}"

passes:
  lora:
    type: LoRA
    config:
      r: 16
      lora_alpha: 32
      target_modules: ["c_attn", "c_proj"]
      modules_to_save: ["ln_f", "lm_head"]
```

#### ជំហាន 3: អនុវត្តការបូកបន្ថែម

```bash
# រត់ជាមួយការកំណត់ផ្ទាល់ខ្លួន
olive run --config olive-config.yaml --setup
```

### ឧទាហរណ៍ 3៖ ការបូកបន្ថែម QLoRA សម្រាប់ប្រសិទ្ធភាពកំណត់ចំណាំ

```bash
# តម្រែតម្រង់ជាមួយ QLoRA សម្រាប់ប្រសិទ្ធភាពអង្គចងចាំកាន់តែប្រសើរ
olive finetune \
  --method qlora \
  --model_name_or_path meta-llama/Meta-Llama-3-8B \
  --data_name nampdn-ai/tiny-codes \
  --train_split "train[:4096]" \
  --eval_split "train[4096:4224]" \
  --text_template "### Language: {programming_language} \n### Question: {prompt} \n### Answer: {response}" \
  --per_device_train_batch_size 16 \
  --per_device_eval_batch_size 16 \
  --max_steps 150 \
  --logging_steps 50 \
  --output_path adapters/tiny-codes
```

## អនុសាសន៍និងមគ្គុទេសក៍

### ការរៀបចំទិន្នន័យ

**1. គុណភាពទិន្នន័យលើបរិមាណ**
- ផ្តល់អាទិភាពឧទាហរណ៍មានគុណភាពខ្ពស់ និងចម្រុះជាងចំនួនដ៏ច្រើននៃទិន្នន័យខ្សោយ
- ប្រាកដថាទិន្នន័យតំណាងឱ្យករណីប្រើប្រាស់របស់អ្នក
- ធ្វើអោយស្អាត និងធ្វើតាមបំណង់ទិន្នន័យយ៉ាងច្បាស់

**2. របៀបវាយបញ្ចូល និង គំរូ**
- ប្រើទម្រង់ស្របគ្នារវាងគំរូហ្វឹកហាត់ទាំងអស់
- បង្កើតគំរូបញ្ជូល-បញ្ចេញដែលច្បាស់លាស់ ហើយសមរម្យនឹងករណីប្រើប្រាស់របស់អ្នក
- រួមបញ្ចូលការបង្ហាញលក្ខណៈបញ្ជាក់សម្រាប់ម៉ូឌែលដែលបានបង្រៀនដោយការណែនាំ

**3. ការបំបែកសំណុំទិន្នន័យ**
- រក្សាទុក ១០-២០% នៃទិន្នន័យសម្រាប់បញ្ជាក់
- រក្សាភាគរយដូចគ្នារៀងៗគ្នារវាងចំណែកហ្វឹកហាត់ និងបញ្ជាក់
- គិតពីការជ្រើសរើសតាមចំណាត់ថ្នាក់សម្រាប់ភារកិច្ចចាត់ថ្នាក់

### ការកំណត់ការបណ្តុះ

**1. ជ្រើសរាល់អត្រាការសិក្សា**
- ចាប់ផ្តើមជាមួយអត្រាការសិក្សាតិច (1e-5 ទៅ 1e-4) សម្រាប់ការបូកបន្ថែម
- ប្រើកក្កដិការ៉េអត្រាការសិក្សាសម្រាប់ការសម្រួលល្អ
- តាមដានកោងការបាត់បង់ដើម្បីកែទម្រង់អត្រា

**2. បរិមាណលំហូរ**
- តុល្យភាពរវាងទំហំលំហូរនិងកំណត់ចំណាំមាន
- ប្រើការសរុបស្រាំងច្រើនសម្រាប់ទំហំលំហូរធំជាង
- ពិចារណាទំនាក់ទំនងរវាងទំហំលំហូរ និងអត្រាការសិក្សា

**3. រយៈពេលហ្វឹកហាត់**
- តាមដានវិលាការបញ្ជាក់ដើម្បីជៀសវាងការហ្វឹកហាត់លើស
- ប្រើការបញ្ឈប់ជាមុនពេលពាក់ព័ន្ធប្រសិទ្ធភាពបញ្ជាក់នៅកម្រិតខ្ពស់
- រក្សាទុកចំណុចបញ្ឈប់ជាប្រចាំសម្រាប់ការបាលភាគ និងពិនិត្យ

### ជ្រើសរើសម៉ូឌែល

**1. ជ្រើសរើសម៉ូឌែលមូលដ្ឋាន**
- ជ្រើសរើសម៉ូឌែលដែលបានហ្វឹកហាត់លើដែនខ្លឹមសារដូចគ្នា បើអាចធ្វើទៅបាន
- ពិចារណាទំហំម៉ូឌែលទាក់ទងនឹងកំណត់គណនារបស់អ្នក
- វាយតម្លៃតម្រូវការលីសង់សម្រាប់ការប្រើប្រាស់ពាណិជ្ជកម្ម

**2. ជ្រើសរើសវិធីសាស្ដ្របូកបន្ថែម**

- គិតពីវិធីសាស្ត្រតាម អាដាប់ធ័រសម្រាប់ករណីភារកិច្ចច្រើន

- ប្រើចងចាំ GPU យ៉ាងមានប្រសិទ្ធភាពជាមួយការត្រួតពិនិត្យកោងបាត់បង់
- ពិចារណាជម្រើសមេឃសម្រាប់ម៉ូឌែលធំៗ

**2. គ្រប់គ្រងកំណត់ចំណាំ**
- ប្រើការហ្វឹកហាត់ជាលំនៀងលាយម៉ាស្សា (mixed precision) នៅពេលមាន
- អនុវត្តសរុបស្រាំងសម្រាប់ការកំណត់ចំណាំ
- តាមដានការប្រើប្រាស់ចងចាំ GPU ក្នុងរយៈពេលហ្វឹកហាត់

### មុខងារបាត់បង់ផ្ទាល់ខ្លួន

អនុវត្តមុខងារបាត់បង់ជាផ្ទាល់ដែន៖

```python
# custom_loss.py
import torch
import torch.nn as nn

class CustomContrastiveLoss(nn.Module):
    def __init__(self, margin=1.0):
        super(CustomContrastiveLoss, self).__init__()
        self.margin = margin
        
    def forward(self, output1, output2, label):
        euclidean_distance = nn.functional.pairwise_distance(output1, output2)
        loss_contrastive = torch.mean((1-label) * torch.pow(euclidean_distance, 2) +
                                    (label) * torch.pow(torch.clamp(self.margin - euclidean_distance, min=0.0), 2))
        return loss_contrastive
```

## ការវាយតម្លៃនិងការតាមដាន

### មាត្រដ្ឋាននិងការវាយតម្លៃ

**1. មាត្រដ្ឋានស្តង់ដារ**
- **ភាពត្រឹមត្រូវ (Accuracy)**៖ ការត្រឹមត្រូវរួមសម្រាប់ភារកិច្ចចាត់ថ្នាក់
- **Perplexity**៖ វិមាត្រគុណភាពម៉ូឌែលភាសា
- **BLEU/ROUGE**៖ គុណភាពការបង្កើតអត្ថបទ និងការរំពឹង
- **ពិន្ទុ F1**៖ ការត្រឹមត្រូវនិងការទាញយកសមមាត្រ សម្រាប់ចាត់ថ្នាក់

**2. មាត្រដ្ឋានផ្ទាល់ខ្លួនឡើងវិញ**
- **គន្លងភារកិច្ចជាក់លាក់**៖ ប្រើគន្លងអនុវត្តដែលបានបង្កើតសម្រាប់ដែនរបស់អ្នក
- **ការវាយតម្លៃមនុស្ស**៖ រួមបញ្ចូលការវាយតម្លៃដោយមនុស្សសម្រាប់ភារកិច្ចមានចំណាំខាងចិត្តវិទ្យា
- **មាត្រដ្ឋានអាជីវកម្ម**៖ សម្របសម្រួលជាមួយគោលបំណងអាជីវកម្មពិតប្រាកដ

## បញ្ហាទូទៅនិងដំណោះស្រាយ

### បញ្ហា 1៖ ហ្វឹកហាត់លើស (Overfitting)

**រោគសញ្ញា:**
- កាត់ប្លង់បាត់បង់ហ្វឹកហាត់បន្តកាត់បន្ថយ ខណៈបាត់បង់បញ្ជាក់កើនឡើង
- ជន្លោះធំរវាងប្រសិទ្ធភាពហ្វឹកហាត់ និងបញ្ជាក់
- ធ្វើការត្រួតពិនិត្យសំណុំទិន្នន័យថ្មីបានអន់

**ដំណោះស្រាយ:**
```yaml
# Regularization techniques
passes:
  lora:
    type: LoRA
    config:
      r: 16  # Reduce rank to prevent overfitting
      lora_alpha: 16  # Lower alpha value
      lora_dropout: 0.1  # Add dropout
      weight_decay: 0.01  # L2 regularization
```

### បញ្ហា 2៖ កំណត់ចំណាំមិនគ្រប់គ្រាន់

**ដំណោះស្រាយ:**
- ប្រើការត្រួតពិនិត្យកោងបាត់បង់
- អនុវត្តការសម្រួលបម្រែបម្រួលកោងបាត់បង់
- ជ្រើសរើសវិធីសាស្ត្រប្រាក់លើប៉ារ៉ាម៉ែត្រ (LoRA, QLoRA)
- ប្រើបច្ចេកទេសផ្សេងៗដើម្បីរបៀបចែកម៉ូឌែល

