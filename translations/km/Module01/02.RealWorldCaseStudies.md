# ភាគទី 2៖ ករណីសិក្សាផ្នែកពិត

កម្មវិធី EdgeAI បង្ហាញពីការអនុវត្ត​ជាក់ស្តែង​នៃ​សមត្ថភាព AI លើឧបករណ៍មួយចំនួននៅចំណុចអ៊ិច (edge devices) ដែលបង្ហាញដំណោះស្រាយពិតប្រាកដ ដើម្បីដោះស្រាយបញ្ហាអំពីភាពឯកជន សម្ពាធពេលវេលា និងថ្លៃដើម។ វាមានសារៈសំខាន់ក្នុងការយល់ពីរបៀបដែលអង្គការប្រើប្រាស់ Samll Language Models (SLMs) ដោយជោគជ័យ ហើយបង្កើនប្រសិទ្ធភាពសម្រាប់ការប្រើប្រាស់ជាក់លាក់ ខណៈដែលរក្សាបាននូវមុខងារលើឧបករណ៍ដែលមានទំហំធនធានកំណត់។

## ការណែនាំ

នៅមេរៀននេះ យើងនឹងសិក្សាអំពីកម្មវិធីនិងការអនុវត្ត EdgeAI នៅពិភពពិត។ យើងនឹងពិនិត្យលើប្រព័ន្ធសំភារៈ Small Language Model របស់ Microsoft រួមទាំងម៉ូឌែល Phi Silica និង Mu ម៉ូឌែល សិក្សាលើករណីជោគជ័យដូចជា ប្រព័ន្ធ AI របាយការណ៍របស់អាកាសចរណ៍ជប៉ុន ហើយយល់អំពីការពិចារណាផ្លូវច្បាស់សម្រាប់ការវាស់វែង EdgeAI សម្រាប់បរិស្ថានសហគ្រាស។

- 🔧 យល់ដឹងអំពីអត្ថប្រយោជន៍ និងបញ្ហាប្រឈមនៃការរត់ SLM នៅក្នុងបរិស្ថានផលិតកម្ម។
- 📊 វាយតម្លៃផលប៉ះពាល់អាជីវកម្ម និងចំណូលលើការវិនិយោគ (ROI) នៃកម្មវិធី EdgeAI នៅឧស្សាហកម្មផ្សេងៗ។
- 🛠️ អនុវត្តិ​បច្ចេកទេសល្អបំផុតសម្រាប់ការវេចខ្ចប់ EdgeAI នៅក្នុងស្ថានการณ์ពិតប្រាកដ។

យើងអូសពិនិត្យរឿង គ្រឿងផ្សំនិងច្នៃប្រឌិតសំខាន់ៗដែលធ្វើឱ្យប្រព័ន្ធ EdgeAI របស់ Microsoft ចេញពីភាពជោគជ័យលើកម្មវិធីនិងករណីប្រើប្រាស់ផ្ទាល់ខ្លួន។

### បច្ចេកវិទ្យាមូលដ្ឋាន EdgeAI របស់ Microsoft

របៀបយក EdgeAI របស់ Microsoft ត្រូវបានបង្កើតលើបច្ចេកវិទ្យាមូលដ្ឋានជាច្រើន ដែលធ្វើឱ្យមានការបញ្ចូល AI លើឧបករណ៍ដោយមានប្រសិទ្ធភាព៖

- **រចនាសម្ព័ន្ធម៉ូឌែល Phi**៖ ម៉ូឌែលភាសាឆ្និត្រូវបានបង្កើតសម្រាប់ការវេចខ្ចប់លើចំណុចអ៊ិចជាមួយការប្រើប្រាស់ប៉ារ៉ាម៉ែត្រអ្វីដែលមានប្រសិទ្ធភាព។
- **QuaRot Quantization**៖ បច្ចេកវិទ្យាគុណភាពលំដាប់ 4-ប៊ីត ដែលរក្សាបានគុណភាពម៉ូឌែល ខណៈបន្ថយទីតាំងធនធាន។
- **ការរួមបញ្ចូល NPU**៖ ការបង្កើនប្រសិទ្ធភាពជាមួយ Neural Processing Unit សម្រាប់ឧបករណ៍ Windows និងបច្ចេកវិទ្យាការល្បឿនហាវ៉ែរ។
- **បច្ចេកទេសបង្កើនសមត្ថភាពសម្រាប់ភារកិច្ចជាក់លាក់**៖ ម៉ូឌែលដែលត្រូវបានកែរូបខ្ពស់សម្រាប់ដែនជាក់លាក់ជាជំនួសវត្ថុទូទៅ។

## Phi Silica៖ ការរួមបញ្ចូល AI លើ Windows

### រចនាសម្ព័ន្ធបច្ចេកទេស និងច្នៃប្រឌិត

- **គាំទ្រភាសាច្រើន:** 8 ភាសា (អង់គ្លេស ចិន បារីស ប៊ែម៉ាស German អ៊ីតាលី ជប៉ុន ព័រទុយហ្កាល់ ហ្ស្ប៉ាញ)

ការចាក់ចេញ EdgeAI តម្រូវឲ្យមានការសម្របសម្រួលយ៉ាងម៉ត់ចត់រវាងសមត្ថភាពម៉ូឌែល ការប្រើប្រាស់ធនធាន និងកំណត់តាំងការវេចខ្ចប់។ អង្គការត្រូវតែវាយតម្លៃចំណាយបំលែងរបស់ភាពត្រឹមត្រូវនិងប្រសិទ្ធភាព ដោយផ្អែកលើករណីប្រើប្រាស់ជាក់ស្តែងរបស់ខ្លួន។

### ភាពស្មុគស្មាញក្នុងការអភិវឌ្ឍនិងវេចខ្ចប់

ការវេចខ្ចប់ EdgeAI ដែលជោគជ័យ តម្រូវឱ្យមានជំនាញពិសេសក្នុងការបង្កើតម៉ូឌែល ការរួមបញ្ចូលហាវ៉ែរ និងហេដ្ឋារចនាសម្ព័ន្ធ edge computing។ អង្គការត្រូវចំណាយថវិកាទៅលើការបណ្តុះបណ្តាល និងការអភិវឌ្ឍ។

### ការថែទាំមូឌែល និងធ្វើបច្ចុប្បន្នភាព

ការរក្សាទុកម៉ូឌែល EdgeAI ឲ្យទាន់សម័យ និងមានប្រសិទ្ធភាព តម្រូវយុទ្ធសាស្ត្រក្នុងការគ្រប់គ្រងកំណែ ការតាមដានសមត្ថភាព និងធ្វើបច្ចុប្បន្នភាពតាមដំណាក់កាលលើឧបករណ៍ចំណុចអ៊ិចបង្ករ។

## សេចក្តីសន្និដ្ឋាន

កម្មវិធី EdgeAI របស់ Microsoft បង្ហាញថា Small Language Models មិនមែនជារូបមន្តតូចមួយនៃម៉ូឌែលធំទេ ប៉ុន្តែជា កំណែបំលែងគោលដៅទៅរកប្រព័ន្ធ AI ដែលមានប្រសិទ្ធភាព និងមានការពិសេសខ្ពស់។ ការជោគជ័យរបស់ Phi Silica ម៉ូឌែល Mu និងការអនុវត្តជាក់ស្តែងដូចជា ប្រព័ន្ធ JAL AI Report បង្ហាញថា EdgeAI អាចផ្តល់តម្លៃអាជីវកម្មជាក់ស្តែង ខណៈដែលដោះស្រាយបញ្ហាសំខាន់ៗ ពីភាពឯកជន សម្ពាធពេលវេលា និងថ្លៃដើម។

អនាគតនៃ EdgeAI ស្ថិតក្នុងការកែលម្អបន្តរូបមន្តម៉ូឌែល ក្បាលគុណភាព និងយុទ្ធសាស្ត្រវេចខ្ចប់ដែលផ្តោតលើប្រសិទ្ធភាព និងការពិសេស ជាជំនួសប្រព័ន្ធទូទៅ។ អង្គការដែលទទួលយកបំណុលនេះ នឹងមានទីតាំងល្អក្នុងការប្រើប្រាស់សមត្ថភាពបំលែងរបស់ AI ខណៈពេលរក្សានូវការត្រួតពិនិត្យលើទិន្នន័យ និងប្រតិបត្តិការរបស់ខ្លួន។
