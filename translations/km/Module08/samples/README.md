# Module 08 ឧទាហរណ៍: មគ្គុទេសក៍អភិវឌ្ឍ Foundry Local

## ទិដ្ឋភាពទូទៅ

ការរៀបចំពេញលេញនេះបង្ហាញពីរបៀបទាំងពីរ **Foundry Local SDK** និង **Shell Command** សម្រាប់សាងសង់កម្មវិធី AI ដែលត្រៀមសម្រាប់ផ្ទុកក្នុងផលិតកម្ម។ គំរូនីមួយៗបង្ហាញពីចំណុចខុសគ្នានានានៃការអភិវឌ្ឍ edge AI ចាប់ពីការរួមបញ្ចូល REST មូលដ្ឋាន ដល់ប្រព័ន្ធ multi-agent កម្រិតខ្ពស់។

## វិធីសាស្រ្តអភិវឌ្ឍនា: SDK និង Shell Commands

### ប្រើ Foundry Local SDK ពេលទាំងនេះ៖

- **Programmatic Control**: អ្នកត្រូវការត្រួតត្រា​តាមកម្មវិធី​ពេញលេញលើវដ្តជីវិត agent, ការវាយតម្លៃ, ឬលំហាត់ការផ្តល់ដាក់ (deployment)
- **Custom Tooling**: ការសាងសង់អូតូម៉េស្យុងជុំវិញ Foundry Local (CI/CD integration, multi-agent orchestration)
- **Fine-Grained Access**: តម្រូវឲ្យមានព័ត៌មានលម្អិតអំពី metadata ធាតុនៃ agent, ការ versioning, ឬការត្រួតត្រា evaluation runner
- **Python Integration**: ធ្វើការនៅក្នុងបរិយាកាសដែលផ្អែកលើ Python ឬបញ្ចូល logic របស់ Foundry ទៅក្នុងកម្មវិធីទូលំទូលាយ
- **Enterprise Workflows**: អនុវត្តដំណើរការ​ម៉ូឌុល និង pipeline វាយតម្លៃដែលអាចធ្វើឡើងវិញ ស្របតាម reference architectures របស់ Microsoft

### ប្រើ Shell Commands ពេលៈ

- **Quick Testing**: ធ្វើតេស្តលឿនក្នុងមូលដ្ឋាន, បើក agent ដោយដៃ, ឬផ្ទៀងផ្ទាត់ការរៀបចំ
- **CLI Simplicity**: ត្រូវការបញ្ជាទូទៅ CLI សាមញ្ញសម្រាប់ចាប់/បញ្ឈប់ agents, ពិនិត្យកំណត់ហេតុ, ឬការវាយតម្លៃមូលដ្ឋាន
- **Lightweight Automation**: ស្គ្រីបអូតូម៉េស្យុងសាមញ្ញដោយមិនចាំបាច់រួមបញ្ចូល SDK ទាំងមូល
- **Rapid Iteration**: វដ្តដើម្បីដោះស្រាយបញ្ហា និងអភិវឌ្ឍន៍យ៉ាងលឿន ជាពិសេសនៅក្នុងបរិយាកាសដែលមានកំណត់ ឬការដាក់បញ្ចូលនៅកម្រិតក្រុមធនធាន (resource group-level deployments)
- **Setup & Validation**: ការកំណត់បរិយាកាសដំបូង និងភារកិច្ចបញ្ជាក់លឿន

## អនុវត្តល្អបំផុត និងវិធីសាស្ត្រដែលផ្ដល់អនុសាសន៍

អាស្រ័យលើការគ្រប់គ្រងវដ្តជីវិត agent, ការតាមដានភាពពឹងផ្អែក (dependency tracking), និងគោលការណ៍នៃការបង្កើតឡើងវិញដោយអានិច្ចតិចបំផុត (least-privilege reproducibility)៖

### ជំហរ 1: មូលដ្ឋាន និង ការរៀបចំ
1. **Start with Shell Commands** សម្រាប់ការរៀបចំដំបូង និងការផ្ទៀងផ្ទាត់លឿន
2. **Verify Environment** ដោយប្រើឧបករណ៍ CLI និងការដាក់ប្រេងម៉ូដែលមូលដ្ឋាន
3. **Test Connectivity** ជាមួយការហៅ REST ងាយៗ និងការត្រួតពិនិត្យសុខភាព

### ជំហរ 2: ការ​អភិវឌ្ឍ និង ការរួមបញ្ចូល
1. **Transition to SDK** សម្រាប់ដំណើរការដែលអាចពង្រីក និងអាចតាមដានបាន
2. **Implement Programmatic Control** សម្រាប់អន្តរកម្ម agent ស្មុគស្មាញ
3. **Build Custom Tools** សម្រាប់ទំព័រគំរូដែលត្រៀមសម្រាប់សហគមន៍ និងការរួមបញ្ចូល Azure OpenAI

### ជំហរ 3: ផលិតកម្ម និង កម្រិត
1. **Hybrid Approach** ដែលរួមបញ្ចូល CLI សម្រាប់ប្រតិបត្តិការ និង SDK សម្រាប់លក្ខណៈបច្ចេកទេសនៃកម្មវិធី
2. **Enterprise Integration** ជាមួយការត្រួតពិនិត្យ, កំណត់ហេតុ, និង pipeline នៃការដាក់ដំណើរការ
3. **Community Contribution** តាមរយៈទំព័រគំរូដែលអាចប្រើឡើងវិញ និងអនុវត្តល្អបំផុត
