### ហេតុអ្វីបានជា Jupyter Notebooks?

ខុសពីមេរៀនប្រពៃណី សៀវភៅទាំងនេះផ្គើត:

- **ការសិក្សាអន្តអ្នកប្រើ**: ប្រតិបត្តិខ្នែរកូដ និងមើលលទ្ធផលភ្លាមៗ
- **ការសាកល្បង**: ផ្លាស់ប្ដូរពីរ៉ាម៉ែត្រ និងសង្កេតការផ្លាស់ប្ដូរនៅពេលពិត
- **ឯកសារ**: ពណ៌នាទៅក្នុងខ្នែរកូដ និងកូនសៀវភៅ markdown ជួយដឹកនាំអ្នកតាមមូលដ្ឋាន
- **អាចចម្លងឡើងវិញបាន**: ឧទាហរណ៍ដែលដំណើរការពេញលេញដែលអ្នកអាចយោង និងប្រើឡើងវិញ
- **ការមើលឃើញ**: មើលវិនិយោគិតកម្មវិធី ប្រវែង embeddings និងលទ្ធផលនៅក្នុងបន្ទាត់

## 🎯 គោលបំណងការសិក្សា

### ជំនាញមូលដ្ឋានដែលអ្នកនឹងអភិវឌ្ឍ

ដោយធ្វើតាមសៀវភៅទាំងនេះ អ្នកនឹងមានជំនាញខាង:

1. **ការគ្រប់គ្រងសេវា AI នៅក្នុងស្រុក**
   - កំណត់រចនាសម្ព័ន្ធ និងគ្រប់គ្រងសេវាកម្ម Microsoft Foundry Local
   - ជ្រើសរើស និងផ្ទុកម៉ូឌែលដែលសមស្របសម្រាប់រឹងរឿងរបស់អ្នក
   - តាមដានការប្រើប្រាស់ឧបករណ៍ និងបង្កើនប្រសិទ្ធភាព
   - គ្រប់គ្រងការស្វែងរកសេវា និងពិនិត្យសុខភាពសេវា

2. **ការអភិវឌ្ឍកម្មវិធី AI**
   - អនុវត្តការបញ្ចប់ជជែកក្នុងលំនាំដូចជា OpenAI នៅក្នុងស្រុក
   - សាងសង់ចំណុចចាក់ចမြិចសម្រាប់លំហឆាប់សម្រាប់បទពិសោធន៍អ្នកប្រើល្អប្រសើរ
   - រៀបចំ prompt ដ៏មានប្រសិទ្ធភាពសម្រាប់ Small Language Models
   - បញ្ចូលម៉ូឌែលក្នុងស្រុកទៅក្នុងកម្មវិធី

3. **Retrieval Augmented Generation (RAG)**
   - បង្កើតការស្វែងរកអារម្មណ៍ជាមួយ embeddings វិកទ័រ
   - គ្រប់គ្រងចម្លើយ LLM ដោយពឹងផ្អែកលើឯកសារដែនកំណត់ពិសេស
   - វាយតម្លៃគុណភាព RAG ដោយប្រើមាត្រដ្ឋាន RAGAS
   - ពង្រីកពីគំរូដំណើរការទៅផលិតកម្ម

4. **បង្កើនប្រសិទ្ធភាព**
   - វាស់ប្រសិទ្ធភាពម៉ូឌែលច្រើនយ៉ាងប្រព្រឹត្ត
   - វាស់កម្រិតពេលវេលា latency, throughput, និងពេលត្រូវបាន token តំបន់ដំបូង
   - ប្រៀបធៀប Small Language Models ទល់នឹង Large Language Models
   - ជ្រើសម៉ូឌែលដែលសមស្របដោយផ្អែកលើការជជែកប្រសិទ្ធភាព/គុណភាព

5. **ការសម្របសម្រួលមួយចំនួន Agent**
   - រចនាអ្នកតំណាងឯកទេសសម្រាប់ភារកិច្ចផ្សេងៗ
   - អនុវត្តចងចាំ agent និងការគ្រប់គ្រងបរិបទ
   - សម្របសម្រួល agent ដល់គ្នាក្នុងចំណុចធ្វើការងារស្មុគស្មាញ
   - សាងសង់គំរូជាសម្នាក់របស់អ្នកដឹកនាំសម្រាប់ការសហការ agent

6. **ការបញ្ជូនម៉ូឌែលយ៉ាងឆ្លាត**
   - អនុវត្តការរកកំណត់ចេតនា និងការបaphneគំរូ
   - ធ្វើការបញ្ជូនសំណួរទៅម៉ូឌែលសមរម្យដោយស្វ័យប្រវត្តិ
   - សាងសង់ផ្លូវច្រើនជំហាន (ផែនការ → អនុវត្ត → កែប្រែ)
   - រចនាស្ថាបត្យកម្មម៉ូឌែលជាឧបករណ៍ដែលអាចពង្រីកបាន

### ការផ្ទៀងផ្ទាត់យ៉ាងលឿន

រត់នេះក្នុងកោសិកា Python ដើម្បីផ្ទៀងផ្ទាត់ការកំណត់:

```python
from foundry_local import FoundryLocalManager
import openai

# ចាប់ផ្តើមអ្នកគ្រប់គ្រង (ស្វែងរកសេវាកម្មដោយស្វ័យប្រវត្តិ)
manager = FoundryLocalManager("phi-4-mini")

# កំណត់រចនាសម្ព័ន្ធកម្មវិធីភ្ញៀវ OpenAI
client = openai.OpenAI(
    base_url=manager.endpoint,
    api_key=manager.api_key
)

# សាកល្បងការបំពេញការជជែក
response = client.chat.completions.create(
    model=manager.get_model_info("phi-4-mini").id,
    messages=[{"role": "user", "content": "Hello!"}]
)

print(response.choices[0].message.content)
```

**លទ្ធផលដែលរំពឹងទុក**: ការឆ្លើយស្វាគមន៍ពីម៉ូឌែលក្នុងស្រុក។

- ✅ ចែករំលែកធនធានដោះស្រាយបញ្ហាបន្ទាន់
- ✅ លើកទឹកចិត្តឱ្យសាកល្បង និងកែប្រែ
- ✅ ប្រើពេលសម្រាកយ៉ាងយុទ្ធសាស្ត្រ (បន្ទាប់ពីរាល់ 2 សម័យ)
- ✅ មានអ្នកជំនួយ (TAs) សម្រាប់ជំនួយផ្ទាល់ 1-on-1

**បន្ទាប់ពីវគ្គបណ្តុះបណ្តាល**:
- ✅ ចែករំលែកកំណត់ត្រាដែលដំណើរការបានពេញលេញ និងដំណោះស្រាយ
- ✅ ផ្ដល់តំណភ្ជាប់ទៅធនធានបន្ថែម
- ✅ បង្កើតសវនាការមតិសម្រាប់កែលម្អ
- ✅ ផ្ដល់ម៉ោងពិគ្រោះសម្រាប់សំណួរតាមដាន

### សម្រាប់អ្នករៀន

**កែលម្អការរៀនរបស់អ្នក**:
- ✅ សម្រេចការតំឡើងមុនការចាប់ផ្តើមវគ្គ
- ✅ រត់កោសិកាកូដនិមួយៗដោយខ្លួនឯង (កុំតែអានប៉ុណ្ណោះ)
- ✅ សាកល្បងជាមួយប៉ារ៉ាម៉ែត្រ និង prompts
- ✅ កត់ត្រាអំពីចំណេះដឹង និងចំណុចដែលត្រូវប្រុងប្រយ័ត្ន
- ✅ សួរសំណួរពេលបានជាប់ (អ្នកដទៃប្រហែលជាមានសំណួរដូចគ្នា)

**ចន្លោះកំហុសទូទៅដែលគួរជៀសវាង**:
- ❌ លោតលើលំដាប់ការរត់កោសិកា (រត់លំដាប់តាមលំដាប់)
- ❌ មិនអានសារកំហុសយ៉ាងយកចិត្តទុកដាក់
- ❌ រត់រហ័សដោយគ្មានការយល់ដឹង
- ❌ មិនយកចិត្តទុកដាក់ចំពោះការពន្យល់ក្នុង Markdown
- ❌ មិនរក្សាទុកកំណត់ត្រាដែលបានកែប្រែ

**ក្បួនដើម្បីដោះស្រាយកំហុស**:
1. **សេវាកម្មមិនដំណើរការ**: ពិនិត្យ `foundry service status`
2. **កំហុស Import**: ផ្ទៀងផ្ទាត់បរិយាកាស virtual ត្រូវបានអាស្រ័យ
3. **រកមិនឃើញម៉ូឌែល**: រត់ `foundry model ls` ដើម្បីបញ្ជីម៉ូឌែលដែលបានឡោដ
4. **ប្រសិទ្ធភាពយឺត**: ពិនិត្យការប្រើប្រាស់ RAM, បិទកម្មវិធីផ្សេងទៀត
5. **លទ្ធផលមិនរំពឹងទុក**: ចាប់ផ្តើម kernel ម្តងទៀត និងរត់កោសិកាទាំងអស់ពីចាប់ផ្ដើម
